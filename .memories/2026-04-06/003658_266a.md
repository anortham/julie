## Fix: Sidecar MPS GPU memory leak (6.6GB -> 1.9GB)

### What
Fixed excessive memory usage in the Python embedding sidecar on Apple Silicon. Process footprint dropped from 6.6GB to 1.9GB.

### Why
PyTorch's MPS caching allocator was holding ~6GB of Metal GPU buffers for a 522MB model. On Apple Silicon's unified memory, this directly consumed system RAM.

### How
Three changes:
1. **`_empty_device_cache()` method** on `SentenceTransformerRuntime` - calls `torch.mps.empty_cache()` / `torch.cuda.empty_cache()` after every `embed_batch` and during probe-fallback cleanup
2. **Float16 model loading** on GPU devices via `model_kwargs={"torch_dtype": torch.float16}` with graceful fallback to float32 if loading fails
3. **Embedding format version bump** (v2 -> v3) to force re-embed of existing fp32 vectors

### Impact
- 72% memory reduction on Apple Silicon (verified via `footprint`)
- Codex adversarial review caught two issues pre-release: fp16 constructor crash risk (fixed with try/except fallback) and mixed-precision vector drift (fixed with format version bump)
- All tests green: 42 Python, 8 Rust xtask buckets
