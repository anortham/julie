## Removed WatcherPool Raw Ref-Count Hooks

### WHAT
- Deleted public `WatcherPool::increment_ref`.
- Merged `WatcherPool::decrement_ref` into `WatcherPool::detach` and kept the saturated decrement plus grace-period startup behavior.
- Rewrote watcher tests to use the product lifecycle (`attach`, `detach`, `reap_expired`, `update_all_provider`) instead of raw ref-count hooks.
- Updated dead-code cleanup docs, baseline ledger, and workspace service split plan wording.

### WHY
The raw ref-count methods were test fabrication hooks after session ownership moved to `WorkspaceSessionAttachment`. Production code should attach and detach workspace resources, not manually mutate watcher refs.

### HOW
- Used Julie `deep_dive` on `WatcherPool::increment_ref`, `WatcherPool::decrement_ref`, and `WatcherPool::attach` plus subagent audit evidence.
- Confirmed raw ref-count methods had no remaining references with Julie `fast_refs` and raw `rg`.
- Added a lightweight test helper that attaches dummy workspaces with no DB/search index, creating real pool entries without starting file watchers.

### IMPACT
WatcherPool now has a smaller public lifecycle API. Tests cover the same ref-count and grace behavior through the daemon-facing path instead of preserving dead methods.
