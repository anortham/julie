## Fixed get_symbols missing-file failure reporting

- **WHAT:** Changed `get_symbols` missing-file handling so primary and target workspace paths return real errors instead of successful text responses. Added a shared missing-file message helper and updated tests for direct tool behavior plus handler metrics persistence.
- **WHY:** Missing-file calls were being treated as successful tool calls, which hid failures from metrics and made dogfooding data less trustworthy.
- **HOW:** Replaced the old `Ok(CallToolResult::text_content(...))` branch with `bail!(...)` in both symbols backends, while preserving the existing user-facing guidance text. Extended handler metrics coverage to verify `success = 0`, `output_bytes = 0`, and missing-file metadata.
- **IMPACT:** `get_symbols` now behaves consistently with handler failure accounting, so failed calls are visible in metrics and MCP/CLI error paths.
