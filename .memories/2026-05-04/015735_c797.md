## Removed Legacy IPC Transport

### WHAT
Deleted the daemon IPC transport path and moved daemon clients to the localhost Streamable HTTP discovery contract.

- Removed `src/daemon/ipc.rs`, `src/daemon/ipc_session.rs`, and legacy IPC adapter tests.
- Added shared HTTP client config in `src/daemon/http_client.rs` for adapter and CLI daemon calls.
- Collapsed `TransportEndpoint` to HTTP-only discovery and readiness probing.
- Moved workspace disconnect ordering into `src/daemon/mcp_session.rs` with direct tests.
- Updated xtask transport bucket and docs to describe HTTP-only daemon MCP transport.

### WHY
The old IPC compatibility path had become dead weight after the daemon and adapter successfully dogfooded HTTP transport. Keeping both paths increased lifecycle risk and test cost.

### HOW
Removed the listener, header protocol, ready-line handshake, stream forwarding helpers, and IPC path helpers after adding or preserving tests for HTTP admission, stale-binary disconnect behavior, launcher readiness, CLI tool-error semantics, and session cleanup.

### IMPACT
The daemon now has one MCP transport boundary: localhost Streamable HTTP. Stdio remains only as the external MCP client adapter. Verification used focused tests, `cargo xtask test changed`, `cargo xtask test reliability`, and `cargo xtask test bucket transport`.
