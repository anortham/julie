## Planned Legacy IPC Removal

### WHAT
- Created `docs/plans/2026-05-04-remove-legacy-ipc.md` in branch `codex/remove-legacy-ipc`.
- Plan identifies CLI daemon IPC usage as a required migration before deleting IPC modules.
- Tasks cover CLI HTTP client migration, HTTP-only discovery/launcher, daemon IPC listener removal, adapter fossil deletion, and xtask/docs cleanup.

### WHY
HTTP is now the canonical daemon MCP transport and live dogfood shows sessions arriving over HTTP. Keeping IPC around preserves a second transport stack, a second session handshake, and many fossil tests.
