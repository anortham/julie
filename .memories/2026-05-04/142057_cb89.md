## Hardened Watcher Queue And Repair Paths

### WHAT
Implemented the watcher hardening plan for queue coalescing, duplicate suppression, pause-aware ingestion, targeted overflow repair, and persisted repair filtering.

### WHY
Recent daemon logs showed repeated watcher processing, queue overflow bursts, and repair retries for unsupported files. The old flow could grow logs and redo large amounts of unchanged work after backpressure.

### HOW
- Added `src/watcher/queue.rs` for path-based coalescing and chunked overflow drain to a 750 item target.
- Changed runtime duplicate handling to drop recent create/modify duplicates and emit batch summaries.
- Added pause-aware event ingestion so catch-up indexing drains notify events without filling the queue, while scheduling one repair pass.
- Changed overflow repair to hash-compare indexed files and dispatch only deleted, modified, and new supported files.
- Changed persisted repair retry filtering to clear unsupported extensionless and unsupported extension paths before dispatch.
- Recorded verification in `docs/plans/2026-05-04-watcher-hardening.md`.

### IMPACT
Watcher bursts should now produce bounded queue growth, fewer repeated extraction attempts, fewer info logs per file, and repair scans proportional to actual changed work. Verification included watcher exact tests plus `cargo xtask test changed` and `cargo xtask test dev`.
