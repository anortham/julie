## Fixed post-merge build break from dashboard action route dependency scope

### WHAT
Repaired a production build failure on `main` caused by the new dashboard workspace-action route depending on `tempfile` from normal library code while `Cargo.toml` declared it only under `dev-dependencies`.

- Moved `tempfile` into `[dependencies]`
- Removed an unused `Html` import in `src/dashboard/routes/projects_actions.rs`
- Rebuilt the crate and reran the dashboard action tests plus the changed-suite gate

### WHY
The merged code compiled under test runs because dev builds for tests can see dev-only crates, but `cargo build` for the shipped library target could not resolve `tempfile`. That left the repo in a broken state after merge.

### HOW
- Reproduced with `cargo build`
- Read the compiler error at `src/dashboard/routes/projects_actions.rs`
- Confirmed `tempfile` was used in runtime code and declared only in `dev-dependencies`
- Applied the minimal fix by promoting the crate to a normal dependency

### IMPACT
`cargo build` works again on `main`, dashboard action routes keep their temp-anchor behavior, and the branch gate is green after the repair.
