## Reduced Julie skill surface

### WHAT
Reduced the repo-local skill set from 11 to 5 and updated the kept skills to match the current tool surface.

- Kept: `editing`, `explore-area`, `impact-analysis`, `search-debug`, `web-research`
- Removed: `architecture`, `call-trace`, `dependency-graph`, `logic-flow`, `metrics`, `type-flow`
- Rewrote `editing` around `edit_file`, `rewrite_symbol`, and `rename_symbol`
- Tightened `search-debug` as a Julie-dogfooding skill
- Added a design note at `docs/plans/2026-04-18-skill-set-reduction-design.md`

### WHY
The tool surface changed and the skill set drifted behind it. `edit_symbol` and `query_metrics` were gone, but skills still taught them. Several remaining skills were thin report templates over existing tool descriptions and did not justify separate maintenance cost.

### HOW
Deleted the cut skill files, rewrote the kept `editing` skill to the live editing workflow, and kept `search-debug` as repo-only dogfood rather than distributed product surface.

### IMPACT
The julie repo now has a smaller, cleaner skill set that matches the current tool roster and preserves only the workflows with distinct value.
