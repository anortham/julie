## Hardened post-review workspace cleanup and dashboard actions

### WHAT
Applied the Claude review fixes on top of the hard-C workspace-management branch.

- Added a grace-window recheck before auto-pruning missing workspaces
- Reused that safer missing-path check in `open` before pruning a stale row
- Stopped silently swallowing cleanup sweep failures in `list` and `register`
- Added dashboard action CSRF tokens and confirmation prompts for destructive actions
- Added focused regression tests for transient path disappearance and dashboard action protection

### WHY
Claude surfaced two real risks: one-shot missing-path checks could delete a valid index during a transient filesystem blip, and dashboard delete/prune actions were unprotected POSTs with no confirmation. The cleanup sweep also hid failures in a couple of places.

### HOW
- Introduced `path_missing_after_grace()` in workspace cleanup and used `Path::try_exists()` twice with a short delay
- Threaded a per-dashboard action token through `DashboardState`, template rendering, forms, and POST handlers
- Returned `403` with a rendered error page when the token check fails
- Added confirmation prompts to delete and prune buttons in dashboard templates
- Extended focused tests in `workspace_cleanup` and `dashboard::projects_actions`
- Re-ran targeted tests and `cargo xtask test changed`

### IMPACT
Auto-prune is less trigger-happy, dashboard mutations are harder to trigger by accident or cross-site POST, and cleanup failures are no longer hidden. The branch stays green after the post-review hardening.
