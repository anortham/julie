## Captured agent tool surface redesign spec

### WHAT
Created a design doc for the next Julie MCP tool-surface pass at `docs/plans/2026-04-17-agent-tool-surface-design.md`.

### WHY
The conversation settled several coupled product decisions that would be easy to lose or blur if they stayed only in chat:
- remove `query_metrics` from the public MCP surface
- leave `manage_workspace` unchanged
- replace `edit_symbol` with a narrower AST-backed `rewrite_symbol`
- add a bounded shortest-path `call_path` tool
- keep token discipline as the governing constraint for any tool change

### HOW
The spec records:
- current usage signals from dashboard metrics
- the one-tool-one-job design rules for token efficiency
- the `rewrite_symbol` contract, operations, and non-goals
- the `call_path` contract and the decision to ship shortest-path only in v1
- dashboard follow-through needed after removing `query_metrics`
- scoped follow-up tuning for `fast_refs`, `get_context`, and `rename_symbol`

### IMPACT
This gives the next implementation pass a concrete design target, keeps the scope narrow, and preserves the rationale behind each decision so the work does not drift into kitchen-sink tools.
