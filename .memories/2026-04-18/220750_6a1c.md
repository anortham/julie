## Completed Task 2: remove pairing storage and add shared cleanup

Implemented the cleanup/storage slice of the workspace-management hard-C plan in the worktree branch.

- Added daemon DB migration 003, which creates `workspace_cleanup_events` and drops `workspace_references`
- Added cleanup-event persistence plus capped retention and query helpers in `DaemonDatabase`
- Updated workspace ID migration to rewrite cleanup events instead of the dead pairing table
- Added shared cleanup helpers in `src/tools/workspace/commands/registry/cleanup.rs`
- Wired that cleanup path into `register`, `list`, `clean`, `open`, and `remove`
- Manual delete now blocks active workspaces and records `manual_delete` events on success
- `open` now auto-prunes missing inactive known workspaces instead of trying to refresh a dead path
- Added a background daemon cleanup sweep on a timer
- Converted legacy pairing-focused tests to known-workspace/current-active-known semantics where the lifecycle rule still mattered
- Updated stdio registry metadata to use `known_workspaces` and `WorkspaceType::Known` with serde aliases for older JSON

## WHY

The old pairing table added stale semantics, made cleanup reasoning messy, and left deleted worktrees hanging around until someone ran cleanup by hand. This slice centralizes prune eligibility and makes cleanup behavior visible through a small event log.

## HOW

- Added migration 003 without rewriting historical migrations
- Introduced `delete_workspace_if_allowed`, `prune_missing_workspaces`, `prune_orphan_index_dirs`, and `run_cleanup_sweep`
- Added `WorkspacePool` helpers for indexing snapshots and eviction, plus immediate inactive watcher removal in `WatcherPool`
- Reworked list/stats outputs to current/active/known language and removed pairing-derived counts
- Updated focused tests in daemon DB, daemon session cleanup, and workspace global-targeting flows

## IMPACT

- Deleted inactive worktrees can now disappear through list/open/clean/register traffic and through a background daemon sweep
- Manual delete will no longer rip out live workspace state under an active session
- Cleanup actions are now observable through persistent event rows, which sets up the dashboard cleanup log in Task 3
- The codebase no longer depends on `workspace_references` as real storage semantics
