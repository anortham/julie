## WHY

The user wanted a hard semantic break from the old reference-workspace model before continuing workspace/dashboard work. The codebase still had live routing types, helper modules, docs, test buckets, and daemon helpers encoding the dead concept. That mismatch would keep confusing future workspace work and made the `xtask` runner lie about what test names exist.

## HOW

- Used Julie symbol/context tools to inspect the shared workspace-routing layer before renaming the enum and helper modules.
- Applied the rename at the routing primitive (`WorkspaceTarget`) first, then propagated it through search/navigation/editing/symbol tooling so the model stayed coherent.
- Preserved backward-compatibility only where it still matters:

  - `WorkspaceRegistry.known_workspaces` keeps `#[serde(alias = "reference_workspaces")]` for legacy stdio registry JSON.
