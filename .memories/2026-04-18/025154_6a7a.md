## WHAT

Completed the agent tool-surface implementation batch in the `codex/agent-tool-surface` worktree.

- Removed the public `query_metrics` MCP tool while keeping dashboard-backed metrics visibility intact.
- Replaced `edit_symbol` with `rewrite_symbol` and removed the old handler wiring, module, and tests.
- Added the new `call_path` navigation tool with bounded shortest-path behavior.
- Updated agent guidance and site examples to reflect the live roster.

## WHY

The previous public surface had one low-value admin distraction and one weak symbolic edit primitive. The goal was to tighten the agent-facing loop around tools that save tokens and answer one job each.

## HOW

- Deleted `QueryMetricsTool` routing from `src/handler.rs` and `src/tools/metrics/mod.rs`, removed dead code-quality helper wiring, and replaced tool-level coverage with dashboard integration tests.
- Built `src/tools/editing/rewrite_symbol.rs` around indexed symbol resolution, freshness checks, live extraction via `ExtractorManager`, tree-sitter span recovery, atomic writes, and compact diff previews.
- Ported safety coverage from `edit_symbol` into `rewrite_symbol` tests, including stale-index rejection and primary rebind / swap-window behavior.
- Added `src/tools/navigation/call_path.rs` as a bounded BFS over indexed outgoing relationships with compact JSON output and hop labels.
- Registered `call_path` in navigation exports, handler routing, and tool-list assertions.
- Updated `JULIE_AGENT_INSTRUCTIONS.md` and `docs/site/*` to remove `query_metrics` and `edit_symbol`, add `rewrite_symbol`, and add `call_path` examples.

## IMPACT

- Public MCP surface now exposes `rewrite_symbol` and `call_path`, and no longer exposes `query_metrics` or `edit_symbol`.
- Symbol rewriting is now live-parse-backed instead of line-range-based.
- The new path tool answers dependency-chain questions with bounded output.
- Verification is clean across targeted tests, `cargo xtask test changed`, and `cargo xtask test dev`.
