## Applied exact-definition cleanup to locations mode

### WHAT
Fixed a live dogfood inconsistency where `return_format="locations"` ignored the exact-definition cleanup used by full output. Exact definition locations now use the same declaration/import demotion and loose-token suppression path as full output.

### WHY
After rebuilding, full output for `format_definition_search_results` was clean, but locations mode still returned a loose `src/tools/navigation/formatting.rs:48` result ahead of exact related matches. Locations mode should be compact, not semantically weaker.

### HOW
- Added `test_locations_mode_uses_exact_definition_cleanup`.
- Extracted shared `definition_display_parts` for exact definition display partitioning.
- Updated `format_locations_only` to flatten exact + related display parts when exact matches exist, otherwise preserve the existing generic formatting behavior.
- Adjusted the generic locations formatter test to use a no-exact query so it continues to cover the fallback path.

### IMPACT
Full and locations definition output now agree on exact-match cleanup: real declarations first, exact imports as related, loose token-only results suppressed.
