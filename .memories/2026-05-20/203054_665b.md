## Improved exact-definition related match snippets

### WHAT
Fixed another live dogfood formatting issue in exact definition searches: related `Other matches` now prefer symbol signatures over weak `code_context` snippets.

### WHY
After the rebuild, `fast_search(query="SearchIndex", search_target="definitions")` correctly kept related `SearchIndex*` matches, but displayed mostly `}` / `...` context lines. The matches were real; the presentation was not actionable.

### HOW
- Added `test_exact_match_other_matches_prefer_signature_over_empty_context`.
- Added `write_definition_other_match_snippet` so promoted-definition `Other matches` print signatures when available, falling back to code context only when needed.
- Preserved grouped line anchors and the earlier filtering of loose token-only matches.

### IMPACT
Exact definition output keeps useful related symbols while showing readable signatures instead of low-information context.
