## Fixed exact definition lookup noise found during live dogfood

### WHAT
Dogfooding the rebuilt MCP showed exact definition searches still worked but could print low-value `Other matches` entries when unrelated symbols matched only loose query tokens. Added filtering so exact-definition formatting keeps related name/signature matches but drops loose token-only matches.

### WHY
A lookup for `rank_file_search_result` returned the correct promoted definition, then cluttered the answer with unrelated anchors from symbols like `run_get_context_pipeline` because the formatter dumped every non-exact result after the promoted section.

### HOW
- Added a regression test for `rank_file_search_result` + token-only `run_get_context_pipeline`.
- Filtered promoted-definition `Other matches` through a normalized full-query match against symbol name/signature.
- Preserved existing useful behavior where related names like `search_index` remain visible for `SearchIndex`.

### IMPACT
Exact symbol lookups are tighter and less distracting while conceptual/related definition output remains intact.
