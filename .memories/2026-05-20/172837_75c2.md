## Completed fast_search dogfood follow-up fixes

### WHAT
Implemented two dogfood fixes found after the live restart:
- `exclude_tests=auto` now keeps test files when `file_pattern` explicitly targets a test path.
- File-mode searches for hidden directory names, such as `.cargo`, now rank matching hidden-directory paths before root `Cargo.*` basename fragments in both index-level and tool-level output.

### WHY
The live dogfood session showed two remaining result-quality defects: explicit test scopes could be filtered away by the NL-query auto test filter, and tool-level file sorting lost the hidden-directory ranking that users expect for dot-directory queries.

### HOW
- Updated `line_mode` auto-filter derivation to treat test-like `file_pattern` values as explicit test intent unless `exclude_tests` is set.
- Added shared file-search ranking in `search/index.rs` and reused it from `tools/search/execution.rs` so CLI/MCP file-mode output preserves the same ordering as the Tantivy file search.
- Added regression coverage at the line-mode, index, and tool-output layers.

### IMPACT
`fast_search` dogfood results now behave correctly for explicit test-path scopes and hidden directory file queries. Verified with narrow tests, `cargo xtask test changed`, `cargo xtask test dev`, and standalone debug CLI dogfood checks.
