## Demoted exact import matches below real definitions

### WHAT
Fixed live dogfood output where exact definition searches put import symbols inside the promoted `Definition found:` block when a real function/struct definition was also present.

### WHY
After the latest rebuild, `fast_search(query="format_definition_search_results", search_target="definitions")` showed both the function declaration and an exact test import under `Definition found:`. That made exact definition output noisier and less semantically accurate.

### HOW
- Added `test_exact_match_promotes_declaration_before_import_matches`.
- Kept exact import/export-only searches compatible.
- When exact non-import/export declarations exist, exact import/export matches are demoted into the related `Other matches` section instead of the promoted block.

### IMPACT
Exact definition output now highlights the actual declaration first while still surfacing exact import matches as related context.
