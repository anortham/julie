## v6.0.2 Tagged and Pushed (Windows Fixes)

### What
Released v6.0.2 with critical Windows daemon and DirectML embedding fixes. Two fix commits + version bump, pushed to origin with tag.

### Fixes in v6.0.2

**1. Windows daemon connection (ec031bf6)**
- `is_process_alive()` was a TODO stub returning `false` on Windows, causing every adapter launch to delete PID file and spawn duplicate daemons
- `IpcConnector::connect()` didn't retry on `ERROR_PIPE_BUSY` (231), the standard Windows named pipe transient error
- `wait_for_socket()` only checked for success (not pipe-busy), and had a 10s timeout vs 26s ORT cold-start
- Fixed with: `OpenProcess` Win32 API, exponential backoff retry, ERROR_PIPE_BUSY handling, 45s timeout

**2. DirectML VRAM overflow (272d2de2)**
- `embed(texts, None)` sent all 250 texts in one ORT inference call
- One long text (up to 8192 tokens for Jina-code-v2) forced padding for all 250 texts, creating tensors exceeding 6GB VRAM
- DirectML silently fell back to CPU for oversized batches
- Fixed by passing `Some(32)` as sub-batch size to fastembed
- **Result**: 40 minutes down to 70 seconds (34x faster), all batches consistent ~4s on GPU
