## Code Review: Embeddings/Workspace/Watcher/Analysis Layer

Completed a systematic read-only code review of the embeddings, workspace, watcher, analysis, and utils subsystems as part of a coordinated deep-review agent team.

### CRITICAL Findings

- **Linux rename dead code path** (`src/watcher/events.rs`): `FileChangeType::Renamed` is fully handled in `mod.rs` but `events.rs` never produces it. Linux renames arrive as `EventKind::Modify(ModifyKind::Name(Both))` — old path fails `is_file()`, new path is queued as plain `Modified`. Stale index entries from old paths accumulate permanently. macOS and Windows are unaffected.
- **Fake disk space check** (`src/workspace/mod.rs`): `check_disk_space()` is hardcoded to return 1000MB regardless of actual disk state. Low-space warnings can never fire. This is a stub that was never completed.

### HIGH Findings

- **Unbounded event queue** (`src/watcher/mod.rs`): `VecDeque<FileChangeEvent>` has no capacity cap — a rapid burst of FS events (e.g., large `git checkout`) could exhaust memory.
- **Duplicated watcher logic** (`src/watcher/mod.rs` lines 150-359 vs 367-483): ~120 lines duplicated between the background loop and `process_pending_changes`, with behavioral divergence (atomic-save dedup clearing is absent from the manual path).
- **Fire-and-forget stop** (`src/watcher/mod.rs`): `stop()` cancels the AtomicBool but has no join handles — no way to verify graceful shutdown or drain the pending queue.
- **Language-agnostic violation** (`src/utils/path_relevance.rs`): `path_str.contains("src") || path_str.contains("lib")` returns 1.0, missing Go (`cmd/pkg/internal/`), Java test paths (`src/test/java/`), and other layouts.
- **Relaxed atomic ordering** (`src/watcher/mod.rs`): `cancel_flag` uses `Relaxed` ordering; should be `Release` on write and `Acquire` on read to establish proper happens-before.

### MEDIUM Findings

- **Enriched symbols always re-embedded** (`src/embeddings/pipeline.rs` line 291): ~2000 enriched symbols unconditionally re-embedded on every pipeline run, even when unchanged. Comment admits this is known.
- **Blocking I/O in Drop** (`src/embeddings/sidecar_provider.rs`): `Drop` impl calls `shutdown_and_terminate` which sends JSON-RPC and waits 500ms — blocking I/O in `Drop` can deadlock async runtimes.
- **Fragile JSON unwrap** (`src/analysis/change_risk.rs`, `security_risk.rs`): `meta.as_object_mut().unwrap()` is safe given the `unwrap_or_else(|_| json!({}))` fallback but brittle if that invariant changes.
- **Path ID lowercasing on Linux** (`src/workspace/registry.rs`): `normalize_path` calls `.to_lowercase()` on all platforms — on Linux, two case-different paths hash to the same workspace ID.
- **Spurious Tantivy commits** (`src/watcher/mod.rs`): commit decision uses pre-dedup `queue_size`, so commits fire even when all events are filtered as duplicates.
- **Public blacklist fn with wrong arg** (`src/watcher/filtering.rs`): `contains_blacklisted_directory()` takes an absolute path while internal callers correctly use `_relative()`.

### LOW Findings

- **Rust INPUT_PATTERNS false positives** (`src/analysis/security_risk.rs`): patterns include `"&str"` and `"String"` — near-universal matches in Rust code, producing meaningless security scores.
- **Unencapsulated queue** (`src/watcher/mod.rs`): `VecDeque` is accessed directly across multiple methods instead of through an abstraction.
- **unwrap vs expect** (`src/workspace/mod.rs`): `self.db.as_ref().unwrap()` in `initialize_file_watcher` — safe given prior guard but should use `.expect("db must be set before watcher init")` for diagnostics.
- **Stderr-only sidecar errors** (`src/embeddings/sidecar_bootstrap.rs`): `run_command` captures only stderr in error messages, losing stdout which may contain relevant failure context.
