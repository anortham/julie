## v6.0.3 Windows Hardening (7 fixes)

Comprehensive Windows audit of v6.0.0-v6.0.2 changes, found and fixed 7 issues across critical/important tiers.

### Critical Fixes
- **Graceful shutdown on Windows restart**: Replaced `std::process::exit(0)` with `Arc<Notify>` that feeds into the existing `tokio::select!` cleanup path. Sessions now drain, embeddings shut down, and PID files are cleaned up on Windows restart. Also eliminated the `unsafe libc::kill` on Unix.
- **Console window flash**: Added `CREATE_NO_WINDOW` to `spawn_daemon()`, matching all other background process spawns.
- **Cache dir `/tmp` fallback**: Replaced hardcoded `/tmp` with `USERPROFILE` then `std::env::temp_dir()` fallback chain in `default_cache_dir()`.

### Important Fixes
- **Scoped pipe names**: `daemon_ipc_addr()` now hashes `julie_home` into the Windows pipe name for isolation. Different installations get different pipes.
- **Force-kill for console**: Added `/F` to `taskkill` in `stop_daemon()`.
- **Handle exhaustion**: Added `WSAEMFILE` (10024) to `is_transient_accept_error()` on Windows.
- **RootDir path component**: Changed `push("/")` to `push(component.as_os_str())` in `secure_path_resolution()`.
