## Clarified Codex configuration docs

### WHAT
Updated the Codex setup documentation to distinguish Codex CLI from Codex Desktop and corrected the Codex copy snippet on the website.

### WHY
The previous docs implied a single Codex configuration path and omitted the `JULIE_WORKSPACE` requirement that Codex Desktop needs when its working directory is unreliable. That led to Julie being launched with `/` as the inferred workspace.

### HOW
- Added separate README guidance for Codex CLI and Codex Desktop
- Documented the project-local `.codex/config.toml` pattern for Codex Desktop
- Kept the Codex CLI guidance minimal because it usually inherits the project cwd correctly
- Updated the website copy snippet to show the Desktop-safe Codex config
- Matched the TOML example to the inline `env = { ... }` style used in the local working config

### IMPACT
The README and website now point users at the configuration that works in Codex Desktop without implying that Codex CLI has the same requirement. This should prevent the `workspace=/` startup failure and reduce future support churn.
