## Fixed dashboard embedding service status race condition

Dashboard showed "Not configured" for the Embedding Service on initial page load, but showed "Available" after refresh. Two bugs fixed:

### Root Cause
1. **Missing state**: Template had binary logic (available/not configured), but the `EmbeddingService` has three states: `Initializing`, `Ready`, `Unavailable`. During the ~36s lazy init window, `Initializing` was displayed as "Not configured".
2. **Stale polling**: The 5s polling JS updated uptime/sessions/workspaces but never touched the embedding status cell, so even after init completed the panel stayed frozen.

### Fix
- Added `DashboardState::embedding_initializing()` method using `!svc.is_settled()`
- Passed `embedding_initializing` to both Tera template context and `/status/live` JSON
- Template now shows three states: green "Available", yellow "Initializing", grey "Not configured"
- Polling JS updates `#embedding-status` cell every 5s from live response

### Files Changed
- `src/dashboard/state.rs` - new method
- `src/dashboard/routes/status.rs` - wire through both routes
- `dashboard/templates/partials/services_panel.html` - tri-state UI
- `dashboard/templates/status.html` - live polling update
