## Fixed manage_workspace for reference workspaces

### Bug 1: `index` silently returns primary's stats for reference workspaces
- **WHAT**: `manage_workspace(operation="index", path="/path/to/ref")` returned "Workspace already indexed: 31158 symbols" where 31158 was the PRIMARY workspace's symbol count, not the reference workspace's. The reference workspace was never touched.
- **WHY**: The `is_indexed` guard in `handle_index_command` (index.rs:147) checks a handler-level boolean and queries the primary's database. It doesn't distinguish between primary and reference workspace targets.
- **HOW**: Added `&& !is_reference_workspace` to the guard condition. The `is_reference_workspace` variable was already computed at line 104 but wasn't used to skip the guard.
- **IMPACT**: Agents can now use `index` on reference workspace paths without getting silently wrong results.

### Bug 2: `refresh` didn't advertise `force` parameter
- **WHAT**: Tool schema said `force` was "used by: index" only. Refresh example omitted `force`. Agents didn't know they could force a full re-index on reference workspaces.
- **HOW**: Updated schema description to `(used by: index, refresh)` and updated refresh example to show `force: true`.

### Doc gap: Agent instructions had no reference workspace coverage
- **WHAT**: `manage_workspace` section in JULIE_AGENT_INSTRUCTIONS.md was 2 lines with no mention of `add`, `refresh`, reference workspaces, or `force`.
- **HOW**: Expanded with examples for add, list, refresh (incremental and force), and explanation of when to use `force=true`.

### Live verification (pre-fix)
- `index` on flask without force: returned "already indexed: 31158 symbols" (Julie's count, not Flask's 4291)
- `refresh` on flask without force: correctly showed Flask's 226 files, 4291 symbols but "Already up-to-date"
- `index` on flask with `force=true`: correctly processed 226 files, 4291 symbols
