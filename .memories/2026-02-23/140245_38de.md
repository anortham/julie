## Completed exploration for ignore crate migration plan

### What
Thoroughly explored both file walkers (discovery.rs, startup.rs), shared utilities (utils/ignore.rs), constants (shared.rs), watcher filtering, and all related tests to design a migration plan from manual file walking to the BurntSushi `ignore` crate.

### Key findings
- **Walker 1** (`discovery.rs`): Manual `fs::read_dir` recursion, 551 lines, 7-layer filtering, two-phase vendor detection
- **Walker 2** (`startup.rs`): Uses `walkdir` crate, 353 lines, duplicate blacklisted dirs, `is_ignored_path()` 
- **Shared utils** (`utils/ignore.rs`): 327 lines with `load_julieignore()` + `is_ignored_by_pattern()` + inline tests (violates project rules)
- **Critical finding**: `ignore` crate with `hidden(false)` does NOT auto-exclude `.git/` -- need `filter_entry` workaround
- **Watcher** (`watcher/filtering.rs`, `watcher/events.rs`): Has its own `should_index_file` and glob patterns -- OUT OF SCOPE
- **22 tests** in discovery.rs test file, 7 tests in stale_index_detection.rs, inline tests in ignore.rs
- `globset` used only in `src/tools/search/query.rs` -- cannot remove
- `walkdir` used only in `startup.rs` -- can remove after migration

### Impact
Plan covers 7 tasks with TDD methodology, exact file paths, and dead code cleanup.
