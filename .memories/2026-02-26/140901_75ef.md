## Task 4 Spec Compliance Review Completed

Reviewed current working-tree changes for Task 4 requirements focused on weighted query builders.

- **Inspected implementation:** `src/search/query.rs`
- **Inspected tests:** `src/tests/tools/search/tantivy_query_weighting_tests.rs`, `src/tests/tools/search/mod.rs`
- **Checked non-integration constraint:** verified no call-site wiring changes in `src/search/index.rs`; weighted builders only referenced from wrappers/tests.
- **Verified behavior evidence:** ran targeted tests for weighting and query preprocessor regressions.

### Key finding
Grouped-term handling does **not** consistently implement weighted SHOULD behavior in AND mode: when `require_all_terms=true`, grouped terms are emitted as `Must` at both term and group levels.

### Why this matters
Spec wording requires grouped weighted SHOULD behavior while preserving MUST semantics for structural filters (`doc_type`, optional `language`/`kind`). Current code preserves filter MUST semantics but over-constrains alias/normalized groups under AND mode.
