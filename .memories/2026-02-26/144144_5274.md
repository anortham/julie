## Implemented Task 6 NL-only path prior in search scoring

Added `apply_nl_path_prior(results: &mut [SymbolSearchResult], query: &str)` in `src/search/scoring.rs` and kept it intentionally conservative and deterministic.

- **What changed:**
  - Added NL path-prior constants for `src/**` boost and `docs/**`, `src/tests/**`, `fixtures/**` penalties.
  - Added NL gate heuristic (`is_nl_like_query`) with early return for empty/whitespace and identifier-like queries.
  - Added deterministic score sorting helper after multiplier application to preserve descending order.
- **Why:** Task 6 requires an NL-only prior that nudges production code above docs/tests/fixtures without disturbing identifier searches.
- **How:**
  - Gate rejects identifier-like tokens (snake_case or camelCase), including mixed NL+identifier queries.
  - Path classification order ensures `src/tests/**` is penalized before general `src/**` boost.
  - Re-sort by score descending with deterministic tie-break by symbol id.
