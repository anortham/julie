## Continued Phase 3: batched graph expansion and file-size compliance refactor

### WHAT
- Refactored `expand_graph` in `src/tools/get_context/pipeline.rs` to eliminate per-pivot relationship queries:
  - Replaced N+1 calls (`get_relationships_to_symbol` + `get_outgoing_relationships`) with batched queries:
    - `get_relationships_to_symbols(...)`
    - `get_outgoing_relationships_for_symbols(...)`
- Added/kept batch DB API and coverage:
  - `src/database/relationships.rs::get_outgoing_relationships_for_symbols`
  - `src/tests/core/database.rs::test_get_outgoing_relationships_for_symbols_batch`
- Reduced `src/tools/get_context/pipeline.rs` below the project’s 500-line implementation limit:
  - Extracted content helpers into new module `src/tools/get_context/content.rs`
  - Added `pub mod content;` in `src/tools/get_context/mod.rs`
  - Re-exported truncation utility from pipeline for test compatibility

### WHY
- Phase 3 goal is lower latency by removing hot-path DB round trips.
- Pipeline file grew above the 500-line hard limit and required module extraction.

### HOW
- Converted pivot ID handling to batched vectors/sets, then merged relationship results into the existing neighbor map.
- Moved `abbreviate_code` + `truncate_to_token_budget` into `content.rs` to preserve behavior while shrinking pipeline.
- Kept all existing test call-sites stable by re-exporting `truncate_to_token_budget` from `pipeline`.

### IMPACT
- Further reduces query overhead in `get_context` graph expansion for multi-pivot queries.
- Restores compliance with project file-size policy for implementation files.
- Behavior remains green across targeted and fast-tier test suites.
