## Final quality review for Task 4 after fixes

Reviewed `src/search/query.rs` and `src/tests/tools/search/tantivy_query_weighting_tests.rs` for previously reported AND-mode weighted-group semantics.

- **What changed:** `require_all_terms=true` now applies `Must` only to original-term group, keeping alias/normalized groups as optional boosts.
- **Validation:** Ran `cargo test --lib tantivy_query_weighting`; all 4 weighting tests pass.
- **Strength:** Regression tests now include executable in-memory Tantivy queries that prove optional expansion behavior in AND mode for both symbol and content weighted builders.
- **Remaining risk:** `build_content_query_weighted` still treats underscore terms as `Should` even in AND mode; if caller passes only underscore terms in `original_terms`, query can devolve to filter-only matching (because Should is optional when Must filters exist). Not covered by current tests.

This checkpoint captures approval posture: prior blocker appears fixed, but one edge-case test/fix is still recommended before broad integration.
