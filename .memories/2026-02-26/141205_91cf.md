## Fixed Task 4 weighted-query semantics for AND mode

Updated query builders so **original terms remain strict requirements** in `require_all_terms=true`, while **expanded alias/normalized groups are optional boost-only SHOULD clauses**.

- **What changed:**
  - In `build_symbol_query_weighted`, grouped terms now track whether they are original vs expanded.
  - In AND mode, only original group terms/groups are `Must`; expanded groups are `Should`.
  - Removed the scoring shim (`NAME_FIELD_BOOST + 0.01`) and kept deterministic group-scaled boosts.
  - Applied the same original-vs-expanded behavior to `build_content_query_weighted`.
- **Why:**
  - Reviews flagged over-constrained AND semantics where expansions were treated as required, causing false negatives.
  - Removed undocumented scoring hack to keep transparent, stable scoring behavior.
- **How verified:**
  - Added targeted behavior tests in `src/tests/tools/search/tantivy_query_weighting_tests.rs` using in-memory Tantivy indexes to validate that docs matching only original terms still match in AND mode.
  - Updated stale RED wording/comments in weighting tests.
  - Ran required commands and confirmed pass.

- **Impact:**
  - Task 4 semantics now align with intended strictness + optional expansion boosts.
  - Backward compatibility wrappers remain intact.
  - No index integration changes introduced.
