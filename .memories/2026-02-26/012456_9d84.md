## Phase 1 fully completed; Phase 2 expanded with compact-mode semantics and dedup

### WHAT
- Finalized Phase 1 relevance work by adding **auxiliary code path de-boost**:
  - `examples/`, `example/`, `benches/`, `bench/`, `scripts/`
  - Implemented in `src/tools/get_context/scoring.rs` via `AUXILIARY_CODE_PENALTY` + `is_auxiliary_path`
- Extended Phase 2 formatting work:
  - Added `OutputFormat::{Readable, Compact}` and `format_context_with_mode(...)`
  - Added compact-mode support in tool params (`GetContextTool.format`) and pipeline wiring (`run_pipeline` + handler path)
  - Compact rendering now respects `NeighborMode` (includes/omits signatures/docs accordingly)
  - Added caller/callee deduplication in both readable and compact outputs
- Updated agent-facing instructions:
  - Added `format` parameter docs in `JULIE_AGENT_INSTRUCTIONS.md`

### WHY
- Phase 1 needed one final relevance boundary: production `src/` code should outrank demo/bench/script code by default.
- Phase 2 required compact output to be semantically faithful (not just shorter), plus explicit documented usage for agents.

### HOW
- Added failing test for auxiliary-path ranking, then implemented de-boost and verified.
- Added formatting tests for:
  - dedup behavior
  - compact structure
  - compact neighbor verbosity in `NameAndLocation`
  - compact output size vs readable output
- Added pipeline integration coverage for `format="compact"`.

### IMPACT
- `get_context` now has stronger code-first relevance defaults (including auxiliary path boundary cases).
- Compact mode is production-usable for token-sensitive agent loops and aligns with allocation neighbor semantics.
- Documentation now exposes compact mode to agent users.
