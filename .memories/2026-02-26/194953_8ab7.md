## Fixed definition search over-fetch and exact-name promotion

### WHAT
Added 5x over-fetch + stable-partition promotion for `search_target="definitions"` in `text_search_impl()`.

### WHY
Actual definitions (mentioned once at definition site) ranked below references (mentioned many times across consumers) in Tantivy BM25 scoring. With small limits like `limit=5`, the actual definition could fall outside results entirely.

### HOW
1. **Over-fetch**: Changed `tantivy_limit` from `limit_usize` to `limit_usize.saturating_mul(5).max(50)` for the non-file_pattern case
2. **Promote**: Added `promote_exact_name_matches()` in `scoring.rs` — stable partition that moves case-insensitive exact name matches to the front
3. **Truncate**: `filtered_results.truncate(limit_usize)` after promotion
4. Applied in **both** code paths (primary workspace + reference workspace)

### IMPACT
- Fixes the bug where `fast_search("UserService", definitions, limit=5)` missed the actual class definition
- 7 unit tests for the promotion logic
- 698 fast-tier tests pass, 0 regressions
- `text_search.rs` stays at 498 lines (under 500-line limit)
