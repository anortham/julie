## Fixed title-case matching and consolidated duplicate path classifiers

### Issue 1: Case-Sensitive Matching
- **Problem:** `is_test_path`, `is_docs_path`, `is_fixture_path` only matched lowercase directory names. Swift uses `Tests/`, some projects use `Docs/`, `Fixtures/`, etc.
- **Fix:** Added title-case variants to all three match arms:
  - `is_test_path`: `"Test" | "Tests" | "Spec"`
  - `is_docs_path`: `"Docs" | "Doc" | "Documentation"`
  - `is_fixture_path`: `"Fixtures" | "Fixture" | "Snapshots"`
- **Tests:** Added `test_swift_project_layout` and `test_title_case_path_segments_detected`

### Issue 2: Duplicate Path Classifiers
- **Problem:** `src/tools/get_context/scoring.rs` had its own broken `is_test_path` (substring-based), `is_non_code_path` (`starts_with("docs/")`), and `is_auxiliary_path` (`starts_with` for fixtures)
- **Fix:** 
  - Removed local `is_test_path` definition, imported from `crate::search::scoring`
  - Rewrote `is_non_code_path` to use canonical `is_docs_path` + kept unique file-extension checks
  - Rewrote `is_auxiliary_path` to use canonical `is_fixture_path` + kept unique examples/benches/scripts checks
  - Updated `pipeline.rs` to import `is_test_path` from `crate::search::scoring`
  - Added `test-data` (hyphen variant) to canonical `is_fixture_path` to preserve old coverage
