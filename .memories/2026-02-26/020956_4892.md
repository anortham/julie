## Live testing revealed fixture/memories indexing gaps — fixed both

### Live test results
Ran `get_context` through MCP against real Julie index. 4 queries tested:
- **"get_context pipeline"** → Excellent (run_pipeline with full call graph)
- **"workspace routing"** → Bad (docs-only, no code in search results)
- **"symbol extraction"** → Bad (all test files, fallback can't help)
- **"search scoring"** → Mixed (real struct + bash fixture leak)

### Bug found: `fixtures/` paths unpenalized
`is_auxiliary_path` didn't include `fixtures/`, `fixture/`, `testdata/`, `test-data/`, `test_data/`. Added all to scoring penalty.

### `.memories/` excluded from indexing
Added `.memories` to:
1. `BLACKLISTED_DIRECTORIES` in `shared.rs` (primary indexing gate)
2. `build_ignore_patterns` in `watcher/filtering.rs` (watcher layer)

Goldfish now owns memory files exclusively — Julie won't index or surface them.

### Dogfood suite
43/43 `search_quality` tests passed (253s). No regressions.

### Remaining limitation
The code-first fallback can't help when the search index itself returns no code results. Queries like "workspace routing" match doc headings better than code symbols. This is a search-layer issue, not solvable at the scoring layer.
