## Task 2 Complete: KNN-to-SymbolSearchResult Conversion

Implemented `knn_to_search_results` function that bridges sqlite-vec KNN output to the `SymbolSearchResult` type used by the existing search pipeline.

- **What**: `knn_to_search_results(knn_results, db)` in `src/search/hybrid.rs`
- **How**: Batch-fetches symbols via `db.get_symbols_by_ids()`, builds HashMap for O(1) lookup, iterates KNN results in order, converts `(id, distance)` → `SymbolSearchResult` with `score = (1.0 - distance) as f32`
- **Edge cases**: Empty input returns empty vec; missing symbol IDs silently skipped (expected during incremental updates)
- **Tests**: 3 TDD tests in `conversion_tests` module — converts correctly, skips missing, handles empty
- **No regressions**: 751 fast-tier tests pass
