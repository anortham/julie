## Completed code-quality review for Task 2 search expansion changes

Reviewed `src/search/expansion.rs` and `src/search/mod.rs` for determinism, correctness, maintainability, heuristic brittleness, and regression risk.

- **Strengths:** Deterministic insertion order preserved, dedupe behavior explicit, cap enforcement deterministic, module exposure minimal and low-risk.
- **Key issue found:** `normalize_term()` uses `trim_end_matches("ing")` and `trim_end_matches('s')`, which can over-strip (`"process" -> "proce"`, `"thing" -> "th"`, `"class" -> "cla"`) and inject noisy query terms.
- **Impact:** Risk of relevance regression via malformed normalized expansions, especially for common `-ing` and trailing-`s` words.
- **Evidence:** Existing Task 2 tests pass but do not cover over-stripping edge cases.

### Recommended required fix
Replace aggressive `trim_end_matches` stemming with conservative single-suffix stripping (`strip_suffix`) plus guardrails (minimum stem length and avoid plural stripping for `ss` endings), and add regression tests for words like `process`, `class`, `thing`, `routing`.
