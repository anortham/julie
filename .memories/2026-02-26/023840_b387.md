## Fixed Task 2 query normalization overreach with conservative suffix stripping

Implemented the Task 2 code-quality fix in deterministic query expansion and added regression tests for noisy edge cases.

- **What changed:**
  - Replaced `trim_end_matches`-based normalization in `normalize_term()` with single-suffix stripping via `strip_suffix`.
  - Added stricter, conservative guards:
    - `"ing"` stripping only when the remaining stem length is at least 5.
    - `'s'` stripping only when stem length is at least 4 and the source term does **not** end with `"ss"`.
  - Added edge-case tests for `thing`, `process`, `class`, and `routing` to block malformed stems and noisy normalization output.

- **Why it matters:**
  - `trim_end_matches` was removing repeated trailing characters and over-normalizing nouns, producing malformed stems (`th`, `proce`, `cla`, `rout`).
  - This polluted deterministic term expansion and risked noisier downstream query behavior.
