## Fixed C# Extractor Constructor DI Relationship Gaps

### WHAT
Added `extract_constructor_parameter_relationships` to the C# extractor to capture DI (Dependency Injection) constructor parameter types as `Uses` relationships.

### WHY
C# classes like `LuceneIndexService` had `ref_score: 0` because the extractor didn't capture constructor injection patterns. In C#/.NET, DI constructor injection is THE primary wiring mechanism — nearly every service class takes dependencies as constructor parameters. Without these relationships, Julie missed the most important connections in C# codebases.

### HOW
- Added `constructor_declaration` match arm to `visit_relationships` in `relationships.rs`
- Implemented `extract_constructor_parameter_relationships` function that:
  - Walks up the tree to find the containing class/struct/record
  - Extracts parameter types from the constructor's `parameter_list`
  - Creates `Uses` relationships from the class to each parameter type
  - Handles generic types (`ILogger<T>` -> `ILogger`), nullable types (`ICache?` -> `ICache`)
  - Skips predefined types (`string`, `int`, `bool` etc.)
  - Creates `PendingRelationship` for cross-file types not defined locally
- Used two-phase approach to satisfy borrow checker (collect data first, then apply mutations)
- Added `extract_parameter_type_name` helper for robust type name extraction

### IMPACT
- C# services with DI constructors will now have proper centrality scores
- 7 new tests covering: basic DI, generics, predefined type skipping, nullable types, cross-file types, multiple constructors, confidence levels
- All 39 C# tests pass, 698 fast-tier tests pass with 0 failures
- Also fixed pre-existing `test_utils.rs` missing file that was blocking extractor test compilation
