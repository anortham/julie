## Phase 0 Complete: All 3 Bug Fixes Implemented and Reviewed

All three Phase 0 tasks from the semantic embedding layer design doc are implemented, spec-reviewed, code-quality-reviewed, and passing.

### Task 1: Language-Agnostic NL Path Prior
- **What:** Replaced Rust-specific `starts_with("src/")` path matching with segment-based classifiers (`is_test_path`, `is_docs_path`, `is_fixture_path`)
- **Impact:** NL path prior now works for C#, Python, Java, Go, JS/TS, Ruby, Swift — not just Rust
- **Also:** Consolidated duplicate `is_test_path` in `get_context/scoring.rs` to use canonical implementation
- **Files:** `src/search/scoring.rs`, `src/tools/get_context/scoring.rs`, `src/tools/get_context/pipeline.rs`

### Task 2: Definition Search Over-Fetch and Promotion
- **What:** Added 5x over-fetch (min 50) for definition search, stable partition to promote exact name matches, truncate to user limit
- **Impact:** Searching `"UserService"` with limit=10 now returns the actual definition at the top, not just references
- **Also:** Fixed case-sensitivity mismatch between scoring and formatting layers
- **Files:** `src/tools/search/text_search.rs`, `src/search/scoring.rs`, `src/tools/search/formatting.rs`

### Task 3: C# Constructor DI Relationship Extraction
- **What:** Added `extract_constructor_parameter_relationships` to C# extractor — creates `Uses` relationships from classes to constructor parameter types
- **Impact:** C# services using DI injection now get proper centrality scores (previously `ref_score: 0`)
- **Also:** Handles generic types, nullable types, tuple type skip, deduplication across constructor overloads
- **Files:** `crates/julie-extractors/src/csharp/relationships.rs`, extractor test utils
