## Released v3.3.3 — NL query expansion, weighted builders, path prior

Reviewed and committed GPT-authored search-layer NL relevance changes after deep-dive verification.

### What shipped
- **Query expansion** (`src/search/expansion.rs`): deterministic alias/normalization for NL queries
- **Weighted query builders** (`src/search/query.rs`): original/alias/normalized term groups with tiered boost weights (5.0/3.5/2.5)
- **NL path prior** (`src/search/scoring.rs`): conservative src-boost (1.08) / docs-tests-penalty (0.95) reranking with 4x overfetch
- Regression tests for identifier stability, path prior behavior, query expansion, and query weighting
- Updated workspace discovery test to match `.memories` blacklisting
- Development memories and NL relevance design/impl plan docs

### Review findings
- Initially flagged 4 issues; deep-dive verification showed all were overstated or non-issues
- Duplicate `is_nl_like_query` functions have different strictness levels appropriate to their contexts
- "Double stemming" concern was unfounded — normalization adds useful intermediate forms
- Code is architecturally sound
