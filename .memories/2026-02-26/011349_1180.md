## Phase 1 continued: added fallback safety test + deterministic pivot tie-breaking

### WHAT
- Expanded Phase 1 relevance tests in `src/tests/tools/get_context_relevance_tests.rs`:
  - Guardrail fires when initial pivots are non-code-only.
  - Guardrail does **not** override primary ranking when selected pivots already include actionable code.
  - Equal-score pivot ordering is deterministic.
- Added deterministic tie-breaking in `src/tools/get_context/scoring.rs`:
  - Sort now uses `compare_pivots` (score desc, then actionable-code preference, then lexical stability by name/path/id).

### WHY
- Needed to prevent ranking jitter for equal/near-equal scores and ensure fallback behavior is precise (fix bad cases without harming good cases).

### HOW
- Introduced `compare_pivots` and switched sort from inline score-only comparator to this stable comparator.
- Added tests that model mixed doc+code selection and equal-score ordering.
- Tuned mixed-case test with high centrality doc score to keep scenario realistic under existing penalties.

### IMPACT
- `get_context` pivot ordering is now stable across equal-score scenarios.
- Fallback logic remains targeted: it rescues docs-only pivot sets, but does not force code-only re-ranking if at least one actionable pivot already exists.
- Phase 1 behavior now has stronger regression protection.
