## Fixed three issues in GPT's get_context changes

Reviewed and cleaned up GPT-authored changes to the get_context pipeline.

### Fixes applied:
- **`format_neighbor_compact`**: `SignatureAndDoc` and `SignatureOnly` arms were identical — now `SignatureAndDoc` appends `doc="..."` inline, matching the readable mode's behavior
- **`select_pivots_with_code_fallback`**: Restructured to avoid cloning the full results vec in the common case (all-code results). Mixed-corpus path only clones the code candidate subset
- **`build_pivot_entries`**: Extracted batch data fetching into `PivotBatchData` struct + `fetch_pivot_batch_data()` helper, separating the "fetch everything" concern from the per-pivot entry construction loop
- **Token savings threshold**: Lowered from 20% to 15% since compact mode now correctly includes doc summaries

### Impact
- All 656 fast-tier tests pass, no regressions
- All files under 500-line limit (pipeline.rs at 496)
