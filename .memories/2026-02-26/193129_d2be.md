## Made NL path prior language-agnostic

Replaced Rust-specific path matching in `apply_nl_path_prior()` with language-agnostic heuristics that work across all project layouts.

### What changed
- **`src/search/scoring.rs`**: Replaced hardcoded `src/`, `src/tests/`, `docs/`, `fixtures/` checks with three new `pub(crate)` helper functions:
  - `is_test_path()` — detects test dirs (`test/`, `tests/`, `spec/`, `__tests__`, `.Tests`) and test file patterns (`*_test.go`, `*.test.ts`, `*.spec.js`, `test_*.py`)
  - `is_docs_path()` — detects `docs/`, `doc/`, `documentation/` path segments
  - `is_fixture_path()` — detects `fixtures/`, `testdata/`, `test_data/`, `__fixtures__/`, `snapshots/`, `__snapshots__/` segments
- Anything NOT classified as test/docs/fixtures now gets the source boost (was previously only `src/`)
- Same constants (1.08 boost, 0.95 penalties) preserved

### Tests added (11 new, 20 total)
- C#, Python, Java, Go, JS/TS, Ruby layout tests
- Generic docs and fixtures penalization tests
- Direct `is_test_path`, `is_docs_path`, `is_fixture_path` verification
- All 9 existing tests still pass
