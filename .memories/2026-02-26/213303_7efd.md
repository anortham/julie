## Phase 1 Batch 1 Complete: Tasks 1-3

Foundation layer for semantic embeddings is in place.

### Task 1: Dependency Validation
- Added `fastembed = "5"`, `sqlite-vec = "0.1"`, `zerocopy = { version = "0.7", features = ["derive"] }` to Cargo.toml
- All deps compile cleanly with rusqlite 0.37 bundled
- **Key finding:** fastembed's `embed()` takes `&mut self` — confirmed need for `Mutex` wrapper
- **Key finding:** fastembed uses relative `.fastembed_cache/` CWD — must use absolute `~/.cache/fastembed/` in production and tests
- **Key finding:** fastembed `InitOptions` is `#[non_exhaustive]` — use builder pattern `InitOptions::new(model).with_cache_dir().with_show_download_progress()`
- 4 smoke tests + `#[serial(fastembed)]` to avoid ONNX runtime download race

### Task 2: EmbeddingProvider Trait + OrtEmbeddingProvider
- `src/embeddings/mod.rs` — trait with `embed_query`, `embed_batch`, `dimensions`, `device_info`
- `src/embeddings/ort_provider.rs` — fastembed wrapper with `Mutex<TextEmbedding>`
- Trait uses `&self` with interior mutability (Mutex) for `Send + Sync`
- 6 tests including semantic similarity sanity check

### Task 3: sqlite-vec + Migration 010 + Vector CRUD
- Registered sqlite-vec via `Once` + `sqlite3_auto_extension` in `SymbolDatabase::new()`
- Migration 010: `CREATE VIRTUAL TABLE symbol_vectors USING vec0(symbol_id TEXT PRIMARY KEY, embedding float[384])`
- `src/database/vectors.rs` — store, delete_for_file, knn_search, count, clear_all
- **Key finding:** vec0 virtual tables don't support `INSERT OR REPLACE` — use DELETE then INSERT
- 11 tests all passing
