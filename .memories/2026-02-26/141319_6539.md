## Completed Task 4 weighted query builders with spec + quality approval

Implemented and validated the weighted query-builder layer for deterministic NL expansion support.

- **What:**
  - Added weighted builders in `src/search/query.rs`:
    - `build_symbol_query_weighted(...)`
    - `build_content_query_weighted(...)`
  - Preserved compatibility wrappers (`build_symbol_query`, `build_content_query`) by delegating through weighted builders with empty expansion groups.
  - Updated weighting tests in `src/tests/tools/search/tantivy_query_weighting_tests.rs` to verify behavior with real in-memory Tantivy queries.
- **Why:**
  - Provide the query-construction primitives needed for later SearchIndex integration while keeping existing callers stable.
- **How:**
  - Enforced strict matching on original groups in AND mode, while alias/normalized groups remain optional boost-only SHOULD clauses.
  - Kept `doc_type` and language/kind constraints as MUST clauses.
  - Removed ad-hoc score shim and kept deterministic weight constants.
- **Impact:**
  - Task 4 is green and approved; codebase now has production-ready weighted query primitives for Task 7 integration.
