## Task 3 Complete: hybrid_search Orchestrator Function

Implemented `hybrid_search` orchestrator in `src/search/hybrid.rs` that composes Tantivy keyword search + KNN semantic search into a single entry point.

- **What**: `hybrid_search(query, filter, limit, search_index, db, embedding_provider)` function
- **How**: Runs Tantivy keyword search, then optionally runs semantic search (embed → KNN → convert), merges via `rrf_merge(k=60)`. Internal `run_semantic_search` helper isolates the semantic pipeline so errors are caught in a single match.
- **Graceful degradation**: 
  - `None` provider → returns Tantivy results unchanged
  - Any semantic step failure → `debug!()` log + empty semantic results → RRF falls through to keyword-only
- **Over-fetch**: 2x limit from both sources when provider is present
- **Relaxed flag**: Preserved from tantivy_results (AND→OR fallback indicator)
