## Cross-workspace dogfood testing — 8 issues documented

Systematically tested `fast_search`, `get_context`, `deep_dive`, `fast_refs`, and `get_symbols` across all 3 workspaces (julie/Rust, coa-codesearch-mcp/.NET, miller/Python+Rust).

### Key findings
- **`get_context` on coa is broken** — NL queries return test files, stub methods, and `Program.cs` god object instead of relevant implementation code
- **Definition promotion fails on reference workspaces** — `fast_search` definitions mode doesn't show "Definition found:" for C# classes, and promotes Python imports over actual class definitions
- **`fast_refs` misclassifies imports** — Python imports shown as "Definitions" (122 imports + 1 real definition = "Definitions (123)")
- **`get_context` on miller is excellent** — consistently returns relevant pivots with rich neighbor context
- **`deep_dive` and `get_symbols` work great** across all 3 workspaces

### Root cause hypotheses
1. NL path prior may not apply to reference workspace queries (or coa has weak centrality data)
2. `Program.cs` acts as a "god object" matching every query — needs deprioritization
3. Definition promotion logic may be primary-workspace-only or language-specific
