## Embedding Input Enrichment: Implementation Complete, Pending Live Verification

### What
Implemented three changes to enrich the text fed to embedding models for better semantic search quality:

1. **Multi-sentence doc extraction** (`extract_doc_excerpt`): New function that collects multiple doc comment lines (up to 300 chars) instead of just the first sentence. `first_sentence()` preserved for display use in `get_context`.
2. **Callee name enrichment**: Functions and methods now get `" calls: insert_tool_call, get_total_file_sizes"` appended to their embedding text, mirroring the existing child enrichment for containers. Built via `build_callee_map` helper that queries `get_outgoing_relationships_for_symbols`.
3. **MAX_METADATA_CHARS raised from 600 to 1200**: Gives room for expanded docs + callees. Still safe for BGE-small (~300 tokens of 512 max).
4. **Force-re-embed fix**: Added `Enum` to the container list (pre-existing bug), scoped function re-embedding to only those with callees (avoids re-embedding 70% of symbols).

### Why
Dogfooding revealed that `get_context(query="metrics save database persist")` completely missed `record_tool_call` because:
- `first_sentence()` truncated "Record a completed tool call. Bumps in-memory atomics synchronously, then spawns async task for source_bytes lookup + SQLite write." to just "Record a completed tool call."
- No callee info meant "insert_tool_call" and "get_total_file_sizes" were invisible to the embedding model

### How
- TDD with subagent-driven development (3 implementation tasks dispatched to subagents, spec + quality review between each)
- Plan reviewed by code-reviewer agent, caught 3 critical issues (first_sentence display caller, force-re-embed perf, cross-file callee resolution)
