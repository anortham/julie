## Field Access Enrichment for Embedding Search Quality

### What
Added member_access field names (e.g., `self.session_metrics`, `this.db`) to embedding input text for functions/methods. Also fixed force re-index to clear embeddings so format changes take effect immediately.

### Why
`get_context("metrics save database persist")` couldn't find `record_tool_call` because there was **zero vocabulary overlap** between the query and the embedding text. The function uses "record", "SQLite write", "tool call" while the query says "save", "database", "metrics". Code embedding models match tokens, not semantic concepts across synonym boundaries.

### How
- **DB layer**: Added `get_member_access_identifiers_grouped()` to `identifiers.rs` (mirrors existing `get_call_identifiers_grouped()`)
- **Pipeline**: Added `build_field_access_map()` in `pipeline.rs`, passed to `prepare_batch_for_embedding()`
- **Metadata**: `prepare_batch_for_embedding()` now appends `" fields: session_metrics, db, output_bytes"` for functions/methods
- **Force re-embed**: `handle_index_command` and `handle_refresh_command` now call `clear_all_embeddings()` when `force=true`
- **Enriched skip logic**: Functions with field accesses are included in `enriched_ids` so they get re-embedded on incremental runs
- Language-agnostic: `member_access` identifiers are extracted across all 33 languages (verified Rust, C#, TypeScript, Vue)
