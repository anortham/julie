## Session Summary: Model-Switch Fix + Plugin Distribution Research

### Bug Fix: Embedding model name change now wipes vectors
- Previously, switching between models with same dimensions left stale vectors in place (silently wrong KNN results)
- Merged the dimension-change and model-name-change branches in `run_embedding_pipeline` so any mismatch wipes and re-embeds
- TDD: verified RED without fix, GREEN with fix. Dev tier all green (8 buckets, 332s)
- Updated README and sidecar docs to document automatic re-embedding on model switch

### Per-Language Config Benchmark
- Re-embedded all 19 workspaces with BGE-small + per-language TOML configs
- Coverage increased 30-136% for affected languages (Zod +136%, JS +58%, C++ +52%, Ruby +41%)
- Quality held steady (avg top similarity 0.929 unchanged, diversity +0.002)
- Results saved to `benchmarks/results/bge-small-per-lang-configs.*`

### Claude Code Plugin Distribution Research
- **Conclusion**: Viable via separate `julie-plugin` repo bundling pre-built binaries
- **Key constraint**: MCP servers connect before SessionStart hooks, so binary download at runtime won't work. Must bundle binaries.
- **Architecture**: Platform binaries in `bin/`, launcher script for platform detection, sidecar bundled (484KB), skills bundled, hooks for auto-index
- **Pattern**: Force-push releases to keep repo lean (~100MB). Goldfish plugin at `~/source/goldfish` is the working reference.
- **Not a replacement**: Manual installation path stays for non-Claude-Code users
- **Open questions**: Windows launcher (bash not guaranteed), whether to create as marketplace or standalone repo
