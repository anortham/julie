## Cross-repo Julie improvement audit

- **WHAT:** Compared live Julie, Miller, Miller's semantic integration worktree, and `julie-semantic-sidecar` to identify transferable lessons and current friction.
- **WHY:** Miller's lexical engine is fast but misses natural-language intent; Julie has stronger semantic retrieval but its development loop and default routing make progress and discovery worse than they should be.
- **HOW:** Audited repository state, search/query routing, test orchestration, immutable index publication, telemetry, semantic benchmarks, and both sidecar approaches.
- **IMPACT:** The highest-value first slice is to make Julie's test control plane genuinely fast. Separately, semantic retrieval is already proven valuable; the next search-quality slice should change automatic query routing in shadow mode before replacing the current runtime. The Rust sidecar is a promising protocol/packaging alternative but is not yet a proven replacement.
