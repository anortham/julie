## Commit 5: Codex review follow-up — all 3 findings closed

After committing 1–4 and invoking `/codex-cli` for adversarial review, Codex returned 3 material findings. All three were valid; validated each against the code and fixed them with TDD (RED → GREEN cycles).

### Finding #1 (HIGH): adapter retry window too short during upgrade
- **Bug:** `RejectAndFlagForRestart` only set `restart_pending`, daemon kept advertising `ready`. Adapter's 3 retries × 2 s sleep = ~6 s fail against daemon still serving old sessions.
- **Fix:** Extracted `flag_restart_pending_after_version_reject()` which (on first rejection) transitions `daemon_state` to `"stopping"`. Adapter's existing `ensure_daemon_ready` handles `Stopping` via `wait_for_pid_exit` (60 s deadline). New adapters now wait for old daemon to drain instead of bouncing off.
- 2 unit tests in `src/tests/daemon/state.rs`.

### Finding #2 (HIGH): Commit 2 `add` fix was bypassed on real MCP path
- **Bug:** `manage_workspace_request_targets_primary` classified `"add" | "list" | "remove"` as primary-targeting, so preflight ran before tool body and silently bound startup-hint as primary. My Commit 2 actionable error for `add` was unreachable on the real RMCP path.
- **Fix:** Removed `"add"` from the classification. Kept `list`/`remove` (they use preflight for label resolution, 25+ roots tests depend on this). Added regression test via `ServerHandler::call_tool` + `serve_directly` that fires in deferred Cwd session.

### Finding #3 (MEDIUM): `.julieignore` fix was too narrow
- **Bug:** Indexer accepts ALL extensionless text files (`Dockerfile`, `Makefile`, `Containerfile`, etc.) but `scan_workspace_files` rejected them. Every repo with a Dockerfile got phantom "deleted file" on reconnect.
- **Fix:** Extracted `is_likely_text_file` to `src/utils/mod.rs::file_utils` as shared utility. `is_code_file` now uses it for extensionless files + applies `BLACKLISTED_FILENAMES`. Discovery's private method delegates to the same function (lockstep).
- New test with `Dockerfile` + `Makefile` + `.rs` files.
