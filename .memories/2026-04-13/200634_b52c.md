## Fixed startup-time `roots/list` hang in weak `cwd` sessions

- **WHAT:** Removed request-time client roots probing from `JulieServerHandler::on_initialized()` and locked the new behavior down with daemon roots regression tests.
- **WHY:** Weak `cwd` startup sessions were calling `roots/list` during `notifications/initialized`, which could stall Claude Code during handshake and leave the server connected without capabilities or tools.
- **HOW:** `on_initialized()` now records deferred auto-index intent and returns without querying the client. Primary roots resolution stays request-bound via `ensure_primary_workspace_for_request()`. Updated daemon roots tests to reflect the new contract: no init-time probe, first primary-scoped request performs roots resolution, and first-request roots failures fall back to startup `cwd` because there is no initialized roots snapshot anymore.
- **IMPACT:** Removes the startup deadlock path, keeps primary binding aligned with the workspace-roots design spec, and preserves coverage across startup, request-time bind, dirty-roots refresh, and reference-scoped first-request behavior.
