## Commit 3: un-ignore fresh-index test + fix `.julieignore` scan/index asymmetry

Un-`#[ignore]`d `test_fresh_index_no_reindex_needed` (Finding #17 from `ROOTS_IMPL_REVIEW_NOTES.md`), discovered and fixed an unrelated pre-existing bug in the process.

### What
- **Test fix:** Replaced the 10ms sleep with explicit `File::set_modified()` call (stable since Rust 1.75) that backdates the source file's mtime by 10s after indexing. Eliminates dependence on sub-second FS clock resolution.
- **Real bug found:** Diagnostic instrumentation revealed the test wasn't failing due to mtime resolution at all — it was failing because `.julieignore` (auto-generated by discovery) was stored in the symbols DB as a tracked file, while `scan_workspace_files` filtered it out (extension check rejects extensionless files). Every freshly indexed workspace had `indexed_files = {.julieignore, ...}` but `scanned = {...}`, so the "deleted files" diff always flagged a phantom deletion.
- **Fix:** Added `.julieignore` to `BLACKLISTED_FILENAMES` in `src/tools/shared.rs`. Both sides now agree Julie's own config is not code.

### Why
- Test was the load-bearing happy-path ("fresh index → don't reindex"). Every other test in the file asserts the TRUE branch; shipping the FALSE branch untested meant a regression flipping `check_if_indexing_needed` to always return `true` would pass CI silently.
- The `.julieignore` asymmetry was a latent daemon-mode bug: on every reconnection, `check_if_indexing_needed` returned `true`, triggering unnecessary re-indexing of untouched workspaces.

### Impact
- 11/11 tests in `tests::integration::stale_index_detection` pass; 33/33 tests in `tests::tools::workspace::discovery` pass.
- Full `cargo xtask test dev` green: 9 buckets, 180 daemon tests pass, 331.8s.
- Daemon mode no longer does phantom re-indexing on reconnection.
