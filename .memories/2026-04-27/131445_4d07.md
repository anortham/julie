## Julie logging timestamp audit

I corrected the scope from Hermes to Julie and audited the Julie codebase for user-visible timestamp handling in logs and dashboard views.

- **What:** Verified that the main per-project log writer already uses local time (`chrono::Local::now()`), and found two dashboard display sites that turn Unix timestamps into human-readable strings without an explicit local conversion.
- **Why:** Murphy asked how much work it would take to switch Julie's logging timestamps from UTC to local time, and the first pass had analyzed the wrong repository.
- **How:** Inspected `src/daemon/project_log.rs`, `src/main.rs`, `src/dashboard/routes/projects.rs`, `src/dashboard/routes/events.rs`, and the related templates/tests. Searched for `Utc::now`, `Local::now`, `DateTime::from_timestamp`, and tracing subscriber timestamp configuration.
- **Impact:** The change scope appears small: the primary project log is already local, so the real work is concentrated in the dashboard cleanup/last-indexed formatting path rather than the logging pipeline itself. Existing dashboard tests do not assert the timestamp string format, so the refactor should be low risk.
