## Switched daemon logs from UTC to local time

Daemon logs used `tracing_appender::rolling::daily` which writes UTC timestamps and rotates files at UTC midnight. For a local MCP server, this made debugging confusing.

- **New module:** `src/logging.rs` with `LocalTimer` (implements `FormatTime` via `chrono::Local`) and `LocalRollingWriter` (implements `Write` with local-date file rotation)
- **Wiring:** `src/main.rs` daemon branch now uses `LocalRollingWriter` + `.with_timer(LocalTimer)` instead of `rolling::daily`
- **Codex review fix:** Rotation failure no longer advances `current_date`, so the old file stays open as fallback and the next write retries rotation
- **Consistency:** Both logging paths (daemon tracing + per-project `ProjectLog`) now use `chrono::Local`
- **7 tests** covering timer format, trait satisfaction, file creation, appending, directory creation, date rotation, and rotation-failure fallback
