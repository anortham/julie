## Fixed infinite repair retry loop in daemon watcher

The daemon's `retry_persisted_repairs` mechanism was dispatching 5 `.ogg` files from the sealab workspace every 1-second cycle, producing 293K log lines/day (56MB), and growing daemon memory to 1.2GB.

### Root cause

- `.ogg` files leaked into the sealab database during initial indexing (missing from `BLACKLISTED_EXTENSIONS`)
- Extraction failed with "stream did not contain valid UTF-8", persisting repair entries
- On retry, the handler's Blake3 hash matched (file unchanged), returning early at line 111
- The early return bypassed the `clear_indexing_repair` call at line 238
- The repair entry's `updated_at` never changed, so the 30s age check always passed
- Result: infinite 1-second retry loop

### Three-layer fix

1. **Handler** (`handlers.rs:113`): Hash-match early return now clears stale repairs with `?` propagation
2. **Retry filter** (`runtime.rs:219`): `retry_persisted_repairs` checks `supported_extensions` before dispatch; extensionless files (Dockerfile, Makefile) pass through
3. **Blacklist** (`shared.rs`): Added ~30 missing binary formats (`.ogg`, `.flac`, `.opus`, `.heic`, `.wasm`, `.zst`, etc.)

### Cleanup

- Removed 5 stale `.ogg` repair entries and file records from sealab_ab2cd5a6 database
- Codex review caught two valid issues: swallowed errors and extensionless file handling, both fixed
