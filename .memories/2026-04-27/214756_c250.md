## Reviewed and fixed spillover paging

- **WHAT:** Audited `spillover_get`, its store, producers, public marker text, and tests. Fixed default format handling, explicit format validation, and paging call hints.
- **WHY:** Spillover pages were useful only if agents noticed opaque handles and trusted the follow-up output. The old flow could switch compact pages to readable on the next page, which made pagination feel inconsistent.
- **HOW:** Added failing tests for compact format preservation, unknown format rejection, and visible call hints. Updated the pager to preserve stored format by default, parse explicit formats strictly, and reuse one marker helper across `get_context`, `blast_radius`, and `spillover_get`.
- **IMPACT:** Agents now see an executable `spillover_get(...)` hint when rows are paged, and follow-up pages retain the producer format unless a caller overrides it.
