## Phase 1: Daemon + HTTP Foundation — Complete

All 10 tasks of Phase 1 implemented and reviewed via subagent-driven development.

### What was built
Julie transformed from a per-session stdio MCP server into a **persistent daemon** with:

- **CLI** (`src/cli.rs`) — clap-based with `daemon start|stop|status` subcommands
- **Daemon lifecycle** (`src/daemon.rs`) — PID file, signal handling, foreground mode
- **HTTP server** (`src/server.rs`) — axum with tower middleware, CORS
- **MCP Streamable HTTP** (`src/mcp_http.rs`) — rmcp StreamableHttpService at `/mcp` + per-workspace `/mcp/{workspace_id}`
- **Global registry** (`src/registry/`) — `~/.julie/registry.toml` with CRUD, atomic persistence
- **Multi-project workspace loading** (`src/daemon_state.rs`) — DaemonState with workspace map, lazy loading
- **Vue + PrimeVue web UI** (`ui/`, `src/ui.rs`) — embedded via rust-embed, Dashboard + Projects views
- **File watching** (`src/daemon_watcher.rs`) — per-project notify watchers, debounced, gitignore-aware
- **Background indexing** (`src/daemon_indexer.rs`) — mpsc channel, sequential processing, status transitions
- **Integration tests** (`src/tests/daemon_integration_tests.rs`) — 18 end-to-end tests

### Key stats
- **1034 lib tests pass** (up from 953 at session start)
- **15 commits** on `feature/julie-platform` branch
- **~2500 lines of new Rust code** + Vue UI scaffold
- Every task had code quality review, issues fixed before proceeding

### Architecture decisions
- Per-workspace MCP endpoints (`/mcp/{workspace_id}`) for multi-project support
- `RegisterResult` enum for conflict detection without duplicated logic
- `remove_workspace` made async to avoid watcher race conditions
- Indexing worker uses `tokio::select!` for shutdown-aware processing
- Duplicate indexing guard (409 Conflict) on trigger endpoint
