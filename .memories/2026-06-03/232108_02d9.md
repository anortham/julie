## T6: Prune stale top-crate bucket refs after T3 relocation

### WHAT
Removed 9 stale test bucket commands from `test_tiers.toml` that targeted test modules T3 relocated into `crates/julie-index/`. Also removed the fully-stale `analysis` top-crate bucket. Updated all 4 lockstep files in sync.

### WHY
After T3 relocated tests into `julie-index`, the top-crate buckets still had commands filtering for modules that no longer exist there. nextest exits with code 4 ("no tests to run") on those filters, breaking bucket execution. The `analysis` bucket was entirely empty — all analysis tests moved to `crates/julie-index/`.

### HOW
**Pruned from `test_tiers.toml`:**
- Removed `[buckets.analysis]` section entirely
- Removed `analysis` from `dev` and `full` tiers
- Removed `language_config_embedded_tests` from `tools-search-tantivy`
- Removed `reranker_tests` from `tools-search-query`
- Removed 6 stale lines from `tools-search-unified`: `c3_enriched_schema_tests`, `compat_marker_v4_test`, `projection_search_doc_test`, `reranker_ordering_tests`, `schema_phase2_fields_test`, `tokenizer_simple_test`

**Updated 4 lockstep files:**
- `manifest_contract_expected.rs`: removed `analysis` from `expected_buckets()` + `expected_bucket_metadata()`; pruned stale commands from tantivy/query/unified entries
- `manifest_contract_tests.rs`: removed `"analysis"` from dev + full tiers in `expected_tiers()`
- `changed.rs`: updated `crates/julie-index/src/analysis/` routing to `&["core-index"]` only; removed `src/analysis/` + `src/tests/analysis/` routing block; removed `"analysis"` from `sort_bucket_names`
- `changed_tests.rs`: removed `src/analysis/symbol_quality.rs` assertion; updated `early_warnings.rs` assertion to `vec!["core-index"]`; removed dead `changed_tests_analysis_paths_route_to_analysis_bucket` test; removed `[buckets.analysis]` from `sample_manifest()`
