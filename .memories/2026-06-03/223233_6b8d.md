## T2 Complete: julie-index crate extracted — search+analysis merged

### WHAT
Extracted a new `crates/julie-index` crate containing the full `search` and `analysis` modules, plus the `languages/` TOML configs that `search/language_config.rs` embeds at compile time.

**Moved via git mv:**
- `src/search/` → `crates/julie-index/src/search/`
- `src/analysis/` → `crates/julie-index/src/analysis/`
- `languages/` → `crates/julie-index/languages/`

### WHY
Phase 1 of Julie Rescue: extracting the search+analysis layer into a sub-crate so the top-level crate doesn't need to relink everything when only search/analysis code changes.

### HOW

**Scaffolding:**
- Created `crates/julie-index/Cargo.toml` with deps: julie-core, julie-extractors (same pin), tantivy, tracing, anyhow, thiserror, serde, serde_json, toml, rusqlite, regex, fs2 (+ tempfile in dev-deps)
- Created `crates/julie-index/src/lib.rs` with `pub mod search; pub mod analysis;`
- Added `crates/julie-index` to workspace members in root `Cargo.toml`
- Added `julie-index = { path = "crates/julie-index" }` dep to top-level `Cargo.toml`

**Import repointing (via Python bulk replacement):**
- `crate::database::*` → `julie_core::database::*`
- `crate::extractors::*` → `julie_extractors::*`
- `crate::embeddings::EmbeddingProvider` → `julie_core::embeddings_contract::EmbeddingProvider`
- `crate::tools::search::matches_glob_pattern` → `julie_core::glob::matches_glob_pattern`
- Inline `crate::extractors::*` and `crate::database::*` in index.rs

**Top-crate shims in src/lib.rs:**
- `pub mod search;` → `pub use julie_index::search;`
- `pub mod analysis;` → `pub use julie_index::analysis;`

**Visibility promotion** (pub(crate) → pub for cross-crate boundary):
- scoring.rs: CENTRALITY_NOISE_NAMES, is_test_path, is_docs_path, is_fixture_path, classify_role, test_subrole, is_nl_like_query
- index.rs: compact_alnum_lc
- projection.rs: project_documents_with_locks + all pub(crate) use re-exports
- projection/apply.rs: collect_relationship_names_bounded, collect_relationship_partner_symbol_ids, apply_uncommitted_documents_from_symbols, apply_documents_with_db, reproject_partner_symbols

**include_str! paths:** ../../languages/ resolves correctly from crates/julie-index/src/search/language_config.rs → crates/julie-index/languages/ — no edit needed.

### IMPACT
- All top-level `crate::search::*` and `crate::analysis::*` consumers work unchanged via the pub use shims
