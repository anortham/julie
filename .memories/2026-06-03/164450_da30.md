## Task 7: Pure database test slice relocated to julie-core — complete

### WHAT
Moved 70 database tests from `src/tests/core/database*` into `crates/julie-core/src/tests/`. The top-crate files were deleted and their module declarations removed from `src/tests/mod.rs`.

### WHY
Phase 0 of Julie Rescue: relocating tests into the leaf crate so editing database code only relinks `julie-core`, not the full `julie` binary.

### HOW (including the non-obvious parts)

**Dev-dep cycle rlib mismatch**: `file_info_builder().build()` returns `julie_core::database::types::FileInfo`. Using `julie-test-support` builders from WITHIN `julie-core`'s own tests causes a type mismatch because Cargo compiles `julie-core` as two distinct rlib artifacts (one for the test binary, one as `julie-test-support`'s dep). Fixed by creating a LOCAL `crates/julie-core/src/tests/helpers.rs` with the same builder API but using native `crate::` types. External-type helpers (`open_test_connection` → `rusqlite::Connection`, `symbol_builder` → `julie_extractors::Symbol`) still come from `julie-test-support` — those unify fine.

**Fixture path**: `test_bulk_store_symbols_for_existing_file_paths` used `env!("CARGO_MANIFEST_DIR")/fixtures/real-world/go/main.go`. In julie-core, CARGO_MANIFEST_DIR is `crates/julie-core/` not the workspace root. Fixed: `../../fixtures/real-world/go/main.go`.

**Import rewrites applied** (Python script):
- `crate::extractors::` → `julie_extractors::`
- `crate::tests::helpers::db::` → `julie_test_support::` (for external-type builders)
- `crate::tests::test_helpers::open_test_connection` → `julie_test_support::open_test_connection`
- `crate::language::get_tree_sitter_language` → `julie_extractors::get_tree_sitter_language`
- `crate::utils::paths::` → `crate::paths::`
- `crate::database::*` stays (we're in julie-core)

**Dev-deps added to julie-core**: `tempfile = "3.13"`, `tree-sitter = "0.26.8"`
