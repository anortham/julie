## Task 5 Complete: database module moved to julie-core

### WHAT
Moved all 37 files of `src/database/**` (including `bulk/` and `symbols/` sub-trees) into `crates/julie-core/src/database/`. Updated `src/lib.rs` to `pub use julie_core::database;` shim. Deleted original `src/database/`.

### WHY
Phase 0 of the Julie rescue — moving heavyweight code into the leaf `julie-core` crate so isolated edits to database code don't trigger a relink of the full `julie` binary.

### HOW
1. `cp -r src/database crates/julie-core/src/database/`
2. Three import rewrites via `sed` across all 37 files:
   - `crate::daemon::connection_pool::` → `crate::connection_pool::` (mod.rs only)
   - `crate::extractors::` → `julie_extractors::` (14 files)
   - `crate::utils::paths::` → `crate::paths::` (files.rs only)
3. `pub(crate)` → `pub` promotions for boundary-crossing items:
   - `mod bulk` in database/mod.rs
   - All 8 sub-mods in bulk/mod.rs
   - `SymbolDatabase.conn`, `.file_path`, `SymbolDatabaseConn` enum, `ReadSnapshot`
   - `into_read_snapshot`, `is_autocommit_for_test` (removed `#[cfg(test)]` gate — needed cross-crate)
   - All methods in repairs.rs, index_engine.rs, schema.rs
   - `CanonicalWriteSet`, `AtomicPersistenceMetadata` and their fields in write_set.rs
   - `TypeArgumentRow`, `flatten_type_argument_usages` in type_arguments.rs
   - Atomic methods: `incremental_update_atomic_with_metadata`, `bulk_store_fresh_atomic_with_metadata`, `replace_workspace_data_atomic`, `delete_single_file_atomic`
4. Added missing deps to `julie-core/Cargo.toml`: `serde`, `serde_json`, `blake3`, `md5`, `zerocopy`, `chrono`
5. `#[cfg(test)]` guard removed from `is_autocommit_for_test` — `cfg(test)` in library crates hides items from dependent crates' test builds
