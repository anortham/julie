## julie-extractors 2.0.2 migration: design hardened, ready for plan

**WHAT:** Designed the migration of julie off its vendored `crates/julie-extractors` (v1.22.0) to consume the external `anortham/julie-extractors` library crate (v2.0.2) as a git-dep. Design doc at `docs/plans/2026-06-02-julie-extractors-migration-design.md` (NOT yet committed — awaiting user review).

**WHY:** External repo is now the single source of truth for extractors; all future language/extractor work happens there. Stop double-maintaining the fork.

**HOW (decisions):**
- **Integration model = in-process library git-dep** (`{ git, tag = "v2.0.2" }`), keep julie's own DB/Tantivy/daemon/indexing. NOT the out-of-process artifact model.
- **Perf is a SEPARATE follow-up track** — verified the v2.0.x perf wins (rayon ~2x, SQLite write-path) live in `julie-extract-cli` + `julie-extract-artifact`, NOT the library. A version bump captures none of it. Track 2 = port the techniques into julie's own `src/indexing_core` (rayon, already a dep) + `src/database/bulk` (defer-FK, fold-into-INSERT, hoisted prepared stmts).

**Recon + Codex review (gpt-5.5 xhigh, adversarial) findings, all incorporated:**
- API is drop-in compatible (byte-identical defining files; same `EXTRACTION_CONTRACT_VERSION`); path→git swap touches ~0 src files.
- 🔴 **Reindex drift (Codex-caught):** upstream did NOT bump `EXTRACTION_CONTRACT_VERSION` despite changing Swift/JS/TS extraction. Must bump julie's `SEMANTIC_INDEX_ENGINE_VERSION` with `+extractor-dep=v2.0.2` to force reindex + tie engine version to the pinned tag.
- Add julie-owned `extractor-dep-integration` xtask bucket (replaces deleted `parser-upgrade` re-pin gate); route Cargo.toml/lock to it.
- Remove FULL `certify tree-sitter` surface (cli/main/lib/tests, not just impl modules).
- Fixture keep-set is broader than `fixtures/real-world/**`: also `fixtures/qml/real-world/**` + `fixtures/r/real-world/**`.
- Self-navigation into extractor internals lost (source moves to ~/.cargo/git) — accepted.
- v2.0.2 tag confirmed on remote (`61b225a`).

**Big shed:** delete in-tree crate + `extractors`/`extractor-units`/`parser-upgrade` buckets + ~2291 extractor unit tests + `fixtures/extraction/**` + cert docs — all now owned upstream.
