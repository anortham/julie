## Parallelized tree-sitter extraction; housekeeping batch landed

### WHAT
Two commits landed today completing the housekeeping + parallel extraction plan from the hermes-agent perf investigation:

- **00ab017f** `perf(indexing): close SQL hot spots, add projection observability` — bundled the in-flight 21:05 SQL fixes (battle-tested on hermes-agent), Tantivy projection split timing (#1a), markdown long-line bypass (#3), and embedding selection log clarity + named GLOBAL_VARIABLE_EMBEDDING_CAP const (#4).
- **12c8992f** `perf(extraction): parallelize tree-sitter parsing across files` — refactored `extract_index_batch` from sequential outer-loop awaits to `futures::stream::buffer_unordered(N)` flattened across all languages.

### WHY
Hermes-agent cold-start was 148s today (down from effectively unfinishable yesterday). Investigation surfaced four follow-up findings:
- #1 Tantivy projection 29.3s with no instrumentation (couldn't split load vs apply)
- #2 Tree-sitter extraction 57s SERIAL — `process_file_with_parser(...).await` inside nested loops, so `spawn_blocking` benefit was negated
- #3 Long-line minified heuristic firing on prose `.md` files
- #4 Embedding selection log misreporting per-language cap as "budget_cap"

### HOW
**#2 (the prize):** Flattened `files_by_language` into a tagged work list, then processed via `futures::stream::buffer_unordered(available_parallelism())`. Each in-flight future internally still does `spawn_blocking` for the parse, so concurrent polling translates into N cores of CPU work. State and batch mutation stay sequential after collection (cannot share &mut across futures).

**Regression test** (reverse TDD): `test_extract_index_batch_aggregates_results_from_all_files` creates 30 small Python files and asserts files_processed/all_file_infos/files_to_clean/all_symbols counts hold regardless of execution order. Passed against sequential code first (proving assertions sane), still passes against parallel code (proving correctness preserved).

**Visibility changes:** `extract_index_batch` and `ExtractedBatch` lifted to `pub(crate)` to enable the regression test. Added new `ExtractOutcome` enum (private) to carry parser/non-parser result types through the parallel stream.

**Logging:** New `🚀 Extracting N files in parallel (concurrency=K, languages=M)` line + per-language file counts at debug level + final `⏱️ parallel extraction complete: T.TTs` marker. Closes the per-language visibility gap from finding #2's secondary goal.

### IMPACT — PROJECTED
- Cold-start indexing on hermes-agent (155k symbols, 2,406 files, 16 languages):
  - Tree-sitter extraction: **57s → ~10-15s** on Apple Silicon (8+ perf cores)
  - Total cold-start: **148s → ~100s (~30% faster)**
- Tantivy projection (29.3s) is now the largest remaining single phase. Split timing markers will reveal whether next focused effort should target `load_symbol_contexts_from_database` or `apply_documents_with_context`.
