## v7.5.0 ready to ship — cold-start 148s → 31.86s

### What's in the release

**Tantivy IndexWriter heap 50MB → 256MB** (`src/search/index.rs:37`). Tantivy 0.26's `Index::writer()` auto-clamps thread count when per-thread budget falls below the 15MB floor; at 50MB total we were running 3 threads at ~16.67MB each. At 256MB we get all 8 worker threads at 32MB each.

**Three-run measurement progression on hermes-agent (2406 files, 155942 symbols):**

| Run | Heap | Wall clock | apply_documents | Notes |
|---|---|---|---|---|
| 1 | 50MB | 50.05s | (markers in dead code) | Validated parallel extraction |
| 2 | 50MB | 49.37s | 28.37s | Markers fixed, baseline established |
| 3 | 256MB | **31.86s** | **10.71s** | Heap bump validated |

**Total reduction vs original 148s baseline: 78%** (combined with the parallel tree-sitter extraction from commit 12c8992f).

### Why a minor version (7.5.0)
User-visible perf improvement (cold-start 49s → 32s), no API breaks, new internal observability (projection timing markers). Patch (7.4.2) understates the impact; major (8.0.0) overstates given zero breaking changes.
