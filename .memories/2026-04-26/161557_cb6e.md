## Implemented Rust Scoped Call Resolution

- **WHAT:** Preserved Rust scoped call namespace metadata from extractor output through indexing and into resolver candidate selection. Added `resolve_structured_batch`, structured pending plumbing in indexing, namespace-aware resolver scoring, and an indexed `call_path` regression.
- **WHY:** The old scoped TODO was stale in the narrow extractor sense, but the remaining bug was worse: scoped calls such as `std::collections::HashMap::new()` could resolve by terminal name to a local `fn new()`, creating bogus call graph edges.
- **HOW:** Added extractor tests for `scoped_identifier` namespace preservation, carried `StructuredPendingRelationship` through `process_file_with_parser` and `ExtractedBatch`, resolved structured pendings by terminal name while scoring with namespace context, rejected external Rust namespace roots without project matches, and split namespace helper logic into `src/tools/workspace/indexing/resolver/namespace.rs` to keep `resolver.rs` under the line target.
- **IMPACT:** Rust `crate::module::function()` calls now resolve to namespace-consistent project symbols, external `std/core/alloc` scoped calls no longer attach to local same-name functions, and the TODO item is marked complete.
