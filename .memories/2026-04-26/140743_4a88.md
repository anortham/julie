## Capped Dry-Run Diff Previews

- **WHAT:** Added a shared dry-run diff preview formatter and wired `edit_file` plus `rewrite_symbol` dry-run output through it.
- **WHY:** Large dry-run edits were dumping huge unified diffs, which wastes agent context when agents preview before applying edits.
- **HOW:** Added `format_dry_run_diff` in `src/tools/editing/validation.rs` with head/tail retention and an omitted-line summary. `format_unified_diff` remains unchanged for applied-edit output. Added an async regression for `edit_file` dry-run output and checked off the TODO item.
- **IMPACT:** Dry-run previews stay useful for inspection while avoiding oversized token payloads. `rewrite_symbol` also benefits because its span header was capped but its diff was not.
