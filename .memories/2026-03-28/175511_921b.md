## Fixed .julieignore not created on plugin updates

Two stacked bugs prevented .julieignore from being created on fresh installs:

1. **Julie binary bug** (c21a3a4e, already fixed): `generate_julieignore_file` was inside `if !detected_patterns.is_empty()`, so clean projects never got the file
2. **Plugin launch script bug** (new fix): launch script only extracted the binary if it didn't exist (`if [ ! -x "$BINARY" ]`). After plugin updates, new archives sat unused while the old extracted binary kept running. Fixed by comparing archive mtime vs binary mtime.

- **Regression test added:** `test_discover_creates_julieignore_even_without_vendor_patterns`
- **Version bumped to v6.1.8**, release built and deployed
- **Plugin launch script fixed** in julie-plugin repo (c78f0ad)
- **Confirmed working** on other test machine after both fixes deployed
