## Investigated .julieignore not created on fresh index

User reported .julieignore missing after fresh index on another machine. Traced the full code path from adapter -> daemon -> handler -> indexing -> discovery.

- **Root cause:** Bug in v6.1.6 where `generate_julieignore_file` was inside `if !detected_patterns.is_empty()`, so clean projects (no vendor dirs) never got the file
- **Fix existed:** Commit `c21a3a4e` (today 8:45am) moved the call outside the conditional
- **But never deployed:** Plugin repo still has v6.1.6 archives. CI trigger workflow was broken (two fix commits `a2fab930`, `5cc015d0` after version bump)
- **Added regression test:** `test_discover_creates_julieignore_even_without_vendor_patterns` -- verified RED with old code, GREEN with fix
- **Next:** Bump to v6.1.8, build archives, update plugin repo, confirm on other machine
