## WHAT

Implemented Julie's complete `patterns` consumer surface for julie-extractors v2.16 structural facts across MCP and named/generic CLI paths.

## WHY

The upgraded extractor/database dependency exposes generic structural facts, but Julie had no public tool for agents to list, search, filter, summarize, or facet that data.

## HOW

- Added typed `PatternsTool` operations for list, search, and summary.
- Used explicit target-workspace pooled database selection and deterministic SQLite-backed output.
- Added path, language, metadata equality, query substring, grouping, facet, limit, compact, and JSON behavior.
- Registered the read-only MCP router, telemetry metadata, generic tool dispatcher, and named CLI command.
- Reused Julie's existing global CLI `--workspace` and `--format` flags to avoid duplicate clap arguments while retaining those fields in MCP/generic JSON.
- Added handler, CLI parsing, telemetry, and public-surface contract tests.
- Fixed a discovered query-substring zero-match leak with a dedicated RED/GREEN assertion.

## IMPACT

Agents can now consume the new cross-language structural-fact data directly through Julie. Results are workspace-safe, bounded, stable, and available through both primary public interfaces.
