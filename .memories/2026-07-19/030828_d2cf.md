## Removed the only upgrade-caused clippy warning

- **WHAT:** Boxed the successful parser extraction payload inside `ExtractOutcome::WithParser`.
- **WHY:** v2.16 enrichment data enlarged `ParserFileProcessResult`, making the private outcome enum trigger `clippy::large_enum_variant`.
- **HOW:** Changed the variant to `Result<Box<ParserFileProcessResult>>`, boxed the successful async result, and destructured through the box at aggregation.
- **IMPACT:** The feature branch now has exactly the same normalized clippy warning set as `origin/main`, with no upgrade-introduced warning and no public API change.
