## WHAT

Added a focused target-workspace isolation test for the new `patterns` tool and made `FakeToolContext` capable of returning distinct databases by workspace ID.

## WHY

Lead review found that the original target assertion was invalid: `FakeToolContext::get_pooled_database_for_workspace` ignored its workspace ID and reopened the configured primary database, so the test did not prove isolation.

## HOW

- Added `workspace_db_paths` and `with_workspace_db_path` to `FakeToolContext`.
- Cross-workspace pooled and mutex DB methods now use the explicit mapped path when present and preserve the existing primary-path fallback otherwise.
- Added `patterns_respects_target_workspace`, which seeds separate primary and target SQLite files with disjoint facts, requests the target workspace, asserts the target fact is present, and asserts the primary fact is absent.

## IMPACT

The patterns consumer now has real regression protection for workspace isolation, and future tool tests can use the shared fake to model distinct target databases honestly.
