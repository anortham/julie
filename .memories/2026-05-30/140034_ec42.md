## #66 Qualified-callee false-positive audit — DONE (11/11 green)

Systemic correctness pass across all call-style adapters. `classify_call` (test_calls.rs:42) keys on `callee.split('.').next()` — designed for JS `it.only`/`describe.skip`. Any non-JS adapter feeding a dotted callee whose LEADING segment is a vocab word false-positives.

### Two false-positive mechanisms found
- **Mech A (member access in function position):** `it.register("x")` → `field_expression`/`dot_index_expression` text "it.register" → split → "it" → bogus is_test. (Lua, Scala, Dart, C, C++.)
- **Mech B (dotted bare name — '.' is NOT a member operator):** R S3 names (`describe.default`), Bash/PowerShell command names (`It.helper`, `Context.Helper`) are single dotted tokens → split → vocab prefix → bogus symbol. A node-kind guard CANNOT catch this; only a text guard does. This is the audit's key catch.

### Empirical partition (RED run: 8 failed, 3 passed)
- **VULNERABLE → guarded (8):** C, C++, Lua, R, Dart, Scala, PowerShell, Bash
- **SAFE (3, locked by passing negative control):**
  - PHP — method/static calls are `member_call_expression`/`scoped_call_expression` (filtered by node-kind); PHP fn names can't contain '.'
  - Swift — callee = first DIRECT `simple_identifier` child; member receiver is nested in `navigation_expression`, never found
  - Go — already has `function.kind() != "identifier"` guard (rejects `selector_expression`)
- **Excluded:** Kotlin (vue-html-fixer #64, found+fixed); JS/TS (intended exception — keeps the split for `.only`/`.skip`)

### Fix
Guard `if <callee>.contains('.') { return None; }` after computing callee text, before `classify_call`, in the 8 vulnerable adapters. This is the precise complement of classify_call's `split('.')` and closes BOTH mechanisms without per-grammar node-kind assumptions. No DSL in the 8 uses a dotted modifier chain (JS-only idiom), and all positive callees are dotless → zero false-negatives, no regression to existing positive tests.
