## #33 closed: workspace-rebind "bug" was test non-hermeticity, not a product bug

**WHAT:** Resolved task #33. The 4 `tools-workspace-targeting` rebind failures (the only red bucket in the dev tier) were **not** a product primary-resolution bug. Root-caused to **test fixtures depending on a clean `$TMPDIR`** + local `/private/tmp` pollution.

**WHY it failed (root cause):** `find_workspace_root` (`src/tools/workspace/paths.rs`) walks UP from a path looking for markers (`.git`/`Cargo.toml`/...) and returns the first ancestor with one, or the start path if none. Test fixtures created **marker-less** temp workspaces under `$TMPDIR`. A stray `/private/tmp/Cargo.toml` (a `zig-debug` scratch project) made the walk climb out of the temp dir and resolve **every** workspace to `tmp_*` instead of `target_*`. Passes on clean CI; fails only on the polluted dev box. Proven: move the file → 6/6 green with **zero product changes**; restore → 4 fail.

**HOW fixed:**
- Added shared `make_isolated_workspace_root` / `mark_workspace_root` helpers (`src/tests/helpers/workspace.rs`) that drop a `.git` marker so `find_workspace_root` stops at the temp workspace.
- Hermeticized 10 resolution-critical + collateral test files (rebind_index, refresh_routing, edit_file_tests, rewrite_symbol_tests + cross_language, metrics_recording, editing_metrics, call_path_tests, call_path_disambiguation_tests, global_targeting helper).
- Cleaned the `/private/tmp` debris (user-authorized): `Cargo.toml`, `Cargo.lock`, `.julie/`, `.julieignore`, `*.julie-extract.lock`.
- Corrected the CLAUDE.md/AGENTS.md #33 note (was wrongly described as a product rebind bug; now documents the real cause + the clean-`$TMPDIR` expectation).

**Scope decision (user chose):** "Clean debris + keep my fixes" over hermeticizing all ~57 temp-workspace test files. The suite broadly assumes a clean `$TMPDIR` (as CI always has); full hermeticity was judged disproportionate for an abnormal local condition.

**IMPACT:** Dev tier fully green again (35 buckets, 549.7s) WITH the resolution-critical fixtures now immune to ancestor-marker pollution. My earlier characterization of #33 as a product bug was wrong — corrected.
