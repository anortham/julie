## Phase 3b literals breadth — final 2 legs (Bash + PowerShell) + interpolation normalization

Completes the carrier-gated string-literal capture to **26/26 applicable languages** (was 24). Miller bridge Phase 3.

### What landed
- **Bash leg** (`bash/mod.rs` arm + `bash.toml` carriers + test registration): command-grammar capture; carrier = command name; curl/wget/http/https → url, psql/mysql/mariadb/sqlite3 → sql. The real reason the WIP was "1/5" earlier: the `mod literals;` test module was never registered in `tests/bash/mod.rs`.
- **PowerShell leg** (`powershell/identifiers.rs` arm + `powershell.toml` + registration): command-grammar capture over `command_elements`, skipping `command_argument_sep`; carrier = cmdlet name; invoke-restmethod/invoke-webrequest/irm/iwr → url, invoke-sqlcmd/invoke-sqlcmd2 → sql.
- **Shared decoder interpolation normalization** (`base/extractor.rs decode_string_literal`): rewrote the named-child flatten to (1) check content FIRST, (2) recurse through wrapper layers (Dart `string_literal_double_quotes`), (3) treat interpolation/substitution **expressions** as `{}` holes while EXCLUDING delimiter sub-tokens (`_start`/`_end`/`_quote`/`_brace`), (4) recognize bash shell-expansion kinds (`simple_expansion`/`expansion`/`arithmetic_expansion`) as holes.

### Bugs fixed (found on the path)
- **Bash truncation** (real): `curl "https://api/users/$id"` decoded to `https://api/users/` — the `simple_expansion` was recursed-into-nothing, silently dropping the rest of the URL. Now → `https://api/users/{}`.
- **C# leading `{}`** (regression I introduced then fixed): `interpolation_start` (`$` sigil) contains "interpolat"; the first broad hole rule wrongly emitted a leading `{}`. Fixed by content-first ordering + delimiter-suffix exclusion.
- **Swift `\(x)`** → was silently dropped (`interpolated_expression` unrecognized) → now `{}`.
- **Dart `$x`/`${x}`** → was raw `$id` via fallback → now `{}` (wrapper recursion + `template_chars` content).

### PowerShell-specific decode
PS tokenizes expandable-string static text as ANONYMOUS bytes (only `variable`/`sub_expression` holes are child nodes), so the named-child walk can't rebuild surrounding text. Added a PS-local byte-reconstruction (`decode_ps_string_literal` + `collect_ps_interpolation_holes` + `strip_ps_string_delimiters`) that blanks outermost holes → `{}`, so `"https://api/users/$id"` → `https://api/users/{}` and `$($u.Id)` collapses to one `{}`.
