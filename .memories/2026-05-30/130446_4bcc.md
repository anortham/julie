### Key grammar finding (verified via AST probe)
`test("name") { body }` is a CURRIED nested call, not flat:
- OUTER call_expression: function=INNER call_expression, arguments=block
- INNER call_expression: function=identifier "test", arguments=arguments→string
Adapter guards on `arguments.kind()=="block"` + `function.kind()=="call_expression"` → fires only on OUTER; INNER returns None (arguments≠block) → no double-materialization. Nested `it` parents to `describe` via existing visit_node threading.

FlatSpec `"subj" should "beh" in { }` = `infix_expression(op=in, right=block, left=infix(op=should, string, string))`. Guards: op==in + block + left-verb ∈ should/must/can/will. Name = "subj should beh".
