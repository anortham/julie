## Implemented blast radius, task-shaped get_context, spillover, and test linkage cleanup

### WHAT
- Added a new `blast_radius` MCP tool with deterministic inbound graph ranking, revision-based file seeding, likely-test collection, and spillover paging for long result sets.
- Extended `get_context` with task-shaped inputs: `edited_files`, `entry_symbols`, `stack_trace`, `failing_test`, `max_hops`, and `prefer_tests`.
- Added shared spillover storage plus public `spillover_get` so large graph-heavy responses can page instead of truncating.
- Renamed static test metadata from `test_coverage` to `test_linkage`, wired `compute_test_linkage()` into indexing, and kept fallback reads for old metadata.
- Added revision file-change tracking for indexed updates.
- Folded in external Claude review fixes: stale `test_linkage` metadata is now cleared before recompute, linkage now stores test paths as well as names, `failing_test` hydration now scans linkage rows only and matches path-aware data, and the rebound-primary `get_context` test now seeds the daemon pool through `ManageWorkspaceTool` instead of relying on a shadow handler path.

### WHY
- Julie needed an agent-native change-impact tool and a task-aware context tool without swallowing SDL's heavier workflow machinery.
- Large graph outputs needed paging so token savings do not come from hard truncation.
- Static linkage was useful for likely-tests and risk hints, but stale linkage and the old `test_coverage` naming made the feature misleading.
- Claude review caught two real correctness bugs and one bad handler detour taken to satisfy a test.

### HOW
- Added new impact and spillover modules, handler registrations, tool metadata, and session metrics enums.
- Stored revision file deltas during fresh, incremental, and orphan-delete indexing paths.
- Reworked `get_context` pipeline scoring and formatting to consume task signals and spillover handles.
- Changed `compute_test_linkage()` to clear old linkage keys before recompute, emit `linked_test_paths`, and preserve old-key fallback for readers.
- Reworked `hydrate_failing_test_links()` to query only linkage-bearing symbols and match file paths with `path_matches_signal()`.
- Reverted the temporary `primary_workspace_snapshot()` fallback change after review, then rewrote the rebound `get_context` test to index the rebound workspace through the pool-backed path.

### IMPACT
- Julie now has the planned V1 feature set for blast radius, task-aware context, spillover, revision deltas, and renamed linkage metadata.
- Agent-facing outputs are smaller and more actionable for graph-heavy tasks.
- Post-review correctness is stronger: no stale linkage drift, no dead `failing_test` boost path, and no daemon-pool bypass in the primary snapshot logic.
- Both `cargo xtask test changed` and `cargo xtask test dev` passed after the review fixes.
