## Reviewed and patched commit 2fb89819 (impact-analysis follow-up batch)

Reviewed `2fb89819 fix(tools): harden impact analysis spillover and context` against the written design + plan in `docs/plans/2026-04-21-blast-radius-review-fix-*.md`. Confirmed Tasks 1-4 met their acceptance criteria. Found three concerns and fixed all of them in this turn.

### What was wrong

- **SQL LIKE wildcards unescaped** in `hydrate_failing_test_links` (`src/tools/get_context/task_signals.rs`). The new `EXISTS + json_each` query passed `linked_path.value` and `?1` directly into LIKE patterns without escaping, so `_` and `%` (which are common in real test paths like `payment_service_tests.rs`) acted as wildcards. Empirically verified: `'tests/paymentXserviceXtests.rs' LIKE '%tests/payment_service_tests.rs'` returns `1` in SQLite. Same class of bug as the named regression already encoded in `build_name_match_clause`.
- **Dead SQL clause** at the same site: `?2 LIKE '%::' || linked_test.value`. `?2` is `failing_test_name` (the leaf after `rsplit_once("::")`), so by construction it has no `::` and the pattern can never match.
- **Plugin skill copies stale.** Task 5 acceptance criterion required `~/source/julie-plugin/skills/impact-analysis/SKILL.md` and `explore-area/SKILL.md` to match the repo copies. They didn't. Plugin users would still see the broken `<symbol_name>` workflow and Git-ref revision examples.

### How I fixed each

- **TDD for the SQL fix:** wrote `test_hydrate_failing_test_links_treats_underscores_as_literals` in `src/tests/tools/get_context_task_inputs_tests.rs` first, verified RED (test failed against unescaped SQL), then wrapped both LIKE patterns with `REPLACE(REPLACE(REPLACE(x, '\\', '\\\\'), '%', '\\%'), '_', '\\_')` + `ESCAPE '\\'` matching the convention from `relationships.rs:289` and `identifiers.rs::build_name_match_clause`. Verified GREEN.
- **Dropped the dead `'::' ||` clause** in the same edit.
- **Strengthened multi-seed assertion** in `test_walk_impacts_preserves_identifier_call_kind_and_resolved_target` (`src/tests/tools/blast_radius_determinism_tests.rs`). Now asserts `alpha_adapter` resolves with `RelationshipKind::References` + `via_symbol_name = "AlphaStore"` alongside the existing beta-side checks. Catches a future regression that re-introduces frontier-first fallback in multi-seed walks.
- **Synced plugin skills** by copying the two SKILL.md files into `~/source/julie-plugin/skills/`. `diff -q` reports byte-identical.
