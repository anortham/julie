## Implemented blast_radius / spillover / get_context review fixes

### WHAT
- Preserved identifier-edge semantics in shared impact/context graph walking.
- Made spillover handles replayable and deterministic across repeated identical requests.
- Added lenient boolean deserialization for `blast_radius.include_tests` and `get_context.prefer_tests`.
- Split `src/tools/get_context/pipeline.rs` into smaller modules under the repo file-size target by extracting graph expansion and entry-building helpers.
- Reworked failing-test hydration so SQLite filters candidate linked symbols instead of Rust scanning and parsing every linkage row.
- Updated Julie agent instructions, handler descriptions, and Claude skills so the documented surface matches the tool contract.
- Added targeted regression tests for identifier-call semantics, spillover determinism, and lenient bool parsing.

### WHY
The review found correctness drift in three places that mattered to tool value:
- identifier-only calls were downgraded to generic references and could point at the wrong seed target
- spillover paging broke the deterministic/idempotent story once results overflowed
- docs and skills taught parameter names and workflows the tools could not support

The `get_context` pipeline file also still violated the repo's file-size rule, and failing-test hydration had avoidable per-call overhead.

### HOW
- Introduced target-aware identifier incoming edges with normalized relationship kinds in `src/database/impact_graph.rs`.
- Updated `src/tools/impact/walk.rs` to carry optional resolved targets, avoid lying when a multi-frontier identifier target is ambiguous, and build the next frontier from sorted impacts.
- Updated `src/tools/get_context/graph.rs` and the pipeline caller to consume shared identifier edges.
- Replaced UUID-based spillover handles with stable content-derived handles in `src/tools/spillover/store.rs` and made reinsertion idempotent.
- Extracted `src/tools/get_context/graph.rs` and `src/tools/get_context/entries.rs`, leaving `pipeline.rs` at 262 lines.
- Replaced full-row metadata hydration in `task_signals.rs` with SQL `json_each` existence checks against linked test paths and names.
- Fixed docs and skill frontmatter, including `spillover_handle`, `manage_workspace`, `max_hops`, `prefer_tests`, and revision-number wording.

### IMPACT
- `blast_radius` now preserves identifier call semantics and resolved targets.
- `spillover_get` and overflow-producing tools are replayable for identical inputs instead of minting new handles each time.
- `get_context` is smaller, easier to inspect, and its failing-test signal path does less useless work.
- Agent docs no longer teach a broken `spillover_get(handle)` call or unreachable `symbol_ids`/Git-ref flows.
- The implementation passed focused tests and `cargo xtask test changed`, which fell back to a full `dev` bucket run and passed all 10 buckets.
