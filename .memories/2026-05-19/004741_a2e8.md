## Fixed Claude review findings for external extract CLI

### WHAT
Implemented the review fixes on the external extract branch:

- Moved external analysis stale-marker writes into the same SQLite transaction as canonical persistence.
- Made analysis-current/stale metadata helper writes transactional.
- Restored dangling `parent_id` sanitization during bulk symbol inserts.
- Replaced repo-wide `extract update --file` indexability checks with scoped walking that still respects parent `.gitignore`, `.julieignore`, extra ignore files, and hard-blacklisted ancestor directories.
- Documented the 1 MiB external extraction file cap and SQLite WAL sidecar contract.
- Made watcher single-file persistence use a deterministic root-derived workspace id now that canonical persistence requires workspace ids for revision tracking.
- Kept touched implementation files under the 500-line cap.

### WHY
Claude review identified correctness and operational risks in the external extractor interface:

- Failed metadata writes could leave revisions/files committed with stale analysis state not updated.
- Partial analysis marker writes could leave contradictory metadata.
- Bulk inserts could persist invalid symbol parent links.
- Single-file updates scanned the whole repo and risked getting slow for filewatcher integrations.
- Documentation omitted limits and WAL sidecar behavior external hosts need to know.

The system tier then exposed the related watcher regression caused by the new canonical workspace-id requirement.

### HOW
Added RED tests for transactional rollback, analysis marker atomicity, and dangling parent ids. Added scoped walker coverage for parent ignore behavior. Refactored persistence metadata to carry `mark_external_analysis_stale`, added transaction-local stale metadata writes, added a panic-safe FK PRAGMA guard, checked symbol parents against both batch and existing DB ids, and made scoped indexability validate each path component from the root down. The watcher handler now derives the same stable workspace id from `workspace_root` used elsewhere.

### IMPACT
External `scan`/`update`/`delete` now keep canonical revisions and external analysis metadata consistent, single-file updates avoid repo-wide rediscovery while preserving ignore semantics, and watcher incremental persistence works with revision tracking.
