## Fixed deferred `manage_workspace(open,path)` routing bug

### WHAT
Added a regression test for `manage_workspace(open, path=...)` in a true deferred daemon session with no bound primary, and fixed path-backed index routing so daemon-mode target workspaces derive storage paths from `WorkspacePool::indexes_dir()` instead of requiring a current primary storage anchor.

### WHY
Opening a ready registered workspace by path could fail before activation with `Primary workspace not initialized` because `handle_open_command` refreshed the target before `switch_primary_workspace_with_root`, and refresh routed through `workspace_storage_anchor()` despite no primary being bound yet.

### HOW
Changed `IndexRoute::path_backed_route` in `src/tools/workspace/indexing/route.rs` to use the daemon workspace pool's shared index root when available, falling back to the existing handler anchor logic for non-pool/stdout-style cases. Added `src/tests/tools/workspace/deferred_open.rs` and registered it in `src/tests/mod.rs`.

### IMPACT
Explicit target workspace routes no longer depend on an unrelated primary binding in daemon mode. The exact regression and diff-scoped dev fallback gate pass.
