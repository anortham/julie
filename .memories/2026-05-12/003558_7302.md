## v7.9.1: Codex review fixes + canonicalize hardening

### WHAT
- Patch release on top of v7.9.0. Addresses all five findings from a `gpt-5.3-codex high` review of the v7.9.0 lifecycle/safety batch, plus an in-flight canonicalize bug surfaced during live dogfood.

### Codex findings resolved
1. **Drop try_lock leak** — `HttpJulieService::drop` now logs `warn!` on each `try_lock()` contention so missed cleanup is visible in `daemon.log` instead of silently leaving tracker rows behind.
2. **Spawn-on-drop trade-off** — Added a 20-line invariant comment on the `Drop` impl explaining honest-drain-accounting vs cleanup-on-abrupt-shutdown.
3. **Sensitive root alias gaps** — Added `/private/var/root` literal entry on macOS so rejection no longer depends on `canonicalize()` succeeding. Windows list is now env-driven (`SystemDrive`, `SystemRoot`, `ProgramFiles`, `ProgramFiles(x86)`, `ProgramW6432`, `ProgramData`, `PUBLIC`) so non-`C:` installs are still caught. Extracted `sensitive_root_candidates()` helper.
4. **HTTP busy+restart_pending test** — New `test_http_julie_session_rejects_new_sessions_after_restart_pending_with_active` proves a third session is rejected after staleness flips and the second session has already marked `restart_pending`.
5. **Test silent skip** — Replaced early-return on canonicalize failure with `assert!(raw.exists())` + `expect(...)` so prerequisites fail loudly.

### Canonicalize hardening (pre-codex live find)
- `is_sensitive_workspace_root` was raw-string-comparing paths. On macOS, `resolve_workspace_root` canonicalizes `/var/X` → `/private/var/X`, but the forbidden list and `dirs::home_dir()` stayed uncanonicalized. The guard MISSED `/var/root` (canonicalized to `/private/var/root`); only filesystem permission denial caught it during dogfood. Fix canonicalizes both sides with raw-string fallback.

### WHY
- v7.9.0 shipped lifecycle/safety improvements but had a quietly-broken sensitive-root guard and a few rough lifecycle edges that codex review flagged. Patching now so we don't ship a known security-guard hole and so drain accounting failures are observable instead of silent.

### IMPACT
- Daemon drain/restart correctness is now observable in `daemon.log` if the Drop fast-path is ever broken.
- Sensitive-root rejection is defense-in-depth: literal `/private/var/root` + canonicalize fallback + Windows env-driven discovery.
- HTTP integration test pins the restart_pending+busy rejection behavior end-to-end.
- 5 new root_safety unit tests + 1 new HTTP integration test added.
