## Typed Workspace Resolution Failures

### WHAT
Added typed metadata for resolver-created workspace resolution failures while preserving the exact existing display strings.

- Added `WorkspaceResolutionFailureKind` and `WorkspaceResolutionFailure` in `src/tools/navigation/resolution.rs`
- Wrapped only resolver-created failures: unknown workspace, not-ready workspace, primary swap in progress, and auto-activation failed
- Added `workspace_resolution_failure_kind(&anyhow::Error)` for downcast-based classification
- Updated deep_dive wrapper classification to use typed resolver failures instead of substring matching
- Added focused tests for typed kind + exact message and deep_dive invalid_params behavior

### WHY
The deep_dive wrapper was classifying workspace errors by message substrings, which is brittle and can misclassify unrelated errors. Candidate 6 first slice needs typed metadata without changing user-visible error text or normalizing other wrappers.

### HOW
Resolver-created errors now use a small `std::error::Error` type whose `Display` returns the same message text as before. Existing `anyhow::Result<WorkspaceTarget>` signatures remain unchanged, and callers can detect resolver failures via `anyhow::Error::downcast_ref`.

### IMPACT
Typed workspace target failures can now drive wrapper behavior without text parsing. Scope intentionally stays limited to resolver-created failures and deep_dive classification; other wrapper normalization and metrics attribution remain untouched.
