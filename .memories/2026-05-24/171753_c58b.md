## Unified Symbol Test Exclusion For Semantic KNN

### WHAT
Implemented Candidate 7 first slice for symbol-result test filtering and KNN role projection.

- Added a shared `is_test_symbol_result(file_path, role)` policy in `src/search/index.rs`.
- Changed `SearchFilter::matches_symbol_result` and unified symbol hit filtering to exclude when path is test OR `role == "test"`.
- Added `symbol_role_and_test_role(...)` helper matching Tantivy symbol projection's path + metadata policy.
- Updated `knn_to_search_results` in `src/search/hybrid.rs` to populate `role` / `test_role` from `Symbol.metadata` instead of path-only classification.
- Added focused regression coverage for direct filter behavior, KNN conversion metadata projection, and hybrid semantic result filtering for metadata-only inline test symbols.

### WHY
Semantic/hybrid KNN results could leak inline test symbols from production paths because KNN conversion ignored `Symbol.metadata` and `SearchFilter::matches_symbol_result` filtered tests by path only.

### HOW
Used TDD with exact nextest filters. Verified the new tests failed first, then implemented the shared policy/projection helper and reran exact filters plus requested guard checks.

### IMPACT
`exclude_tests=true` now consistently removes metadata-classified test symbols across lexical symbol results and semantic/hybrid KNN symbol results. Line-mode filtering was intentionally not changed.
