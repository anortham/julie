## Test Symbol Exclusion from Embeddings: Complete

### What
- Added `is_test_symbol_for_embedding()` to `src/embeddings/metadata.rs` using both metadata `is_test` and `is_test_path()` detection
- Filters applied to both `prepare_batch_for_embedding()` and `select_budgeted_variables()`
- Added stale embedding purge to `run_embedding_pipeline()` that cleans up any previously-embedded symbols no longer in the eligible set
- 7 new tests covering all paths

### Results (19 workspaces, 356k symbols)
- **47.2% embedding reduction**: 176,996 → 93,384 embeddings (83,612 test embeddings eliminated)
- **Express: 89.8% reduction**, Flask: 72.5%, Julie: 67.8%, Newtonsoft: 67.0%
- Quality held steady: top sim -0.003, cross-language UP +1.0%
- Stale purge correctly cleaned old test embeddings on re-index

### Files Changed
- `src/embeddings/metadata.rs` - `is_test_symbol_for_embedding()`, filter in `prepare_batch_for_embedding` and `select_budgeted_variables`
- `src/embeddings/pipeline.rs` - generalized stale purge (replaces variable-only stale deletion)
- `src/tests/core/embedding_metadata.rs` - 7 new tests

### Key Design Decision
Generalized the stale purge: instead of only deleting stale variables, the pipeline now deletes ANY previously-embedded symbol not in the current eligible set. This handles test exclusion, future filter changes, deleted symbols, and variable budget changes in one mechanism.
