## Per-Language Embedding Config via TOML Files

### What
- Added `[embeddings]` section to all 33 language TOML files with `extra_kinds` and `variable_ratio` fields
- `EmbeddingsConfig` struct added to `LanguageConfig` with serde defaults
- `prepare_batch_for_embedding` and `select_budgeted_variables` now accept `Option<&LanguageConfigs>` and use per-language settings
- Dimension migration, stale purge, and test exclusion all working together

### Per-Language Overrides
- **Constructor added**: Java, C#, Dart, Kotlin, Swift, Scala, C++, Go, PHP, TS, GDScript
- **Export added**: TypeScript, JavaScript, GDScript
- **Variable ratio raised**: JS/TS/Ruby/PHP/Lua (0.50), Python/C++/GDScript (0.40), C/Zig (0.30)
- Default languages (Rust, Elixir, etc.) use global defaults (0.20 ratio, no extra kinds)

### Architecture
- Config loaded via `LanguageConfigs::load_embedded()` (already used for tokenizer/scoring)
- Passed through `run_embedding_pipeline` -> `prepare_batch_for_embedding` / `select_budgeted_variables`
- Watcher incremental path passes `None` (global defaults fine for single-file re-embeds)
- `is_embeddable_for_language()` checks global EMBEDDABLE_KINDS first, then per-language extra_kinds

### Files Changed
- `src/search/language_config.rs`: EmbeddingsConfig struct, embeddings_config() accessor
- `src/embeddings/metadata.rs`: is_embeddable_for_language(), updated signatures
- `src/embeddings/pipeline.rs`: lang_configs parameter threading
- `src/tools/workspace/indexing/embeddings.rs`: loads LanguageConfigs for pipeline
- `src/watcher/mod.rs`: passes None for incremental path
- All 33 `languages/*.toml` files: [embeddings] section added
- Multiple test files: updated call signatures with None
