## CodeRankEmbed Benchmark Complete

### What
- Successfully evaluated nomic-ai/CodeRankEmbed (768d, 137M params) against BGE-small-en-v1.5 (384d, 33M params)
- Fixed sidecar to support arbitrary dimension models (removed hardcoded 384 guard)
- Added automatic dimension migration to embedding pipeline
- Added `trust_remote_code=True` and `einops` dependency for CodeRankEmbed

### Benchmark Results (19 workspaces, 95 pivots)
- **Namespace overlap**: 0.356 -> 0.394 (+10.7%) - better at finding related symbols
- **Same-kind ratio**: 0.839 -> 0.873 (+4.1%) - more precise kind matching
- **Cross-language**: 10.5% -> 12.6% (+20%) - surfaces more cross-language matches
- **Top similarity**: 0.929 -> 0.840 (-9.6%) - expected, code models have wider score spread
- **Diversity**: 0.640 -> 0.641 (unchanged)
- MPS GPU acceleration worked on Mac, all 19 workspaces embedded successfully
- Dimension migration (384 -> 768) fired automatically on first run

### Qualitative Improvements
- `cobra/Error` matched `flagCompError` instead of `execute` (semantic vs English word overlap)
- `phoenix/inspect` matched `Inspect` protocol instead of a test class
- `cobra/Name` matched `DisplayName` instead of `UsageString`

### Infrastructure Changes
- `python/embeddings_sidecar/sidecar/runtime.py`: removed `EXPECTED_DIMS=384`, added `_SUPPORTED_DIMS` set, added `trust_remote_code=True`
- `src/embeddings/pipeline.rs`: added dimension-change detection with automatic `recreate_vectors_table`
- Installed `einops` in sidecar venv for CodeRankEmbed custom model code
