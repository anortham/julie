## A1.4 complete: token_file write_token/read_token with 0600 mode

### WHAT
Implemented `src/daemon/token_file.rs` with `write_token` and `read_token` primitives. Added `token_file()` getter to `DaemonPaths` in `src/paths.rs`. Registered module in `src/daemon/mod.rs` and tests in `src/tests/daemon/mod.rs`.

### WHY
Daemon bearer token at `~/.julie/daemon.token` must be readable only by the owning user. POSIX mode 0600 prevents other local users from stealing the token via filesystem reads.

### HOW
- `write_token`: `OpenOptionsExt::mode(0o600)` at open time + `set_permissions(0o600)` post-write (defends against umask stripping)
- `read_token`: `read_to_string` + `trim_end()` (strips trailing newline from `writeln!`)
- Windows: no ACL restriction (no clean stdlib path without external crate); emits `tracing::warn!` with TODO A1.4 follow-up reference
- Modeled after existing `write_token_file` in `src/daemon/http_transport.rs` for style consistency
- `token_file()` getter placed immediately after `discovery_file()` in paths.rs: returns `julie_home.join("daemon.token")`

### IMPACT
5 tests all green: round-trip, 0600 mode (unix-gated), overwrite+mode, paths helper, trailing whitespace trim. Unblocks A1.6/A1.8 which wire startup to call write_token.
