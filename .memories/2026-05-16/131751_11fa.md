## A1.2 complete: DaemonLockGuard kernel advisory lock

Built the new `DaemonLockGuard` in `src/daemon/discovery.rs` for the daemon-split architecture. Replaces nothing yet ŌĆö coexists with `SingletonLock` until A1.5 (migration gate) and A1.8 (wiring) handle the cutover.

### What
- **New module:** `src/daemon/discovery.rs` ŌĆö hosts `DaemonLockGuard`, `LockAlreadyHeld`, `AcquireError`.
- **Wired into** `src/daemon/mod.rs` as `pub mod discovery;`.
- **Test file:** `src/tests/daemon/lock_test.rs` ŌĆö 5 new tests, all passing.

### How
- Used `fs2::FileExt::try_lock_exclusive` for cross-platform locking (POSIX `flock` + Windows `LockFileEx`). Same pattern as the existing `SingletonLock` ŌĆö proven precedent in the codebase. Plan suggested `nix`/`rustix` or `windows-sys` directly, but `fs2` already satisfies all the plan invariants (drop releases, kernel releases on crash, second concurrent acquire fails).
- API shape: `try_acquire(&Path) -> Result<DaemonLockGuard, AcquireError>` where `AcquireError::AlreadyHeld(LockAlreadyHeld { path })` is the typed "another daemon" variant.
- Lock file NEVER unlinked on drop ŌĆö stable-inode invariant. Lock state only is per-process.

### Test coverage (all 5 pass)
- `test_daemon_lock_try_acquire_succeeds_when_unheld`
- `test_daemon_lock_try_acquire_fails_when_held` ŌĆö verifies `AlreadyHeld` with correct path
- `test_daemon_lock_released_on_drop` ŌĆö re-acquire after drop succeeds

### Judgment calls
- `src/daemon/discovery.rs:33` ŌĆö chose `fs2` over direct `fcntl`/`LockFileEx` because Julie already depends on it (Cargo.toml:104) and `singleton.rs` proves the pattern works for the same invariants the plan needs.
- `src/paths.rs:105` ŌĆö reused existing `paths.daemon_lock()` (already returns `~/.julie/daemon.lock`). The adapter launcher currently locks this same path for startup serialization; that's a known coexistence concern handled by A1.5/A1.8, not this task.
- Added `AcquireError` enum wrapping `LockAlreadyHeld` so callers get a clean `From<LockAlreadyHeld>` conversion without losing the typed "already-held" case. Tests match on `AcquireError::AlreadyHeld(LockAlreadyHeld { path })`.

### Cross-platform verification
Single-process tests are valid on both POSIX and Windows because `fs2::try_lock_exclusive` contends across open-file-descriptions/handles within the same process on both platforms. Code compiles cleanly on Darwin (verified). No `#[cfg(unix)]`/`#[cfg(windows)]` branches needed ŌĆö the `fs2` wrapper handles the platform dispatch.
