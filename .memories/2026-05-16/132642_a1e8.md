## A1.3: DiscoveryRecord + atomic discovery-file IO

### WHAT
Implemented Task A1.3 of the daemon-split plan: `DiscoveryRecord`, `DiscoveryFile`, and `DiscoveryState` in `src/daemon/discovery.rs`. Added `discovery_file()` getter to `src/paths.rs`. Bumped `process_creation_time_micros` to `pub(crate)` in `src/daemon/pid.rs`.

### WHY
The adapter needs to read `~/.julie/discovery.json` to find the daemon's HTTP endpoint and verify the daemon's identity (pid + creation time) before connecting. Atomic write prevents readers from ever seeing a half-written file.

### HOW
- `DiscoveryRecord`: serde `Serialize + Deserialize`, all plan-specified fields; constructor `for_current_process(port, token_path, log_path)` populates pid/creation_time/version fields automatically.
- `DiscoveryFile::write_atomic`: temp file → fsync → rename → dir fsync (POSIX only, skipped Windows).
- `DiscoveryFile::read_and_validate`: returns `DiscoveryState::{Live, Stale, Missing, Corrupt}`. PID liveness via `PidFile::is_process_alive`; creation-time mismatch → `Stale`.
- `hostname()`: `libc::gethostname` on unix, `"127.0.0.1"` fallback on Windows.
- Tests in `src/tests/daemon/discovery_test.rs`: 5 tests, all pass.

### IMPACT
- All 5 `test_discovery_*` tests GREEN.
- `cargo check --tests` clean.
- A1.4 (token file) can now reference `paths.discovery_file()` and build on this record.

### Judgment calls
- **Field name `pid_creation_time_micros` not `_ns`**: existing `process_creation_time_micros` helper returns micros; renaming to match avoids lossy conversion. Plan explicitly blessed this.
- **`pub(crate)` on `process_creation_time_micros`**: minimal visibility increase; only 3 internal callers in pid.rs, now + discovery.rs.
- **`DiscoveryFile` as a unit struct with `impl` methods** (not free functions): matches `PidFile` style in the same module.
- **`hostname()` via `libc::gethostname` without a new dep**: project already depends on `libc` (confirmed via pid.rs). Avoids adding a `hostname` crate.
- **`None` on creation-time lookup → `Stale`** (not `Live`): conservative — better to restart daemon than connect to wrong process.
