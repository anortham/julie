## Phase A.1 complete: daemon split shipped

A.1 of `docs/plans/2026-05-16-daemon-split-and-search-reranker-plan.md` is
fully landed across the julie + julie-plugin repos on branch
`feat/daemon-split-and-reranker`.

### What landed
- **A1.1** `[[bin]]` julie-adapter and julie-daemon entries; thin wrappers
  in `src/bin/julie-{adapter,daemon}.rs` calling into `daemon::cli::run`.
- **A1.2** `DaemonLockGuard` over `daemon.lock` via `fs2::FileExt`
  kernel-held advisory lock; cross-platform (POSIX flock, Windows
  LockFileEx). Guarantees one daemon per JULIE_HOME.
- **A1.3** Discovery file (`daemon.discovery.json`) with atomic
  temp+fsync+rename+parent-dir fsync write and pid+creation_time identity
  validation. Survives pid reuse.
- **A1.4** Token file with mode 0600 on POSIX (OpenOptions::mode +
  post-write set_permissions). Windows ACL gap surfaced via tracing::warn!
  pending B-phase work.
- **A1.5** Hard legacy-migration gate. `MigrationDecision` enum dispatches
  refuse-with-PID vs proceed-and-unlink. Conservative on Indeterminate.
- **A1.6** `DaemonApp` + `DaemonHandle` factored from `run_daemon`
  (510â†’31 lines). Locked interface contract: `DaemonConfig` /
  `DaemonRuntimeContext` (empty stub for A.1; B.1 fills it).
- **A1.7** Bounded shutdown drain with recovery markers; transport flips
  to 503 (draining) â†’ 502 (aborted). Markers surfaced in `/status/live`
  dashboard route.
- **A1.8** `julie-server` compat shim dispatches via shared
  `daemon::cli::{start,stop,status}_daemon` helpers. Launcher releases
  `daemon.lock` before spawn to avoid migration-gate collision (in-flight
  defect fix).
- **A1.9** Plugin entry point switched from `julie-server` to
  `julie-adapter`. `maybeStopExistingDaemon` prefers `julie-daemon`,
  falls back to legacy binary for pre-A1.8 installations. Release
  workflow bundles all three binaries with codesign.
