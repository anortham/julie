## Fixed project-local Julie state entering source policy

A1 from Eros continuous testing showed Julie-generated repo-local state could be treated as source input, especially `.julie/indexes/...` files written during test/tool runs.

- **Root cause:** `should_watch_path`/`should_index_path_candidate` rejected blacklisted filenames and extensions but did not reject paths with a `.julie` component. Recursive walkers already filtered `.julie`, but direct/single-path policy could still accept supported text formats like `.json` under `.julie/indexes`.
- **Fix:** Added a narrow `.julie` path-component guard in `crates/julie-core/src/file_policy.rs` before filename/extension handling.
- **Test:** Added `test_should_watch_path_rejects_project_local_julie_state` covering `.julie/indexes/workspace/metadata.json`.
- **Impact:** Generated Julie state no longer becomes watchable/indexable source through the shared path policy, reducing CT self-trigger loops without excluding real fixture/source files.
