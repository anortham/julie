## Dashboard Intelligence Layer - Design Spec Complete

Brainstormed and designed a new "Intelligence Layer" for the Julie dashboard with input from Claude, Gemini, and Codex.

### What
- New `/intelligence/{workspace_id}` page with 5 sections: Codebase Fingerprint (hero stats), Top Symbols by Centrality ("Main Characters"), Symbol Kind Distribution (SVG donut), Complexity Hotspots (top files by composite score), and Story Cards (auto-generated observations)
- Enrichments to existing pages: Top Symbol column in Projects table, kind breakdown bar in detail modal, success rate card on Metrics, centrality badges on Search results

### Why
- Dashboard currently surfaces ~30-40% of available data, showing counts but not interpretation
- New users onboarding week of 2026-03-31 need strong first impressions
- Three-model consensus: "surface interpretation, not counts" is the key gap

### Key Decisions
- **Approach A** chosen over "Enrich Everything" (B) and "Full Observatory" (C)
- Companion panel usage (glanceable, not interactive explorer)
- Server-side SVG donut charts, no JS charting libraries, no build step
- All data already in SQLite, zero schema migrations needed
- Out of scope: call graph explorer, architecture map, historical sparklines

### Data Layer
- 3 new queries on SymbolDatabase: `get_top_symbols_by_centrality`, `get_file_hotspots`, `get_aggregate_stats`
- 1 new query on DaemonDatabase: `get_tool_success_rate`
- 4 new lightweight structs: CentralitySymbol, FileHotspot, AggregateStats, DonutSegment
