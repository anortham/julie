## Task 4 Completed in Tree-Sitter Hardening Worktree

Finished the shared unresolved-relationship foundation work in `.worktrees/treesitter-world-class-hardening`.

### What
- Added `crates/julie-extractors/src/base/relationship_resolution.rs` with the new structured unresolved-target model.
- Introduced `UnresolvedTarget` with receiver, terminal name, namespace path, and import-context slots.
- Introduced `StructuredPendingRelationship` as the shared unresolved relationship model that preserves structured target data plus optional caller-scope context.
- Added `BaseExtractor::create_pending_relationship()` as the shared helper for building unresolved relationships from a `Node`.
- Kept `PendingRelationship` as the compatibility record for existing language extractors and added `PendingRelationship::legacy()` as the compatibility constructor.
- Added `crates/julie-extractors/src/tests/relationship_precision.rs` with helper-level regression tests for member-call context, duplicate terminal names, legacy compatibility, and degradation to the legacy pending record.

### Why
Task 4 needed to establish a stronger unresolved-relationship model before the language-specific migrations in Tasks 5 and 6. The first attempt to make `PendingRelationship` itself carry the new fields caused a wide constructor fan-out across many language files. That was the wrong trade for this phase.

### How
- Wrote failing helper-level tests first.
- Tried a direct `PendingRelationship` field expansion, saw it explode into broad constructor churn, then rolled that back.
- Landed the contained foundation instead: shared model + helper APIs + compatibility glue.
- Verified all Task 4 narrow tests in `julie-extractors`.
- Ran `cargo xtask test dev` in the worktree after the fix.

### Impact
- There is now a shared structured unresolved-target model ready for language migrations.
- Existing extractors still compile and run because the legacy pending record remains intact.
- Task 5 can now migrate JS and TS onto the new structured helper model without inventing its own shape.
