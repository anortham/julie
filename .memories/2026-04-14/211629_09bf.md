## Completed remaining pending-call wave after Task 6

Continued the tree-sitter hardening branch beyond the OO-wave migration and finished the remaining languages that still emitted legacy-only pending relationships.

- **Languages migrated:** Bash, PowerShell, Lua, QML, R
- **What changed:** each of these extractors now stores and returns `structured_pending_relationships` alongside legacy `pending_relationships`
- **Registry plumbing:** canonical extraction now preserves structured pending data for Bash, PowerShell, Lua, QML, and R instead of dropping it at the registry layer
- **Call precision:** unresolved call targets now preserve receiver and qualified-call context where available, including cases like `calc.double`, `Calculator.new`, and `Utils.helperFunction`
- **R-specific behavior:** unresolved namespace and extract-operator cases now build structured targets rather than only flat callee names
- **Cleanup:** removed fresh dead-code warnings introduced by the migration wave; remaining warning noise is back down to the pre-existing unused `warn` import in `crates/julie-extractors/src/base/extractor.rs`

## Why

The branch was still inconsistent after Task 6: the remaining pending-call languages were collapsing unresolved targets to flat strings and canonical extraction was still discarding structured data for those languages. That would have left the extractor stack half-upgraded, which is not world-class by any definition.

## How

- Added structured-pending storage/getters to the remaining extractors
- Switched unresolved emitters to `BaseExtractor::create_pending_relationship`
- Preserved compatibility by degrading structured records into the legacy `pending_relationships` list
- Added regression assertions to the existing cross-file suites for Bash, PowerShell, Lua, QML, and R
- Added debugging on a transient `core-fast` bucket timeout, verified the exact handler bucket independently, removed the fresh warning noise, and re-ran the full dev tier until it was green

## Impact

- Structured unresolved relationship support now spans JS/TS, the OO wave, and the remaining pending-call wave
- Canonical/API parity remains green after the expanded registry coverage
- `cargo xtask test dev` passed after the migration and follow-up cleanup
