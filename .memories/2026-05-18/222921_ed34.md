## Implemented external extract operation locking

- **WHAT:** Added per-DB external extract operation locking with `fs2`, timeout-rich errors, lock path helper, and a write-operation DB wrapper that holds the lock while the DB handle is open.
- **WHY:** Task 3 requires `scan`/`update`/`delete`/`analyze` write operations to serialize per database while leaving `info` read-only and unblocked.
- **HOW:** Created `src/external_extract/lock.rs`, exported lock APIs, added `open_external_extract_database_for_operation`, and added focused tests in `src/tests/external_extract/locking.rs` for contention, timeout details, lock-file persistence, read-only info behavior, and DB-open lock lifetime.
- **IMPACT:** Future external extract write commands can use the operation wrapper to acquire the extract lock before DB schema validation/open/migration. Exact worker tests pass.
