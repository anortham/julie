## Local Julie Harness Setup

- **WHAT:** Reconfigured local development machine so Claude Code, Codex, and OpenCode point at `/Users/murphy/source/julie/target/release/julie-adapter`, disabled redundant Julie hook layers, and fixed `cargo xtask dev-link` to link all split binaries (`julie-server`, `julie-adapter`, `julie-daemon`) in Claude plugin cache architecture directories.
- **WHY:** Claude Code plugin cache had only `julie-server`; the split adapter looked for sibling `julie-daemon`, failed to spawn it, and timed out. Codex worked because it used the local target directory with all binaries present.
- **HOW:** Disabled the Claude Julie plugin for daily dev, added project-local Claude MCP `julie` to the local adapter, switched Codex/OpenCode configs to the local adapter, removed Codex/OpenCode Julie hooks, emptied repo-local `.claude/hooks/hooks.json`, and updated `xtask/src/dev_workflow.rs` with regression tests for split binary linking and ignoring `bin/archives`.
- **IMPACT:** Fresh Claude Code, Codex, and OpenCode sessions should use the same local dev adapter/daemon. `cargo xtask dev-link` now repairs installed plugin caches without creating stray archive symlinks.
