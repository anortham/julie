## Completed external extract CLI plan implementation

### WHAT
Implemented `julie-server extract ...` for caller-owned SQLite extraction, including scan/update/delete/analyze/info operations, report formatting, CLI wiring, docs, and review-driven contract fixes.

Key changes:
- Added `src/external_extract/` for CLI args, metadata/info, locking, path normalization, reports, operations, and data-loss guard.
- Added `src/indexing_core/` for shared extraction, discovery, persistence, and analysis helpers reused by workspace indexing and external extract.
- Split oversized database bulk persistence code into focused `src/database/bulk/*` modules.
- Wired top-level `Command::Extract` through `src/main.rs` without daemon/MCP startup.
- Added `docs/EXTERNAL_EXTRACT.md` and implementation/design plan docs.
- Applied review fixes: info report exposes schema/contract metadata, reports latest canonical revision separately from analyzed revision, root mismatch requires force rebuild, parser-backed empty/failing extraction preserves known-good symbols, docs cover lock timeout and schema policy.

### WHY
Other runtimes need to invoke Julie as a normal process from Go/C# without Rust FFI or cross-language layers. The host owns the DB path and file watcher; Julie provides idempotent scan and single-file update/delete into SQLite.

### HOW
- Kept the first implementation inside the main crate and created module boundaries that can later split mechanically if useful.
- Reused Julie extractor and canonical SQLite persistence paths instead of duplicating extraction logic.
- Added per-DB file locking before DB open/migration.
- Used atomic persistence primitives for force rebuild, incremental scan, single-file replace, and delete.
- Added report contract fields for `julie_version`, `schema_state`, `extract_contract_version`, latest `revision`, `analyzed_revision`, totals, and metadata gaps.

### IMPACT
The external extraction boundary is now process-friendly and idempotent. Workspace indexing remains stable while sharing the new extraction core. Review issues that could confuse consumers or risk data loss were fixed before handoff.
