## External Extract CLI Foundation Implemented

- **WHAT:** Implemented foundation tasks for the external extractor CLI: isolated parser/report contract, schema metadata/read-only info, per-DB operation lock, and external path/discovery helpers.
- **WHY:** These pieces establish the public process contract and safety boundaries before moving shared extraction and SQLite mutation logic into reusable seams.
- **HOW:** Added `src/external_extract/{cli,report,info,metadata,lock,paths}.rs`, `src/indexing_core/discovery.rs`, registered tests, bumped SQLite schema to migration 26 for `external_extract_metadata`, and extended the shared walker so `.gitignore` works as a custom ignore filename outside git repos too.
- **IMPACT:** External callers now have typed args and stable report/info/metadata/lock/path APIs. Exact tests prove read-only info does not migrate, strict schema rejects older DBs before migration, locks serialize writes while info remains readable, and external discovery respects `.gitignore`, `.julieignore`, repeatable ignore files, path containment, symlink containment, and hard blacklists.
