## Phase 2: Cross-Project Intelligence — Complete

All 8 tasks of Phase 2 implemented and passing on `feature/julie-platform`.

### What was built
- **WorkspaceTarget enum** — `Primary`, `Reference(String)`, `All` replacing `Option<String>` for workspace routing
- **DaemonState on handler** — `Option<Arc<RwLock<DaemonState>>>` injected into `JulieServerHandler`
- **Federation module** (`src/tools/federation/`) — Generic N-list RRF merge, `WorkspaceSearchEntry`, parallel fan-out via `JoinSet`
- **Federated fast_search** — definition + content search across all projects with `[project:]` tags
- **Federated fast_refs** — cross-project reference search with per-project grouping
- **Federated deep_dive** — cross-project symbol investigation with cross-project callers section
- **Federated get_context** — cross-project pivot search with proportional token budgets
- **Integration tests** — 4 E2E tests creating real workspaces with SQLite + Tantivy
- **Tool descriptions** — All 4 tools document `workspace="all"` option

### Key architecture
- Lock discipline: DaemonState read lock dropped before async work
- `std::sync::Mutex` for SearchIndex/SymbolDatabase (used inside `spawn_blocking`)
- RRF merge with k=60 for score-agnostic ranking fusion across projects
- Results tagged with `[project: name]` in output formatters
