## Phase 3: Memory Integration — Complete

Replaced the Goldfish MCP plugin with native Rust memory tools in Julie. 10 tasks implemented via subagent-driven development, all reviewed and cleaned up.

### What was built
- **`src/memory/` module** (8 files, ~2,500 lines): Types, YAML frontmatter storage, git context capture, checkpoint save, plan CRUD, recall (filesystem + Tantivy search), memory Tantivy index, date filtering
- **`src/tools/memory/` wrappers** (4 files, ~590 lines): CheckpointTool, RecallTool, PlanTool registered in handler's tool_router
- **9 test files** (~6,000 lines): 222 memory-specific tests, all passing

### Key decisions
- Tantivy BM25 for memory search (replaces Goldfish's fuse.js)
- Default tokenizer (not CodeTokenizer) since memories are natural language
- Best-effort index-on-save (file written first, Tantivy indexed after)
- Cross-project recall decoupled from DaemonState (takes Vec of workspace paths)
- Schema validation guard on MemoryIndex open to prevent stale index issues

### Goldfish compatibility
- Reads existing `.memories/` directories without migration
- Handles BOM, CRLF, legacy field names (files_changed/filesChanged)
- Same checkpoint ID format, same filename format, same directory structure
