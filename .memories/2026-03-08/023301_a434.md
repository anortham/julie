## Phase 4 Design Complete — Management UI + Agent Dispatch

Brainstormed and wrote design doc for Phase 4. Covers agent dispatch (Claude CLI, single-shot with generous context assembly), search playground with full BM25/centrality scoring debug, memory timeline with plan viewer, dashboard expansion, and 13 new REST endpoints.

Key decisions:
- Single-shot dispatch (no interactive sessions) — value is in context assembly
- No artificial token budget on context — generous assembly for single-shot
- No new MCP tools — agent dispatch is UI-driven to avoid context bloat
- Full search debug mode — BM25 scores, centrality, field matches, token breakdown
- Dependency graph visualization deferred to later phase
