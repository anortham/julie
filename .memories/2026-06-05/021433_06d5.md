## Codex F3 fix — bound read-only manage_workspace ops in hang guard

### WHAT
Fixed `manage_workspace` being blanket-exempt from the per-request deadline guard.

**Before:** `EXEMPT_WRITER_TOOLS` exempted the entire `manage_workspace` tool, including read-only ops (List, Stats, Health) — a hung stats/health/list was unbounded.

**After:**
- Added `is_write_exempt(tool_name, arguments)` — operation-aware logic:
  - Always exempt: `edit_file`, `rename_symbol`, `rewrite_symbol`
  - `manage_workspace` exempt only for mutating ops: Index, Register, Remove, Clean, Refresh, Open
  - `manage_workspace` deadline-bounded: List, Stats, Health, unparseable
- `dispatch_with_deadline` now takes `exempt: bool` (caller decides, not the function)
- Removed `EXEMPT_WRITER_TOOLS` const
- `call_tool` computes `exempt` via `is_write_exempt` before `request` is moved

### HOW
`ManageWorkspaceOperation::parse` is `pub(crate)` so I inlined `from_arguments` logic directly in `is_write_exempt` (kept change to `handler.rs` only — `commands/mod.rs` not in ownership).

### IMPACT
- 16/16 deadline tests GREEN (10 new is_write_exempt tests + 2 updated dispatch tests + 4 original parse tests)
- cargo check -p julie clean
