## T7 complete: loser write-refusal + metrics gate (D1 + D3)

Implements the full T7 gate invariant for the julie-rescue Phase 3c in-process model.

### What changed

- **`src/leadership.rs`**: Added `in_process: bool` field + `follower()` constructor + `is_follower()` method. `leader()` sets `in_process=true`, `follower()` sets `in_process=true, lock=None`, `none()` sets `in_process=false`. This distinguishes in-process losers from regular non-election handlers.
- **`src/handler.rs`**: Added `is_in_process_follower()` — returns `true` only when created via `LeadershipState::follower()`. Critical: does NOT fire for regular pre-3c handlers (which use `none()`), so daemon/stdio mode unaffected.
- **6 D1 write entry points gated** with `if handler.is_in_process_follower() { return Ok(CallToolResult::error(...)) }`:
  - `handle_index_command` (index.rs)
  - `handle_register_command` (register_remove.rs)
  - `handle_remove_command` (register_remove.rs)
  - `handle_refresh_command` (refresh_stats.rs)
  - `edit_file`, `rename_symbol`, `rewrite_symbol` (handler/tools/*.rs — returns `Err(McpError)`)
- **`src/handler/tool_metrics.rs`**: Wrapped `metrics_tx.try_send(MetricsTask {...})` in `if !self.is_in_process_follower()` — suppresses persistent DB write on followers; in-memory parts (session_metrics, project_log, dashboard_tx) unchanged.
- **New test**: `src/tests/core/handler/loser_refuses.rs` — 5 tests, all GREEN.

### Key architectural decision

Used `LeadershipState::follower()` instead of `::none()` for losers so that `!is_leader()` does NOT become the gate. Regular pre-3c handlers (daemon mode, stdio mode) use `none()` and have `is_in_process_follower() == false` — they continue writing metrics and accepting all tool calls unchanged.
