## Phase 3d.1 Task #1 complete: Route CLI to standalone-only, delete daemon-IPC client

Deleted the daemon-IPC path from the CLI tool runner as the first cut of Phase 3d (daemon teardown).

### What changed

- **Deleted:** `src/cli_tools/daemon.rs` — `DaemonClient`, `run_via_daemon`, `ensure_daemon_ready`, `try_connect_daemon`, `DaemonCallError`, `build_startup_hint`
- **Modified:** `src/cli_tools/mod.rs` — removed `pub mod daemon`, daemon imports, `Daemon`/`DaemonFallback` `CliExecutionMode` variants, the daemon-then-fallback branch in `run_cli_tool`, `run_via_daemon`, `build_cli_startup_hint`, `summarize_error`. `run_cli_tool` now calls `validate_standalone()` + `run_standalone()` unconditionally; `_standalone: bool` param kept for CLI compat but ignored.
- **Modified:** `src/tests/cli_execution_tests.rs` — removed daemon import, `CliExecutionMode::Daemon/DaemonFallback` tests, daemon detection tests, `DaemonCallError` error-mapping tests, daemon-fallback integration test, and http-client-config test (which called `daemon::build_startup_hint`).

All 6 target symbols confirmed no external callers outside `src/cli_tools/`:

- `http_client_config_for_endpoint` — that fn lives in `src/daemon/http_client.rs` (Task 2 scope), not deleted here ✅

### Invariant proven
