## T3 — Handler per-request deadline / hang guard

### WHAT
Added a per-request timeout to `JulieServerHandler::call_tool` that bounds read/query tools
and exempts canonical writer tools, preventing session hangs from slow tools.

### Items added to `src/handler.rs`
- `DEFAULT_REQUEST_TIMEOUT_SECS` (120s), `REQUEST_TIMEOUT_ENV` (`JULIE_INPROCESS_REQUEST_TIMEOUT_SECS`)
- `EXEMPT_WRITER_TOOLS` const slice: `edit_file`, `rename_symbol`, `rewrite_symbol`, `manage_workspace`
- `pub(crate) fn parse_request_timeout(raw: Option<String>) -> Option<Duration>` — `"0"` → disabled, garbage → 120s, `"N"` → N seconds
- `pub(crate) async fn dispatch_with_deadline(tool_name, fut, deadline)` — exempts writers, wraps readers in `tokio::time::timeout`, returns `McpError::internal_error` naming the tool on expiry
- Modified `call_tool`: reads env var, calls `dispatch_with_deadline` instead of direct `.await`

### WHERE the deadline wraps
`call_tool` after `ensure_primary_workspace_for_request` completes (unbounded), ONLY around `tool_router.call(...)`.

### Test file
`src/tests/core/handler/deadline.rs` — 6 tests:
- 4 sync tests for `parse_request_timeout` (`None`→120s, `"0"`→None, garbage→120s, `"30"`→30s)
- `test_dispatch_with_deadline_read_tool_times_out` (`start_paused=true`, 50ms deadline, 300s stall → timeout error)
- `test_dispatch_with_deadline_writer_exempt_not_bounded` (`start_paused=true`, edit_file exempt → stall runs to completion)
- Registered in `src/tests/core/handler.rs` as `mod deadline;`

### HOW
- `tokio::time::timeout` + `start_paused = true` in tests (zero real-wall time, tokio auto-advances virtual clock)
- Writer exemption is a compile-time const slice; checked before any timeout logic
