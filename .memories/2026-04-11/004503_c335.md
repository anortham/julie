## Approved global workspace targeting spec

Wrote and committed the approved design spec at `docs/superpowers/specs/2026-04-10-global-workspace-targeting-design.md`.

- **What:** Captured the shift from legacy reference-workspace routing to global workspace targeting.
- **Why:** The daemon-owned `~/.julie` storage model made the old primary-to-reference mental model misleading and created confusion around routing, freshness, and watcher behavior.
- **How:** Documented the approved design: one explicit ensure/open activation path, mandatory freshness before query/edit use, session-level active workspace sets, watcher coverage only for active workspaces, and optional pairings as convenience metadata only.
- **Impact:** The spec now gives a clean target architecture for planning and implementation, and it calls out stale assumptions in current code such as removal-path logic inherited from the old nested layout model.
