## Completed Task 1 of global workspace targeting

Implemented session-scoped active workspace tracking for daemon handlers and closed the teardown leak for activated secondary workspaces.

- **What changed:** `JulieServerHandler` now tracks `active_workspaces`, seeds the primary workspace into that set, and exposes helper methods for active-state queries and activation. `handle_ipc_session` now disconnects every active workspace on teardown, including failure paths. `WorkspacePool` session count updates are now awaited instead of fire-and-forget.
- **Why:** The previous daemon session lifecycle only cleaned up the primary workspace and could leak session counts on fast failures due to increment/decrement reordering.
- **How:** Added handler state in `src/handler.rs`, updated daemon session wiring in `src/daemon/mod.rs`, and made `WorkspacePool` count updates deterministic in `src/daemon/workspace_pool.rs`. Added targeted daemon handler tests and a malformed-MCP regression test for cleanup.
- **Impact:** Task 1 is complete and review-passed. Task 2 can now build on reliable per-session active workspace state instead of shaky primary-only teardown logic.
