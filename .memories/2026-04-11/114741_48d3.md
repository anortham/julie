## Implemented global workspace targeting

Completed the workspace-targeting implementation and verified it on the branch.

- **What:** Added explicit `manage_workspace(operation="open")`, session-scoped active workspace tracking, daemon-mode routing gated on active workspaces, pairing metadata cleanup, and updated workspace docs/comments. Also fixed follow-up regressions uncovered by system-tier verification.
- **Why:** The daemon-owned `~/.julie` registry made the old reference-workspace mental model misleading. Julie now needs one clear front door for cross-workspace access, with freshness checks before query/edit use and session-scoped activation instead of implicit attach behavior.
- **How:** Implemented the open flow, reused indexing/refresh machinery with shared per-path locking, demoted persisted pairings to convenience metadata only, removed startup auto-attach, corrected global remove/list/stats semantics, and repaired integration tests that still assumed old stdio or daemon lifecycle behavior.
- **Impact:** Cross-workspace behavior now matches the approved design, build warnings are cleared, and both calibrated verification tiers pass on the branch (`cargo xtask test dev`, `cargo xtask test system`).
