## Threaded daemon runtime mutation gate registry

- **WHAT:** Added runtime mutation-gate plumbing from `DaemonRuntimeContext` into daemon session handlers and watcher runtimes. Handler writer paths now acquire gates through `JulieServerHandler::acquire_mutation_gate`; watcher queue/repair/drain paths use `timed_acquire_gate_with_registry` with the injected registry.
- **WHY:** Review found `DaemonRuntimeContext::mutation_gate_registry` was decorative: production daemon writer paths still used the process-global mutation gate registry, so test runtime isolation did not apply to real writer plumbing.
- **HOW:** Introduced a small explicit interface: `DaemonSessionDependencies` carries `Arc<MutationGateRegistry>`, handlers store it, `WatcherPool`/`IncrementalIndexer`/`QueueRuntime` carry it for background writer paths. Existing stdio/standalone constructors still default to `Registry::global()`.
- **IMPACT:** Daemon runtime registry is now live state for index/register/refresh/startup repair and watcher writer loops. Added regression test proving a daemon session handler created with an isolated runtime is not blocked by a held global gate.
