## Phase A + B + C complete + fast_refs fixture fix

Closing out the world-class-systems Phase A/B/C plan on `feat/daemon-split-and-reranker` and clearing a latent fixture bug that A2.2c had exposed.

### What landed this session

- **B.3** InProcessDaemon test harness (`27a42bf7`) — `src/tests/harness/{mod,in_process}.rs` wraps `DaemonApp::new + serve` behind a 3-line builder; two self-tests prove B.1's per-test gate registry + B.2's idempotent tracing hold under concurrent fixtures.
- **C.1** parse_query (`f9b9507c`) — pure-function `ParsedQuery { intent: Free | Symbol(SymbolKind) | Test, terms, target_terms }` + 24 inline unit tests. Intent threshold is ≥3 tokens.
- **C.2** rerank (`fe1b711f`) — pure-function scorer with named boost constants (`EXACT_TITLE_BOOST=100`, `PHRASE_BOOST=260`, etc.), `Candidate::builder()`, and per-rule unit tests. `rerank_score(query, candidate)` exposed for in-place scoring without forcing the Vec<Ranked> shape on callers. Sort tie-break is title-asc, path-asc.
- **C.3** schema enrichment (`dc237f2e`) — three new STRING|STORED Tantivy fields (`role`, `test_role`, `capability_flags`), derived during `add_symbol_with_context` via new `classify_role / test_subrole / is_vendor_path / is_generated_path / is_source_language` helpers in `src/search/scoring.rs`. Schema sig changes; the auto-recreate path in `SearchCompatMarker` rebuilds old indexes transparently on first start.
- **C.4** wire reranker (`9848748f`) — `apply_reranker_to_symbol_results` + `apply_reranker_to_content_results` hooked into both definition_search (hybrid + keyword) and content_search paths, before centrality/promote. Reranker reweights, never filters; target-specific `exclude_tests` defaults preserved. New fixture `fixtures/search-quality/test-helper-discoverability.json` (10 queries) + `c4_test_helper_discoverability.rs` (with-flag + without-flag, `#[serial]` env isolation).
- **C.5** flip default-on (`200233a8`) — `reranker_enabled()` now returns `true` unless `JULIE_RERANKER_ENABLED=0`. Sign-off evidence: 33/35 dogfood pass with flag ON, identical to flag-OFF baseline.
- **Fixture fix** (`97aa8400`) — `setup_handler_inner` in `src/tests/tools/search_quality/helpers.rs` was rebuilding workspace ID as `{tempdir_name}_{first-8-of-full-id}`, which sliced the safe_name prefix instead of the hash. Fast_refs (migrated to pooled-DB lookup in `28e79d7b` A2.2c-followup) couldn't find the workspace SQLite. Replaced the rebuild with the canonical full ID from `generate_workspace_id`. Dogfood now 35/35.
