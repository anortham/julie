## Dogfood batch: 6 daemon/search findings resolved

After merging the daemon-split and C.3-C.5 reranker work, surfaced 6 issues during dogfooding and worked all of them in one session.

**Changes:**
- **#1 Reranker dead for mixed queries** (`src/search/scoring.rs:339`) — `is_nl_like_query` veto flipped from `any` to `all`; "how does fast_refs find callers" now engages reranker + path prior. Tests in `hybrid_search_tests.rs` and `tantivy_path_prior_tests.rs` inverted from codifying-the-bug to asserting-correct.
- **#2 Per-tool-call SymbolDatabase cold-open** (`src/handler/tool_metrics.rs:64`) — `run_metrics_writer` now acquires from `WorkspacePool` instead of `SymbolDatabase::new(db_path)` every tool call. Added LogCapture-based regression test (`test_metrics_writer_reuses_pooled_connection_no_cold_open_per_call`).
- **#3 12-daemon startup thundering herd** (`src/daemon/mod.rs:308` + `app/helpers.rs:acquire_or_yield_to_existing_daemon`) — singleton lock acquired BEFORE port bind. Losers exit `Ok(())` silently with status 0. `DaemonConfig.daemon_lock: Option<DaemonLockGuard>` threads guard through; back-compat for tests that construct DaemonConfig without pre-acquired lock.
- **#4 dev-restart kills calling Claude Code session** (`xtask/src/dev_workflow.rs::run_dev_restart`) — now soft by default (no SIGTERM). `--force` opts into legacy SIGTERM. Adapter classifies post-output transport errors as `Terminal` (intentional, safe), so soft mode keeps the calling session alive while daemon auto-restarts on next session disconnect.
- **#5 Drain timeout** — already 60s default with env override [1,120] as of 2026-05-06 (PR3 Task 3.2 in daemon-windows-lifecycle plan). TODO.md updated with resolution evidence pointing at all callsites going through single `drain_timeout()` helper.
- **#6 Tantivy schema rebuild on upgrade** — verified `index_is_compatible` → `RecreatedIncompatible` → `repair_recreated_open_if_needed` chain auto-rebuilds from canonical SQLite when C.3's new fields (role/test_role/capability_flags) mismatch the persisted `julie-search-compat.json` signature. Documented operator impact in `docs/OPERATIONS.md`.
