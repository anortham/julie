## Connection-per-request read snapshot fixes

- **WHAT:** Added pooled read snapshot support for `SymbolDatabase`, rollback-on-drop cleanup for pooled connections, primary read handler migration for `get_symbols`, `deep_dive`, `get_context`, and `blast_radius`, and idle connection eviction from `WorkspacePool` sweeps.
- **WHY:** Primary MCP reads still used the legacy shared `Arc<Mutex<SymbolDatabase>>`; pooled connections also could be returned with an open transaction.
- **HOW:** Introduced `ReadSnapshot` owning a pooled `SymbolDatabase` and beginning `BEGIN DEFERRED TRANSACTION`; snapshot drop rolls back. `PooledConn::drop` now rolls back or discards leaked transactions before returning to the pool. Added primary read regression coverage holding the legacy mutex while primary tools run.
- **IMPACT:** Primary/default workspace reads no longer block on the legacy DB mutex in covered handlers. Idle sweeper now trims stale DB connections. Remaining structural risks: runtime mutation gate registry is not threaded into writer APIs; `WorkspacePool::get_or_init` still holds the global write lock across slow init.
