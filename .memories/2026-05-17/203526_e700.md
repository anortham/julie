## P2.1 done: labeled query corpus authored (v1, 45 queries)

**WHAT:** Wrote `docs/eval/julie-search-corpus-v1.json` — 45 queries across 6 categories, all targeting symbols/files that exist on this repo's main branch.

**Categories:**
- exact-symbol (8): CamelCase + snake_case identifier lookups
- symbol-intent (8): multi-word mixed prose+identifier that should resolve to one symbol
- test-intent (8): queries where the user wants test code (tests the test-intent override on NL path prior)
- file-path (6): path/basename lookups
- concept (10): natural-language behavior questions (tests NL path prior to demote docs/tests/fixtures)
- qualified-name (5): partial qualified names (tests DB-rescue path)

**Schema per query:** `{id, query, category, expected_paths[], source, notes?}`. The ablation harness will match by substring of file_path against any expected path.

**WHY:** Plan P2.1 requires a labeled corpus to drive ablation. Not the eros corpus (overfit). Real symbols + real path layouts so we measure actual quality, not test-set memorization.

**HOW:**
- Verified every expected_path via `fast_search` (corrected ManageWorkspaceTool, ParentReferenceContext, swapped Router for WatcherPool because Router was an axum import, not a Julie-domain symbol).
- 45 queries vs the plan's 40–60 target.
- Cross-language queries deferred (Q1) — Julie is monolang Rust.

**IMPACT:** Unblocks P2.2 (ablation harness) and P2.3 (baseline run). Corpus is small enough to run all four ablation modes inside seconds-to-minutes once the harness exists.
