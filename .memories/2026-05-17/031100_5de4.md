## Reviewed and hardened daemon split/search reranker branch

### WHAT
Completed a broad branch review against the daemon split and search reranker plan/design, then fixed review findings across daemon upgrade safety, discovery, search reranking, connection/read snapshots, mutation gate routing, xtask routing, plugin extraction, and watcher shutdown.

### WHY
The branch changes daemon startup/adapter ownership, user upgrade paths, search ranking defaults, and shared workspace mutation coordination. These are high-risk areas for existing users upgrading in place and for agents using Julie under concurrent MCP sessions.

### HOW
- Fixed plugin pre-extraction daemon stop behavior for daemon-split upgrades and made archive-version markers rerun stop preflight.
- Reworked daemon singleton/discovery handling so the new daemon owns `daemon.lock`, no longer writes legacy PID artifacts, and publishes loopback discovery host matching the bound MCP listener.
- Fixed legacy attach to read `daemon-mcp-transport.json` instead of dashboard `daemon.port`, with deterministic fake legacy-daemon E2E tests.
- Routed daemon session/watcher mutation gates through the runtime registry and added read-snapshot connection handling.
- Fixed search reranker/test-default behavior and moved reranker tests out of implementation.
- Split daemon app handle/runtime modules to keep touched files under project limits.
- Added a watcher shutdown regression test for pending queue + held mutation gate, then made watcher gate acquisition cancellation-aware and rescan-marking to avoid force-reindex shutdown deadlocks.
- Split documentation indexing into its own xtask system bucket so the integration bucket does not hide long watcher/doc checks behind a single timeout.

### IMPACT
Upgrade path is safer for existing users: old daemon artifacts are respected for attach/refusal, new daemon discovery uses a valid loopback host, plugin extraction stops prior daemon binaries before replacement, and watcher shutdown no longer wedges when force reindex owns the mutation gate. Verification now covers exact regressions plus dev/system/dogfood/parser-upgrade/plugin gates.
