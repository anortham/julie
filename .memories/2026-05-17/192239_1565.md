## Search consolidation plan committed (post-codex review)

Following the test-intent path-prior fix (commit `83189a45`), the user asked for a sober architecture review of julie's search to find the overcomplicated facade. I gave an opinion, then ran it past codex-cli for adversarial review. Real consensus emerged plus codex caught defects I missed.

### Plan committed at `805dd85a`

Path: `docs/plans/2026-05-17-search-consolidation-plan.md`

Structure:
- **Phase 1**: Fix confirmed bugs
  - B1: NL path prior applied twice (`index.rs:602` + `text_search.rs:592`)
  - B2: `capability_flags` is dead schema baggage
  - B3 (informational): reranker has two integration paths (`rerank()` vs `rerank_symbol_score`)
- **Phase 2**: Build ablation harness against a labeled corpus authored from real queries (NOT the eros corpus — overfit)
- **Phase 3**: Cut suspected non-earners gated on ablation numbers
  - S1: Reranker (in-code comment at `text_search.rs:237` says flag-on dogfood matched flag-off 33/35)
  - S2: Hybrid/embeddings default-on (PyTorch + CodeRankEmbed + sidecar)
  - S3: RRF→BM25 ×200 rescaling band-aid
- **Phase 4**: Daemon refactor — explicitly OUT of scope, separate plan

### Codex corrections I had to absorb

- Targets are 3 (content/definitions/files) not 4 — annotations is query syntax in definitions
- "Collapse to one packed doc" misses content's intentional neutral-score 0.0 (`execution.rs:225,262`)
- DB rescue is qualified-name truncation (`text_search.rs:613`), NOT centrality
- Daemon complexity is load-bearing (watcher pool, drain, singleton startup) — refactor not delete

### Explicit non-goals captured

- No schema rewrite
- No target collapse  
- No daemon cuts
- No chasing eros's 0.98 MRR (benchmark-fit, not search quality)
