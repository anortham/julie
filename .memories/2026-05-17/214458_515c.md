## Completed search-consolidation plan end-to-end (P1.1 → P3.3 + Codex review)

Implemented all of `docs/plans/2026-05-17-search-consolidation-plan.md` and applied a Codex pass over the result. User's next priority (verbatim): the daemon fragility deep dive — "we can't ship a single update until it's fixed."

### Phase 1 — confirmed bugs

- **P1.1** (`00b71b37`): NL path prior was applied twice — inside `SearchIndex::search_symbols` / `_relaxed` AND inside the assembly layer (`definition_search_with_index`). Removed the retrieval-layer call. Pinned the contract with 5 tests in `src/tests/tools/search/nl_path_prior_pipeline_tests.rs` (3 dynamic ratio tests + 2 static `include_str!` source-text contract tests added post-Codex review).
- **P1.2** (`88dc0930`): Dropped the `capability_flags` Tantivy field — dead schema, zero readers. Schema-version fingerprint already covers auto-rebuild on existing user installs.

### Phase 2 — measurement infrastructure

- **P2.1** (`1f77cd93`): authored 45-query labeled corpus at `docs/eval/julie-search-corpus-v1.json` covering 6 categories (exact-symbol, symbol-intent, test-intent, file-path, concept, qualified-name).
- **P2.2** (`287a64d9`): shipped `cargo xtask eval ablation` — drives the same `definition_search_with_index_for_ablation` production traverses. Reranker toggled via `JULIE_RERANKER_ENABLED` env, embedding provider passed as per-call param. File-path queries routed through `SearchIndex::search_files` to match production. Writes per-query / per-category JSON at `docs/eval/julie-search-ablation/<date>-<commit>-baseline.json`.
- **P2.3**: captured baseline at HEAD. Current numbers: keyword-only MRR@10=0.324, keyword+reranker MRR@10=0.332 (+2.5% global, +33% concept-category but driven by *one* query). Hybrid modes structurally wired but skipped (fixture has no `symbol_vectors`).

### Phase 3 — decisions

- **P3.1 reranker**: **DEFER** (Codex reframed from "KEEP"). Data too thin: per-category n=10, the concept lift collapses to a single moving query. A t-test wouldn't rescue this. Recommend corpus expansion to 25-30 queries per category + fixture regeneration before revisiting.
- **P3.2 hybrid**: **data-blocked** (not silent-deferred). Documented unblock path: add `--source ~/.julie/indexes/<id>` flag to harness, point at a daemon-indexed workspace with embeddings.
- **P3.3 consolidation**: **verified clean** by code audit. After P1.1, pipeline has exactly one pass per concern (centrality / NL path prior / language affinity / exact-name promotion / reranker) in both hybrid and keyword branches. `apply_important_patterns_boost` flagged as a minor arch quirk (lives in retrieval) but not duplicated.

### Codex review (`925577a9`) — all findings addressed

1. **Critical harness bug**: `db_has_embeddings` checked `embeddings`/`symbol_embeddings` tables — Julie uses `symbol_vectors` (sqlite-vec virtual table per `src/database/vectors.rs:3`). Hybrid modes would have silently skipped even on workspaces WITH embeddings. Fixed + now also counts rows.
2. **Env-restore leak**: replaced post-loop restore with `EnvGuard` (RAII drop).
3. **MRR@10 mislabeled when `--limit != 10`**: renamed `mrr_at_10` → `mrr_at_k` in JSON, added top-level `k` field, recompute uses `limit`.
4. **Hybrid NL-prior test gap**: added 2 static `include_str!` source-text contract tests that fail loudly if `apply_nl_path_prior` reappears in `hybrid.rs` or `index.rs`.
5. **P3.1 framing**: reframed "KEEP" → "DEFER" — honest about the evidence.
