## Search Reranker Default Fixes

- **WHAT:** Implemented content-search default handling and reranker integration for the real line-mode path. Added regression coverage for NL content default test exclusion, explicit test inclusion, test-intent ranking, and inline test-helper metadata ranking under centrality pressure.
- **WHY:** FastSearch content requests were routed through line-mode, so they bypassed the C.4 content reranker and kept the old `exclude_tests=None => include tests` behavior. Symbol reranking also classified candidates by path only, discarding SQLite/extractor metadata such as `is_test` and `test_role`.
- **HOW:** Added `effective_content_exclude_tests` in `src/tools/search/line_mode.rs`, reranked `ContentSearchResult` file candidates before line expansion, enriched symbol reranker candidates from SQLite via `get_symbols_by_ids`, and moved centrality before additive reranker boosts in definition search.
- **IMPACT:** Content NL queries now hide tests by default unless caller opts in, `test ...` intent keeps/ranks tests, definition search still includes tests by default, and inline test helpers in source files can be classified from metadata rather than path heuristics.
