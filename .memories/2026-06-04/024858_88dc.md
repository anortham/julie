## Julie Rescue Phase 2 — planning workflow results (11-agent, adversarially verified)

Ran an ultracode planning workflow (6 investigate → 1 synthesize → 4 adversarial verify; ~1M tokens, 316 tool calls) to design Phase 2 of the crate split: extract `julie-tools` / `julie-runtime` / `julie-pipeline` from the top crate, gated on a `ToolContext` facade that frees tools from the 24-field `JulieServerHandler` god-object (`src/handler.rs:191`).

### Recommended design (corrected post-verify)
- **DAG (7 crates):** julie-core → julie-index → **julie-context (NEW)** → julie-pipeline → julie-tools → julie-runtime → top crate `julie`.
- **`ToolContext` = `#[async_trait]` trait** (3 hottest accessors are async: `embedding_provider` h.rs:1282, `primary_pooled_database` :1889, `get_search_index_for_workspace` :2261). `JulieServerHandler` impls it in the top crate; tools hold only `&dyn ToolContext`.
- **`julie-context`** = new tiny leaf holding the trait + relocated `SpilloverStore` + value types (ResolvedTarget, RefactorMetrics). Keeps julie-core a pure data leaf.
- **`julie-pipeline` extracts FIRST and clean** — `indexing_core/**` + `embeddings/**` have ZERO `JulieServerHandler` refs (verified).
- **Test relocation** via `FakeToolContext` in julie-test-support; ~498 handler-free tool unit tests relocate (relink cure), 159 handler-instantiating tests stay in top-crate binary.

### 5 BLOCKERS the verify pass caught (must be in the plan)
1. **search→ManageWorkspaceTool upward edge** (`search/mod.rs:671,679`): fast_search instantiates ManageWorkspaceTool (which moves UP to runtime) to index-on-pending → add facade method `ensure_target_workspace_indexed_if_pending()`, impl in top-crate handler.
2. **resolve_workspace_filter facade gap** (`navigation/resolution.rs:135`, called by all 11 read+edit tools): needs 5 handler caps incl. a session-activation MUTATION (`activate_workspace_with_root` :174) → make `resolve_workspace_target()` encapsulate the WHOLE resolver verbatim in top crate.
3. **P0 relocation list incomplete**: tool source consumes `crate::utils::{serde_lenient×29, file_utils, token_estimation, language, cross_language_intelligence, string_similarity}` + `crate::mcp_compat` → all must relocate to julie-core/down or julie-tools can't compile.
4. **commands subtree daemon coupling**: moving `tools/workspace/commands/**` wholesale to runtime breaks — `registry/cleanup.rs` + `registry/mod.rs` construct daemon types and daemon sits ABOVE runtime. Only 8/10 command files move; those 2 stay top-crate.
5. **system_readiness default-method + file_policy→file_utils**: confirm `check_system_readiness` (health/checker.rs:104) truly only needs the 2 accessors before making it a default trait method; `file_policy`→`utils::file_utils` is an unlisted edge (file_utils → core).

Reality check: facade is ~20 methods, NOT the synthesizer's claimed 16; "only 2 raw handler-field derefs" was undercounted.

All 4 verify lenses = **design-needs-revision** (not broken); core thesis + lowest-first ordering HOLD.</description>
