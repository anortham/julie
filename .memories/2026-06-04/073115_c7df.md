## U1 follow-up: test binary compile fix + warning cleanup

### WHAT
Two post-relocation compile fixes committed at `435fc2f6`.

### WHY
`cargo build` skips `#[cfg(test)]` so the original U1 gate gave a false green. The real gate `cargo nextest run -p julie --no-run` revealed the test binary couldn't compile.

### HOW
1. **`sanitize_name` visibility**: Was `pub(crate)` in `julie_core::workspace::registry`. Test at `src/tests/tools/workspace/registry.rs` uses glob import `use crate::workspace::registry::*` and calls it directly. Fixed: `pub(crate)` → `pub` in julie-core; added `#[cfg(test)] pub use julie_core::workspace::registry::sanitize_name` in top-crate re-export.

2. **Unused import warning**: `is_sensitive_workspace_root` and `sensitive_root_candidates` in `src/workspace/root_safety.rs` were re-exported unconditionally but only used in test code. Fixed by gating their re-export with `#[cfg(test)]`.

### IMPACT
- `cargo nextest run -p julie --no-run` — PASS, zero errors, zero warnings
- `cargo nextest run -p julie-core --no-run` — PASS
