## Updated Julie Agent Tier Policy

- **WHAT:** Updated `RAZORBACK.md` so model routing distinguishes docs-only work from evidence interpretation and gate ownership.
- **WHY:** A prior delegated run showed that a medium mechanical worker can mishandle a failing replay gate by treating stale evidence as acceptable.
- **HOW:** Mechanical workers are now barred from owning failing tests, replay evidence, metrics, or acceptance gates. Bounded implementation workers route to `gpt-5.4-mini xhigh`; gate-review lanes route to `gpt-5.3-codex high`; workers must state gate invariants and hard-gate versus report-only metrics.
- **IMPACT:** Future Julie plans should no longer route acceptance-gate interpretation to low/medium mechanical workers, and assigned verification failure must stop the worker unless the plan says to update that gate.
