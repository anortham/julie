## Request-level multi-glob diagnostics

### WHAT
Added an early fast-search diagnostic for whitespace-separated multi-glob `file_pattern` values such as `src/** docs/**`.

### WHY
These malformed scopes were landing in the zero-hit bucket after readiness checks and search work, which hid the root cause and wasted work opening the workspace index.

### HOW
- Added `src/tools/search/input_diagnostics.rs` to detect request-level malformed `file_pattern` values before workspace resolution.
- Wired `FastSearchTool::execute_with_trace()` to return an early `FastSearchExecution` with zero hits, `strategy_id = fast_search_input_diagnostic`, and the same syntax-hint text already used by the formatter.
- Added `SearchExecutionResult::input_diagnostic()` in `trace.rs` so the diagnostic result can stamp `file_pattern_diagnostic` and `hint_kind` while preserving `execution: Some(...)` telemetry.
- Added regression tests for content, files, and telemetry serialization.

### IMPACT
- `file_pattern="src/** docs/**"` now returns the syntax correction without reaching readiness checks or Tantivy search.
- Telemetry records `file_pattern_diagnostic=whitespace_separated_multi_glob` and `hint_kind=file_pattern_syntax_hint` for the diagnostic path.
- Literal-space glob behavior stays intact because the existing whitespace-separation heuristic is reused.
