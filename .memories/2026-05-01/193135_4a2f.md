## WHAT
Added content zero-hit hints that steer path-like queries to `search_target="files"` and symbol-like queries to `search_target="definitions"`.

## WHY
Content misses were still leaking through when the query shape already showed the target choice was off. That left the agent repeating the same wrong search mode.

## HOW
- Added `HintKind::FileTargetHint` and `HintKind::DefinitionsTargetHint` with `snake_case` serde labels.
- Added path-shape detection in `src/tools/search/hint_formatter.rs` for slash and backslash paths, known file extensions, and directory markers.
- Added symbol-shape detection for `::`, dotted identifiers, call-shaped queries ending in `(`, CamelCase, and snake_case single-token queries gated on `LineMatchMiss` or `TantivyNoCandidates`.
- Updated the content zero-hit precedence ladder in `src/tools/search/mod.rs` and kept trace stamping on the selected hint kind.
- Added tests for target selection and telemetry serialization.

## IMPACT
- Path-like misses such as `fixtures/real-world/php/index.php` and `src/tools/search/mod.rs` now point at `files`.
- Symbol-like misses such as `ArgAction::SetTrue`, `OS.has_feature`, `format_line_mode_output`, and `fast_refs_metadata(` now point at `definitions`.
- The trace now records the new hint kinds cleanly for telemetry consumers.

## EVIDENCE
- `cargo nextest run --lib task2_target_hints 2>&1 | tail -10` passed after formatting.
