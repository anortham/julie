## Task 3 trace promotion and strategy fields

### WHAT
Added `SearchTrace` metrics fields for `line_match_strategy`, `definition_exact_match`, and `target_hint`, then wired them through content search, definition search, and telemetry serialization.

### WHY
Future metric passes need direct trace data for content strategy, definition promotion, and zero-hit hint targets. The old behavior forced downstream code to infer too much from formatter output and prose.

### HOW
- Extended `SearchTrace` with neutral defaults and serialization support.
- Exposed the definition-name matching helper from `formatting.rs` so formatter logic and telemetry share one rule.
- Stamped content traces from the live `LineModeSearchResult.strategy` label.
- Stamped definitions traces when the promoted exact-match formatter path would apply.
- Stamped zero-hit target hints for `files` and `definitions`.
- Added focused tests for default serialization, content telemetry, definition promotion, and helper reuse.

### IMPACT
Telemetry now has direct fields for the search behavior Task 3 cares about, and the trace shape stays source-compatible for older callers through explicit defaults.
