## Watcher Pipeline Fixes (A through G) Complete

All 7 watcher bugs fixed, 16 tests green (1 ignored).

### WHAT
Fixed 7 reliability bugs in `src/watcher/` as part of the pipeline-fixes team task:

- **Fix A** - macOS `RenameMode::Any` now checks `path.exists()` to route correctly to Deleted vs Modified; previously always emitted Modified causing orphaned DB/Tantivy entries for gone paths
- **Fix B-a** - Removed redundant `path.exists()` guard from `handle_file_deleted_static`; atomic-save detection belongs solely in `dispatch_file_event` to avoid TOCTOU race
- **Fix B-b** - `handle_file_created_or_modified_static` returns `Result<bool>` (Tantivy ok flag); added `tantivy_dirty` HashSet to `IncrementalIndexer`; queue processor retries dirty files each tick
- **Fix C** - `queue_file_change` sets `needs_rescan` AtomicBool on overflow instead of silently dropping; processor runs full indexed-file staleness pass after queue drains; dedup re-queue changed from `break` to `continue` (HOL blocking fix)
- **Fix D** - `stop()` drops watcher first (closes channel tx), then joins tasks via `.await` instead of `.abort()`; in-flight work completes gracefully
- **Fix E** - `reembed_symbols_for_file` (sync blocking call) wrapped in `tokio::task::spawn_blocking` everywhere in `dispatch_file_event`
- **Fix F** - `dispatch_file_event` returns `Option<PathBuf>` for atomic-save skip path; caller inlines `last_processed.lock().remove()` instead of spawning detached task
- **Fix G** - Replaced stub `test_blake3_change_detection_placeholder` with real test: tampers DB hash, verifies skip, writes new content, verifies re-index, asserts new symbol present

### WHY
Pre-existing reliability bugs causing: orphaned DB entries on macOS renames, missed deletions during atomic saves, Tantivy/SQLite desync on errors, queue overflow silently dropping events, HOL blocking when dedup re-queues, async runtime starvation from blocking reembed calls.

### HOW
- `events.rs`: Added `needs_rescan: &Arc<AtomicBool>` param to `process_file_system_event` and `queue_file_change`
- `handlers.rs`: Changed return type to `Result<bool>`, removed TOCTOU guard
- `mod.rs`: New `tantivy_dirty` field, rewritten `dispatch_file_event` (no more generic callback), dirty-file retry loop, overflow rescan, graceful stop

### IMPACT
- One pre-existing test updated: `test_delete_handler_skips_when_file_still_exists` renamed to `test_delete_handler_always_cleans_up` with inverted assertion (old test was validating the bug)
- Task #2 of pipeline-fixes team completed; task #4 (full test run) now unblocked pending task #3
