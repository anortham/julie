## Edit Tool Enforcement: Spec + Plan Complete

Brainstormed and designed a 3-layer enforcement system to make agents use Julie's edit_file/edit_symbol tools instead of the Read+Edit antipattern.

### What was done
- Researched enforcement patterns from Serena (tool exclusion, per-context descriptions) and Superpowers (rationalization prevention, hard gates, iron laws)
- Designed 3 independent layers: upgraded SessionStart instructions, leaner PreToolUse hook on Edit, rewritten editing skill
- Wrote spec: `docs/superpowers/specs/2026-04-05-edit-tool-enforcement-design.md`
- Wrote implementation plan: `docs/superpowers/plans/2026-04-05-edit-tool-enforcement.md`

### Key decisions
- **No Read hook**: rejected because Read fires 20-50x per session, token cost of nagging undermines Julie's value prop
- **No hard blocks**: tool names differ across harnesses, fallback path must stay open
- **Skill description rewrite**: old description matched tool names, new one matches user intent words (fix, change, update, modify, refactor...)
- **Rationalization prevention tables** in both SessionStart and skill body, inspired by Superpowers pattern

### Plan structure
- Task 1: Rewrite editing skill (both repos)
- Task 2: Upgrade SessionStart instructions + JULIE_AGENT_INSTRUCTIONS.md
- Task 3: Upgrade PreToolUse Edit hook (both repos)
- Task 4: Verify and test
