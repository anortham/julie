## Pipeline Review Complete: 27 Findings Across 4 Reviewers

Thorough review of Julie's indexing/embedding/watcher/editing pipeline using 3 Sonnet agents + Codex cross-cutting review. Motivated by IU onboarding.

### Review Structure
- **index-reviewer**: Initial indexing + session-connect catch-up (5 findings)
- **watcher-reviewer**: Filewatcher + incremental indexing (8 findings)
- **embed-edit-reviewer**: Embeddings lifecycle + editing tools (9 findings)
- **Codex cross-cutting**: Interaction bugs at component boundaries (5 findings)

### Most Critical Discovery
**Edit tools permanently break watcher re-indexing**: `update_file_hash` after edit commit means the watcher's blake3 check sees "unchanged" and skips symbol re-extraction. `last_indexed` is also bumped, so catch-up skips it too. Edited files never get symbols re-extracted unless manually force-reindexed. Every subsequent edit_symbol on that file operates on stale line numbers.

### Severity Summary
- 2 Critical (force-reindex ref clears primary embeddings; edit hash breaks watcher)
- 9 High (wrong workspace_id, macOS rename orphans, queue overflow, delete TOCTOU, stale vectors after cancel, consecutive edit corruption, catch-up vs watcher race, SQLite/Tantivy atomicity, force reindex vs ref watcher)
- 9 Medium
- 7 Low

### Key Interaction Bugs (Codex)
1. Edit hash update prevents watcher re-extraction (Critical)
2. Catch-up and watcher can race on same file, older snapshot overwrites newer (High)
3. SQLite/Tantivy updates not atomic, Tantivy failures warn-only with no repair (High)
4. Force reindex on reference workspace doesn't stop its watcher (High)
5. Bulk and incremental embedding paths can race on symbol_vectors (Medium)
