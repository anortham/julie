## Optimized read-tool broad materialization paths

- **WHAT:** Added batched file-content loading for content search verification and a filtered identifier lookup for incoming impact/get_context edges.
- **WHY:** `content_search_with_index` fetched one file blob per Tantivy candidate, and `identifier_incoming_edges` materialized identifier rows that could be filtered by kind/container earlier.
- **HOW:** Added `SymbolDatabase::get_file_contents_by_paths` and `SymbolDatabase::get_identifiers_by_names_kinds_excluding_containers`; routed `content_search_with_index` through a path-to-content map and `identifier_incoming_edges` through SQL kind/container filtering with a bind-count guard.
- **IMPACT:** Reduces N single-row content fetches to chunked `IN` queries, and narrows identifier rows before Rust-side edge resolution. Focused RED/GREEN tests cover both helpers plus the graph and content-search call paths.
