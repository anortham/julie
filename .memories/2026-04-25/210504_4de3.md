## SQL performance review follow-up

### WHAT
Closed the remaining SQL and indexing hot spots from the review pass.

- Added language-scoped temp-table matching for test linkage fallback.
- Added batched file-content lookup and SQL-filtered identifier expansion.
- Replaced per-row symbol existence probes in bulk persistence with chunked symbol ID prefetch.
- Split Tantivy projection so SQLite is released while the index mutex is held or while documents are applied.
- Split post-index analysis into separate SQLite lock windows.

### WHY
The hermes-agent indexing run exposed query shapes and lock scopes that could scale poorly on larger workspaces.

### HOW
Used focused regression tests for the language ambiguity guard, batched reads, identifier expansion, missing-symbol normalization, and projection lock behavior. Verified with the changed test gate, which fell back to dev.

### IMPACT
Indexing should spend less time in cartesian joins, N+1 query loops, and long SQLite mutex holds. Search and graph expansion paths now push more filtering into SQLite instead of materializing broad candidate sets.
