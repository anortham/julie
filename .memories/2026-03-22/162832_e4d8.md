## v6 Daemon + Adapter Architecture Design Complete

### What
Wrote and reviewed the design spec for Julie v6: evolving from per-session stdio processes to a persistent local daemon with thin stdio adapters. Spec at `docs/superpowers/specs/2026-03-22-v6-daemon-adapter-architecture-design.md`.

### Why
Julie's current per-process architecture breaks under modern dev workflows:
- Agent teams spawn multiple MCP clients, each hitting Tantivy `LockBusy`
- VRAM multiplied per process (~270MB x N for embedding models)
- Redundant file watchers and duplicate reference workspace indexes
- User already runs 3+ concurrent Claude Code sessions

### How
- Researched MCP transport options (3 parallel agents): Streamable HTTP is current spec, all 7 major clients support it, rmcp crate has built-in support
- Researched LSP/MCP multi-client patterns: ra-multiplex daemon+adapter is the proven pattern
- Reviewed v4.x git history (49K lines added, 23K ripped out) to learn from past failure
- Designed daemon + adapter architecture: single binary with subcommands, auto-start, IPC via Unix sockets/named pipes
- Two spec review passes with all critical issues addressed (startup race condition, IPC protocol, Windows binary lock, handler refactoring for shared workspaces, safe migration)

### Key Design Decisions
- **Daemon + stdio adapter** (not HTTP-only): Universal MCP client compatibility
- **Centralized indexes** (`~/.julie/indexes/`): Eliminates reference workspace duplication
- **WorkspacePool**: Shared `JulieWorkspace` instances across sessions, new `new_with_shared_workspace()` constructor
- **Advisory file lock** on `daemon.lock`: Prevents concurrent adapter startup race
- **Byte-forwarding adapter**: No JSON-RPC parsing, one socket per adapter, simplest possible design
- **4 phases**: Daemon core, shared refs, shared embeddings, plugin distribution

### Impact
Design spec committed and ready for implementation planning. This is the architectural foundation for supporting agent teams, parallel sessions, and efficient resource sharing.
