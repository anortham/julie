# Track B: WatcherPool B1-B3 Complete

## WHAT
Implemented `src/daemon/watcher_pool.rs` — the full WatcherPool for v6 Phase 2 shared workspaces. All three Track B tasks (B1, B2, B3) are done in one cohesive module.

## WHY
Prevents N duplicate OS-level file watchers when N sessions share the same workspace. One `IncrementalIndexer` per workspace, reference-counted across sessions with a 5-minute grace period before shutdown.

## HOW
- `WatcherEntry { watcher: Option<IncrementalIndexer>, ref_count: usize, grace_deadline: Option<Instant> }`
- `increment_ref` / `decrement_ref` — testable without real workspaces (watcher stays None)
- `attach()` / `detach()` — creates real IncrementalIndexer on first attach, reuses on reattach
- `reap_expired()` — collects expired entries into a Vec first, drops write lock, then spawns async stop tasks (avoids holding write lock across await)
- `spawn_reaper()` — background task on `Arc<Self>`, calls `reap_expired` every N seconds
- 5 unit tests, all green

## Key Design Decisions
- Used `Option<IncrementalIndexer>` not `Arc<IncrementalIndexer>` because `stop()` requires `&mut self`
- `reap_expired` drops write guard before spawning stop tasks — prevents deadlock if stop() tries to re-acquire any lock
- `decrement_ref` uses `saturating_sub` — safe to call extra times without underflow
