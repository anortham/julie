## C5: Replace WorkspaceRegistryService in manage_workspace registry commands

Added daemon_db-first branches to all manage_workspace registry commands so they use `DaemonDatabase` when running under the daemon (handler.daemon_db is Some), with full `WorkspaceRegistryService` fallback for stdio mode.

### Files Changed

- `src/tools/workspace/commands/registry/add_remove.rs`
- `src/tools/workspace/commands/registry/list_clean.rs`
- `src/tools/workspace/commands/registry/refresh_stats.rs`

### Daemon Mode Behavior (new)

- **add**: instant-attach if ref workspace already has status "ready" in daemon.db; otherwise upsert as "indexing", call index_workspace_files, update_workspace_stats, mark "ready", add_reference
- **remove**: get_workspace → delete index dir → remove_reference → delete_workspace
- **list**: list_references(primary_workspace_id) for ref workspaces, get_workspace(primary_id) for primary row
- **clean**: list_workspaces, delete any whose path doesn't exist on disk
- **refresh**: get_workspace → index_workspace_files → update_workspace_stats
- **stats**: get_workspace for specific ID, or primary+refs summary for overall

### Key Design Decisions

- `generate_workspace_id(path)` used to compute ref_workspace_id (matches what `index_workspace_files` uses internally, so IDs are consistent)
- `handler.workspace_id` is the primary workspace ID in daemon mode (set by `new_with_shared_workspace`)
- `index_workspace_files` calls are unchanged — they route to the right DB based on path canonicalization
- Index size is NOT tracked in DaemonDatabase (update_workspace_stats has no index_size_bytes column); only symbol_count, file_count are updated
