## Added Tree-sitter real-world certification evidence runner

- **WHAT:** Added `cargo xtask certify tree-sitter --real-world` with a checked-in corpus fixture, generated smoke evidence JSON/Markdown, and certification report consumption of that evidence.
- **WHY:** Restored historical verification docs were useful but still manual and stale. The next confidence step needed current, repeatable evidence from real OSS repos instead of another handwritten checklist.
- **HOW:** Implemented an xtask runner that indexes local repos through an isolated Julie home, records deterministic repo evidence (git SHA, file/symbol/relationship/identifier/type/diagnostic counts), and writes `docs/LANGUAGE_REAL_WORLD_EVIDENCE.json`. Added a no-project-writes handler/pool path so external repo indexing does not create `.julie` or `.julieignore` in the repo under test.
- **IMPACT:** `docs/LANGUAGE_CERTIFICATION_REPORT.md` now includes current smoke evidence for Julie/Rust, Zod/TypeScript, and Flask/Python. The extractor bucket checks the generated certification report, and `cargo xtask test changed` exercised the broad dev fallback after shared indexing changes.
