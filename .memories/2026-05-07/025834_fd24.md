## Task 4: Migrated 3 writers to shared mutation gate

All three uncoordinated workspace writers now acquire the shared `MutationGuard` proof token before touching workspace state.

### What changed

- **`src/tools/workspace/commands/index.rs`**
  - Deleted `indexing_lock_cache()` and `indexing_lock_for_path()` (path-keyed local lock, now dead)
  - Replaced `indexing_lock_for_path(&canonical_path)` + `lock().await` with `acquire_gate(&gate_workspace_id).await`
  - Removed `pause_force_reindex_watchers` / `resume_force_reindex_watchers` call sites
  - Added `index_workspace_inner(_guard, handler, path, force)` proof-token method (thin wrapper over `index_workspace_files`)
  - Replaced `test_shared_index_lock_reuses_lock_for_same_path` with two workspace_id-keyed async tests

- **`src/tools/workspace/commands/registry/refresh_stats.rs`**
  - Removed `indexing_lock_for_path` import (line 5)
  - Removed `pause_force_reindex_watchers` / `resume_force_reindex_watchers` imports and call sites
  - Added `acquire_gate(workspace_id).await` before indexing work
  - Calls `index_workspace_inner(&mutation_guard, ...)` instead of `index_workspace_files`

- **`src/tools/workspace/commands/registry/register_remove.rs`**
  - Added `acquire_gate` import
  - Wrapped the previously-uncoordinated `index_workspace_files` call (line 83) with `acquire_gate(&workspace_id).await` + `index_workspace_inner`
  - This is the key correctness fix: register was completely outside the lock contract before

- **`src/tools/workspace/commands/force_safeguards.rs`**
  - Deleted `ForceReindexWatcherPause` struct
  - Deleted `pause_force_reindex_watchers()` and `resume_force_reindex_watchers()`
  - Added `acquire_force_reindex_gate(workspace_id) -> MutationGuard<'static>` (thin wrapper over `acquire_gate`)

### Build state

- `cargo build` (lib): **clean, 0 errors**
- `cargo test --no-run`: 3 errors in `src/tests/integration/watcher_queue.rs` — these call `handle_file_created_or_modified_static` with old arity; caused by Task 2's in-progress changes to `src/watcher/handlers.rs`. **Not our files.**

### Judgment calls

- Gate workspace_id for `handle_index_command`: non-primary target uses `generate_workspace_id(canonical_path)`; primary uses `current_primary_id` with fallback to canonical path string (matches existing stat-registration logic)
- `index_workspace_inner` return type: `crate::tools::workspace::indexing::index::IndexResult` — not re-exported from `indexing/mod.rs` root
- `acquire_force_reindex_gate` kept despite being currently unused — plan explicitly asked for it; downstream tasks will use or Task 5 will retire it
