### Non-obvious operational notes (worth re-finding)
- **`Waited Nms for mutation gate` is watcher-only.** Wired at 4 sites in `src/watcher/runtime.rs` via `timed_acquire_gate(workspace_id, Duration::from_millis(100))`. Force-reindex (`src/tools/workspace/commands/index.rs`), startup repair (`src/startup.rs`), register/refresh-stats (`src/tools/workspace/commands/registry/`) all use plain `acquire_gate` and emit no wait warnings. **The absence of a warning under contended force-reindex is correct, not a wiring bug.** The serialization signal for those paths is the timestamp gap between the previous run's `stage=completed` log and the next run's first phase log.
- **Cross-session WIP can leak into main.** This session caught 6 modified files in main (gdscript identifiers, powershell test rewrite, vue script_setup + fixture, handler probe) that belonged to a parallel codex agent's `codex/tree-sitter-extractor-audit-remediation` branch. Confirmed the audit-remediation worktree had the same files modified with much larger diffs (123-line GDScript, 149-line PowerShell, 40-line handler refactor), so the work was safe to drop from main. Mechanism: when multiple worktrees are open simultaneously, AI-driven edits can land in whichever cwd the harness was last rooted in.
- **`.memories/` is a flat shared directory across worktrees.** Goldfish keys checkpoints by project root, not branch, so any active session writes to the same `.memories/` tree.
