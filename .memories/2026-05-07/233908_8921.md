## Fixed regex fallback symbol spans

The regex extractor now assigns exact spans to fallback lookaround and unicode-property symbols instead of inheriting the root regex node span.

- **WHAT:** Added `NormalizedSpan::from_content_range`, a reusable `BaseExtractor::create_symbol_from_span`, and updated regex fallback recovery to attach exact text-match spans.
- **WHY:** Fallback regex symbols were getting root-node spans, which made their byte/column positions incorrect and broke span assertions for the fallback recovery path.
- **HOW:** The fallback scanners now return both the matched text and its exact `NormalizedSpan`, and the regex patterns module uses that span when creating symbols.
- **IMPACT:** `tests::regex::task15::test_regex_constructs_have_distinct_symbol_kinds` now passes with exact span checks for `(?=foo)`, `(?!bar)`, `\p{Greek}`, and `\p{Letter}`.
