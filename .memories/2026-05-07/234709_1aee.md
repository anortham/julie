## Fixed HTML embedded body offset detection

HTML `<script>` and `<style>` embedded bodies now derive their host byte offset from the real tree-sitter `text`/`raw_text` child node instead of substring search over the parent node text.

- **WHAT:** Verified and completed the TS-RF-005 fix in `crates/julie-extractors/src/html/scripts.rs`, and added a regression for duplicate script/style body text in `crates/julie-extractors/src/tests/html/script_style.rs`.
- **WHY:** Substring-based offset detection could bind to the copy of the embedded body that appeared in an attribute, producing incorrect host spans.
- **HOW:** The extractor now uses the embedded child node's start byte as the host offset, and the regression asserts both byte offsets and line/column mappings against duplicated attribute/body content.
- **IMPACT:** The duplicate-body collision is covered by a focused failing-then-passing test, and TS-RF-005 is marked closed in the review status doc.
