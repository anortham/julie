## Fixed HTML embedded body offset detection

Replaced the HTML script/style embedded-body offset lookup with tree-sitter child-node bounds instead of substring search against the full host element text.

- **WHAT:** Updated `crates/julie-extractors/src/html/scripts.rs` and added a regression in `crates/julie-extractors/src/tests/html/script_style.rs`.
- **WHY:** Substring-based offset discovery could pick the wrong occurrence when the script/style body text also appeared in an attribute.
- **HOW:** The extractor now locates the `text`/`raw_text` child node and derives the embedded offset from that node's start byte plus leading trim, then applies that offset to extracted symbols.
- **IMPACT:** Inline HTML script/style symbols now anchor to the real embedded body instead of a duplicate string elsewhere in the tag; the new regression passes.
