## Bash extractor audit remediation

### WHAT
Reworked the Bash extractor so external command calls stop being emitted as fake `Function` symbols, while `alias` and `source`/dot-source now emit searchable `Import` symbols with metadata.

### WHY
The Bash lane was polluting the symbol index with command invocations and also treating shell builtins too broadly, which broke call tracing and made search results lie about what was actually defined.

### HOW
- Centralized Bash builtin filtering in `crates/julie-extractors/src/bash/commands.rs`
- Changed command extraction to only emit symbols for `alias` and `source`/dot-source
- Added structured import pending relationships for sourced scripts in `crates/julie-extractors/src/bash/relationships.rs`
- Tightened environment-variable classification in `crates/julie-extractors/src/bash/variables.rs` so `local ALL_CAPS=...` does not become a constant
- Added/updated Bash regression tests for external commands, alias/source imports, and local ALL_CAPS handling

### IMPACT
Bash command calls now flow through identifiers plus pending relationships instead of fake function definitions. The remaining verification is blocked by unrelated compile errors in concurrent GDScript test work, but the Bash files themselves no longer show compile errors in the test build output.
