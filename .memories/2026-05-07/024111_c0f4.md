## Task 1 Complete: mutation_gate.rs proof-token API

### What
Implemented `src/workspace/mutation_gate.rs` — a new shared workspace mutation gate with a proof-token API. Added one-line `pub mod mutation_gate;` to `src/workspace/mod.rs`.

### Why
Root cause of 5+ "stale index" bug patches in 2 months: multiple uncoordinated workspace writers (watcher event-processor, repair scan, catch-up indexer, force-reindex) with no shared serialization primitive. The prior pause/resume approach was lossy. This gate gives the type system enforcement of correct usage.

### How — API Surface

- **`MutationGuard<'a>`** — wraps `OwnedMutexGuard<()>` with a private `_guard` field. No public constructor; can only be obtained via `acquire_gate`. Dropping releases the lock.
- **`acquire_gate(workspace_id: &str) -> MutationGuard<'static>`** — looks up or creates `Arc<AsyncMutex<()>>` in a `OnceLock<StdMutex<HashMap<String, Arc<AsyncMutex<()>>>>>` cache, awaits `lock_owned()`, wraps in `MutationGuard`.
- **`clear_cache_for_test()`** — `#[cfg(test)]` only; clears per-workspace cache for test isolation.
- **`compile_fail` doctest** on the module doc proving external construction is a compile error.

### Cache Pattern
Mirrors `indexing_lock_cache()` in `src/tools/workspace/commands/index.rs:15` exactly — same `OnceLock<StdMutex<HashMap<...>>>` idiom. Key difference: keyed by `workspace_id: String` (stable identifier) instead of `PathBuf` (which has lock-identity bugs from path-spelling differences).

### Tests — 5/5 Passing
- `test_same_workspace_id_returns_same_arc` — `Arc::ptr_eq` proves same-key cache hit
- `test_different_workspace_ids_return_different_arcs` — different keys → different Arcs
- `test_guard_drop_releases_lock` — drop releases; timeout-gated re-acquire succeeds
- `test_concurrent_acquisition_serializes` — second acquire blocks while first held; timeout proves it
- `test_proof_token_required_to_call_gated_fn` — gated fn only callable with valid guard

### Impact
Foundation for Tasks 2-4. Any mutation function wired up with `_guard: &MutationGuard<'_>` becomes impossible to call without holding the gate — compile-time enforcement, not a convention.
