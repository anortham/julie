## Completed Task 13 scripting language extraction

### WHAT
Implemented Task 13 across Bash, PowerShell, Lua, GDScript, and shared test detection.

Key changes:
- Bash external commands are call identifiers and pending call relationships, not fake function symbols.
- Bash `alias`, `source`, and dot-source commands emit import symbols and structured pending import relationships.
- Bash local all-caps variables stay variables instead of environment constants.
- PowerShell command calls no longer emit function symbols, builtin cmdlet filtering is centralized, and method signatures are declaration-only.
- Lua function signatures no longer include bodies, and bare `require` emits a real import symbol plus structured pending import relationship.
- GDScript signatures are declaration-only, and attribute-call identifiers and relationships use the rightmost method name with the full receiver prefix.
- Shared Bash, PowerShell, and Ruby test detection now recognizes common framework forms without marking generic production names as tests.

### WHY
The extractor audit found scripting-language correctness gaps that polluted symbol indexes with command definitions, hid import/source constructs, leaked bodies into signatures, and over-relied on path-only test detection.

### HOW
Started with parallel implementation workers. Two mini-worker lanes produced contract or compile issues, so Bash was redeployed to `gpt-5.3-codex` high and the lead fixed the remaining Lua and GDScript correctness issues directly after Julie-assisted review. Stale helper functions made unreachable by the command-symbol cleanup were removed. Golden fixtures were refreshed for GDScript, Lua, and PowerShell.

### IMPACT
Task 13 is ready to commit. Task 14 is next. Verification is strong: focused exact tests, golden, format, diff check, changed-file buckets, parser-upgrade, and dev tier all pass.
