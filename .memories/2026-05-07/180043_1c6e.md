## Added Executable Tree-sitter Certification Gate

- **WHAT:** Added `cargo xtask certify tree-sitter` and `--check`, generated `docs/LANGUAGE_CERTIFICATION_REPORT.md`, and wired the check into the `extractors` bucket.
- **WHY:** Manual verification docs kept drifting and could be deleted. This creates a deterministic current-HEAD report from checked-in capability, golden fixture, and historical verification doc state.
- **HOW:** Implemented the xtask command across `xtask/src/cli.rs`, `xtask/src/main.rs`, and focused modules `tree_sitter_certification.rs`, `tree_sitter_certification_data.rs`, and `tree_sitter_certification_report.rs`. The loader hard-fails on missing gap evidence, missing fixture files, malformed expected JSON arrays, pending/structured pending parity drift, invalid gap metadata, and target/implementation capability contradictions. The report summarizes 36 registry rows, 37 golden fixtures, 33 historical matrix rows, 13 raw historical OSS reports, fixture evidence counts, open gaps, and current rows missing from the historical matrix. Updated docs and quick references to point release claims at the generated report.
- **IMPACT:** The extractor bucket now fails if the generated certification report is stale or if the capability matrix points at missing evidence. This turns the restored verification docs into input history instead of an unverifiable source of truth.
