## Verified regex fallback span fix for TS-RF-007

Confirmed the regex extractor working tree already contains the fallback-span remediation for lookaround and unicode-property symbols.

- **WHAT:** Fallback lookaround/unicode-property extraction now carries exact spans from text match ranges into symbol creation, and the regex regression test asserts exact byte/column spans for both constructs.
- **WHY:** The open finding TS-RF-007 reported that fallback symbols could inherit whole-pattern/root-node spans, which would make symbol locations inaccurate.
- **HOW:** The extractor now threads `NormalizedSpan` through the fallback source-scan path and uses `create_symbol_from_span(...)` instead of the default node-derived span path; the direct regex fixture in `crates/julie-extractors/src/tests/regex/task15.rs` checks exact spans.
- **IMPACT:** Targeted `cargo nextest run -p julie-extractors --lib test_regex_constructs_have_distinct_symbol_kinds`, `test_lookarounds_still_extracted`, and `test_extract_lookahead_with_doc_comment` all passed in the current workspace.
