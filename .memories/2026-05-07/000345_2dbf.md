## Fixed Vue script setup line offsets

Script setup symbols were using `VueSection.start_line + row` for several node paths. `VueSection.start_line` is the `<script setup>` tag line, while symbol spans and `byte_for_position` expect 1-based file lines, so symbols on the first content line pointed at the tag instead of source code.

- **What changed:** Added the missing content-line offset for script setup function declarations, variable declarations, default imports, named imports, and standalone Vue macro calls.
- **Why:** The tightened regression test exposed off-by-one spans for script setup symbols.
- **How:** Expanded `test_script_setup_line_numbers_are_file_relative` to assert import, variable, and macro call line numbers; updated the Vue basic golden fixture after the corrected spans shifted symbol keys.
- **Impact:** Vue script setup symbols now point to real source lines and generated relationship/type keys match the corrected spans.
