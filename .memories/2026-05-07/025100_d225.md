## Task 3: Catch-up indexer gated/inner split — complete

Split `run_primary_workspace_repair` into a gated wrapper + inner helper to eliminate the deadlock risk identified in the v2 plan codex review.

### What changed

- **`run_primary_workspace_repair`** (public, gated): calls `acquire_gate(workspace_id)` then delegates to `_inner`. If `require_primary_workspace_identity()` fails (no workspace bound yet, first-run path), falls through to `_inner_ungated` directly — no concurrent writer can exist in that state.
- **`run_primary_workspace_repair_inner`** (pub(crate)): takes `&MutationGuard<'_>` as compile-time proof token, delegates to `_inner_ungated`. For callers already holding the gate.
- **`run_primary_workspace_repair_inner_ungated`** (private): holds the actual repair body. Shared by both paths.
- **Deleted**: `pause_primary_workspace_updates`, `resume_primary_workspace_updates`, `PrimaryWatcherPauseTarget` enum — all three had only one caller (`run_primary_workspace_repair`) and are made obsolete by the gate.

### Why this structure

`tokio::sync::Mutex` is non-reentrant. The old v2 plan would have deadlocked because `run_primary_workspace_repair` holds the gate AND calls into `handle_index_command` which also tries to acquire it. The `_gated`/`_inner` split makes the guard a compile-time parameter — callers inside a held gate use `_inner` variants and never call `acquire_gate` again.

### Verification

- No compile errors from `startup.rs` (confirmed with `cargo build`)
- 7 remaining compile errors are all in `src/watcher/runtime.rs` and `src/tools/workspace/commands/index.rs` — Tasks 2 and 4 in-progress on same branch
- Tests blocked by those parallel-task errors; not regressions from Task 3

### Impact for downstream tasks

- **Task 5** (pause/resume removal): `pause_watcher` and `resume_watcher` on the handler are now dead code — nothing calls them from startup anymore. Task 5 can delete them cleanly.
- **Task 4** (`handle_index_command` migration): when it calls into the repair path while holding its own gate, it must use `run_primary_workspace_repair_inner` (passing `&guard`), never `run_primary_workspace_repair` (which would deadlock).
