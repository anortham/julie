## Shared scripting test-framework detection

### WHAT
Added shared detection for Bash, PowerShell, and Ruby test-framework shapes, plus a focused regression test in the shared detector suite.

### WHY
The detector only recognized Ruby via a narrow `test_` prefix and leaned on path heuristics. That missed common scripting test shapes like BDD-style names in non-test paths.

### HOW
- Added a shared keyword matcher for scripting test names in `crates/julie-extractors/src/test_detection.rs`.
- Routed Bash, PowerShell, and Ruby through language-specific keyword lists.
- Kept Bash and Ruby `test_` prefixes recognized without depending on path names.
- Added `test_bash_powershell_and_ruby_test_framework_detection_covers_common_frameworks` in `crates/julie-extractors/src/tests/test_detection.rs`.

### IMPACT
Bash, PowerShell, and Ruby test symbols now recognize common framework-style names directly, which reduces path-only misses and makes shared test detection behave more like the actual scripting DSLs.
