## Daemon Windows lifecycle hardening landed (PR #17)

Merged 16 commits + 1 report commit covering all 11 findings from the daemon-windows-lifecycle review plus 3 codex-discovered cross-cutting bugs. Single combined PR after user emphasized parallel throughput over staged rollout.

### What shipped (4 thematic PRs collapsed to 1 branch)

- **Shutdown ordering (LIFO)**: explicit `WorkspacePool::shutdown` (commits Tantivy writes, releases file locks), `WatcherPool::shutdown` (drops OS file-watcher handles), async `EmbeddingService::shutdown` with bounded sidecar wait (3s), refactored `run_daemon` shutdown sequence with new `ShutdownArtifacts<'a>` helper struct.
- **PID file hardening**: extended format from `<pid>` to `<pid> <creation_time> <binary_mtime>`. PID-reuse defense via creation_time match, exponential backoff in `create_exclusive` (50ms·2^n cap 5000ms, max 10 retries), proper Windows `ERROR_SHARING_VIOLATION` propagation. Real macOS sysctl impl using 648-byte raw buffer + offset parsing (avoids libc-0.2.x `kinfo_proc` removal).
- **State file + drain + admit**: atomic `write_daemon_state` via temp+rename, configurable drain timeout `JULIE_DAEMON_DRAIN_TIMEOUT_SECS` (default 10s, range [1,120], drain-timeout event raised WARN→ERROR), instrumentation locking in the existing `?`-based admit_initialize short-circuit.
- **Docs**: honest `binary_mtime` Windows caveat, new `docs/OPERATIONS.md` covering fs2 lock semantics + lock/PID/state file behavior, julie_home_hash UTF-8 collision note.

### Codex pre-merge review caught 3 cross-cutting bugs (commit 40f63442)

All real, all fixed. None of these would have surfaced in per-PR review focused on each diff in isolation.

1. **(high) Legacy PID file deletion broke single-daemon invariant during upgrade** — `check_running` deleted any non-3-field file, so a v7.7.x adapter starting against a live legacy daemon would erase its PID file and spawn a duplicate. Fixed with `check_running_legacy_fallback` + new `OwnerState` enum.
2. **(medium) Creation-time lookup failure accepted recycled Windows PIDs** — `Some(actual)` vs `None` was conflated via `unwrap_or(0)`. Fixed with explicit three-way match: match/mismatch/indeterminate; indeterminate preserves the file for retry.
3. **(medium) `WorkspacePool::shutdown` could block forever post-drain** — `std::sync::Mutex::lock()` on a hung tokio task would bypass the drain timeout. Fixed with per-workspace `tokio::time::timeout(2s)` wrapping `spawn_blocking`.
