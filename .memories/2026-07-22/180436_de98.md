## Defined the Phase 3 evaluation-first implementation plan

The approved roadmap now has an executable plan for opt-in mixed impact traversal.

- **WHAT:** Added `docs/plans/2026-07-22-impact-mixed-traversal.md` with a versioned corpus, locked default snapshot, report-only latency scorecard, TDD implementation slice, and exact promotion/branch gates.
- **WHY:** Current `blast_radius` web mode runs the ordinary reverse walk and then appends only direct web callers, so it cannot cross ordinary to web to ordinary paths.
- **HOW:** Preserve `BlastRadiusTool.mode` and make the existing reverse BFS policy-driven. Default remains byte-identical; web mode shares one frontier, visited set, depth budget, ranking path, and deterministic order across ordinary, identifier, HTTP, and SQL edges.
- **IMPACT:** Phase 3 can proceed without guessing, silent default changes, language-specific traversal branches, or a precision-blind promotion.
