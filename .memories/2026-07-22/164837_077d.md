## Recalibrated the get-context graph bucket timeout

The final `cargo xtask test dev` gate twice exhausted the bucket's cumulative 45-second timeout without any assertion failure.

- **WHAT:** Raised `tools-get-context-graph` from 12 expected / 45 timeout seconds to 25 expected / 90 timeout seconds and updated its exact manifest contract.
- **WHY:** The bucket has four sequential commands. Under current machine load, commands three and four each took about 33 seconds, producing a valid 68.7-second pass that the old cumulative ceiling could never allow.
- **HOW:** TDD first changed the expected contract and proved it failed against the old manifest, then updated the manifest. The exact bucket passed 4/4 under the new ceiling and all 179 xtask tests passed.
- **IMPACT:** `dev` remains bounded while tolerating measured loaded-machine runtime instead of reporting a false branch-gate failure.
