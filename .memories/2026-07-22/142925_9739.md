## Phase 2D SearchIndex boundary planned

The active Julie quality-roadmap brief now marks Phase 2C complete and Phase 2D planning active. Live impact analysis found a 2,326-line `SearchIndex` implementation with 62 methods and a broad caller surface, so the implementation plan preserves the facade and all caller paths.

- **Approach:** Keep `SearchIndex`, its eight fields, `SearchIndexHandle`, and public reexports in `search/index.rs`; move types, compatibility, lifecycle, mutation, and query inherent implementations into private children.
- **Boundary:** Keep the existing `search/schema.rs` responsibility and `search/mod.rs` reexports unchanged. Query work is internally divided into wrapper, execution, term, and file/rerank modules only to meet the 500-line limit.
- **TDD:** Add one structural RED/GREEN test that reports the current 2,326-line facade and then enforces every Phase 2D production file at or below 500 lines.
- **Verification:** Baseline `cargo nextest run -p julie-index` passes 380/380 with one skipped test. Changed selection maps search paths to `core-index`, 13 search-tool buckets, and `search-quality`; dogfood, dev, and full gates remain required.
- **Impact:** Phase 2D is approval-ready without API migration, schema changes, behavior changes, or new abstractions.
