## Fixed search-quality fixture disk exhaustion

Phase 2D's `cargo xtask test full` reached the search-quality bucket after 47 passing buckets, then failed with SQLite disk I/O and disk-full errors.

- **WHAT:** Replaced the fixture helper's leaked `TempDir` with an owning `FixtureHandlerGuard` that preserves existing borrowed handler call sites through `Deref` and `AsRef`.
- **WHY:** Each nextest test process leaked a copied SQLite fixture and Tantivy index, leaving about 138 MiB behind per test. Three search-quality runs accumulated 168 directories and 22.1 GiB.
- **HOW:** Added an exact cleanup regression, proved the old helper failed it, retained the handler until guard drop, then let `TempDir` remove the workspace after handler resources close. Also silenced the intentional test-only `classify_file_match` facade re-export at the narrow import site.
- **IMPACT:** Search-quality gates no longer consume permanent temp storage, the full gate can run without progressively exhausting disk, and Phase 2D remains behavior-preserving.
