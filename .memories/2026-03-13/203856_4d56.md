## Released v5.0.4 — Context-aware recovery hints for all tool error messages

### What
Added smart recovery hints to every Julie MCP tool's "nothing found" error path. When a tool returns empty results, it now suggests the right next tool based on the caller's likely intent.

### Why
Observed a 4-call recovery sequence when `get_symbols` was called with a wrong file path — the LLM got a dead-end "File not found" error with no guidance. Audited all tools and found the same pattern across `fast_refs`, `fast_search`, `get_context`, and various "No workspace initialized" errors.

### How
- **get_symbols**: detects whether `target` was set (symbol-first → suggest `deep_dive`) or not (file-first → suggest `fast_search`)
- **fast_refs**: suggests `fast_search` to verify the symbol exists
- **fast_search**: suggests switching `search_target` mode (definitions ↔ content)
- **get_context**: suggests `fast_search` for exact matches
- **All "No workspace initialized" errors**: standardized to include `manage_workspace(operation="index")` hint
- **get_symbols tool description**: added "Requires exact file path — use deep_dive if you don't know the path"
- **Design decision**: no file guessing or auto-redirect (avoids the `constants.cs` ambiguity problem with multiple same-named files)

### Impact
- 14 files changed, 114 insertions, 26 deletions
- 140 tests passed, 0 failed
- Should reduce LLM round-trips when tools return empty results
