## Codebase Report Skills — Complete

### What
Added `query_metrics` MCP tool and three report skills (`/codehealth`, `/security-audit`, `/architecture`) that produce human-readable reports from Julie's existing analysis data.

### Why
Julie had rich analysis data (centrality, test quality, change risk, security risk) only accessible to AI agents via MCP tools. No way for humans to get high-level reports. The security audit skill is particularly valuable in the age of AI-generated code where non-technical users may ship code without understanding its security implications.

### How
- **`query_metrics` tool** (`src/tools/metrics/`): SQL query layer over existing SQLite metadata with sorting by security_risk/change_risk/test_coverage/centrality, filtering by min_risk/has_tests/kind/file_pattern/language/exclude_tests. Uses `json_extract` for metadata, `COALESCE` for NULL handling, post-filters by globset for file patterns.
- **12 tests** covering all query paths, filters, and formatting.
- **3 SKILL.md files** (`.claude/skills/codehealth/`, `security-audit/`, `architecture/`): User-invocable, model-invocation disabled. Each skill instructs Claude on which tools to call and how to format the output.

### Impact
- Users can now run `/codehealth`, `/security-audit`, `/architecture` for instant reports
- Phase 2 deferred: complexity metrics + `/hotspots` skill
