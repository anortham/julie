## Batch tech debt cleanup — 4 items from TODO priority list

Audited TODO.md against current codebase, identified stale/completed items, then fixed 4 tech debt items via parallel subagent dispatch:

### Changes

- **Elixir ExUnit extraction** — Added `"test"` and `"describe"` arms to `dispatch_call`. Test blocks extract as `SymbolKind::Function` with `is_test: true`, describe blocks as `SymbolKind::Namespace` with child traversal. New `extract_first_string_arg` helper in `helpers.rs`.
- **Test quality comment stripping** — Added 5-state `strip_comments_and_strings()` state machine before regex scanning. Handles `//`, `#`, `--`, `/* */`, `"..."`, `'...'`. Fixed 2 existing tests that were themselves false positives.
- **PHP/Swift test detection path guard** — `detect_php` and `detect_swift` now require `is_test_path()` for name-prefix matching. PHP `@test` doc annotation still works anywhere.
- **Line-mode `exclude_tests` regression test** — New integration test verifying the filtering pipeline. Discovered it was already triple-layered (Tantivy, post-filter, defense-in-depth).

### Approach
- All 4 dispatched as parallel worktree agents (no file overlaps)
- Each agent did TDD: RED → GREEN
- Integrated by copying files from worktrees, verified with `cargo xtask test dev` (8/8 buckets green)
- Live dogfood verification confirmed all symbols indexed correctly

### Dogfooding discovery
- Zig `extract_test` → Elixir `extract_test` false caller edge in `deep_dive`. Pre-existing resolver limitation: import constraint only helps when candidates have a `parent_id`. Free functions with identical names across modules still get spurious edges.
