## Fixed 6 Language Verification Issues — P1 Class Extraction + P2 Centrality

### WHAT
Fixed all remaining P1 and P2 issues from the 15-language verification sweep:

**P1 — Class Extraction Gaps (3 fixes):**
- **Swift:** `open class Session: @unchecked Sendable` not extracted → replaced fragile child-kind search with `child_by_field_name("name")`. Also fixes extensions with complex types (`[Int]`, `[String: Int]`, `(Int, String)`).
- **Kotlin:** `sealed class JsonReader` wrapped in ERROR node → added ERROR recovery that detects `class`/`interface` keywords + identifier inside ERROR nodes and passes them to existing extraction functions.
- **Dart:** `base class`, `sealed class`, `final class`, `interface class` produce ERROR nodes in tree-sitter-dart v0.0.5 → added Dart 3 modifier recovery with consumed-block tracking to prevent double-visiting.

**P2 — Centrality Issues (3 fixes):**
- **Python:** Test subclass `class Flask(flask.Flask)` in test file stealing centrality → added Step 4 to `compute_reference_scores()` that de-weights test-file symbols by 0.1x using language-agnostic path patterns.
- **Ruby:** Class/constant dual symbols splitting centrality → skip `constant` extraction when node is a class/module name field or superclass reference.
- **C:** Header/implementation centrality split (`jq_next` in `.h` gets all refs, `.c` gets zero) → added Step 5 that propagates 70% of header centrality to same-named implementation functions.

### WHY
The 15-language verification sweep identified 9 remaining issues. 6 were P1/P2 priority — all now fixed.

### HOW
Dispatched 6 parallel worktree agents (one per bug), each following TDD. All agents investigated tree-sitter AST structures, wrote failing tests, then fixed. Integrated all 6 patches into main, resolved the `relationships.rs` overlap (Steps 4+5), and ran `cargo xtask test dev` — all 8 buckets passed.

### IMPACT
- Swift, Kotlin, Dart class extraction now handles edge cases that real-world projects hit
- Python, Ruby, C centrality scoring is now more accurate
- Key discovery: tree-sitter-swift uses `class_declaration` for ALL types (class/struct/enum/extension/actor) — dead code in `visit_node` match arms
- Key discovery: Kotlin and Dart ERROR recovery can rescue classes from tree-sitter parse failures
