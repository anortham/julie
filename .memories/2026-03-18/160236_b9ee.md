## Watcher `.gitignore` Support — Complete

Replaced the file watcher's hardcoded `glob::Pattern` list with the `ignore` crate's `Gitignore` matcher so the watcher respects `.gitignore` the same way the walker does.

### What Changed
- **`src/watcher/filtering.rs`**: Rewrote filtering layer — `build_gitignore_matcher()` builds a composite matcher from root `.gitignore` + `.julieignore` + synthetic patterns. Added `contains_blacklisted_directory()` and `is_gitignored()` helpers. New `should_index_file()` and `should_process_deletion()` with three-layer filtering.
- **`src/watcher/events.rs`**: Updated `process_file_system_event` to take `&Gitignore` + `&Path` instead of `&[glob::Pattern]`. Removed duplicate private filter functions.
- **`src/watcher/mod.rs`**: Changed `IncrementalIndexer.ignore_patterns` field to `gitignore: Gitignore`. Updated constructor and `start_watching()`.
- **`src/tools/shared.rs`**: Added `.gradle` and `.dart_tool` to `BLACKLISTED_DIRECTORIES`.
- **Tests**: 9 new/updated tests covering gitignore patterns, negation, .julieignore merging, blacklisted directories, deletion events.

### Why
The walker (`build_walker`) used the `ignore` crate with full `.gitignore` support, but the watcher used ~25 hardcoded glob patterns. Any `.gitignore` pattern not in that list leaked through. With Julie supporting `.json`, `.yaml`, `.toml` extensions, the surface area for leaks was growing.

### Key Decisions
- **Root `.gitignore` only** (not nested) — `GitignoreBuilder` scoping semantics for nested files are subtle. Root covers 95% of value.
- **`matched_path_or_any_parents()`** over `matched()` — needed for directory patterns like `build/` to catch files inside.
- **Synthetic patterns** for `.julie/`, `.memories/`, `cmake-build-*/`, `*.min.js`, `*.bundle.js` — safety net for patterns that won't be in a user's `.gitignore`.
- **`BLACKLISTED_DIRECTORIES` kept as independent safety net** — catches directories even without `.gitignore`.
