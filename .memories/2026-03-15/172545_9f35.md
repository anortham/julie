## Released v5.1.1 — Code Health Bugfixes

### What
Tagged and pushed v5.1.1 with 14 commits: 8 bug fixes, 1 feature, 3 doc/process improvements, plus a critical performance fix discovered during dogfooding.

### Key Fixes
- **deep_dive**: Change Risk no longer printed twice, labels say "dependents" not "callers", test locations capped/deduped
- **get_context**: no-results respects format param, SignatureOnly mode no longer drops risk/security labels
- **search**: exclude_tests filtering before truncation, threaded through line_mode_search
- **test_coverage**: language filter prevents cross-language mislinking (applied in Rust, not SQL — see incident below)
- **test_detection**: Go Fuzz/Example entry points recognized
- **docs**: subagent test rules in CLAUDE.md/AGENTS.md, doc contract tests fixed

### Critical Incident: SQL Query Plan Regression
Adding `AND s_test.language = s_prod.language` to test_coverage's Step 2b query caused SQLite to use `idx_symbols_language` instead of `idx_symbols_name`, turning a <1s query into a 3+ minute 100% CPU hang. Fixed by moving the language filter to Rust code. **Lesson: cross-table column comparisons in SQL can silently destroy query plans — always check EXPLAIN QUERY PLAN.**
