## Added subagent Julie tool propagation via hooks

Investigated why subagents (from superpowers subagent-driven-development) fall back to basic Glob/Grep/Read instead of Julie tools. Root cause: SessionStart hook doesn't fire for subagents, and the superpowers implementer prompt template is Julie-unaware.

- **Fix:** Two-layer approach: SessionStart guidance now includes subagent dispatching instructions, plus a PreToolUse hook on Agent nudges at dispatch time
- **Files changed:** `JULIE_AGENT_INSTRUCTIONS.md`, `.claude/hooks/hooks.json`, `~/source/julie-plugin/hooks/{hooks.json,session-start.cjs}`
- **Also updated plugin cache** for immediate testing
- **Decision:** Don't revive razorback (superpowers fork). Hook-based fix works with any workflow plugin.
