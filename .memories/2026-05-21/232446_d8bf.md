## T3: Tokenizer ablation env-var gates — COMPLETE

### WHAT
Added two runtime env-var gates to `CodeTokenizer` for A/B bakeoff comparisons:

- `JULIE_ABLATE_STEMMING=1` — skips the English stemmer step in `tokenize_code`
- `JULIE_ABLATE_CAMEL_EMIT=1` — skips CamelCase splitting; emits only the full lowercased identifier

### HOW

**`src/search/tokenizer.rs`**
- Added `ablate_stemming: bool` and `ablate_camel_emit: bool` fields to `CodeTokenizer` (with doc comments pointing to env var names)
- Added same two fields to `TokenizerCompatibilitySignature` so index rebuild is forced on toggle
- `CodeTokenizer::new` reads env vars once at construction; "0" and unset are both treated as off
- `token_stream` passes the flags down through `CodeTokenStream::new` → `tokenize_code`
- `tokenize_code` branches: camel-split block guarded by `!ablate_camel_emit`, stemmer block wrapped in `if !ablate_stemming {}`
- `compatibility_signature` includes the two new bool fields

### IMPACT
- Default path is byte-identical to pre-change behavior (proved by existing tests unchanged)
- Index rebuild is forced automatically when either flag is toggled
- Task 4 bakeoff harness can set env vars, wipe the Tantivy index, rebuild, and measure
