## JULIE_HOME override implementation complete

Shipped the `JULIE_HOME` env override for the daemon root, plus the consolidation cleanup the plan called for.

### What
- `DaemonPaths::try_new()` now reads `JULIE_HOME` via `var_os`. Empty value → `InvalidInput`. Set non-empty → that path verbatim (no `.julie` appended). Unset → existing `dirs::home_dir().join(".julie")` fallback.
- Added `DaemonPaths::is_julie_home(&Path) -> bool` with best-effort canonicalize + macOS case-insensitive comparison.
- Deleted the two duplicate global-home resolvers (`src/tools/workspace/paths.rs::julie_home` + `is_global_julie_dir`; inline lookup in `src/workspace/mod.rs::find_workspace_root`). Both walkers now share `DaemonPaths::is_julie_home`, fixing the macOS case-insensitivity drift between them.
- Fixed `src/dashboard/routes/intelligence.rs::open_workspace_db` fallback (found during Task 4 audit) that also joined `dirs::home_dir()` with `.julie/indexes`.
- Documented the override and migration workflow in `docs/WORKSPACE_ARCHITECTURE.md`, `docs/OPERATIONS.md`, `docs/SEARCH_FLOW.md`.

### Why
Operators want to relocate daemon state and workspace indexes off `~/.julie` (e.g. to a fast SSD). The macOS case-insensitivity divergence between the two duplicate walkers was a latent bug consolidated away.

### How
Subagent-driven development: Task 1+2 (TDD red/green) and Task 5 (docs) dispatched in parallel; Task 3 (delete duplicates) sequential after Task 2; Task 4 (audit) lead-owned, caught the dashboard regression that would have shipped silently otherwise.

### Verification
`cargo xtask test dev`: 32 buckets passed in 511.9s. `cargo fmt --check` clean. Polluted-env test `JULIE_HOME=/tmp/x cargo nextest run --lib tests::daemon::paths`: 19/19 pass. Inline review caught one test-race regression (existing path tests asserting `~/.julie` failed under polluted env) and one production audit hit (dashboard fallback joining `dirs::home_dir()` with `.julie/indexes`); both fixed.
