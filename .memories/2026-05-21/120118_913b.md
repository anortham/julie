## Codex Review: 5 JULIE_HOME Findings Fixed

Implemented all 5 fixes from Codex adversarial review of `docs/plans/2026-05-20-julie-home-override.md`.

### What
- **F1a:** Added `daemon.token`, `daemon-mcp.token`, `daemon-mcp-transport.json`, `daemon.state` to `BLACKLISTED_FILENAMES`. Deliberately omitted `discovery.json` and `migration.json` (too generic — `is_under_julie_home` covers them).
- **F1b:** New `DaemonPaths::is_under_julie_home(&Path) -> bool` using canonicalize-with-fallback + `Path::starts_with`. Wired into BOTH `tools/workspace/discovery.rs::should_index_file` AND `watcher/filtering.rs::should_index_file` + `should_process_deletion`.
- **F2:** `DaemonPaths::try_new()` rejects non-absolute `JULIE_HOME` with `InvalidInput` (no silent absolutize).
- **F3:** New associated function `DaemonPaths::is_any_known_julie_home(&Path) -> bool` that ALWAYS matches `~/.julie` even when the configured override fails. Wired into both walker call sites: `tools/workspace/paths.rs` and `workspace/mod.rs`.
- **F4:** Replaced unconditional macOS lowercasing in `is_julie_home`. New logic: canonicalize both → if both succeed, compare canonical PathBufs; otherwise raw PathBuf eq (no lowercasing — correct on case-sensitive APFS).
- **F5:** Rewrote 10 path-derivation tests to use `DaemonPaths::with_home(<tmp>/explicit-test-home)` instead of `dirs::home_dir()`. The remaining 2 tests that legitimately read `dirs::home_dir()` (`test_julie_home_uses_home_dir`, `test_default_impl`) now use `#[serial(julie_home_env, home_env)]` to join BOTH serial groups (serial_test 3.x supports comma-separated keys per docs/lib.rs:37).

### Tests added (14 new)
- 4 new daemon path tests (relative `.`, relative path rejection, is_under_julie_home accept/reject/sibling-prefix, is_any_known_julie_home configured/default/empty/unrelated)
- 1 macOS-only canonicalize-existing test
- 1 case-sensitive-FS correctness test (non-existent paths)
- 2 discovery walker tests (daemon-state filenames, paths under JULIE_HOME)
- 2 watcher filtering tests (paths under JULIE_HOME for both index + deletion)

### Impact
Closes the operator footgun where `JULIE_HOME=<repo>/julie-home` would leak daemon tokens, transport metadata, discovery state, and migration markers into the index. Restores defense-in-depth: filename blacklist + `is_under_julie_home` exclusion (two-layer). Hardens walker against env misconfiguration: `~/.julie` is ALWAYS skipped now, even if `JULIE_HOME=""`. Fixes case-sensitive APFS correctness.
