## Hardened JULIE_HOME against Codex adversarial review

Fixed all 5 findings from Codex's adversarial review in commit `fe39bd4a`.

### Findings addressed
- **F1 (HIGH)** — Workspace-contained `JULIE_HOME` could leak daemon state. Added `DaemonPaths::is_under_julie_home(&Path)` and wired it into the discovery walker (`src/tools/workspace/discovery.rs`) and watcher (`src/watcher/filtering.rs::should_index_file` + `should_process_deletion`). Added `daemon.token`, `daemon-mcp.token`, `daemon-mcp-transport.json`, `daemon.state` to `BLACKLISTED_FILENAMES` as belt-and-suspenders. Deliberately did NOT blacklist `discovery.json` / `migration.json` (too generic; the path-under-home check already covers them).
- **F2 (MEDIUM)** — Relative `JULIE_HOME` rejected with `InvalidInput`.
- **F3 (MEDIUM)** — Added `DaemonPaths::is_any_known_julie_home(&Path)` associated fn that ALWAYS checks default `~/.julie`, even when env is broken. Wired into both walkers.
- **F4 (MEDIUM)** — Removed unconditional macOS lowercase in `is_julie_home`. Now canonicalizes both sides; falls back to raw `PathBuf` equality. Correct on case-sensitive APFS.
- **F5 (LOW)** — Path-derivation tests now use `DaemonPaths::with_home(<tmp>)`, no `dirs::home_dir()` dependency. Tests that legitimately read it use multi-key serial `serial(julie_home_env, home_env)`.

### Verification
- `cargo xtask test dev`: 32 buckets PASS in 495.0s.
- Polluted-env regression `JULIE_HOME=/tmp/julie-x cargo nextest run --lib tests::daemon::paths`: 30/30 PASS.
- 14 new tests added across paths.rs, discovery, watcher.

### Impact
The `JULIE_HOME` override is now safe to point inside a workspace tree without leaking daemon state, safe to set to a relative path (rejected loudly instead of silently behaving wrong), case-correct on case-sensitive APFS, and the walker guard holds even with a broken env. No further "follow-up" work — the user explicitly requested reliability over deferral and got it.
