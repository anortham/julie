## T2: Cross-target title-exact and basename-exact short-circuit — COMPLETE

### WHAT
Extended Julie's `EXACT_TITLE_BOOST` to fire across all three search targets (definitions, files, content), not just definitions. Added batched DB lookup for symbol names.

### WHY
Pattern-A bakeoff losses (duplicate-file scenarios like Alamofire's jazzy.search.js copies): wrong copy wins by BM25 because `EXACT_TITLE_BOOST` only fired on definitions path.

### HOW

**New DB method** (`src/database/symbols/queries.rs`):
- `titles_for_files(paths: &[&str]) -> HashMap<String, Vec<String>>` — batched SQL `SELECT file_path, name FROM symbols WHERE file_path IN (...)`, returns lowercase symbol names per file, chunked at 500 to stay inside SQLite's 999-param limit.

**Content path** (`src/tools/search/text_search.rs`):
- `apply_reranker_to_content_results` gained `db: Option<&SymbolDatabase>` parameter.
- Batched `titles_for_files` call capped at 200 pre-rerank candidates.
- Applies `+EXACT_TITLE_BOOST` to any result whose file has a symbol whose lowercase name matches any query target term.
- Both callers updated: `content_search_with_index` (passes `db`) and `run_line_mode_workspace_fetch` in `line_mode.rs` (passes `Some(db)`).

**Files path** (`src/search/index.rs`, `src/tools/search/execution.rs`):
- New `apply_symbol_title_boost_to_file_results(query, results, db)` in `index.rs` — same batched pattern, re-sorts after boosting.
- Wired in `execute_file_search` in `execution.rs`: fetches DB via `handler.get_pooled_database_for_workspace`, best-effort (silently skips on failure).

**Definitions path**: verified correct — `promote_exact_name_matches` stable-partitions by exact name then file tier (source > test > doc), independent of BM25. No change needed.
