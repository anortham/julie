## Added manifest-driven xtask test runner and calibrated tier buckets

### WHAT
Built a new `xtask` workspace member that owns Julie's canonical test interface via `cargo xtask test ...`, added a checked-in `xtask/test_tiers.toml` manifest for tiers/buckets/timeouts, migrated `CLAUDE.md` / `AGENTS.md` / `README.md` to the new runner, and calibrated first-pass bucket budgets against measured runtime.

### WHY
The old docs-only `cargo test --lib -- --skip search_quality` guidance had drifted into nonsense: the documented "fast" tier no longer behaved predictably, progress visibility was bad, and it was hard to tell slow runs from hangs. A real source of truth was needed.

### HOW
- Added `xtask` crate + cargo alias in `.cargo/config.toml`
- Implemented manifest parsing/validation with exact contract tests for tier membership, commands, and budgets
- Built CLI/runner flow for `cargo xtask test {smoke|dev|system|dogfood|full|list}` and `cargo xtask test bucket <name>`
- Fixed a real timeout bug: killing the top-level shell left nested cargo/test processes running; the runner now tears down the full process tree on timeout
- Calibrated buckets from measured runs: `smoke` ~12.8s, `dev` ~228.6s, `core-fast` ~7.3s, `tools-misc` ~171.7s, `dogfood` ~415.3s, `integration` ~26.3s
- Confirmed `workspace_init` is a pre-existing problem, not introduced by this branch: one test already fails on `main`, and the bucket remains pathological even with larger timeout

### IMPACT
- Test commands now have one checked-in source of truth instead of drifting docs snippets
- Tier runs show bucket-level progress and timeout boundaries, so slow vs. hung is much clearer
- Canonical tiers now exercise xtask's own tests too, preventing false-green runner regressions
- Remaining known follow-ups are about the pre-existing `workspace_init` problem, a previously observed get_context-related failure, and softening the docs caveat assertion once the workspace issue is fixed
