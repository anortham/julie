## Split daemon roots tests

### WHAT
Split `src/tests/daemon/roots.rs` from a 3,082-line monolithic daemon roots test file into a small parent module plus five behavior-focused child modules under `src/tests/daemon/roots/`.

- Parent `roots.rs`: shared imports/helpers and module wiring only
- `initial_binding.rs`: first bind and attach/fallback behavior
- `list_changed_refresh.rs`: dirty roots refresh/retry/fallback behavior
- `secondary_targets.rs`: secondary target requests while roots are dirty
- `startup_deferral.rs`: explicit/env/cwd startup deferral paths
- `deferred_auto_index_sensitive.rs`: deferred auto-index and sensitive cwd guards

### WHY
The file exceeded the project test-file cap and mixed several daemon roots behavior clusters. The split reduces review surface without changing runtime behavior or helper semantics.

### HOW
Kept shared helpers in the parent module and moved tests into child modules using `use super::*`, matching existing test split patterns. Updated the cleanup plan and verification ledger based on local and subagent re-review.

### IMPACT
The daemon roots test surface is now below the file-size cap, while existing `src/tests/daemon/` changed-file routing still selects the daemon bucket. The re-review also identified `src/tests/dashboard/intelligence.rs` raw setup rows as the next lower-risk fixture cleanup candidate.
