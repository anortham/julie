## Removed stale handler workspace_root field

### WHAT
Removed `JulieServerHandler::workspace_root` and its constructor initializers, then updated tests to assert the session current root or startup hint directly.

### WHY
Candidate 13 dead-code audit found the field was retained only for test assertions while production code already uses `current_workspace_root()` and `session_workspace` as the root source of truth.

### HOW
Deleted the field from `src/handler.rs`, removed initialization in the stdio and daemon constructors, removed the redundant equality assertion in `handler_construction_sets_workspace_root`, replaced the explicit-root test assertion with `current_workspace_root()`, and replaced the swap-gap fallback comparison with `workspace_startup_hint().path`.

### IMPACT
Removes duplicate root state from the handler. Tests now exercise the production session-root API directly. The changed gate fell back to dev because `src/handler.rs` is shared infrastructure and passed.
