## Added reference-score fixture helper and migrated change-risk setup

### WHAT
- Added `set_symbol_reference_scores` to the shared DB test helper surface and re-exported it.
- Added helper tests for successful reference-score updates and missing-ID rejection.
- Migrated `test_compute_change_risk_scores` from raw SQL inserts to `symbol_builder` plus explicit reference-score updates.
- Updated the cleanup plan progress notes and verification ledger.

### WHY
- `compute_change_risk_scores` reads `symbols.reference_score`, but `Symbol` does not carry that field and `store_symbols` resets it to `0.0`. A dedicated fixture helper preserves centrality-sensitive test behavior without keeping raw symbol insert SQL.

### HOW
- The helper updates `symbols.reference_score` directly and errors when an ID does not exist, preventing silent fixture drift.
- The migrated test preserves symbol kinds, spans, visibility, test-linkage metadata, test metadata, and reference-score inputs.
- Added an extra missing-ID guard test during lead review so the helper's error path is covered.
- Fresh Claude review approved the helper and migration with no findings.

### IMPACT
- Change-risk fixture setup is builder-backed while keeping centrality explicit.
- Shared test helper changes were validated by the dev fallback selected by `cargo xtask test changed`.
