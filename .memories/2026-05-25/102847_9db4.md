## Split Embedding Metadata Enrichment Tests

### WHAT
Split `src/tests/core/embedding_metadata_enrichment.rs` from a 1176-line monolith into a 59-line root helper/module file plus four child modules:
- `tests/doc_excerpt.rs`
- `tests/variable_budgeting.rs`
- `tests/symbol_filtering.rs`
- `tests/batch_enrichment.rs`

Updated `docs/plans/2026-05-24-codebase-cleanup-plan.md` with the completed slice, remaining oversized inventory, verification ledger row, and Claude review row.

### WHY
Candidate 12 cleanup is reducing oversized test files while preserving behavior and changed-file routing. A read-only audit ranked this as the lowest-risk remaining oversized split because it is pure test code under existing `src/tests/core/embedding*` routing.

### HOW
Moved test bodies by behavior family and left shared symbol helpers in the root module. Fixed two issues caught during verification:
- Rust resolves submodules declared inside inline `mod tests` under `embedding_metadata_enrichment/tests/`, so child files were moved there.
- `test_symbol_filtering.rs` matched the repo-wide ignored `test_*.rs` pattern, so it was renamed to `symbol_filtering.rs`.

### IMPACT
All files are now under the 1000-line test cap, all 42 test functions remain covered, no child file is ignored, and `cargo xtask test changed` stays narrow on `core-embeddings`.
