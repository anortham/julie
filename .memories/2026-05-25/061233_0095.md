## Candidate 12 Hybrid Fixture Migration

### WHAT
- Added `store_file_info_if_missing` to the shared DB test helpers so fixture code can preserve raw `INSERT OR IGNORE` file-row semantics without duplicating `get_file_hash` guards.
- Added `SymbolBuilder::doc_comment` with a focused RED/GREEN helper test.
- Migrated `src/tests/tools/hybrid_search_tests.rs` file/symbol setup from raw SQL to `file_info_builder` / `symbol_builder`, preserving doc comments, signatures, metadata roles, spans, bytes, and explicit confidence defaults.
- Replaced copied file-row guard usage in `src/tests/core/vector_storage.rs` and `src/tests/tools/workspace/index_embedding_tests.rs` with the shared helper.
- Updated `docs/plans/2026-05-24-codebase-cleanup-plan.md` with the completed slice, verification ledger rows, and the closed hybrid doc-comment caveat.

### WHY
- Candidate 12 is reducing test fixture churn and raw setup SQL while keeping intentional raw SQL visible for schema/default/malformed-row cases.
- `hybrid_search_tests.rs` could not be safely migrated until doc comments could be represented through the builder.
- The repeated file-row guard was real duplication and also carried an important behavioral invariant: avoid `store_file_info`'s `INSERT OR REPLACE` cascade when multiple symbols share a file.

### HOW
- Followed TDD for `store_file_info_if_missing` and `SymbolBuilder::doc_comment`.
- Used builder APIs for migrated symbols and explicitly set `.confidence(1.0)` where raw inserts previously relied on the `symbols.confidence DEFAULT 1.0` schema default.
- Preserved `INSERT OR IGNORE` file behavior by checking `get_file_hash` before calling `store_file_info` in the shared helper.
- Asked fresh Claude CLI for review; it approved with no actionable findings. Its confidence note was checked against `src/database/schema.rs`, which confirmed the explicit builder confidence preserves raw-insert parity.

### IMPACT
- `hybrid_search_tests.rs`, `vector_storage.rs`, and `workspace/index_embedding_tests.rs` no longer have raw setup inserts for files/symbols in the migrated paths.
- The reusable file-row helper makes future builder migrations less likely to reintroduce cascade-delete bugs.
- Verification passed through exact affected tests, `cargo fmt --check`, `cargo check --all-targets`, `cargo xtask test changed` dev fallback, `git diff --check`, and Claude CLI review.
