## Plan A Complete and Merged: CLI Tool Interface

Julie now has a shell-first CLI interface mirroring all 12 MCP tools. Merged to main via fast-forward from `feat/cli-tool-interface`.

### What shipped (8 commits, 22 files, +4381 lines)

- **7 CLI subcommands**: search, refs, symbols, context, blast-radius, workspace, plus generic `tool <name> --params`
- **Dual execution modes**: daemon (IPC reusing adapter handshake) and standalone (production handler, no daemon required)
- **Output formats**: text (as-is), JSON (pretty-printed), markdown (headers + fenced blocks)
- **122 unit tests + 13 integration tests** all passing, 10/10 dev buckets green

### Codex adversarial review (4 findings, all fixed)

1. Daemon tool errors no longer silently retry in standalone mode (was a semantic bug for write ops)
2. blast-radius `--rev` now resolves via `git diff --name-only` to file paths
3. refs `--file-path`/`--file-pattern` flags removed (tool doesn't support them)
4. Workspace daemon-only ops now exit non-zero

### Post-review refinements (codex)

- `validate_standalone()` trait method rejects daemon-only operations before handler bootstrap
- DRY refactor for blast-radius helpers (shared between daemon and standalone paths)
- `--format` renamed to `--report-format` to avoid global flag collision

### Impact

The dev loop bottleneck is resolved: `cargo build && ./target/debug/julie-server search "query" --standalone` works during active MCP sessions. Plans B (annotation normalization) and C (early warning signals) can now use the CLI for in-session testing.
