## Fixed `fast_search` file-mode scoped filtering regressions

### WHAT
Patched file-mode execution so `fast_search(search_target="files")` no longer starves scoped results when `file_pattern` or `exclude_tests` is applied after the raw Tantivy fetch. Added regression tests for both scoped-hit starvation and bogus `showing X of Y` headers.

### WHY
The old pipeline fetched only `limit` raw file hits, then applied scope filters afterward. That meant valid in-scope files could be dropped if out-of-scope files ranked earlier, and the header counted raw hits instead of filtered hits.

### HOW
- Added deterministic regression fixtures in `src/tests/tools/search/file_mode_tests.rs`
- Updated `execute_file_search` in `src/tools/search/execution.rs` to widen the fetch window up to a hard cap when post-filters are present
- Counted file-mode totals from the filtered result set instead of the raw pre-filter window
- Kept the fix local to file-mode execution rather than changing the Tantivy index API

### IMPACT
- Scoped file searches now return in-scope paths instead of false zero-hit responses
- File-mode headers stop leaking raw out-of-scope totals
- Validation passed for targeted regressions, the broader `fast_search_files` slice, and `cargo xtask test changed` which fell back to `dev` and passed all 10 buckets
