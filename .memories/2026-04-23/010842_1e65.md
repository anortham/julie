## Hid the Search Analysis link from primary dashboard pages

### WHAT
Removed the `Analysis` navigation button from the dashboard search playground and search compare pages, while leaving the `/search/analysis` route intact. Added a regression test that fetches both pages and asserts they do not advertise `/search/analysis`.

### WHY
The analysis page currently looks rough enough to undermine trust in the product surface. Hiding its entry point reduces the release risk without deleting the underlying route.

### HOW
- Added `test_search_and_compare_pages_do_not_link_to_search_analysis` in `src/tests/dashboard/integration.rs`
- Removed the `Analysis` button from `dashboard/templates/search.html`
- Removed the `Analysis` button from `dashboard/templates/search_compare.html`
- Left `dashboard/templates/search_analysis.html` and the route wiring untouched for later cleanup or redesign

### IMPACT
- Main dashboard search flows no longer funnel users into the messy analysis page
- The route still exists for direct access during development
- Narrow regression test passed and the diff-scoped gate fell back to `dev` and passed all 10 buckets
