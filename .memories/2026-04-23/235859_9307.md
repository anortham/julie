## Dashboard Early Warnings Implemented

### WHAT
Implemented annotation-derived early warning reporting for the dashboard and removed the stale security risk surface.

- Added `src/analysis/early_warnings.rs` with entry point, auth coverage candidate, and review marker signals.
- Added `early_warning_reports` cache storage with migration 021 and fresh-schema creation.
- Added language-level annotation classes and early warning config for Python, Java, Kotlin, C#, TypeScript, JavaScript, and Rust test markers.
- Added dashboard Signals route, summary partials, refresh handling, and project detail navigation.
- Deleted `src/analysis/security_risk.rs`, its tests, and the stale metrics trend module.

### WHY
The old security risk feature was guesswork and had been pulled. The new dashboard surface uses annotation data produced by the earlier CLI annotation work, making the warnings evidence-based instead of speculative.

### HOW
Implemented the report generator around persisted normalized annotations and language config classes. Cache keys use workspace, canonical revision, projection revision, config schema version, and file pattern. Claude review surfaced cache and copy risks, so the cache now stores full rows before section limits, prunes superseded entries per workspace and pattern, de-dupes auth candidates, and treats auth bypass markers as auth decisions while still surfacing them as review markers.

### IMPACT
Agents can open `/signals/{workspace_id}` and see annotation-backed signals in the dashboard. Removed remaining user-facing security-risk wording except for a negative regression assertion. Fresh verification ran `cargo fmt --check`, `cargo build`, `cargo xtask test changed`, `cargo xtask test system` after rerun, and the standalone CLI search smoke.
