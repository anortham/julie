## Goldfish backward-compatible parsing — fully verified end-to-end

### Two bugs fixed, all legacy formats now supported

**Bug 1: CRLF line endings (Windows)**
- `parseCheckpointFile` and `parsePlanFile` regex used `\n` which fails on `\r\n` from git checkout with `core.autocrlf=true`
- Fix: `content.replace(/\r\n/g, '\n')` normalization before parsing
- Silent failure — `catch` block in `getCheckpointsForDay` swallowed the error

**Bug 2: Legacy Julie format incompatibility**
- Three old formats discovered: `.json` (JSON body), `.md` with Unix seconds, `.md` with Unix milliseconds + camelCase `filesChanged`
- Extracted shared helpers: `normalizeTimestamp()` (seconds vs ms heuristic: `> 1e10 → ms`), `normalizeGit()` (handles `files`/`files_changed`/`filesChanged`, drops `dirty`)
- Added `parseJsonCheckpoint()` for old `.json` files
- `getCheckpointsForDay` now reads both `.md` and `.json` files

- **Before**: 0 checkpoints recalled from julie workspace
