## Backward-compatible parsing for old Julie memory formats

### What
Extended `parseCheckpointFile` and added `parseJsonCheckpoint` to handle three legacy formats:
1. **Old Julie `.md`** — Unix timestamps (seconds), `files_changed`, `dirty`, `type`
2. **Old Julie `.md` variant** — millisecond timestamps, `filesChanged` (camelCase), `mem_` prefix IDs
3. **Old Julie `.json`** — JSON format with Unix timestamps, description in JSON body

### How
- Extracted `normalizeTimestamp()` helper: `> 1e10 → ms`, `≤ 1e10 → seconds`
- Extracted `normalizeGit()` helper: maps `files_changed`/`filesChanged` → `files`, drops `dirty`
- `getCheckpointsForDay` now reads both `.md` and `.json` files
- Both parsers share the same normalization helpers

### Tests
- 8 new tests (242 total, 0 failures)
- Covers: Unix second timestamps, millisecond timestamps, camelCase `filesChanged`, JSON format with/without git, `.json` + `.md` coexistence in `getCheckpointsForDay`

### Impact
Julie workspace went from 0 recalled checkpoints to finding all historical memories across months of work
