## Deep comparison: Serena vs Julie architecture

Explored the Serena codebase (added as reference workspace `serena_17431b9a`) to understand its current architecture and compare with Julie.

### Key Findings

- **Serena is Python (~43K lines, 123 .py files)**, provides IDE-like semantic tools to LLMs via MCP
- **Two backends**: SolidLSP (wraps real language servers like pyright, rust-analyzer) and JetBrains plugin
- **Core class** `SolidLanguageServer` is a 2,238-line god class wrapping LSP JSON-RPC
- **Multi-language support improved**: `LanguageServerManager` now holds `dict[Language, SolidLanguageServer]`, starts servers in parallel threads. But **fail-fast design** means one bad LS kills all of them.
- **No indexed search**: `SearchForPatternTool` does regex/glob over files. No Tantivy, no ranking, no tokenization awareness.
- **Symbol-level editing** is the "claim to fame": `replace_symbol_body`, `insert_after_symbol`, `insert_before_symbol` let agents edit by symbol name rather than line numbers.

### Comparison Conclusions

| Julie wins | Serena wins |
|------------|-------------|
| Search quality (Tantivy, BM25, centrality) | Semantic precision (real compiler via LSP) |
| Speed (pre-indexed, <20ms queries) | Symbol-level editing tools |
| Zero dependencies (single binary) | LSP rename refactoring |
| Polyglot unified index | Diagnostics/hover (though agents rarely use these) |
| Graph intelligence (centrality, call graphs) | |
| Reliability (no LS crashes) | |

### Decision: No editing tools for Julie right now

- User previously had 15-18 tools including editing, stripped to 7-8 for focus
- Agents ignored custom editing tools in favor of built-in Edit tool
- Serena overcomes this through aggressive behavioral priming (MCP instructions force tool adoption), not organic utility
- Token economics favor Edit (old_string/new_string) over replace_symbol_body (entire function) for typical small changes
- Julie's bottleneck is finding/understanding code, not editing it
- If revisited later, `insert_after_symbol` has the strongest case
