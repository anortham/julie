## Fixed duplicate workspace index directories

`normalize_path` was lowercasing paths on all platforms, then v6.0.4 correctly fixed it to only lowercase on Windows. But daemon.db was never migrated, so every daemon restart generated new workspace IDs, creating duplicate index directories for every project.

- **Root cause:** SHA256 hash of lowercased vs case-preserved path produces different workspace IDs
- **Silent enabler:** `let _ = db.upsert_workspace(...)` swallowed the UNIQUE constraint error on `path`
- **Fix:** Three-layer approach:
  1. `DaemonDatabase::migrate_workspace_ids` -- batch-migrates IDs across all FK tables with PRAGMA scope guard
  2. `upsert_workspace` conflict target changed from `workspace_id` to `path`
  3. `clean` command enhanced with orphan directory scanning
  4. `migrate_stale_workspace_ids` wired into daemon startup before WorkspacePool creation
- **Impact:** Eliminates ~400MB of duplicate index data, prevents future orphans
