## Fixed qualified-name and disambiguation bugs in `call_path` and `rewrite_symbol`

### WHAT
- Repaired Rust `impl Trait for Type` target extraction so impl methods attach to the implementing type, not the trait.
- Taught the shared `find_symbol()` qualified-name path to fall back to `impl_type_name` metadata when `parent_id` is missing.
- Relaxed `call_path` destination resolution so multiple `to` candidates are allowed and BFS can succeed if any target is reachable.
- Added same-file disambiguation for `rewrite_symbol` via `symbol@line` parsing and updated ambiguity messaging to suggest that form.
- Added regression tests for trait-impl qualified names, reachable ambiguous `to` symbols, metadata-based qualified lookup, and overloaded same-file rewrite targets.

### WHY
Dogfood review surfaced two real tool failures:
- `Worker::run` in `impl Trait for Worker` could not resolve in either tool.
- `rewrite_symbol` had no viable same-file disambiguation path, and `call_path` rejected valid graphs when the destination name was duplicated.

### HOW
- Added `extract_impl_target_names()` in the Rust helper layer and reused it in both symbol extraction and relationship extraction.
- Extended `find_symbol()` qualified matching to accept metadata-backed impl type names.
- Split `call_path` matching into shared filtered lookup plus a multi-target resolver for `to`.
- Parsed optional `@line` suffixes in `rewrite_symbol`, filtered candidates by line range, and improved not-found/ambiguous error text.
- Strengthened tool tests so file-path hints and qualified-name behavior are proven by distinct graph shapes.

### IMPACT
- `call_path` and `rewrite_symbol` now resolve trait-impl methods through the shared path.
- `call_path` no longer throws false ambiguity errors when only one duplicated destination is reachable.
- `rewrite_symbol` has a real same-file escape hatch for overloaded or otherwise colliding symbols.
- Regression coverage now pins the exact review findings so they do not sneak back in.
