## Compact project-row dashboard layout

### WHAT
Tightened the Projects table row layout so it stays usable on laptop-width screens.

- Kept only the quick `Open` or `Prune` action inline in the row
- Moved secondary actions out of the crowded row surface
- Added a responsive table shell and path truncation behavior
- Expanded dashboard tests to pin the compact-row contract and detail-panel action ownership

### WHY
The new workspace-management controls made the Projects row too wide. On a 16-inch laptop dashboard view, the action column was clipped and the table felt broken.

### HOW
- Updated `project_table.html`, `project_row.html`, and `app.css`
- Added and updated tests in `src/tests/dashboard/projects_actions.rs`
- Verified with `cargo nextest run --lib projects_actions` and `cargo xtask test changed`

### IMPACT
The dashboard row fits normal laptop widths more cleanly, secondary controls still exist in the detail panel, and the compact layout is guarded by tests.
