## Finished workspace management phase 2

### WHAT
- Added a shared cleanup lifecycle inspector that classifies workspaces as present, stale, or blocked before open and cleanup sweep flows act on them.
- Updated the Projects dashboard to separate session state from path state, surface blocked cleanup holds, and show stale versus blocked workspaces without blowing out row width.
- Added focused tests for missing-path cleanup state, watcher/session reuse, and end-to-end cleanup sweep behavior.
- Updated `WORKSPACE_ARCHITECTURE.md` and `TODO.md` to reflect the shipped workspace-management model.
- Ran Gemini UI review on the Projects dashboard and applied the useful polish fixes it found.

### WHY
- The hard-C cut removed the dead reference-workspace concept, but the worktree lifecycle story and dashboard lifecycle truth were still implicit.
- Missing paths were being handled by a mix of `Path::exists()` checks and ad hoc cleanup wording, which made stale versus blocked rows hard to reason about.
- The dashboard needed to tell the truth about why a dead worktree could or could not be pruned.

### HOW
- Introduced `WorkspaceCleanupState` plus `inspect_workspace_cleanup_state()` in `src/tools/workspace/commands/registry/cleanup.rs`, then wired `open` and cleanup sweep through it.
- Reworked `src/dashboard/routes/projects.rs` to derive async lifecycle state from the cleanup inspector instead of raw path checks.
- Added dashboard view-model fields for blocked cleanup reason and path-state badge, updated the row/detail/summary/cleanup templates, and adjusted CSS for stale versus blocked chips.
- Added tests in `src/tests/daemon/workspace_cleanup.rs`, `src/tests/daemon/workspace_pool.rs`, `src/tests/integration/daemon_lifecycle.rs`, and `src/tests/dashboard/projects_actions.rs`.
- Used Gemini as a read-only UI reviewer, then tightened column width, badge motion, tooltip coverage, wording, and duplicate CSS.

### IMPACT
- Deleted worktrees now have one shared lifecycle model across cleanup, open, and dashboard surfaces.
- The dashboard now makes blocked cleanup obvious instead of leaving users to infer it from session counts.
- Watcher/session reuse and missing-path cleanup flows are covered by regression tests, which cuts the odds of reintroducing zombie rows or misleading UI states.
