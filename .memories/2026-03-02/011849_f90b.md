## Fixed Variable Embedding System Issues (5 fixes)

Executed a 5-fix plan addressing code review findings from GPT's variable embedding implementation (~2,450 lines, 14 commits).

### Fixes Applied

1. **CRITICAL: Removed variable embedding from incremental path** (`pipeline.rs`)
   - `embed_symbols_for_file` was loading the ENTIRE symbol table on every file save via `get_all_symbols()`
   - Deleted `load_symbols_and_variable_reference_scores` and `selected_variable_ids_for_policy`
   - Incremental path is now O(file_symbols) instead of O(workspace_symbols)

2. **Batched `get_reference_scores` for SQLite safety** (`relationships.rs`)
   - Added chunking at 900 params to prevent exceeding SQLite's ~999 bind param limit
   - Follows same pattern as existing `delete_embeddings_for_symbol_ids`

3. **Simplified `has_simple_default_literal`** (`metadata.rs`)
   - Replaced 57-line char-by-char state machine with 20-line byte iterator
   - Extended literal set to include `nil`, `""`, `''`, `{}`, `[]`

4. **Removed domain-specific signal tokens** (`metadata.rs`)
   - Deleted `SIGNAL_TOKENS`, `symbol_tokens()`, `extend_tokens()`
   - Kept only generic descriptiveness heuristic (name has `_` or length >= 12 → +0.15)

5. **Fixed double penalty for short noise identifiers** (`metadata.rs`)
   - `i` was getting -0.30 (short) AND -0.35 (noise) = -0.65
   - Now: known noise → -0.50, unknown short → -0.20, mutually exclusive via if/else if
