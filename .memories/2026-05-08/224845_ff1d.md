## Compacted tool output formatters

### WHAT
Implemented compact output fixes across the previously identified wasteful formatter paths:
- `call_path` now returns compact text instead of pretty JSON.
- `deep_dive` groups adjacent same-file caller/callee/reference rows under one file header.
- `get_context` groups adjacent same-file neighbor rows in readable, compact, and spillover row formatting.
- `blast_radius` groups adjacent same-file impact rows.
- Added the missing `nano` tier expectation in `xtask/tests/manifest_contract_tests.rs` after the dev gate exposed the stale contract.

### WHY
The tool outputs were repeating paths or emitting verbose JSON in places intended for agent-facing text. This increased token use and made MCP responses noisier.

### HOW
Added RED tests for each repeated-path/JSON case, kept single-row formatting compatible where existing tests relied on `file:line`, and grouped only adjacent same-file runs to preserve relevance/rank ordering. For `call_path`, existing tests now parse the compact text response instead of JSON.

### IMPACT
Agent-facing output is denser for the affected tools without reordering results. The broad dev gate passes after fixing the adjacent stale manifest contract for the `nano` tier.
