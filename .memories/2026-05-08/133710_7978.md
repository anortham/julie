## Closed 4 review findings on uncommitted main: index health hardening + extractor polish

Reviewed the uncommitted tree-sitter remediation diff (29 files, +1088/-285) plus index health work, used codex-cli (gpt-5.3-codex high) for an adversarial second pass, reached consensus on findings, and shipped fixes.

### B1 (HIGH): ProjectionFailure was silently cleared by unrelated retry success
- **Root cause:** `IndexingRuntimeState::set_dirty_projection_count(0)` removed both `TantivyDirty` and `ProjectionFailure`. Scenario: file A abandoned at attempt 10 → records ProjectionFailure; file B's later retry succeeds → drops dirty count to 0 → both reasons removed → A's abandonment vanishes from health.
- **Fix:** Added `abandoned_projection_files: BTreeSet<String>` to `IndexingRuntimeState`. ProjectionFailure is now sticky against this set: `record_abandoned_projection` inserts and sets the flag; `clear_abandoned_projection` removes and clears the flag only when the set is empty. `set_dirty_projection_count` no longer touches ProjectionFailure. `begin_operation` clears the abandoned set (so force-reindex resets state).
- **Watcher wiring:** `runtime.rs` abandonment branch now calls `record_abandoned_projection(rel_path)`; success branch calls `clear_abandoned_projection(&rel_path)`.
- **Tests:** 3 new inline tests in `state.rs` (abandoned_projection_tests module): proves projection failure survives unrelated dirty drops, recovers per-file, and resets on begin_operation. All green.

### B2 (MEDIUM): Phantom-fd detail suppressed in text health report
- `render_report` printed only "Database not initialized" when canonical_store level was Unavailable — the rich `detail` ("ON-DISK STATE MISSING: SQLite reports N symbols but db file size is 0 MB. Restart daemon and force-reindex to recover.") never rendered.
- **Fix:** `report.rs` now prints `canonical_store.detail` when level is Unavailable, with fallback to "Database not initialized" only if detail is empty.

### B3 (LOW-MED): Vue containing-symbol-id non-determinism
- `find_containing_symbol_id_for_span` filtered HashMap values, sorted by (priority, size). When two symbols had identical priority and size, stable sort preserved HashMap iteration order — non-deterministic per-process.
- **Fix:** Added `a.id.cmp(&b.id)` lexicographic tiebreaker after priority and size in `crates/julie-extractors/src/vue/identifiers.rs`.

### L1 (DOC): Documented SQL view regex CTE/subquery fallback limitation
- `CREATE_VIEW_SOURCE_RE` in `sql/schema_relationships.rs` non-greedy first-FROM grab captures CTE source instead of view's actual base for `WITH cte AS (... FROM x) SELECT ... FROM y`. Only fires on ERROR-node fallback (AST path is exact).
- Added a code comment up-front documenting this as a known best-effort constraint.

### Known sticky behavior (intentional)
B1 only auto-clears via the retry-success path. If an abandoned file is never re-touched, ProjectionFailure stays sticky until `begin_operation` runs. The error log explicitly tells operators to run `manage_workspace operation=index force=true`, which triggers begin_operation and resets state.
