## OpenCode plugin install path shipped

Added an idempotent OpenCode installer to julie-plugin and updated both READMEs so users can install Julie hooks/skills/precedence into OpenCode the same way they already could into Codex CLI.

### What landed

- **`hooks/opencode-plugin.js`** — bundled ESM plugin module:
  - `tool.execute.before` → throws to deny broad `cargo nextest` / `cargo test` runs without a positional filter
  - `tool.execute.after` → appends sentinel-tagged coaching to `output.output` when grep/find/rg ran on an indexed Julie workspace, or when a >12KB code-file read happened in one
  - Workspace detection: prefers `bun:sqlite`, falls back to `node:sqlite`, then `.julie/` ancestry walk
- **`bin/install-opencode.cjs`** — idempotent installer:
  - Symlinks 6 skills to `~/.config/opencode/skills/julie-*/`
  - Symlinks plugin to `~/.config/opencode/plugins/julie-precedence.js`
  - Appends `<!-- julie-precedence start/end -->` block to `~/.config/opencode/AGENTS.md`
  - `--uninstall` cleanly removes only Julie additions; preserves user AGENTS.md content; deletes empty AGENTS.md
  - Does NOT touch `opencode.json` (same rationale as Codex installer — `JULIE_WORKSPACE` is project-scoped)
- **`julie-plugin/README.md`** — new "OpenCode install" section paralleling "Codex CLI install"; project-structure block updated
- **`julie/README.md`** — Plugin Installation section now lists Codex CLI + OpenCode installer commands; OpenCode row added to workspace-resolution table; OpenCode MCP-config block added in "Connect Your AI Tool"; outdated skills-table OpenCode entry corrected (was "Via instructions config" → now reflects auto-discovery from `~/.config/opencode/skills/` and the installer)

### Why

Previous checkpoint open thread: "Decide whether to invest in OpenCode TS plugin (Phase 5)." User answered yes. OpenCode now has the same first-class plugin install path as Codex.

### How

OpenCode's plugin API surface (verified live from opencode.ai docs and `@opencode-ai/sdk` types on GitHub):
- Plugins are JS/TS modules exporting `async (input) => Hooks`
- Hooks include `tool.execute.before(input, output)` (throw to deny, mutate args), `tool.execute.after(input, output)` (mutate `output.output` to append text)
- Skills auto-discover from `~/.config/opencode/skills/`, also `~/.claude/skills/` natively
- AGENTS.md global lives at `~/.config/opencode/AGENTS.md`
- Tool names lowercase: `bash`, `read`, `edit` (not `Bash`/`Read`/`Edit`)

### Verification

- `node --check` passes; ESM import returns `{ JuliePrecedence }` with the right hook keys
- 5/5 hook smoke tests pass: broad cargo test denied, targeted run allowed, grep on indexed → coaching, grep on `/tmp` → no FP, large read in indexed → coaching
- Installer ran 3× consecutively with no creeping blank lines in AGENTS.md (idempotent)
- Uninstall preserves pre-existing AGENTS.md content; removes file when our block was the only content
- Currently installed in `~/.config/opencode/` for live testing
