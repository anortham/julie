## Julie plugin landed on Claude Code via local-directory marketplace; hook FP fix shipped

Phase 3.4 of `docs/plans/2026-05-08-dev-environment-setup.md` complete plus follow-ups discovered during the install attempt.

### What landed

- **`cargo xtask sync-plugin`** — new xtask command. Skills full-mirror source → plugin (idempotent, with `--dry-run` and `--plugin-root`). Hooks are **report-only** because plugin's `hooks.json` uses `${CLAUDE_PLUGIN_ROOT}` substitution and a plugin-only `lib/` dir that don't apply in source dev mode. 5 unit tests pass; CLAUDE.md Quick Reference + plugin section updated; plan doc Phase 3.4 marked implemented.
- **Plugin install on Claude Code via local-path marketplace** — neither `--plugin-dir` (per-session flag) nor GitHub-marketplace (requires commit/push) was acceptable. Discovered third path: `claude plugin marketplace add ~/source/julie-plugin` then `/plugin install julie@julie-plugin`. Persistent, no flag, no GitHub publish needed. v7.8.1 active in this session — `pretool-grep-redirect.cjs` fired on me as proof.
- **Plugin README** — added Codex CLI install section (`node bin/install-codex.cjs` flow + manual MCP registration steps with the Codex Desktop cwd workaround), expanded skill count 4→6, expanded hook list, updated Project Structure block.
- **Hook FP fix** — extracted `hooks/lib/search-args.cjs` shared between `pretool-grep-redirect.cjs` and `codex-posttooluse.cjs`. 15 new tests pass. Mirrored to runtime cache. End-to-end verified: grep on `~/.config/opencode/opencode.json` no longer triggers the nudge.

### Key non-obvious facts

- **Plugin's `run.cjs` launches a packaged binary** (extracted from `bin/<target>/julie-v*.tar.gz`), NOT the dev `target/release/julie-server`. Dual MCP servers existed briefly until disabled.
- **Disabled the plugin's MCP locally** by removing `mcpServers` from BOTH `~/source/julie-plugin/.claude-plugin/plugin.json` AND `~/.claude/plugins/cache/julie-plugin/julie/7.8.1/.claude-plugin/plugin.json`. Source edit is uncommitted in julie-plugin git status — **must NOT be pushed for distribution** (would remove MCP for marketplace users).
- **`claude plugin update` no-ops on same version** — to refresh the runtime cache from source, either bump version or edit cache file directly.
- **Hook FP root cause:** tildes weren't expanded (`path.resolve(cwd, "~/foo")` produces `/cwd/~/foo`) AND the cwd-fallback fired even when user gave path-shaped args that didn't resolve. Both fixed in `pickSearchTargets()` which returns `null` for "user named paths but none resolved — don't guess".
- **Earlier FP tally was wrong.** Edits on `~/source/julie-plugin/...` files are correct nudges — `julie-plugin` IS registered in daemon DB at `/Users/murphy/source/julie-plugin`. Only ~3 of the 7 I called out were genuine FPs (the tilde greps).
- **OpenCode reality:** julie MCP already in `~/.config/opencode/opencode.json` (lines 70-83) pointing at the same dev binary. Missing: hooks + skills, which would require a TS plugin module like context-mode (~500 LOC). Filed as Phase 5 follow-up.
