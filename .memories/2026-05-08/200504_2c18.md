## Added workspace routing to edit_file

- **WHAT:** Added optional `workspace` support to `EditFileTool`, routed edit-file path resolution through `resolve_workspace_filter`, updated handler target detection/metadata, and added schema + behavior regression tests.
- **WHY:** Codex/Julie advertised cross-workspace editing, but the running `edit_file` schema rejected `workspace` and the implementation hard-pinned primary workspace roots.
- **HOW:** Replaced the stale unknown-field test with `test_edit_file_accepts_workspace_field`, added `test_edit_file_routes_to_target_workspace`, and added `edit_file_tool_schema_describes_workspace_input` to guard generated MCP schema.
- **IMPACT:** After rebuild/restart, `edit_file(..., workspace="<workspace_id>")` can edit opened target workspaces instead of failing schema validation or silently targeting primary.
