## Completed Julie rescue 3d.3 Task 5 slice

### WHAT
- Fixed target-workspace `source_bytes` attribution for `fast_refs` and `get_context` by precomputing source bytes from the resolved workspace binding and carrying them through `ToolCallReport`/`MetricsTask`.
- Deleted the retired dashboard search-compare surface: route, page/templates, dashboard module, database module, and schema creation.
- Added migration 007 to drop legacy `search_compare_*` tables from existing registry DBs.
- Moved the dashboard search helper out of the deleted `projects_actions` module into `routes/search_session.rs`.
- Updated the 3d.3 plan with Task 5 verification evidence and an explicit deferred note for per-workspace metrics-table/API cleanup.

### WHY
- The registry metrics row was recording the requested target workspace id but computing `source_bytes` from the loaded primary workspace DB.
- Search-compare no longer had a useful candidate path after 3d.2b and was a dead dashboard/dev-eval mutation/read surface.

### HOW
- Added RED expectations for target bytes, `/search/compare` absence, and search-compare table removal.
- Computed source bytes via `metrics_source_bytes_for_binding()` before queueing metrics for `fast_refs`/`get_context`.
- Kept the metrics writer fallback for tools that still do not precompute source bytes.
- Replaced search-compare schema creation with a retired migration 004 and a migration 007 drop.

### IMPACT
- Registry telemetry for target-workspace `fast_refs` and `get_context` now records target source bytes and session totals correctly.
- The read-only dashboard no longer exposes or links to search-compare.
- Fresh and migrated registry DBs no longer carry search-compare tables.
- Deferred follow-up remains: fully removing the exported per-workspace `SymbolDatabase` metrics table/API and local write tests is larger than this deletion slice.
