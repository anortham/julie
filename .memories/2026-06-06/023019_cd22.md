## Completed Julie Rescue 3d.3 Task 4 read-only dashboard slice

### WHAT
- Removed dashboard action/SSE route mounts: project register/open/refresh/delete, search-compare POST, signals refresh, and activity SSE.
- Deleted `src/dashboard/routes/events.rs` after confirming `activity_stream` was definition-only.
- Removed visible dashboard mutation forms from project list/detail templates, signals summary, and search compare.
- Rewired `julie-server dashboard` to start a standalone localhost Axum server backed by `DaemonPaths::registry_db()` instead of reading the old daemon port file.
- Removed the old `dashboard_url_from_port_file_contents` helper and its daemon-port tests.
- Updated dashboard route/project-action tests to assert the read-only contract and recorded Task 4 verification rows.

### WHY
3d.3 turns the dashboard into a standalone registry reader. Leaving mutation routes, SSE, or dead forms would preserve the old daemon-era dashboard behavior and confuse users with controls that no longer exist.

### HOW
- Wrote RED router test `test_router_is_read_only_without_action_or_sse_routes`; it failed while `/projects/register` was still mounted.
- Removed route mounts and templates first, then adjusted stale tests from mutation expectations to read-only assertions.
- Added standalone dashboard startup in `src/main.rs` using `DaemonDatabase::open(paths.registry_db())`, `DashboardState::new`, and `axum::serve` on an ephemeral localhost port.

### IMPACT
- `julie-server dashboard` no longer depends on daemon port/discovery files.
- Dashboard GET pages remain available; action/SSE endpoints are absent.
- Remaining dashboard helper cleanup is coupled to the next search/search-compare cleanup because `projects_actions` helpers are still used by dashboard search code.
