## Fixed dashboard zero-workspaces regression

The in-process MCP runtime could load and index the primary workspace, but the standalone dashboard read `~/.julie/registry.db` and saw no registered workspaces.

- **WHAT:** Wired in-process handlers to the workspace registry DB and registered loaded primary workspaces during workspace publication.
- **WHY:** After daemon/adaptor removal, loaded MCP workspaces were healthy but invisible to registry-backed surfaces like the dashboard project list and `manage_workspace list`.
- **HOW:** Added `JulieServerHandler::new_in_process_with_daemon_db`, opened the registry DB in `run_in_process_server`, and synced loaded primary workspaces via `session_attachment().attach_workspace_resources(...)` without holding the workspace lock across await.
- **IMPACT:** A restarted in-process MCP session should populate the dashboard workspace list again instead of showing zero workspaces.
