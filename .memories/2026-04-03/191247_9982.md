## Indexing & Embedding Bugfix Plan Written

Investigated and verified 4 bugs + 2 enhancements reported by another model's review of indexing pipelines and semantic usage. All 6 items confirmed real.

### Bugs Verified
1. **Force-index reference workspace clears primary embeddings** (data loss) -- `handler.get_workspace().db.clear_all_embeddings()` targets primary when force-indexing a reference workspace
2. **Watcher snapshots None embedding provider** -- `IncrementalIndexer` captures `embedding_provider.clone()` at construction (always None), never sees lazy init
3. **vector_count stats under-report** -- `update_vector_count` writes `stats.symbols_embedded` (run count) not actual DB count
4. **matches_filter ignores exclude_tests** -- semantic results from test files consume RRF merge slots before being filtered in caller

### Enhancements Verified
5. **No-op index spawns embeddings** -- `handle_index_command` lacks the `db_mutated` gate that `handle_refresh_command` has
6. **Per-file embeddings miss lang_configs** -- `dispatch_file_event` passes `None` for lang_configs to `reembed_symbols_for_file`

### Plan Structure
- 5 tasks (Bug 2 + Enh 6 combined since coupled)
- Tasks 1-4 independent/parallelizable
- Task 5 internally sequential (shared provider first, then lang_configs)
- Saved to `docs/superpowers/plans/2026-04-03-indexing-embedding-bugfixes.md`
