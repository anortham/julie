## Implemented Doc Coverage and Dead Code Metrics

### What
Added two new `query_metrics` categories: `doc_coverage` and `dead_code`. These are the first "safe" codehealth metrics that report hard numbers from validated data.

### How
- **Database layer** (`src/database/analytics.rs`): Three new query methods (`get_doc_coverage`, `get_undocumented_symbols`, `get_dead_code_candidates`) with shared SQL constants for test path exclusion, fixture/example/doc exclusion, and documentable/callable kind filtering.
- **Formatting layer** (`src/tools/metrics/code_quality.rs`): New module with `format_doc_coverage` (summary + per-language breakdown + top undocumented by centrality) and `format_dead_code` (grouped by kind with signature display).
- **Wiring** (`src/tools/metrics/mod.rs`): Two new category branches in `call_tool` match, updated tool description.

### Key Design Decisions
- Dead code narrowed to functions/methods only (types too noisy due to implicit usage patterns)
- Fixture, example, and docs directories excluded (sample code is intentionally unreferenced)
- Test path exclusion reuses same SQL patterns as centrality pipeline
- Undocumented symbols ranked by centrality (highest-impact gaps first)

### Validation
- Flask workspace: 262/408 documented (64.2%), 188 dead code candidates (library, so external callers not tracked)
- Julie workspace: 380 dead functions/methods out of 17k symbols (2.2%), includes expected false positives for dynamically-dispatched methods (MCP tools, daemon IPC)
- 8/8 xtask buckets pass, zero regressions
