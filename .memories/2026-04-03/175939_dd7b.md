## Embedding Enrichment: Three Bugs Fixed, Similar Symbols Working

### What
Fixed three bugs that were preventing similar symbols from appearing in deep_dive results, completing the embedding enrichment project.

### Bug 1: Wrong distance metric (cosine vs L2)
- sqlite-vec's vec0 table was created WITHOUT `distance_metric=cosine`, defaulting to L2 (Euclidean)
- The similarity code assumed cosine distance (`1.0 - distance` for similarity)
- L2 distances on 768-dim vectors are large numbers, making the formula produce negative/near-zero values
- **Fix:** Added `distance_metric=cosine` to vec0 CREATE TABLE in `src/database/vectors.rs`
- Requires force re-index to recreate the table

### Bug 2: Token budget truncation killing similar section
- `format_symbol_context` appended the similar section LAST, then truncated from the end
- At `context` depth (800 token budget), callers/callees/body consumed everything
- Codex diagnosed this: the neighbors existed above 0.35 threshold (e.g., `hybrid_search` had 68 passing neighbors) but were always truncated away
- **Fix:** Render similar section separately, reserve up to 150 tokens for it, truncate main body first

### Bug 3: Adapter forward_bytes race condition
- `tokio::select!` cancelled daemon-to-stdout copy when stdin finished
- During MCP startup after daemon restart, adapter would exit before daemon's initialize response was sent
- Caused "connection closed: initialize request" errors on every first connection after rebuild
- **Fix:** After stdin EOF, shutdown IPC write side then continue draining daemon->stdout
