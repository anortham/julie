## Token Efficiency Review Complete — Plan Phase

### What
Completed comprehensive 3-model token efficiency review (Claude + Codex + Gemini) of Julie MCP server. User approved 7 changes for implementation.

### Approved Scope
1. `get_symbols` default mode: `"minimal"` → `"structure"` (50-80% savings per call)
2. `fast_search(return="locations")` — file-only results mode (70-90% savings)
3. `deep_dive(full)` — enforce hard token cap (~1500 tokens)
4. Group-by-file output in search/refs formatting
5. Drop kind prefix when signature already contains it
6. `get_context` compact mode tightening (real budget enforcement)
7. New `/efficient` skill for conservative tool defaults

### Key Decisions
- **Vetoed**: Changes to tool descriptions or server instructions (they drive behavioral adoption)
- **Cut**: batch deep_dive (complexity not justified), changed_symbols (git diff + Julie tools sufficient)
- **Cut**: Path aliases (might hurt LLM accuracy), server-side handles/deltas (too complex)
