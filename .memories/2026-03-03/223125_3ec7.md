## Released v3.7.1 — Token Efficiency Cleanup

Shipped the full token efficiency cleanup across all Julie MCP tool outputs.

### What Changed
- **OptimizedResponse**: Stripped from 6 fields to 2 (`results` + `total_found`). Deleted `scoring.rs` entirely — its outputs (`insights`, `next_actions`, `confidence`) were vestigial from a JSON-output era and never reached any lean formatter.
- **get_context**: Removed raw `ref_score` from output (centrality label only), removed redundant file map from compact mode, shortened readable-mode separators, changed default format from readable to compact.
- **Workspace tools**: Removed "Performance Recommendations" filler from health, "enjoy!" line, and cleanup filler.
- **Skills**: Updated explore-area, impact-analysis, search-debug for ref_score removal and format default change.

### Impact
- Every `fast_search` call saves ~50 tokens of dead field computation
- `get_context` compact output is ~30% leaner (no file map, no ref_score, shorter headers)
- Health reports save ~200 tokens of filler per call
- All 901 tests pass on fast tier
