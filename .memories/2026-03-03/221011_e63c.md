## Token Efficiency Cleanup — All 6 Steps Complete

### WHAT
Systematic cleanup of token waste across Julie MCP tool formatting:
- **Step 1**: Stripped `OptimizedResponse` from 6 fields to 2 (`results`, `total_found`). Deleted entire `src/tools/search/scoring.rs` (3 dead functions). Removed dead computation from search pipeline.
- **Step 2**: Removed "Performance Recommendations" filler from health report, compacted search engine status.
- **Step 3**: Removed "Cleanup helps maintain optimal performance" filler from workspace clean.
- **Step 4**: Dropped raw `ref_score` from get_context output (both modes), removed redundant file map from compact mode.
- **Step 5**: Shortened 50-char Unicode separator lines to ~15 chars in readable mode.
- **Step 6**: Changed get_context default format from readable to compact.

### WHY
Every token in tool output eats context window. Audit found dead code computed but never output, redundant information repeated across sections, verbose formatting, and filler text.

### HOW
- Traced `OptimizedResponse` field usage end-to-end — 4 of 6 fields were vestigial from a JSON era
- Identified that `insights`, `next_actions`, `confidence`, `tool` were computed on every search but never reached lean formatters
- File map in compact mode repeated every symbol's file path already shown in PIVOT/NEIGHBOR lines
- Changed default to compact since primary consumers are AI agents

### IMPACT
- CPU savings: eliminated HashMap allocations + string building on every search call
- Token savings: compact default, no redundant file map, no raw ref_score, shorter separators
- Code clarity: `OptimizedResponse` is now 2 fields instead of 6, scoring.rs deleted entirely
- 901 tests pass, 0 failures
