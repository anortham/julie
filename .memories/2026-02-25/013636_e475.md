## De-boost namespace/module/export pivots in get_context (TDD)

### What
Added `STRUCTURAL_KIND_PENALTY` (0.2x) to `select_pivots()` scoring in `scoring.rs`. Namespace, module, and export declarations now get a multiplicative penalty so real functions/structs/classes rank above them.

### Why
The get_context tool was selecting `pub mod token_estimation;` (a useless 1-line namespace declaration) as a top pivot result. These structural declarations match on keywords but carry zero useful code context.

### How
- Made `context_factor` multiplicative: `path_factor * kind_factor`
- `is_structural_kind()` matches `"namespace" | "module" | "export"`
- Penalties stack: namespace in test file = `0.3 * 0.2 = 0.06x`
- Sole namespace still returned (penalized, not filtered)

### Files changed
- `src/tools/get_context/scoring.rs` — Added `STRUCTURAL_KIND_PENALTY`, `is_structural_kind()`, refactored `context_factor` to multiply path and kind factors (140 lines total)
- `src/tests/tools/get_context_scoring_tests.rs` — 4 new TDD tests (created)
- `src/tests/mod.rs` — Registered new test module

- 9 existing select_pivots tests pass (zero regressions)
