## Implemented Token Budget Enforcement in get_context Pipeline

### What
Added `truncate_to_token_budget()` function to `pipeline.rs` and wired it into `build_pivot_entries` to enforce per-pivot token budgets.

### Why
Previously, `pivot_tokens` was computed by the allocation system but never enforced. A single 300-line class body could blow past a 1500-token `max_tokens` setting, defeating the purpose of token budgets.

### How
- **New function** `truncate_to_token_budget(code, max_tokens)` — uses `TokenEstimator` to check size, applies head-biased truncation (2/3 top, 1/3 bottom) when content exceeds budget
- **Wired into** `build_pivot_entries` after content selection: divides `allocation.pivot_tokens` by pivot count to get per-pivot budget
- **Head-biased truncation** preserves function signature + initial logic while keeping closing braces/return statements

### Tests (TDD)
- `test_truncate_small_content_unchanged` — passthrough when within budget
- `test_truncate_large_content_reduced` — verifies truncation with omission marker
- `test_truncate_preserves_head_bias` — signature preserved at top, closing brace at bottom
- `test_truncate_very_short_content_unchanged` — 5 lines or fewer never truncated
- `test_pipeline_respects_token_budget` — integration test with 100-token budget

### Impact
- Fixes the known issue: "get_context token budget not strictly enforced"
