## Implemented token budget enforcement in get_context

The `get_context` tool was computing `pivot_tokens` via the allocation system but never enforcing it — a single 300-line class body could blow past a `max_tokens: 1500` setting.

- **What:** Added `truncate_to_token_budget()` in `pipeline.rs` with head-biased truncation (2/3 top, 1/3 bottom), wired it into `build_pivot_entries` after content selection
- **How:** Uses `TokenEstimator` for size checking, proportional line scaling, preserves function signatures + closing context
- **Tests:** 4 unit tests + 1 integration test, with verified budget assertions
- **Review:** Passed spec compliance review + code quality review (with one fix iteration to strengthen test assertions)
