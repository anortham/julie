## Fixed Tantivy backfill and force-reindex for reference workspaces

### Root Cause
`backfill_tantivy_if_needed` and force-clear were gated behind `if is_primary_workspace`. Reference workspaces never got Tantivy recovery — if their index was empty/stale, incremental updates only added changed files' symbols, leaving most symbols invisible to search.

### Changes
1. **`refresh_stats.rs`**: Pass `self.force.unwrap_or(false)` through instead of hardcoded `false`
2. **`index.rs`**: Added `else` branch for reference workspaces:
   - `force_reindex=true`: deletes reference Tantivy directory, lets `process_files_optimized` rebuild
   - Tantivy missing: calls new `backfill_reference_tantivy()` helper
3. **`index.rs`**: New `backfill_reference_tantivy()` method that opens reference DB + creates Tantivy, reads all symbols/files from DB, populates Tantivy
