## Implemented 3 quality tuning fixes for get_context

After live testing with 5 queries against Julie's own codebase, identified and fixed 3 quality issues:

### Fixes
- **Import filtering**: `kind == "import"` nodes filtered out before pivot selection — imports never contain useful code
- **Test file de-boost**: 0.3x penalty for test paths in `select_pivots()` — tests match well on text but provide wrong context
- **Noise neighbor filtering**: Blocklist of 11 common trait methods (`clone`, `to_string`, `fmt`, etc.) filtered from both neighbors list and callers/callees display

### TDD
- 3 new failing tests written first, then all 3 fixed
- 658 lib tests pass (3 new), 0 regressions

### Key decisions
- `TEST_FILE_PENALTY = 0.3` — aggressive enough that a test with 2x the text score still loses to production code
- `is_test_path()` checks `/tests/`, `/test/`, `_test.`, `_tests.`, `.test.`, `.spec.`
- `NOISE_NEIGHBOR_NAMES`: conservative blocklist of 11 names — only truly-noise trait methods

### Files changed
- `src/tools/get_context/pipeline.rs` — added filtering/de-boost logic
- `src/tests/tools/get_context_tests.rs` — 3 new tests
