## Investigated cross-workspace get_context issue — NOT A BUG

### What happened
`get_context("naming convention", workspace=miller_b275269b)` returned a `NamingConvention` enum that appeared to come from Julie's primary Rust workspace. User asked to investigate.

### Root cause
The miller reference workspace is a **multi-language project** containing 19 Rust files alongside Python code. The `NamingConvention` enum at `src/utils/cross_language_intelligence.rs` is a legitimate symbol in the miller workspace, not leaked from the primary workspace.

### Evidence
- `SELECT DISTINCT language FROM files` in miller DB: javascript, makefile, markdown, python, **rust**, text, toml
- 19 Rust files indexed in miller workspace
- `NamingConvention` enum exists at `src/utils/cross_language_intelligence.rs` in the miller DB
- Workspace isolation via `workspace_db_path` and `workspace_tantivy_path` is correct — both use `indexes_root_path().join(workspace_id)`

### Conclusion
No code fix needed. Workspace isolation is working correctly.
