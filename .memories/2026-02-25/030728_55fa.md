## Fixed reference workspace support for `rename_symbol` and `get_context`

### rename_symbol (Task 1)
- **Problem:** `rename_in_file()` and `update_imports_in_file()` always resolved paths against primary workspace root
- **Fix:** Added `resolve_workspace_root()` helper in `refactoring/mod.rs` that looks up `WorkspaceRegistryService.original_path` for reference workspaces
- **Threading:** Workspace root resolved once in `handle_rename_symbol`, passed as `&Path` to all file-level methods
- **Lazy resolution:** Only resolves if `file_locations` is non-empty (avoids breaking tests with no indexed files)

### get_context (Task 2)
- **Problem:** Explicit hard block rejecting non-primary workspaces in `pipeline.rs`
- **Fix:** Replaced with `resolve_workspace_filter()` + branch pattern (same as `deep_dive`/`text_search`)
- **Reference path:** Opens `SymbolDatabase::new()` + `SearchIndex::open()` from workspace paths
- **Primary path:** Unchanged — uses shared `Arc<Mutex>` from handler
