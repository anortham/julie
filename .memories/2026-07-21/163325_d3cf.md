## Removed outer Arc<Mutex<SearchIndex>>

### WHAT
Shared search index is now `SearchIndexHandle = Arc<SearchIndex>`. Dropped the outer mutex that serialized every reader.

### WHY
`SearchIndex` already uses `&self` + an interior writer `Mutex` and a concurrent Tantivy `IndexReader`. The outer lock caused reader lock starvation (esp. holding lock across hybrid embed sidecar RPC historically).

### HOW
- Type alias + mechanical call-site updates across handler/runtime/tools/tests
- Concurrent-reader barrier test in `search_index_concurrency_test.rs`
- Integration test now holds `acquire_writer_for_test()` instead of outer mutex
- Stale "SearchIndex lock" comments updated

### IMPACT
Concurrent Tantivy searches no longer serialize on a process-wide SearchIndex mutex. Writers still serialize via interior writer mutex.
