## v6 Phase 3 Complete: Shared Embedding Pipeline

### What
Implemented the shared embedding pipeline for daemon mode. 9 commits, 9 tasks, all tests green.

### Key Changes
- **`src/embeddings/init.rs`** (new): Extracted `create_embedding_provider()` from `JulieWorkspace::initialize_embedding_provider()` into standalone function
- **`src/daemon/embedding_service.rs`** (new): `EmbeddingService` struct owns shared provider, created eagerly at daemon startup
- **`src/handler.rs`**: New `embedding_service` field, `embedding_provider()` and `embedding_runtime_status()` helpers that route daemon-vs-stdio
- **`src/daemon/mod.rs`**: `run_daemon()` creates EmbeddingService via `spawn_blocking`, passes through accept_loop to sessions, shuts down on exit
- **`src/daemon/watcher_pool.rs`**: `attach()` accepts shared provider for incremental re-embedding
- **`src/daemon/workspace_pool.rs`**: Stores `EmbeddingService`, passes provider to watcher pool
- **6 tool files migrated**: text_search, nl_embeddings, get_context/pipeline, fast_refs, indexing/embeddings, health

### Impact
- Daemon-mode sessions now have full hybrid search (keyword + semantic)
- Single ORT model load / sidecar process across all sessions
- Incremental re-embedding after file saves works in daemon mode
- Stdio mode fully preserved (falls through to per-workspace provider)
- All 8 dev test buckets pass (440s)
