## v6 Phase 2 Complete + Workspace ID Bug Fixed

### What
Completed the entire v6 Phase 2 implementation (daemon.db, shared workspaces, codehealth trends, WatcherPool) using a 3-teammate agent team. Then dogfooded it and found/fixed two workspace ID bugs.

### Phase 2 Delivered
- `daemon.db` with 4 tables (workspaces, workspace_references, codehealth_snapshots, tool_calls)
- WorkspacePool backed by persistent storage (survives daemon restarts)
- WatcherPool with ref-counted IncrementalIndexers and 5-minute grace period

- WorkspaceRegistryService removed (927 lines)
- Auto-inherit reference workspaces on session connect

### Dogfooding Bugs Found and Fixed
1. **Codehealth snapshot FK failure** (`7fae889e`): `process_files_optimized` used a regenerated workspace_id that didn't match daemon.db's row. Fix: use `handler.workspace_id` for snapshots.
2. **Workspace ID mismatch + path doubling** (`14d4aa58`): `handle_ipc_session` wrapped `generate_workspace_id()` output in another prefix, creating divergent IDs. Plus `workspace_db_path()` doubled the path when `index_root_override` was set. Fix: use `generate_workspace_id()` directly as pool ID; skip ID nesting when override is set.
