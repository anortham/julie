## C6 Complete: Removed WorkspaceRegistryService Entirely

### WHAT
Completed C6 (Remove Deprecated Registry Code) for v6 Phase 2. Deleted `WorkspaceRegistryService` (JSON-based per-project workspace registry) from the entire codebase and fixed all test files that depended on it.

### WHY
`WorkspaceRegistryService` stored workspace state in a per-project `registry.json`. In v6 Phase 2, `DaemonDatabase` (`~/.julie/daemon.db`) is the authoritative registry. The old service was redundant and prevented clean daemon-first architecture.

### HOW

**Files deleted:**
- `src/workspace/registry_service.rs` (927 lines)
- `src/tests/tools/workspace/registry_service.rs`
- `src/tests/tools/workspace/runtime_status_stats.rs`

**Key replacement pattern** — all callers of `get_primary_workspace_id()` replaced with:
```rust
// daemon mode: handler.workspace_id (set at session init)
// stdio mode: generate_workspace_id(&workspace.root.to_string_lossy())
```

**Tests using `manage_workspace add`** — now daemon-only, so tests replaced with:
- `manage_workspace index <path>` + `generate_workspace_id(path)` for integration tests (workspace_isolation_smoke, reference_workspace)
- Manual `SymbolDatabase::new() + bulk_store_symbols()` for unit tests that just need specific symbols present (get_symbols_reference_workspace, reference_workspace_fast_refs_tests)

**Critical insight:** `manage_workspace index` triggers expensive post-indexing analysis (centrality scoring, test quality metrics, change risk, codehealth snapshot) — 300+ seconds on a real repo. Unit tests that only need symbols in a reference workspace DB should use manual DB population instead of live indexing.
