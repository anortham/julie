## v6 Phase 3 Design + Plan Complete

Brainstormed, designed, spec-reviewed, and wrote the implementation plan for Phase 3: Shared Embedding Pipeline.

### What
- Design spec: `docs/superpowers/specs/2026-03-23-v6-phase3-shared-embedding-pipeline-design.md`
- Implementation plan: `docs/superpowers/plans/2026-03-23-v6-phase3-shared-embedding-pipeline.md`
- 9 tasks across 3 tracks, estimated 2-3 hours

### Key Design Decisions
- **`EmbeddingService` struct** at daemon level owns the shared provider (not on WorkspacePool)
- **Eager init** at daemon startup (not lazy per-session)
- **Injected into handler** via new `embedding_service` field, with `handler.embedding_provider()` helper
- **No explicit queue** - provider's internal mutex handles serialization (YAGNI)
- **Extracted `create_embedding_provider()`** from `JulieWorkspace::initialize_embedding_provider()` into standalone `src/embeddings/init.rs`

### Spec Review Caught
- Missing watcher_pool gap (incremental re-embedding after file saves was silently broken in daemon mode)
- Incorrect mutex fairness claim
- Missing stdio fallback in `embedding_runtime_status()`

### Plan Review Caught
- 5+ test call sites of `new_with_shared_workspace` needed updating (not just 1 production site)

### Execution
Going with subagent-driven development (not agent teams) because dependency chain is too tight for parallel worktrees.
