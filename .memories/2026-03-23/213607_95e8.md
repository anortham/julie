## Daemon Stale-Binary Auto-Restart Implemented

### What
Added automatic stale-binary detection to the daemon. When the binary is rebuilt while the daemon is running, it detects the mismatch and exits gracefully after the last session disconnects. The adapter auto-starts a fresh daemon on the next connection.

### How
- Capture binary mtime (`current_exe().metadata().modified()`) at daemon startup
- On each new IPC session connect, compare current mtime to startup mtime
- If stale, set `restart_pending` AtomicBool flag and log warning
- On last session disconnect (active_count == 0 && restart_pending), self-SIGTERM
- Health check surfaces "Binary Status: STALE" or "CURRENT"
- Zero new dependencies; reuses existing shutdown/drain path

### Impact
Eliminates the #1 dev-loop friction point: `cargo build --release` + restart Claude Code now Just Works without manually stopping the daemon. The feature we needed since the Phase 3 dogfood where the daemon was running pre-Phase-3 code.
