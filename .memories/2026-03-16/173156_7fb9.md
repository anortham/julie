## GPT Review — Three Fixes Implemented

All three gaps identified in GPT's external review of Julie are now fixed and merged to main.

### What was fixed

1. **Qualified symbol lookup** — `deep_dive(symbol="SearchIndex::search_symbols")` and `fast_refs` now resolve qualified names by splitting on `::` or `.`, looking up the child name, then filtering by parent ID match. The `parse_qualified_name` parser existed but was dead code — now wired into both tools.

2. **Fixture/benchmark scoring penalty** — `NL_PATH_PENALTY_FIXTURES` increased from 0.95 → 0.75 (25% reduction vs previous 5%). Added `benchmarks`/`benchmark` to `is_fixture_path` path detection. Source code now has 44% relative advantage over fixtures in NL search.

3. **Rename dry-run line-level preview** — `rename_symbol` dry-run now shows `L{n}: - old` / `L{n}: + new` for each changed line (capped at 5 per file), replacing the previous file-level-only summary. New `RenameChange` struct and `compute_line_changes` function replace the bogus line-count heuristic.

### How
- 4 parallel subagents in isolated worktrees (zero file overlap)
- All followed TDD (red → green)
- All 4 merged cleanly, `cargo xtask test dev` — **8 buckets passed, 0 regressions**

### Key discovery
- `Symbol` has `parent_id` not `parent_name` — both agents (Tasks 1 & 2) independently discovered this and adapted by resolving parent names to IDs via a second DB query
