## Built GitHub Pages site for Julie

Single-page dark-terminal-themed site at `docs/site/` showcasing Julie's code intelligence capabilities.

### What
- Full site: `index.html` (31K), `style.css` (19K), `script.js` (13K), `favicon.svg`, `404.html`
- GitHub Actions workflow for deployment from `docs/site/` on main
- 11 sections: Hero with context drain animation, token savings table, how it works, reference graph SVG, code health intelligence, semantic embeddings, 7 tool cards, 31 language badges, performance stats with count-up, tabbed installation (Claude Code/VS Code/OpenCode/Cursor), footer

### Why
Needed a user-facing site that goes beyond the README — visual, demo-forward, showing what Julie does rather than just describing it.

### How
- Brainstorming → design spec → reviewed spec → implementation plan → reviewed plan → built it
- Pure static HTML/CSS/JS, no build step, no dependencies beyond JetBrains Mono font
- Dark terminal aesthetic with scroll-triggered animations (Intersection Observer)
- All token savings numbers dogfood-verified against both Julie (Rust) and LabHandbookV2 (.NET/Vue/TS)

### Key decisions
- `docs/site/` with GitHub Actions (not `docs/` root, which has dev docs)
- Demo-forward design over copy-heavy marketing
- Honest token numbers (75-90% savings) verified by real tool calls, not aspirational
