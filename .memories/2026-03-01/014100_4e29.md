## Implemented C# DI Registration Relationship Extraction

DI-registered classes (`services.AddScoped<SearchFilesTool>()`) previously had **zero graph centrality** because no source code references them directly. Now they get `Instantiates` relationships (weight 2.0) from registration calls.

### What was built
- **New extractor:** `crates/julie-extractors/src/csharp/di_relationships.rs` (~130 lines)
- **Recognizes 10 DI methods:** AddSingleton/Scoped/Transient, AddHostedService, TryAdd*, AddKeyed*
- **Handles:** single/dual generic args, chained member access (`builder.Services.AddScoped<T>()`)
- **Cross-file:** creates `PendingRelationship` when types aren't in the same file

### Dogfood bug found & fixed
- C# class and constructor share the same name → `symbols.iter().find()` could match the constructor
- **Fixed in two places:** same-file extraction (kind filter with fallback) and cross-file resolver (`+10` scoring bonus for Instantiates → Class/Interface/Struct)
