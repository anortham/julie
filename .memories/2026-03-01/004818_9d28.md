## Fixed semantic search quality for C# codebases

### WHAT
Three-layer fix to improve `get_context` NL query results for C# codebases:
1. **Language filter** — Non-code embeddings (markdown, JSON, TOML, YAML, CSS, HTML, regex, SQL) purged from vector space
2. **Container enrichment** — Class/struct/interface/trait embeddings now include child method names (`methods: SearchAsync, IndexDocumentAsync, ...`)
3. **XML doc comment stripping** — `first_sentence()` now skips `<summary>` tags and extracts actual description text

### WHY
Query "Lucene search implementation" on coa workspace returned irrelevant results (SearchFilesTool, HandleSemanticOnlySearchAsync). Root causes:
- Markdown headings ("🚀 Features", "Search Engine") dominated semantic space
- Container embeddings were sparse (~20 words) missing key method names
- C# XML doc comments produced `<summary>` as the doc excerpt instead of actual descriptions

### HOW
- `src/embeddings/metadata.rs`: Added `NON_EMBEDDABLE_LANGUAGES`, `is_embeddable_language()`, `CONTAINER_KINDS`, child method enrichment in `prepare_batch_for_embedding()`, XML tag stripping in `first_sentence()` + `strip_xml_tags()`
- `src/database/vectors.rs`: Added `delete_embeddings_for_languages()` for purging non-code vectors
- `src/embeddings/pipeline.rs`: Non-code purge before incremental check, container symbols always re-embedded (fresh enrichment)
- `src/search/hybrid.rs`: Diagnostic logging at debug level for keyword/semantic top-10

### IMPACT
- "Lucene search implementation" → now returns FileIndexingService + PathResolutionService (genuinely Lucene-related) instead of SearchFilesTool + HandleSemanticOnlySearchAsync
- "LuceneIndexService search index" → returns LuceneIndexService as pivot (previously never appeared)
- Purged 1126 non-code embeddings from Julie, 261 from coa
- 441 container symbols re-embedded with enriched text in coa
