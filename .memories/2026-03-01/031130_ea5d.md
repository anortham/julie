## Fixed sidecar GPU device selection: CUDA vs DirectML mutual exclusivity

### WHAT
Fixed the Python sidecar bootstrap to properly select CUDA (NVIDIA RTX 4080) over DirectML, and fixed a crash caused by GPU memory accumulation.

### WHY
Three compounding bugs prevented the sidecar from using the RTX 4080:
1. **Python 3.14 venv** — `py` launcher defaulted to Python 3.14 which has zero PyTorch wheels
2. **`torch-directml` pins `torch==2.4.1`** — installing it alongside CUDA torch forces a downgrade to CPU-only torch
3. **GPU memory leak** — `convert_to_numpy=False` kept tensors on GPU, causing DirectML to crash after ~16K embeddings

### HOW
- **Python detection**: `detect_bootstrap_python_interpreter` now uses `uv python find 3.12` before system candidates; `ensure_venv_exists` guards against unsupported Python versions
- **Split install strategy**: CUDA torch installed first from `whl/cu124` with `--upgrade`; `.[runtime]` installed separately (no torch-directml); `venv_has_cuda()` probes CUDA availability; `torch-directml` only installed if CUDA is absent
- **Memory fix**: `convert_to_numpy=True` in `embed_batch` releases GPU memory immediately
- **DirectML patch**: `_patch_directml_inference_mode` replaces `torch.inference_mode` with `torch.no_grad` before importing sentence-transformers (microsoft/DirectML#622)

### IMPACT
- NVIDIA machines get CUDA torch → fast, stable GPU embeddings on RTX 4080
- Non-NVIDIA machines get DirectML fallback → works on Intel/AMD GPUs
- Memory leak fix prevents sidecar crashes on long embedding runs
