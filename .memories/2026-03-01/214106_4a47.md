## Step 1 Complete: Property/Field Enrichment in Embedding Metadata

Added property and field names to container symbol embedding text, dramatically improving the semantic fingerprint for DTOs, data classes, and interfaces.

### What Changed
- **`src/embeddings/metadata.rs`**: Added `properties_by_parent` HashMap that collects `Property|Field` children alongside existing `methods_by_parent`
- Container symbols (Class, Struct, Interface, Trait) now embed with `" properties: X, Y, Z"` appended
- **4 new TDD tests** in `src/tests/core/embedding_metadata.rs` covering C# DTOs, TS interfaces, mixed method+property classes, and Rust structs

### Why
Cross-language DTO similarity scores were low (avg 0.52) because embedding text like `"class UserDto public class UserDto"` contained zero semantic content from properties. Properties ARE the structural fingerprint of data types — they enable matching C# UserDto ↔ TS UserDto.

### Baseline Scores (Pre-enrichment)
| TS → C# Query | Score |
|---------------|-------|
| UserDto | 0.47 |
| PageDto | 0.54 |
| ApiResponse | 0.50 |
| ContentBlockDto | 0.58 |
| **Average cross-lang** | **0.52** |
| **Average within-lang** | **0.63** |
| **Gap** | **0.11** |

### Impact
- Before: `"class UserDto public class UserDto"` (no semantic content)
- After: `"class UserDto public class UserDto properties: Id, SamAccountName, DisplayName, Email, IsActive, LastLoginAt, Roles"`
- All 874 fast-tier tests passing
