## Added defensive text sanitization and fallback encoding to sidecar

With the protocol exception handler working, we now see the actual error: `TypeError: TextEncodeInput must be Union[TextInputSequence, Tuple[InputSequence, InputSequence]]` — the HuggingFace tokenizer rejects some text in batch 34.

### Three defensive layers added:
1. **`_sanitize_texts()`** — pre-validates all inputs: replaces empty/whitespace-only strings with `"[empty]"`, strips null bytes, replaces non-string values. Language-agnostic (handles Unicode from all 30+ languages).
2. **`_encode_individually()`** — fallback when batch encode fails. Tries each text one-at-a-time, substituting a zero vector for any unencodable text. Logs the problematic text to stderr for diagnostics.
3. **Pipeline resilience** (from previous fix) — breaks on batch failure instead of aborting, preserving all successful embeddings.
