## Cross-file inheritance ported to 5 languages + fast_refs import classification

### WHAT
- Ported C# cross-file inheritance `PendingRelationship` fix to Java, TypeScript, JavaScript, Kotlin, Swift
- Added `implements_clause` support to TypeScript (was only handling `extends_clause`)
- Fixed `fast_refs` to show imports in separate "Imports" section instead of lumping with "Definitions"

### WHY
- All 5 OOP languages silently dropped cross-file extends/implements relationships
- This broke `get_context` pivot selection for any DI-heavy codebase
- Import noise in `fast_refs` buried real definitions (e.g., 120 imports hiding 3 definitions)

### HOW
- Two-phase borrow pattern (collect data → create relationships) applied to each language
- Changed extractor signatures to `&mut` where needed for `add_pending_relationship`
- Partitioned definitions into `real_definitions` and `import_definitions` in formatting.rs

### IMPACT
- 10 new cross-file tests across 5 languages, 1347 extractors tests pass
- `fast_refs` now cleanly separates Definitions/Imports/References
- Dogfood verified on coa-codesearch-mcp (C#), miller (Python), LabHandbookV2
