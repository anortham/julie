## Completed Track 4 Task 3: indexing and query paths honor settled embedding states

### WHAT
Finished the Task 3 slice that makes indexing and NL-definition search respect settled embedding runtime state instead of blindly retrying provider initialization.

- Added a regression test proving `spawn_workspace_embedding()` must not retry stdio provider init once the workspace already carries an unavailable runtime status
- Added daemon-mode tests proving `spawn_workspace_embedding()` skips cleanly when the shared embedding service is unavailable or times out, without falling through to stdio sidecar startup
- Updated `spawn_workspace_embedding()` so stdio mode short-circuits when `embedding_runtime_status` is already set and no provider exists, preserving the bounded degraded outcome instead of re-running provider bootstrap
- Re-verified the existing NL-definition settled-state behavior and daemon embedding-service behavior after the change

### WHY
Task 3 requires indexing and query paths to consume settled runtime truth instead of surprise fallback behavior.

Before this change:
- NL query already avoided duplicate stdio init when runtime state was settled
- indexing still retried stdio provider init after a prior unavailable runtime status, which wasted work and ignored the recorded degraded outcome
- daemon-mode indexing behavior around unavailable and timeout states was not pinned with direct tests

### HOW
- Wrote a failing workspace test using the slow-health fake sidecar harness to prove stdio indexing was still re-entering provider init after a recorded failure
- Fixed `src/tools/workspace/indexing/embeddings.rs` to check `handler.embedding_runtime_status()` before attempting deferred stdio initialization, logging the settled degraded state and returning early
- Added daemon-mode workspace tests for `Unavailable` and timeout cases, asserting no fallthrough into stdio sidecar startup
- Re-ran the daemon embedding-service tests plus the NL query settled-state tests to verify the shared contract still holds

### IMPACT
- Indexing now behaves like NL query: once runtime state says embeddings are unavailable, Julie stays in the bounded degraded mode instead of hammering sidecar startup again
- Daemon-mode embedding use has direct regression coverage for ready, unavailable, and timeout behavior
- Task 4 can now focus on surfacing embedding degradation through health and dashboard without shaky runtime semantics underneath
