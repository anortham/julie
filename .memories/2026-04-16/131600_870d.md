## Track 1 Task 2 transport contract

- **WHAT:** Added `src/daemon/transport.rs` as the lifecycle-facing IPC contract and routed launcher probing plus daemon bind setup through it. Added transport tests for Unix round-trip, live readiness, stale-socket rejection, and timeout behavior.
- **WHY:** Platform-specific readiness probing lived in `adapter/launcher.rs`, which mixed daemon policy with Unix socket and Windows named-pipe details. That made the launcher harder to reason about and left Windows probe semantics isolated in one brittle helper.
- **HOW:** Introduced `TransportEndpoint` with `bind_listener`, `connect`, `probe_readiness`, and `wait_for_readiness`, while keeping `src/daemon/ipc.rs` as the low-level transport implementation. `DaemonLauncher` now consumes the transport contract for IPC fallback and wait loops, and `run_daemon` binds through the same contract. Integration readiness checks now use the transport contract too.
- **IMPACT:** Control-plane code now reasons about a shared transport abstraction instead of platform branches. Unix stale-socket handling stays pinned, and Windows named-pipe existence probing is centralized for the later restart-handoff cleanup.
