## Completed Track 2 Task 1 route hardening

- **WHAT:** Finished the shared indexing-route work and fixed the detached rebound-current-primary regression that showed up during `tools-workspace` verification.
- **WHY:** The new route seam still had a hidden assumption that a current primary workspace was also the loaded handler workspace. Non-force refresh on a rebound current primary disproved that and failed before indexing could start.
- **HOW:**
  - Added path-backed fallback behavior in `src/tools/workspace/indexing/route.rs` so current-primary routing can resolve database and Tantivy storage without requiring the loaded workspace anchor.
  - Moved indexing-path database and Tantivy opens onto the route snapshot seam instead of mixing `handler.get_workspace()` or handler workspace-specific helpers inside `index.rs`, `processor.rs`, and `incremental.rs`.
  - Re-ran the rebound-primary route tests, then the `system` and `dev` xtask tiers, to make sure the fix held across startup, refresh, and integration flows.
- **IMPACT:**
  - Track 2 Task 1 is now complete and verified.
  - Detached current-primary refresh/index flows no longer fail because the daemon pool or loaded workspace is out of sync with the session binding.
  - The route seam now owns more of the storage-resolution truth, which makes the next stage-boundary refactor less brittle.
