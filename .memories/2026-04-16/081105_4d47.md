## Landed the first world-class systems baseline slice

- **WHAT:** Added a shared health model for Julie with control-plane, data-plane, and runtime-plane vocabulary; rewired `manage_workspace health` to use it; promoted the dashboard status page and live route to surface the same health language; added xtask program tiers for `reliability` and `benchmark`; and added `system_health` integration coverage for projection loss and repair.
- **WHY:** The old health path was fragmented across ad hoc strings and booleans, which made it hard to reason about daemon state, projection state, and runtime degradation consistently. This slice creates a common contract that later control-plane, indexing, and storage work can build on.
- **HOW:** Split the new health module into `src/health/` submodules to stay under file-size limits, introduced `SystemHealthSnapshot` plus shared enums, kept `SystemStatus` for search gating, updated workspace health to reuse the shared watcher/projection/runtime vocabulary, and let the dashboard derive an operational health snapshot from daemon state instead of inventing separate labels. The xtask lane added special program tiers and a `system_health` integration module for projection degradation and repair.
- **IMPACT:** `manage_workspace health`, the dashboard, and xtask now speak a compatible health language; the branch has regression coverage for projection loss/repair; and the program has visible harness entry points instead of buried intent. This gives the next tracks a stable base for daemon lifecycle and indexing work.
