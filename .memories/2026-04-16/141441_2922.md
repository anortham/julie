## Landed Track 1 Task 4: daemon/session lifecycle visibility

### WHAT
Wired explicit daemon and session lifecycle state through the runtime, handler session layer, and dashboard.

- Added `SessionLifecyclePhase` and per-phase aggregation to `SessionTracker`
- Made `SessionWorkspaceState` derive lifecycle phase from binding, swap, serving, and closing state
- Attached daemon IPC sessions to a lifecycle handle so phase changes propagate from handler state into tracker state
- Threaded shared daemon lifecycle phase into `DashboardState`
- Surfaced daemon phase, shutdown cause, and session phase counts in the dashboard health snapshot and status templates
- Added focused tests for tracker phases, handler phase transitions, dashboard health payloads, and rendered status output

### WHY
Task 4 was meant to kill invisible side effects in daemon replacement and request-time primary rebinding. Before this change the dashboard only knew `active_sessions` and `restart_pending`, which hid the interesting control-plane truth and made rebinding/swap gaps impossible to reason about from the UI.

### HOW
- Replaced the tracker's `HashSet<String>` with `HashMap<String, SessionLifecyclePhase>`
- Introduced a cloneable `SessionLifecycleHandle` carried by daemon IPC sessions
- Refactored handler session-state writes through a small helper that recomputes lifecycle phase and publishes it to the tracker
- Modeled session phase in `SessionWorkspaceState` from real state instead of storing an error-prone manual enum value
- Added a shared daemon phase cell beside the state-file writes so the dashboard reads runtime truth, not stale approximations
- Reused the lifecycle types in dashboard health output and templates

### IMPACT
- Dashboard status now shows daemon phase, restart-required state, shutdown cause, and session phase counts
- Request-time primary binding and swap intent now update lifecycle state coherently
- Daemon lifecycle state is shared between runtime control flow and dashboard reporting
- Track 1 now has the visibility needed for deeper lifecycle and Windows restart work without guessing from logs
