## WHAT
Added structured health metadata to the Python embedding sidecar, splitting backend capability probing from runtime load policy.

## WHY
The health response needed explicit capability data and explicit fallback/load decisions so the runtime no longer relied on stderr-only diagnostics.

## HOW
- `sidecar.runtime` now records backend capability booleans and a structured load policy during runtime construction.
- `sidecar.protocol` validates health metadata before returning it, including nested capability and load policy consistency checks.
- Test fixtures and protocol/runtime tests were updated to pin the new shape and reject inconsistent metadata.

## IMPACT
Health responses now surface backend availability and fallback decisions in-band, which makes the sidecar contract easier to validate and less dependent on log scraping.
