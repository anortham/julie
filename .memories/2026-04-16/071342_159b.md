## Designed Julie's world-class systems hardening program

### WHAT
Wrote and aligned a multi-track design for Julie's next foundational improvement wave, covering daemon lifecycle, indexing, canonical storage and projections, embeddings, and observability.

### WHY
The current system works, but core flows carry accumulated deadlock fixes, corruption-window fixes, restart handoff fixes, and platform-specific seams. The goal is to make Julie a high-trust, world-class tool for AI agents by attacking root architecture problems instead of extending fragile hot paths.

### HOW
- Chose a shared-contract approach before subsystem work
- Defined three planes: control, data, and runtime
- Set SQLite as canonical state and Tantivy/vectors as rebuildable projections
- Defined shared contracts for workspace identity, session lifecycle, indexing, projection revisions, and health
- Split execution into parallel tracks:
  - Track 0: contract reset and instrumentation
  - Track 1: daemon/adapter/IPC/Windows lifecycle
  - Track 2: indexing pipeline unification
  - Track 3: canonical storage and rebuildable projections
  - Track 4: embeddings and query-time degradation
- Integrated the dashboard as a first-class health and repair surface
- Added sequencing, phase gates, and acceptance criteria for reliability, benchmarks, and dogfooding

### IMPACT
This creates a concrete foundation for the next planning step. The design now has shared contracts, measurable success gates, and a clear argument for where to cut first without losing sight of the whole program.
