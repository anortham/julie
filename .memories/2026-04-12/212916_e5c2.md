## Completed Task 3 of workspace-startup-roots plan

Finished the large session-state and primary-binding cleanup for handler/daemon/tool routing. This was the demanding task in the plan because it exposed a long chain of stale loaded-vs-current primary assumptions across the codebase.

- **What changed:** Added `SessionWorkspaceState`, explicit current-vs-loaded-vs-attached semantics, swap-in-progress handling, current-primary-aware storage-anchor helpers, truthful loaded-workspace tracking, and handler-owned primary snapshots. Migrated primary consumer paths across search, health, indexing, editing, refactoring, symbols, metrics, startup maintenance, and daemon accounting onto those semantics.
- **Why:** Task 4 needs request-time roots resolution to mutate the session primary safely. Task 3 had to remove the old split-brain where one path used the loaded workspace while another used the current primary, or where swap windows silently fell back to startup/root guesses.
- **How:** Tightened helper APIs, made raw mutators test-only, fixed daemon pool attachment/accounting for multi-hop and bounce-back rebinds, sealed swap windows with explicit invalidation, and added broad regression coverage for rebind, swap-gap, and rebound-primary behavior.
- **Impact:** The session now has a coherent primary-binding model that later roots work can build on. Remaining reviewer comments at the end were design disagreements with already-approved Task 1/2 decisions (explicit missing overrides remain authoritative; malformed explicit `WORKSPACE_SOURCE` values fail loudly), not new Task 3 correctness bugs.
