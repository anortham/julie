## Completed Task 1 of workspace-startup-roots plan

Implemented the startup-hint and handshake groundwork in the isolated `workspace-startup-roots` worktree.

- **What changed:** Added `WorkspaceStartupHint` and `WorkspaceStartupSource`, wired source-aware CLI resolution into `main.rs`, updated adapter handshake serialization, and fixed daemon IPC header parsing to preserve MCP framing.
- **Why:** Task 2 and later roots work need stable startup metadata across CLI, adapter, and daemon. The old single-path handoff lost source intent and the initial adapter changes exposed multiple framing edge cases.
- **How:** Preserved explicit path authority in CLI/env resolution, absolutized nonexistent relative explicit paths against adapter cwd, ordered handshake headers as `WORKSPACE`, `VERSION`, `WORKSPACE_SOURCE`, and switched the daemon parser to deterministic blank-line-terminated startup block parsing with split-write regression coverage.
- **Impact:** The startup transport seam is now solid enough to carry roots/session state work without corrupting the MCP stream or misclassifying explicit workspace hints.
