## Completed Task 2 of workspace-startup-roots plan

Extracted daemon IPC-session parsing and startup-hint plumbing into a dedicated module, then tightened the typing so startup provenance reaches the handler boundary without fabricated defaults.

- **What changed:** Moved header parsing, prefixed-stream replay, and session bootstrap into `src/daemon/ipc_session.rs`; slimmed `src/daemon/mod.rs`; threaded `WorkspaceStartupHint` into handler bootstrap; changed `WorkspaceStartupHint.source` to `Option<WorkspaceStartupSource>`; stopped path-only constructors from inventing `Cwd`.
- **Why:** Task 3 and Task 4 need startup provenance available past the daemon transport seam. Leaving parser/bootstrap logic embedded in `mod.rs` and collapsing hints back to bare paths would force another churn pass later.
- **How:** Reused the Task 1 framing fixes instead of rewriting them, added a handler constructor that accepts the typed startup hint, preserved legacy stream fallback by buffering the first non-startup line as MCP payload, and updated tests to cover parser order, split writes, legacy no-blank streams, and bootstrap preservation.
- **Impact:** The startup hint now survives from CLI through adapter, daemon IPC parsing, and into handler construction. The next tasks can build session state and request-time binding on top of a real typed seam instead of more transport glue.
