## Sidecar Binary Distribution — Implementation Complete

Embedded the Python sidecar source into Julie's binary so distributed binaries and `cargo install` work without a source checkout.

### What
- Added `include_dir` crate to embed `python/embeddings_sidecar/sidecar/` (~24KB) at compile time
- `sidecar_root_path()` now returns `Result<PathBuf>` with 4-level fallback chain:
  1. `JULIE_EMBEDDING_SIDECAR_ROOT` env var
  2. Adjacent to binary (for packagers)
  3. Source checkout (`CARGO_MANIFEST_DIR`)
  4. Extract embedded files to `~/.cache/julie/embeddings/sidecar/source/`

### Why
`env!("CARGO_MANIFEST_DIR")` bakes the build machine's path into the binary. GitHub release binaries and `cargo install` would look for a nonexistent path.

### How
- Precise embedding: `include_dir!` for `sidecar/` package + `include_str!` for `pyproject.toml` (avoids embedding `uv.lock`, tests, caches)
- Version-gated extraction: `.embedded-version` marker piggybacks on `INSTALL_MARKER_VERSION`
- Refactored `sidecar_supervisor.rs` (757→229 lines) by extracting `sidecar_embedded.rs` (115 lines) and `sidecar_bootstrap.rs` (353 lines)

### Impact
- Distributed binaries will auto-extract Python sidecar on first use
- 14 tests (6 new embedding + 8 moved supervisor), 907 fast tier passing
- No changes to install flow, launch protocol, or env var overrides
