## Daemon Reliability: 3 fixes + adapter retry validated

Addressed all 3 daemon reliability issues from TODO.md (reported by Codex during Eros benchmarking) plus a Python test detection false positive.

### What shipped (6 commits + 2 fix commits + 1 extractor fix)

- **Idle workspace eviction** (`WorkspacePool`): LRU sweep evicts workspaces idle >5min. `WorkspaceEntry` tracks `last_accessed`, sweep uses remove-first-then-shutdown protocol, watcher ref_count guards active sessions. Configurable via `JULIE_WORKSPACE_IDLE_TIMEOUT_SECS`.
- **Decouple embedding wait from index response**: `spawn_workspace_embedding` no longer blocks on cold sidecar (~36-50s). Non-blocking `try_settled()` check; deferred task waits in background. New `EmbeddingOutcome { symbols, deferred }` return type across all callers (index, register, refresh).
- **Drain timeout 10s->60s**: `DEFAULT_DRAIN_TIMEOUT_SECS` bumped. `stop_daemon()` aligned to use `drain_timeout()` instead of hardcoded 10s.
- **Adapter retry on pre-output transport errors**: `AdapterError` enum (Transport vs Stdin), `classify_adapter_error`, MAX_RETRIES=5, exponential backoff (1-16s). Post-output and stdin errors are terminal.
- **3 review fixes** from Codex adversarial review: lost request line preservation on send failure, vector count sync on deferred embedding terminal paths, malformed JSON skip instead of retry.
- **Python/Ruby/Bash test detection**: gated name-only `test_*` detection on `is_test_path` in `detect_python` and `matches_script_test_name`.

### Adapter retry validated

Julie MCP survived a daemon rebuild+restart in this fresh session. The previous session's adapter (old binary) died twice on the same scenario. This confirms the retry fix works.
