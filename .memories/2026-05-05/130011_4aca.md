## Live MCP dogfood found stale derived graph data after restart

- **WHAT:** After rebuilding release and restarting Codex, live MCP `call_path(extract_symbols_static -> extract_canonical)` still returned `found=false` until the daemon workspace was force-refreshed. After `manage_workspace(refresh, workspace_id=julie_528d4264, force=true)`, the same live MCP call returned the expected one-hop path to `crates/julie-extractors/src/pipeline.rs:8`.
- **WHY:** The process code was fresh, but persisted daemon graph data was stale. `fast_refs(extract_canonical)` saw the call through identifier fallback, while SQLite had no resolved `relationships` row for `extract_symbols_static -> extract_canonical` before refresh.
- **HOW:** Confirmed via live MCP `fast_refs`, `call_path`, `deep_dive`, direct SQLite inspection of `symbols`, `identifiers`, and `relationships`, then forced daemon refresh. Relationship count moved from 24,599 to 32,787 and health reported canonical/projected revision 3871/3871.
- **IMPACT:** The code fix works live after reindex, but binary upgrades that change extractor/resolver semantics currently need explicit index invalidation when file contents do not change. This is now recorded in `docs/plans/2026-05-05-tree-sitter-gap-closure.md` as an open follow-up finding.
