## Tree-sitter best-in-class branch completed

### WHAT
Completed the broad tree-sitter hardening plan across all supported languages instead of narrowing to a subset.

Key changes:
- Added whole-language production-path extraction fixtures under `fixtures/extraction/` and new extractor tests for golden output, capability matrix coverage, and parser-upgrade invariants.
- Added a root integration contract at `src/tests/integration/real_world_contract.rs` to assert real-world extraction behavior through production `extract_canonical` for the supported language set.
- Upgraded tree-sitter core to `0.26.8` and refreshed parser dependencies where compatible, including replacing the old Harper Dart parser with `tree-sitter-dart = 0.2.0` because Harper's parser crate conflicted on the `tree-sitter` native link target.
- Introduced `LanguageSpec` so registry metadata, extensions, parser names, display names, and capabilities derive from one table instead of duplicated maps.
- Added parse diagnostics through extraction, parse cache, indexing, watcher, and edit/refactor safety paths.
- Unified file indexing policy so startup, batch indexing, watcher filtering, and discovery agree on what is indexable.
- Improved relationship precision with scoped symbol indexing, duplicate-safe symbol maps, and receiver-aware relationship resolution.
- Removed cloned whole-file symbol vectors from identifier allocation walkers by using shared containing-symbol helpers.
- Fixed Dart extraction regressions caused by the new grammar shape and added Go malformed-source symbol recovery for parser-upgrade resilience.
- Documented the parser upgrade workflow in `docs/TREE_SITTER_UPGRADES.md` and updated language/dependency docs and AGENTS/CLAUDE quick references.

### WHY
Tree-sitter is the data layer everything else in Julie depends on. The old setup made parser upgrades risky because behavior drift could hide in any language until a user happened to hit it. The user explicitly rejected targeting a subset of languages, which was the right push: language subsets are how unsupported behavior quietly becomes permanent.

### HOW
Used TDD and broad contracts:
- Started from failing targeted tests for behavior gaps and parser upgrade fallout.
- Added language-wide golden fixtures and capability assertions so every language has an explicit contract.
- Added production-path real-world contract coverage to catch parser-output drift that unit-level extractor tests miss.
- Used `cargo metadata`, `cargo tree`, crates.io API checks, and docs.rs tree-sitter constants to verify dependency choices and ABI compatibility.
- Used Julie dogfood CLI search on the branch after building the debug binary.

### IMPACT
Parser upgrades now have a named gate, `cargo xtask test bucket parser-upgrade`, that exercises all languages through both golden extraction and production real-world contracts. The branch also reduces duplicated language metadata, makes parse errors visible to unsafe edit guards, and closes file-policy mismatches between startup indexing and live watcher paths.
