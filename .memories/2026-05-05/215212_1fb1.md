## Fixed Startup Empty-Database Embedding Skip

- **WHAT:** Changed startup repair so an `EmptyDatabase` repair runs the background embedding pipeline after initial indexing. Added a regression test for fresh workspace startup repair generating embeddings.
- **WHY:** Launching Claude Code with Julie in `/Users/murphy/source/pi-mono` created the symbol index but skipped embeddings because startup auto-index passed `skip_embeddings = true` for all repair reasons except `SemanticVersionChanged`.
- **HOW:** Traced daemon logs showing `Skipping embeddings in auto-index mode` at initial startup, then later catch-up embedding only after `manage_workspace open`. Updated `run_primary_workspace_repair` to embed when repair reasons rebuild embedding inputs: `EmptyDatabase` or `SemanticVersionChanged`.
- **IMPACT:** Fresh workspaces now get embeddings after initial startup indexing instead of waiting for an explicit workspace open or index command. Verified with a failing-first regression test plus changed and dev gates.
