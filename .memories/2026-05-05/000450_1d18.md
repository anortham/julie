## Completed Task 5 Parse Diagnostics And Rewrite Safety

### WHAT

Added tree-sitter parse diagnostics to the extraction contract and wired the metadata through golden fixtures, batch indexing, watcher indexing, and edit/refactor safety checks.

### WHY

Recovered parse trees are useful for indexing, but they are dangerous for AST edits when the error is near the target span. Julie now keeps recovered symbols while making unsafe rewrites fail clearly.

### HOW

- Added `ParseDiagnostic` and `ParseDiagnosticKind` to `julie-extractors` results.
- Added `parse_diagnostics_for_tree()` and included diagnostics in `extract_canonical`, JSONL record offsets, and golden normalization.
- Added a malformed Go recovery fixture that asserts parse diagnostics and recovered symbols.
- Stored parse diagnostics in existing file parse metadata via SQLite `parse_cache`.
- Made `rewrite_symbol` reject target-overlapping parse errors and `rename_symbol` reject AST-aware edits on parse-error trees.
- Persisted parse diagnostics from both batch indexing and watcher updates.

### IMPACT

The extractor layer now exposes parser health as first-class data. Indexing remains tolerant of recoverable parse errors, while editing tools stop before making AST-based changes on unsafe syntax trees.
