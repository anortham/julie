## Added Command Output Efficiency Section to User CLAUDE.md

### What Was Added
- New `## Command Output Efficiency` section in `~/.claude/CLAUDE.md` (lines 63-97)
- Covers Git short formats, language-specific build/test patterns, and general rules
- Based on empirical testing: RTK vs raw vs instruction-based alternatives on Julie repo

### Key Instructions
- **Git**: `git status -s`, `git log --oneline -N`, `git diff --stat` first
- **Rust**: `cargo test 2>&1 | tail -5`, `cargo build 2>&1 | grep -E '^error' -A5`
- **Python**: `pytest -q --tb=short`
- **JS/TS**: `npx vitest --reporter=dot`, `npx tsc --noEmit 2>&1 | head -30`
- **Go**: `go test ./... 2>&1 | grep -E 'FAIL|ok' | tail -10`
- **General**: Use Read/Grep/Glob tools, pipe verbose output, never dump full passing test output

### Context
- RTK testing showed `git status -s` (64 chars) beats RTK (111 chars)
- `cargo test | tail -3` matches RTK for passing tests (101 vs 103 chars)
- Cargo clippy is the only command where RTK is irreplaceable (357K → 3.3K)
- Instructions cover ~60-70% of RTK's savings with zero dependencies
