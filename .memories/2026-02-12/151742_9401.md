## RTK (Rust Token Killer) Project Analysis

### What It Is
- CLI proxy that wraps dev commands and compresses output before it hits LLM context
- v0.14.0, Rust, ~4.1MB binary, 24 command modules, 9 filtering strategies
- Claims 60-90% token savings on common operations

### Key Architecture
- **Command proxy**: Clap CLI → route to module → execute command → filter output → track savings
- **Hook integration**: `rtk init` installs PreToolUse:Bash hook in Claude Code that transparently rewrites commands
- **Token tracking**: SQLite DB, `chars/4` heuristic, 90-day retention, `rtk gain` analytics
- **Session intelligence**: `rtk discover` scans JSONL history for missed savings, `rtk learn` detects CLI correction patterns

### Filtering Strategies
1. Stats extraction (git diff → summary)
2. Failure focus (tests → failures only, 99% reduction)
3. Error grouping (lint errors by rule)
4. Deduplication (log lines with counts)
5. Structure only (JSON → schema)
6. Code filtering (comments/bodies removal)
7. Progress stripping (ANSI bars)
8. Tree compression (file lists → tree)
9. Diff compaction (10 lines/hunk, 100 max)
