## C++ call identifier normalization

### WHAT
Fixed C++ call identifier extraction so `call_expression` nodes whose function child is a `field_expression` now store the rightmost field name, not the full receiver-qualified expression.

### WHY
`obj.method()` and `this->method()` were being indexed as `obj.method` and `this->method`, which meant `find_references("method")` could miss method calls entirely.

### HOW
Updated `crates/julie-extractors/src/cpp/identifiers.rs` to inspect the call's `function` child, detect `field_expression`, and use its `field` child for both the identifier name and span. Added a regression test in `crates/julie-extractors/src/tests/cpp/identifier_extraction.rs` that exercises both `service.process()` and `this->process()`.

### IMPACT
C++ method calls now match the naming behavior used by other languages, and the regression test proves we no longer keep receiver-qualified call names for method invocations.
