- Added tested `IdentifierKind::TypeUsage` extraction for C#, Go, Swift, Python, and Rust typed syntax.
- Added constructor call identifiers for C# object creation.
- Deduped C# member invocation relationships by removing the duplicate member-access extraction path.
- Fixed Python function return type inference to prefer annotation metadata.
- Preserved Elixir qualified calls such as `Logger.info`.
- Fixed Zig method call attribution, Scala call extraction inside vals/givens/extensions, and VB.NET case-insensitive relationship lookup.
- Regenerated canonical golden fixtures for the extractor behavior changes.

### WHY
The compiled tree-sitter audit found missing exact identifier data and call relationships across several language extractors. Those gaps make reference search and relationship navigation miss real usage sites or report noisy duplicate edges.

### HOW
Used TDD with narrow failing tests for each defect, then implemented parser-aware extraction logic in the affected language modules. The Go malformed recovery path needed an extra regression because tree-sitter recovered a missing brace by treating broken function text as struct field declarations. The final rule keeps valid embedded field type usage while rejecting recovered field declarations that follow parser errors.

Task 10 is ready to commit with focused tests, golden checks, changed-bucket checks, and the dev regression tier passing. Reference searches now have better exact identifier coverage for typed languages, constructor calls, qualified calls, and several relationship caller paths.
