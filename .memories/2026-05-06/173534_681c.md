## Completed Task 3 Identity And TypeInfo Contracts

### WHAT
- Changed extractor symbol and identifier IDs to hash normalized file path plus full stored span: start line/column, end line/column, start byte, and end byte.
- Added `BaseExtractor::generate_id_for_node` and routed TypeScript, Python, Markdown, Vue manual symbols, and normal create paths through the same span-based ID contract.
- Preserved relationship ID uniqueness for same-line calls by including node column and byte span at creation, and preserving prior relationship identity when rekeying normalized locations.
- Changed `ExtractionResults::extend` so conflicting duplicate `TypeInfo` rows keep the existing row and emit a warning instead of silently overwriting.
- Added focused regression coverage in path identity, base relationship IDs, TypeScript parent IDs, Python parent IDs, and Markdown semantic extraction.

### WHY
Task 3 from the tree-sitter audit found that ID and type-row contracts were under-entropic and inconsistent. Same-start symbols or identifiers could collide, rekeying could collapse type rows, TypeScript/Python parent lookup used old hash assumptions, and type-info merging could lie by overwriting conflicts.

### HOW
- Verified RED on `test_symbol_ids_do_not_collide_for_same_row_column_different_spans` and `test_extraction_results_extend_does_not_silently_overwrite_typeinfo`.
- Implemented shared span-based ID generation in `stable_location_id`, `Symbol::refresh_id`, `Identifier::refresh_id`, and `BaseExtractor::generate_id_for_span`.
- Removed TypeScript post-create ID regeneration and replaced class parent fallback-by-name with actual parent-node span ID lookup.
- Replaced Python class parent ID recomputation with `generate_id_for_node`.
- Updated Markdown and Vue manual symbol creation to use `generate_id_for_span`.
- Added deterministic tests for same-start symbol/identifier IDs, same-start type rows after rekey, same-line relationship IDs after creation and rekey, conflicting TypeInfo merge, and same-named TypeScript/Python class parent IDs.

### IMPACT
The extractor identity contract is now consistent across initial extraction and normalization. TypeInfo rows are less likely to point at the wrong symbol, same-line calls keep distinct relationship IDs, and parent IDs in TypeScript/Python are no longer tied to brittle row/column guesses or fallback-by-name behavior.
