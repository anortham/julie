## Fixed tree-sitter review blockers and relationship correctness issues

### WHAT
Implemented the review fix batch for the tree-sitter extractor and resolver work:

- Restored Vue legacy/structured pending relationship parity and added Vue to the API surface parity coverage.
- Gated Rust namespace scoring for `std`, `core`, and `alloc` to Rust pending files and Rust candidate symbols.
- Replaced CSS keyframe shorthand extraction with `animation-name:` only and stripped comment text before CSS relationship scanning.
- Added Markdown self-reference filtering and deterministic first-heading behavior for duplicate slugs.
- Added exact YAML anchor metadata matching so `*foo` cannot match `&foobar`.
- Changed QML property binding edges to target real property symbols, not synthetic `property_*` IDs.
- Anchored Rust re-export crate-root detection to workspace crate layouts instead of any path containing `<crate>/src`.
- Split `language_spec.rs` into a small public API module plus `specs.rs`, and split `mod_tests.rs` into include parts below the project test-file cap.
- Removed debug stdout from Bash and Zig cross-file relationship tests and added Rust inner doc-comment support.

### WHY
Opus review identified three release-blocking precision faults and several important correctness gaps. Most findings were accurate; the fixes tighten extractor truthfulness and prevent language-specific assumptions from leaking into generic resolver behavior.

### HOW
Used TDD with exact regression tests first for Vue parity, namespace scoring, CSS, Markdown, YAML, QML, Rust inner docs, and Rust re-export crate-root handling. Updated golden fixtures only where behavior intentionally changed. Used Julie blast-radius and reference checks after implementation.

### IMPACT
The extractor quality gate now catches pending parity drift, fixtures encode real relationship targets, resolver scoring is less Rust-biased for non-Rust code, and oversized touched files now respect project caps. The changed/dev fallback and specialty buckets passed after the batch.
