## PR 1 Task 1.3: EmbeddingService::shutdown async with bounded sidecar wait

### WHAT
Made `EmbeddingService::shutdown` async with a 3-second bounded wait for the Python sidecar child to exit. Added `wait_for_exit` to the `EmbeddingProvider` trait with a default no-op returning `true`. Implemented `wait_for_exit` for `SidecarEmbeddingProvider` using a polling loop. Updated `spawn_process` to use `Stdio::null()` for stderr (was `Stdio::inherit()`) and added `CREATE_NEW_PROCESS_GROUP` on Windows.

### WHY
On Windows, if the sidecar dawdles after the daemon shuts down, the new daemon races the old sidecar for `~/.julie/` file handles. Bounding the wait ensures clean handoff.

### HOW
- `src/embeddings/mod.rs`: Added `fn wait_for_exit(&self, timeout: Duration) -> bool { true }` default to `EmbeddingProvider` trait
- `src/embeddings/sidecar_provider.rs`: Implemented `wait_for_exit` with 10ms poll loop on `child.try_wait()`; changed `stderr(Stdio::inherit())` to `stderr(Stdio::null())`; added `CREATE_NEW_PROCESS_GROUP` (0x00000200) alongside `CREATE_NO_WINDOW` on Windows
- `src/daemon/embedding_service.rs`: Changed `pub fn shutdown` to `pub async fn shutdown`, calls `provider.shutdown()` then `tokio::task::spawn_blocking(move || provider.wait_for_exit(3s)).await`; warns on timeout
- `src/daemon/mod.rs`: Updated `embedding_service.shutdown()` to `embedding_service.shutdown().await` at line 668
- `src/tests/daemon/embedding_service_shutdown.rs`: New test file with two tests
- `src/tests/daemon/mod.rs`: Registered new test module

### IMPACT
- Worker invariant satisfied: "On clean shutdown, the new daemon does not race the old sidecar's handle release: shutdown blocks until the sidecar child has exited or a 3-second bound elapsed."
- Both tests GREEN: `test_embedding_service_shutdown_waits_for_child_exit` and `test_embedding_service_shutdown_returns_on_timeout`
- Existing test `test_embedding_service_shutdown_is_safe_when_no_provider` updated to `#[tokio::test]` async
