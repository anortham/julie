## Fixed Ruby attr properties and cross-file pending edges

- **WHAT:** Ruby `attr_reader`, `attr_writer`, and `attr_accessor` now emit every property argument and preserve the enclosing class as `parent_id`. Ruby superclass, `include`, and `extend` references that are not same-file symbols now emit structured pending relationships.
- **WHY:** The audit found Ruby properties only captured the first argument and lost parent context, while cross-file inheritance and mixin edges were silently dropped.
- **HOW:** Changed Ruby special call extraction to return multiple symbols and thread the current parent into attr properties. Added structured pending emission in the existing inheritance and module inclusion resolution paths when local target lookup misses.
- **IMPACT:** Ruby property searches see all accessor-defined members, and workspace-level relationship resolution can recover Rails-style superclass and concern links instead of losing them at extraction time.
