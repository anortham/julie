## WHAT
Fixed primary `fast_refs` so identifier fallback respects the resolved definition set for qualified lookups and dedupes accepted identifier rows inside the batch.

## WHY
Qualified names like `FastRefsTool::call_tool` were pulling unrelated `call_tool` hits from other parents, and repeated identifier rows on the same file/line could consume limit slots before unique refs were added.

## HOW
- In `src/tools/navigation/fast_refs.rs`, built a resolved definition ID set and gated identifier fallback on `target_symbol_id` when the lookup is qualified.
- Updated the identifier loop to insert accepted `(file_path, start_line)` keys into the dedupe set immediately after pushing a ref.
- Added regression tests in `src/tests/tools/fast_refs_primary_rebind_tests.rs` for qualified-name leakage and within-batch dedupe/limit handling.

## IMPACT
Qualified primary `fast_refs` now stays on the intended definition, and duplicate identifier rows no longer crowd out unique references.
