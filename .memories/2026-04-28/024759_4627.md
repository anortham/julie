## Fixed workspace routing dogfood fallout

- **WHAT:** Hardened workspace root detection so an ancestor `.julie` directory no longer captures an explicit unmarked child workspace, and changed `manage_workspace(open)` to switch the current primary workspace while keeping the previous primary active.
- **WHY:** Dogfooding `rewrite_symbol` with a temp fixture under `/tmp` let `/private/tmp/.julie` capture the fixture root. After that, `open/index` on the Julie repo left default `primary` routing pointed at `/private/tmp`, because `open` only activated a secondary workspace and did not rebind the primary.
- **HOW:** Added regression tests for parent `.julie` capture and open-by-id primary switching. Root detection now trusts `.julie` only at the requested path, using stronger ancestor markers for upward discovery. Added `switch_primary_workspace_with_root` so open attaches the target, binds it as primary, and keeps the old primary in the secondary active set.
- **IMPACT:** Temp dogfood fixtures no longer poison current workspace routing, and `manage_workspace(open)` is now a usable recovery path for switching default tool routing back to the intended workspace.
