## WHAT
Patched target-workspace fast_refs behavior to match the primary path for qualified names, import handling, and identifier kind mapping.

## WHY
Target-workspace searches were diverging from primary fast_refs in three places: qualified names were not split, imports leaked into definitions, and `type_usage` identifiers were downgraded to generic references.

## HOW
- Updated `src/tools/navigation/target_workspace.rs` to parse qualified names with `parse_qualified_name`, resolve child symbols, and filter by matching parent symbol name.
- Added import separation so `SymbolKind::Import` entries become `RelationshipKind::Imports` references and are removed from definitions.
- Expanded identifier conversion so `type_usage` maps to `RelationshipKind::Uses`, alongside the primary mappings for call/import/member_access.
- Updated `src/tests/tools/target_workspace_fast_refs_tests.rs` with a regression integration test that exercises the real target-workspace path and flipped the stale type_usage expectation to `Uses`.

## IMPACT
Target-workspace fast_refs now mirrors primary behavior for the three parity findings, and the new regression test catches the bug on the real implementation path. Full crate verification is currently blocked by unrelated compile errors in other modified test files (`src/tests/cli_execution_tests.rs`, `src/tests/cli_tools_tests.rs`).
