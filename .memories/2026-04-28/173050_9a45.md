## Remaining Tool Deep Review Fixes

### WHAT
Reviewed and fixed issues in the remaining Julie MCP tools: `get_symbols`, `manage_workspace`, and `rename_symbol`.

- Added strict `get_symbols.mode` validation for `structure`, `minimal`, and `full`.
- Marked `manage_workspace` as destructive in public MCP annotations and corrected workspace docs/schema drift.
- Converted selected manage-workspace failure paths to `CallToolResult::error` so failed indexing/register/open refresh paths do not look successful.
- Hardened `rename_symbol` by validating `new_name`, normalizing `file:` scope paths, using qualified-name child identifiers for rewrite, and restricting replacements to reference lines from indexed data.
- Added regression tests for the new contracts.

### WHY
The deep review found tool contract gaps that could mislead agents or damage code: typoed modes were silently accepted, destructive workspace operations advertised a safe contract, failure text returned with `is_error=false`, and rename could over-apply or fail qualified names.

### HOW
Used Julie-first exploration plus parallel audit agents for `manage_workspace` and `rename_symbol`. Followed red/green tests for the main behavior changes, then ran targeted verification and the project dev gate.

### IMPACT
Agents now get clearer errors and safer behavior across the remaining tool surface. `cargo xtask test dev` passes after the changes. Residual rename limitation: receiver-type method calls are only renamed when the index resolves their exact reference line; Julie now avoids over-renaming same-name siblings instead of guessing.
