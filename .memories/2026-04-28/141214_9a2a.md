## Tightened relationship kind filtering

### WHAT
Updated `get_relationships_to_symbols_filtered_by_kind` to join `relationships` against `identifiers` by `r.to_symbol_id = i.target_symbol_id`, and added a regression test that places a `call` and a `type_usage` on the same source line.

### WHY
The old join matched by file path and line number, so any identifier on that line could satisfy the kind filter. That let unrelated relationships slip through when a line carried mixed identifier kinds.

### HOW
Used the structured `target_symbol_id` link as the predicate for the filtered relationship query. Updated the existing relationship-filter test to populate `target_symbol_id`, and added a new regression that proves a same-line `type_usage` does not leak into a `call` query.

### IMPACT
`reference_kind` filtering now tracks the identifier tied to the resolved target symbol, not a loose line match. That removes the false-positive path without adding more path-text guessing.
