## Fixed fast_search review findings

### WHAT
Implemented the fast_search end-to-end review fixes found during the tool-path audit.

- Added bounded `limit` handling for MCP deserialization and direct struct usage.
- Added content `return_format="locations"` output that emits only file and line locations.
- Fixed file-mode missing-index messaging so it names file search instead of definition search.
- Fixed SQLite definition rescue accounting so rescued hits cannot produce totals like `showing 1 of 0`.
- Added target-workspace metrics binding for fast_search and included workspace plus effective limit in telemetry metadata.
- Added focused regressions in `src/tests/tools/search/fast_search_regression_tests.rs`, handler telemetry tests, and handler metrics-binding tests.

### WHY
The fast_search review found response-shape and telemetry bugs that affect agents using compact output, cross-workspace searches, and rescued definition results. These were user-visible enough to fix during the review pass rather than leave as notes.

### HOW
Kept the fixes local to fast_search routing, formatting, telemetry, and definition search accounting. Used existing workspace binding and search execution patterns, with a small metrics binding helper for target workspaces.

### IMPACT
fast_search now honors its public limit contract, produces compact content locations, reports file-mode readiness errors accurately, keeps rescue totals sane, and attributes target-workspace fast_search metrics to the target workspace.
