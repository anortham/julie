## Reviewed and fixed get_context path

### WHAT
- Fixed target-workspace telemetry attribution for `get_context` handler calls.
- Added `format` to `get_context` tool-call metadata.
- Enforced `file_pattern` and `exclude_tests` filters for keyword symbol search results before get_context pivot selection can consume them.
- Fixed code-first fallback so one actionable code candidate can replace a non-actionable primary pivot.
- Prevented second-hop expansion from re-adding the original pivot as a neighbor.
- Added regression tests covering each fixed behavior.

### WHY
The tool review found several quiet correctness issues: target calls could record metrics under primary, filtered get_context calls could leak keyword hits from outside scope, non-code pivots could win despite a code candidate, and second-hop expansion could waste neighbor budget on the pivot itself.

### HOW
Used Julie-first investigation plus parallel explorer agents. Followed RED/GREEN for each defect with focused nextest runs before production edits. Kept changes scoped to handler telemetry, search filtering, get_context scoring, and graph expansion.

### IMPACT
get_context now returns and records data with stronger workspace and filter integrity. Search-related regressions are covered by unit and integration tests. Dev-tier verification passed via `cargo xtask test changed`; dogfood was blocked by a missing historical baseline fixture outside these changes.
