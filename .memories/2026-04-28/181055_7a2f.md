## Fixed Dashboard Activity SSE Loading

### WHAT
Added an initial activity SSE comment frame and Axum keep-alive to `src/dashboard/routes/events.rs`, with a regression test in `src/tests/dashboard/integration.rs`.

### WHY
The Metrics page activity stream opened but emitted no bytes until a tool event occurred. Browsers can show the page as loading during that silent open stream, which matched the dashboard spinner report.

### HOW
- Added a `: connected` SSE comment frame before broadcast events.
- Enabled `KeepAlive::default()` for the stream.
- Added `test_activity_stream_flushes_initial_frame` to prove the stream yields bytes immediately.

### IMPACT
Dashboard navigation no longer gets pinned by an idle activity feed. The rebuilt daemon was restarted and the live endpoint now flushes `: connected` on connect.
