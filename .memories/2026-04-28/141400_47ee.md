## Tightened relationship kind filtering to the exact occurrence

### WHAT
Refined `get_relationships_to_symbols_filtered_by_kind` so a filtered relationship must match the same target symbol, file path, and line number as the identifier row.

### WHY
A target-only join made every relationship to the same symbol eligible for a given `reference_kind`, even when the identifier kind lived on a different line. That let one call site poison the whole symbol.

### HOW
Kept the structured `target_symbol_id` linkage, but added `file_path` and `start_line` equality so the query binds to the same occurrence. Added a regression that seeds two relationships to the same target on different lines with different identifier kinds and verifies only the call occurrence survives a `call` filter.

### IMPACT
Reference-kind filtering now respects per-occurrence identifier data instead of symbol-level spillover. The false positives from same-target relationships on other lines are gone.
