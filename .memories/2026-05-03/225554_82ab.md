## Added shared daemon MCP session wrapper

### WHAT
- Added `src/daemon/mcp_session.rs` with `DaemonMcpSession`, `DaemonSessionDependencies`, and `HttpJulieService`.
- Moved IPC handler construction and workspace cleanup out of `src/daemon/ipc_session.rs` into the shared session module.
- Added HTTP wrapper tests covering missing workspace header, invalid workspace source, valid workspace attach, and DELETE cleanup.
- Updated the HTTP transport plan ledger with exact-test, `cargo check`, and focused transport-bucket evidence.

### WHY
- `rmcp::StreamableHttpService` requires a synchronous service factory, but Julie's real daemon handler construction is async and workspace-header dependent.
- Duplicating IPC lifecycle setup in HTTP would reintroduce the exact daemon/session cleanup bugs we have been fighting.

### HOW
- `HttpJulieService` implements `rmcp::Service<RoleServer>` and lazily creates `JulieServerHandler` on `initialize` after reading `http::request::Parts` from `RequestContext.extensions`.
- `DaemonMcpSession::start` centralizes workspace startup hint handling, `WorkspacePool` attachment, project log start/end, session lifecycle, and cleanup via `WorkspaceSessionAttachment`.
- IPC now keeps only IPC-specific header parsing, ready-line handshake, and stream serving.

### IMPACT
- HTTP has a tested path to run the real Julie handler without daemon startup exposing it yet.
- IPC cleanup behavior is preserved through exact lifecycle tests and the focused `transport` bucket.
- Next slice can wire daemon startup and the stdio shim against a real session wrapper instead of a toy service.
