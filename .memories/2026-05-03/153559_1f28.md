## Fixed tools-search bucket overlap during merge verification

- **WHAT:** During the local merge of `codex/projection-invariants` into `main`, the `cargo xtask test dev` branch gate repeatedly timed out in `tools-search`. Investigation showed the first tools-search command used the substring filter `tests::tools::search`, which also matched `tests::tools::search_context_lines` even though that module was run again as the second bucket command.
- **WHY:** The duplicate command work pushed the bucket against its 90s timeout on the main checkout. This was a test harness defect, not a projection assertion failure.
- **HOW:** Updated the manifest contract expectation first and observed the expected RED failure, then changed `xtask/test_tiers.toml` to use the narrower `tests::tools::search::` filter.
- **IMPACT:** The primary search command now runs the intended search module without duplicating search context lines, reducing unnecessary gate work and making `dev` more likely to reflect real failures instead of bucket timeout noise.
