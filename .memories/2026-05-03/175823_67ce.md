## Added Workspace Registry Store Facade

- **WHAT:** Added `WorkspaceRegistryStore` and routed workspace open/register/remove/list/cleanup cleanup paths through it for registry reads, writes, index directory lookup, and cleanup event recording.
- **WHY:** Task 1 of the workspace service split plan needs registry storage separated from cleanup and command policy, so open auto-prune and cleanup sweep share one registry delete/event path.
- **HOW:** Kept safety checks in `cleanup.rs`, added a storage-only daemon facade, and updated `run_cleanup_sweep`, `delete_workspace_if_allowed`, `prune_missing_workspaces`, and `prune_orphan_index_dirs` to take the store. Added a regression that exercises both open auto-prune and cleanup sweep against concrete rows, index directories, and cleanup events.
- **IMPACT:** Registry deletion behavior is now easier to reuse without pulling raw `DaemonDatabase` and index-dir logic through each command path.
