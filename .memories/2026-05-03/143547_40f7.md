## Implemented projection invariant batch

- **WHAT:** Added a regression for missing projection state with same-count stale Tantivy docs, removed the doc-count-only readiness shortcut, split projection document application helpers into `src/search/projection/apply.rs`, and routed indexing pipeline projection through `SearchProjection` instead of duplicating state/apply logic.
- **WHY:** SQLite must be the source of truth; Tantivy document count is not proof that the projection serves the current canonical revision.
- **HOW:** `SearchProjection` now owns normal incremental projection state transitions and document application, with a lock-aware path that marks `Building`, releases SQLite while waiting for Tantivy, then marks `Ready` or `Stale`. Health readiness now reports `FullyReady` only when projection freshness is `Current`.
- **IMPACT:** Stale/missing projection state fails closed, pipeline lock behavior stays intact, and affected projection/search/dogfood/system-health gates pass.
