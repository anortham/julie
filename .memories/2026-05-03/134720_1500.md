## Fixed workspace isolation smoke assertion

- **WHAT:** Updated `test_search_never_crosses_workspaces` so it distinguishes echoed zero-hit query text from actual `path:line:content` search matches, and added positive controls for each workspace's own marker.
- **WHY:** The system tier exposed a deterministic test failure where `0 content matches for "REFERENCE_WORKSPACE_MARKER"` was misread as leaked reference workspace content.
- **HOW:** Added a small response parser for real content match lines and replaced raw substring checks with content-match assertions.
- **IMPACT:** The isolation test now fails for real cross-workspace leaks but passes for legitimate zero-hit responses that echo the query.
