## Documented CLI Dogfood Contract And Mode Evidence

### WHAT

Finished Task 5 of the testing-contract plan by documenting the standalone CLI dogfood contract and making execution-mode evidence testable.

### WHY

Standalone CLI dogfood is useful for quick tool behavior checks, but it must not be mistaken for daemon transport, restart, or session-routing coverage. Ledger evidence also needs a stable way to cite whether a CLI run used standalone, daemon, or fallback mode.

### HOW

- Added a standalone CLI dogfood section to `docs/TESTING_GUIDE.md` naming what standalone catches and what requires daemon/MCP/transport tests.
- Updated `JULIE_AGENT_INSTRUCTIONS.md` to recommend standalone for quick pre-MCP tool checks and to warn that it does not prove daemon transport/restart/session routing.
- Kept JSON output backward-compatible as raw tool output.
- Added `render_execution_mode_evidence` in `src/cli_tools/mod.rs` and used it for the stderr mode line.
- Added tests for the docs contract, agent instructions contract, mode evidence formatting, and existing standalone `CliToolOutput.mode` behavior.
- Ran `cargo fmt` after code edits.

### IMPACT

The testing-contract plan implementation tasks are now complete. The branch is ready for lead affected-change and branch gates.
