## Added Daemon Lifecycle Controller

- **WHAT:** Introduced `DaemonLifecycleController` in `src/daemon/lifecycle.rs` and rewired `run_daemon` / `accept_loop` to use it instead of passing loose lifecycle primitives.
- **WHY:** The daemon restart, drain, readiness, and state-file behavior was spread across `run_daemon` and `accept_loop`, which made race-sensitive changes hard to reason about.
- **HOW:** Kept the existing pure lifecycle decision helpers, then wrapped side effects in the controller: phase storage, state-file publication, restart-pending mutation, and restart notification. Added controller tests for ready publication, restart-pending idempotence, zero-session stopping, and sessions-drained transition.
- **IMPACT:** Daemon lifecycle ownership is now centralized while preserving adapter readiness strings and existing daemon startup contracts.
