## Implemented Xtask Bucket Metadata And Summary Output

### WHAT

Added optional bucket metadata to the xtask test manifest and surfaced timing details in runner output.

### WHY

The testing-contract plan needs verification buckets to carry enough machine-readable context that agents can see scope, expense, and expected runtime without relying on prose in `AGENTS.md`.

### HOW

- Added `scope_label`, `owner`, `expensive`, and `notes` metadata parsing with defaults in `xtask/src/manifest.rs`.
- Added metadata to checked-in buckets in `xtask/test_tiers.toml`.
- Updated list output to show scope labels, owners, notes, and expensive markers.
- Updated summary output to show expected time, actual time, command count, scope label, and a `SLOW` marker for buckets exceeding 150% of expected runtime.
- Added/updated xtask tests and contract expectations.
- Lead review removed unused metadata copies from the private runtime `BucketPlan` so execution state only carries fields it uses.

### IMPACT

Task 1 of the testing-contract plan is complete. Later changed-selection and specialist-bucket work can now rely on bucket metadata being parsed, validated, and visible in runner output.
