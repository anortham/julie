## HTTP Transport Security Middleware

### WHAT
- Added bearer-token mode to the Streamable HTTP transport module.
- Configured SDK Host and Origin validation for the bound loopback port.
- Added concrete status-code tests for missing token, wrong token, valid token, invalid Host, and foreign Origin.
- Kept discovery token-safe: it publishes the token file path, not the bearer token value.

### WHY
Localhost HTTP is reachable by browser-originated requests, so Streamable HTTP needs real Host/Origin/token policy before any stdio shim points at it.

### HOW
- Added `bearer_token` to `HttpTransportConfig`.
- Wrote `daemon-mcp.token` with Unix `0600` permissions when token mode is enabled.
- Applied Axum bearer-token middleware over `/mcp` and `/mcp/ready`.
- Let `rmcp` 1.6 handle Host and Origin rejection using `allowed_hosts` and `allowed_origins`.
- Verified exact security tests plus the focused `transport` xtask bucket.

### IMPACT
The HTTP module now has the security controls needed for Task 4 adapter token injection and Task 5 real Julie session wiring.
