## Released transient Tantivy writers after projection writes

- **WHAT:** Added `SearchIndex::release_writer()` and wired projection/backfill paths to commit and drop transient Tantivy writers without shutting down search.
- **WHY:** The stricter projection repair path created real Tantivy writers during startup/backfill. A broader system gate exposed that those writers could keep Tantivy lock files alive and make later explicit indexing fail with `LockBusy`.
- **HOW:** `SearchProjection` now releases the writer after successful rebuild/project operations. Startup projection backfill releases any opened writer before continuing into the indexing pipeline, while preserving the original backfill error if cleanup also fails. Added a regression test proving writer release frees the lock while keeping the same `SearchIndex` writable and searchable.
- **IMPACT:** Fixes the `test_v690_upgrade_preserves_existing_tantivy_docs_on_first_edit` system failure without reintroducing watcher scheduling changes or broadening the workaround surface.
