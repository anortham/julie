## Fixed deferred CWD auto-index race

- **WHAT:** Added a regression test for CWD startup with no client roots support where the first primary `fast_search` must return an indexed symbol without requiring `manage_workspace index` first. Updated `JulieServerHandler` so deferred startup repair is awaited before primary code tools dispatch.
- **WHY:** Codex Desktop/user-level MCP registration can start Julie from a weak CWD without `JULIE_WORKSPACE`; deferring auto-index until first request preserved startup safety but broke Julie's automatic indexing contract for first search/navigation tools.
- **HOW:** Added a single-flight `deferred_auto_index_gate`, `complete_deferred_auto_index_if_needed`, and a preflight mode so `manage_workspace index` binds the primary without double-running repair while code tools await repair before dispatch.
- **IMPACT:** First primary search/edit/navigation request after deferred CWD startup now blocks until startup repair/index finishes, avoiding the previous `Workspace not indexed yet` response and preserving automatic freshness semantics.
