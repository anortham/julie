## Added fast_search zero-hit semantic fallback

- **WHAT:** Added an opportunistic semantic fallback for default `fast_search` when lexical search returns zero hits on an unscoped, non-path query. The fallback is labeled in output as `No lexical results. Showing semantic fallback candidates.` Explicit `backend="lexical"` remains pure lexical for bakeoffs and file/path work.
- **WHY:** Dogfooding showed queries like `RetrievalBackend` can have no exact lexical hit while semantic search finds the relevant symbol family. The fallback gives agents useful candidates without making semantic/hybrid the default ranking path.
- **HOW:** Reused existing stored embeddings and the already-published `handler.embedding_provider()` only. The fallback does not wait on provider initialization or trigger embedding generation. Added path/file-pattern guards, extracted path-query detection into `search::query`, and updated tool descriptions plus Julie/Razorback skills/docs.
- **IMPACT:** Default search gets labeled semantic rescue for zero-hit concept/fuzzy-symbol queries while explicit lexical remains a clean baseline for comparisons. Search regression coverage, dev-tier fallback, and search-quality dogfood bucket all pass.
