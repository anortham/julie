## Reduced fast_search MCP default result count

- **WHAT:** Changed the MCP `FastSearchTool` default `limit` from 10 to 6 and updated the schema/docstring plus default-parameter assertion.
- **WHY:** Search ranking quality has improved enough that defaulting to fewer results cuts normal agent context bloat while preserving explicit opt-in breadth via `limit`.
- **HOW:** Used TDD: changed `test_deserialize_params_defaults_applied` to expect 6, verified it failed with the old default, then updated `default_limit()` in `src/tools/search/mod.rs`.
- **IMPACT:** Normal `fast_search` calls without a requested limit now return 6 results instead of 10. CLI defaults remain 10 by design because CLI output is not automatically injected into model context.
