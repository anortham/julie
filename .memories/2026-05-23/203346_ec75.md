## Bounded semantic dogfood and fallback guard tightening

- **WHAT:** Ran a bounded live-MCP semantic dogfood sample across Rust/Python/JS/Swift/Java/C#/Ruby/C/PHP/C++ workspaces, then tightened default `fast_search` semantic fallback to identifier-like zero-hit queries only.
- **WHY:** Dogfood showed real value for fuzzy symbol probes and concept-to-symbol discovery, but also exposed a bad default behavior: arbitrary multi-word lexical-zero noise such as `banana websocket casserole` could produce semantic fallback candidates.
- **HOW:** Kept explicit `backend="semantic"` available for concept search, kept explicit `backend="lexical"` pure for bakeoffs, and changed default fallback gating to require identifier-like probe shape. Added a regression test for plain-language noise not falling back, updated tool descriptions/skills/Razorback wording, and synced plugin skills.
- **IMPACT:** Semantic remains useful as an opt-in concept backend and guarded fuzzy-symbol rescue. Default search is less likely to show confident-looking garbage for nonsense prose. Current recommendation: keep semantic for now, but do not make hybrid/default semantic broader until score thresholds/quality telemetry improve.
