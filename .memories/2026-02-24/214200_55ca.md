## Completed Phase 2 Task 3: Hook compute_reference_scores into indexing pipeline

### WHAT
Added a single call to `compute_reference_scores()` in the indexing pipeline's `process_files_optimized` method, after all symbols, identifiers, and relationships have been stored but before the database lock is dropped.

### WHY
This connects the reference score computation (Task 2) to the actual indexing workflow, so that every time a workspace is indexed, graph centrality scores are automatically computed and persisted. Without this hook, the `reference_score` column would always be 0.0.
