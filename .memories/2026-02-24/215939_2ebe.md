## Completed Phase 2 Task 5: Integrate centrality boost into search scoring

### WHAT
Added `apply_centrality_boost()` function in `src/search/scoring.rs` and hooked it into both code paths in `src/tools/search/text_search.rs` (primary and reference workspace). Wrote 5 comprehensive TDD tests.

### WHY
Search results should promote well-connected symbols — functions/classes that are heavily referenced across the codebase are more likely what the user is looking for. This is the core scoring integration that makes the pre-computed reference scores (Tasks 1-4) actually affect search ranking.

### HOW
- **Formula:** `boosted = score * (1.0 + ln(1 + reference_score) * 0.3)`
- Logarithmic scaling prevents runaway boosting — diminishing returns as reference count grows
- `CENTRALITY_WEIGHT = 0.3` is a tunable constant
- Applied AFTER file_pattern filtering but BEFORE symbol conversion (operates on `SymbolSearchResult` not `Symbol`)
- Only applied to "definitions" search, not content search (content search returns file lines, not symbols)
- Both code paths use the same `apply_centrality_boost()` function — reference workspace opens a fresh `SymbolDatabase`, primary workspace uses the shared `Arc<Mutex<SymbolDatabase>>`
- TDD: 5 tests written first (RED), all pass (GREEN), 610 total lib tests pass with 0 regressions
