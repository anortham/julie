## Completed Phase 3 Task 2: Search + pivot selection with centrality-weighted scoring

### What
Implemented `select_pivots()` function in `src/tools/get_context/pipeline.rs` and comprehensive tests in `src/tests/tools/get_context_tests.rs`.

### How
- `Pivot` struct holds a `SymbolSearchResult` + `combined_score`
- Combined score = `text_relevance * centrality_boost` where boost uses `ln(1 + ref_score) * CENTRALITY_WEIGHT`
- Adaptive pivot count based on score distribution analysis:
  - Top result 2x+ above second -> 1 pivot (clear winner)
  - Top 3 within 30% of each other -> 3 pivots (cluster)
  - Otherwise -> 2 pivots (default)

### Tests (6/6 passing)
1. Empty results -> empty output
2. Single result -> 1 pivot
3. Clear winner (2x+ gap) -> 1 pivot
4. Cluster (top 3 close) -> 3 pivots
5. Default case -> 2 pivots
6. Centrality reranking — lower text score promoted by high reference_score
