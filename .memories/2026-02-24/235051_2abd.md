## Completed Phase 3 Task 6: Wire up full get_context pipeline

### What
Connected all 5 pipeline stages (search, pivot selection, graph expansion, token allocation, formatting) into a working end-to-end flow in `src/tools/get_context/pipeline.rs`.

### How
- **`run_pipeline()`** — testable core function taking raw `&SymbolDatabase` + `&SearchIndex`, executes: search_symbols → get_reference_scores → select_pivots → expand_graph → allocate budget → build_pivot_entries → build_neighbor_entries → format_context
- **`run()`** — handler entry point, extracts workspace resources via `get_workspace()`, clones Arc references, delegates to `run_pipeline()` inside `tokio::task::spawn_blocking` (correct for std::sync::Mutex)
- **`build_pivot_entries()`** — handles all 3 PivotModes: FullBody (code_context from DB), SignatureAndKey (abbreviate_code first+last 5 lines), SignatureOnly
- **`get_pivot_relationship_names()`** — resolves incoming/outgoing names per pivot via neighbor list + DB fallback
- **`build_neighbor_entries()`** — maps GraphExpansion neighbors to NeighborEntry structs
- **`abbreviate_code()`** — takes first 5 + last 5 lines with "..." separator for 12+ line bodies
