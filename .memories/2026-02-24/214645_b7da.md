## Completed Phase 2 Task 4: Batch get_reference_scores query

### WHAT
Added `get_reference_scores(&self, ids: &[&str]) -> Result<HashMap<String, f64>>` method to `SymbolDatabase` in `relationships.rs`. Wrote comprehensive TDD test covering 3 cases (all found, partial match, empty input).

### WHY
The search scoring layer (Task 5) needs to look up pre-computed graph centrality scores for search results. Doing individual queries per result would be an N+1 pattern. This batch method fetches all scores in a single SQL query using an IN clause.

### HOW
- Followed the existing `get_relationships_to_symbols` pattern for parameterized IN-clause queries
- Signature takes `&[&str]` for ergonomic use with search result symbol IDs
- Returns `HashMap<String, f64>` for O(1) lookup during score boosting
- Early return for empty input (avoids invalid SQL)
- TDD: test written first (RED), compilation failure verified, implemented (GREEN), 605 tests pass with 0 regressions

### Files Changed
- `src/database/relationships.rs` — added `get_reference_scores()` method (~30 lines)
- `src/tests/core/database.rs` — added `test_get_reference_scores_batch` test (~55 lines)

- Unblocks Task 5 (integrate centrality boost into search scoring)
