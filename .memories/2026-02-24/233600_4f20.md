## Completed Phase 3 Task 5: Output formatting for get_context

### What
Implemented `format_context()` and supporting data types (`ContextData`, `PivotEntry`, `NeighborEntry`) in `src/tools/get_context/formatting.rs`. This is the final rendering step in the get_context pipeline: search -> rank -> expand -> allocate -> **format**.

### How
- **Data contract**: `ContextData` bundles query, pivots, neighbors, and allocation — decoupling formatting from pipeline internals
- **Sections rendered**: Header (query + summary counts), Pivot sections (location, centrality hint, code body, callers/callees), Neighbors section (3 modes: SignatureAndDoc, SignatureOnly, NameAndLocation), Files section (BTreeMap for sorted file map with pivot/neighbor annotations)
- **Centrality hints**: ref_score >= 20 -> "high", >= 5 -> "medium", else "low"
- **Empty handling**: No pivots -> simple "No relevant symbols found" message
- **Pluralization**: Correct singular/plural for pivot/neighbor/file counts
