## Completed Phase 3 Task 4: Adaptive token budget allocation

Implemented `TokenBudget`, `Allocation`, `PivotMode`, and `NeighborMode` in `allocation.rs`.

- **What**: Token budget allocation module that computes how to distribute tokens across pivots (60%), neighbors (30%), and summary (10%) based on result count
- **How**: Three-tier system — 0-3 pivots get FullBody/SignatureAndDoc, 4-6 get SignatureAndKey/SignatureOnly, 7+ get SignatureOnly/NameAndLocation. Adaptive defaults scale budget: 0-2 → 2000, 3-5 → 3000, 6+ → 4000 tokens
- **Tests**: 9 tests covering all tiers, boundary conditions (0, 3, 7 pivots), budget respect across many combinations, exact 60/30/10 split verification
- **Impact**: Unblocks Task 5 (output formatting) and Task 6 (pipeline wiring)

### Files changed
- `src/tools/get_context/allocation.rs` — 104 lines, full implementation
- `src/tests/tools/get_context_tests.rs` — added `allocation_tests` module (9 tests)

### Design decision
Used `total - pivot - neighbor` for summary tokens instead of a third `(total * 0.1)` floor — guarantees no tokens are lost to rounding while ensuring sum never exceeds budget.
