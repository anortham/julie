## Completed Phase 3 Task 3: Graph expansion with deduplication and centrality ranking

### What
Implemented `expand_graph()` function in `src/tools/get_context/pipeline.rs` with 3 new types (`Neighbor`, `NeighborDirection`, `GraphExpansion`) and 6 comprehensive tests in `src/tests/tools/get_context_tests.rs`.

### How
- `expand_graph(pivots, db)` iterates each pivot, fetching incoming (`get_relationships_to_symbol`) and outgoing (`get_outgoing_relationships`) relationships
- Deduplicates neighbors across pivots using a `HashMap<String, (RelationshipKind, NeighborDirection)>` — first-seen-wins
- Excludes pivot symbols from neighbor results using a `HashSet`
- Batch-fetches symbol metadata via `get_symbols_by_ids` and reference scores via `get_reference_scores`
- Sorts neighbors by `reference_score` descending for importance ranking

### Tests (6/6 passing)
1. Basic incoming + outgoing expansion — verifies both directions discovered
2. Deduplication — shared neighbor across two pivots appears only once
3. Ranking by reference_score — uses `compute_reference_scores()` on DB, verifies sort order
4. Empty relationships — pivot with no rels yields empty neighbors
5. Pivot exclusion — pivots calling each other don't appear as neighbors
6. Empty pivots — no pivots yields empty expansion
