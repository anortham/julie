## Hardened pre-release blockers and updated findings

### WHAT
- Updated `docs/PRE-RELEASE-FINDINGS.md` with a Codex verification addendum that separates confirmed items from overstated ones, then added a working-tree progress note.
- Added `docs/plans/2026-04-16-pre-release-hardening-design.md` to capture the agreed fix scope and acceptance criteria.
- Fixed the projection upgrade-path bug by bootstrapping missing canonical revisions from live SQLite state and by letting `SearchProjection::ensure_current_from_database()` mark a matching, doc-complete index as ready instead of wiping it.
- Fixed the restart-time projection rebuild loop by comparing Tantivy doc count against current SQLite totals instead of the latest revision delta.
- Fixed the dashboard poller typo in `dashboard/templates/status.html`.
- Fixed the workspace-pool write-lock bug by inserting the new workspace before the async session-count update and dropping the write lock before awaiting.
- Added regression tests for the legacy upgrade path, the no-op current-projection check, the dashboard poller script, and workspace-pool lock contention.

### WHY
- The release blockers were real and hit core upgrade, restart, and dashboard flows.
- The findings doc needed verification context so the team does not chase the weaker claims with the same urgency as the live blockers.

### HOW
- Used targeted red/green tests in `src/tests/integration/projection_repair.rs`, `src/tests/dashboard/integration.rs`, and `src/tests/daemon/workspace_pool.rs`.
- Added `SymbolDatabase::ensure_canonical_revision()` and `count_projection_source_docs()` in `src/database/revisions.rs`.
- Updated `src/search/projection.rs` to bootstrap canonical metadata on demand and short-circuit to a ready projection when Tantivy already matches SQLite.
- Moved `WorkspacePool::get_or_init()` to release the pool write lock before the awaited daemon-db session count update.

### IMPACT
- Existing Tantivy docs survive legacy upgrade repair instead of getting wiped.
- A current projected revision no longer rebuilds on every daemon restart.
- The dashboard status page keeps refreshing instead of dying on the first poll tick.
- Existing workspace reads stay available while another workspace waits on `daemon.db`.
- `cargo xtask test dev` passed after the changes.
