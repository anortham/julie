## Closed remaining pre-release findings (B-I3, B-I6) and cleaned build warnings

### WHAT
- **B-I3 fix:** `src/database/workspace.rs:72-117` now sets `PRAGMA foreign_keys = ON` and explicitly deletes from `symbol_vectors`, `identifiers`, `types`, `relationships`, `symbols`, `files`, `indexing_repairs`, `canonical_revisions`, and `projection_states` in dependent-first order. Regression test: `test_delete_workspace_data_clears_all_owned_tables`.
- **B-I6 fix:** new `SearchProjection::ensure_current_with_gate` in `src/search/projection.rs` takes an `AtomicBool` and flips `search_ready` FALSE before `clear_all`, TRUE only when the projection ends `Ready`. `backfill_tantivy_if_needed` wires the handler's `indexing_status.search_ready` into this gated path. Regression tests: `test_ensure_current_with_gate_flips_search_ready_on_rebuild`, `test_ensure_current_with_gate_keeps_search_ready_false_on_missing`.
- **Build warnings:** `pub(crate) use parse_ipc_headers_block` re-export now gated on `cfg(test)` (only used from tests); `parsed_file_count` + `text_only_file_count` methods on `IndexingBatchState` gated on `cfg(test)`; `let (mut client, mut daemon)` in adapter ready test dropped the unneeded `mut daemon`. Lib build and lib-test build are clean.
- **Doc:** `docs/PRE-RELEASE-FINDINGS.md` updated — B-I3 and B-I6 moved from OPEN to FIXED; TL;DR reads 3/3 blockers + 13/13 serious items fixed; ready-to-tag checklist narrowed to manual upgrade + Windows smoke.

### WHY
- B-I3 and B-I6 were the last two soft items blocking v6.10.0.
- B-I3 risk: `delete_workspace_data` was leaving `symbol_vectors`, `identifiers`, `types`, `indexing_repairs` behind — relying on FK cascade that requires a per-connection pragma. Orphaned embedding rows could re-associate incorrectly if a workspace was later recreated with the same path.
- B-I6 risk: during the `clear_all` → `apply_documents` window inside `SearchProjection::rebuild`, Tantivy is committed-empty. No consumer gates on `search_ready`, but the flag is also not maintained as an invariant — tests and future search-path readers can't trust it. Gating inside `ensure_current_with_gate` makes `search_ready == true` imply projection is `Ready`.
- Build warnings on a pre-release branch are noise that hides real regressions.

### HOW
- TDD: RED tests first (both B-I3 and B-I6 confirmed failing before implementation), then minimal GREEN implementation, then rerun.
- Extracted a private `ensure_current_inner(db, index, search_ready: Option<&AtomicBool>)` in `SearchProjection` so the old `ensure_current_from_database` keeps its existing 8 callers untouched while `ensure_current_with_gate` gets the gated behavior. No duplication.
- `cargo xtask test dev` passes (10 buckets, 335s).

### IMPACT
- All 3 release blockers fixed, all 13 serious items fixed.
- v6.10.0 ready to tag pending manual v6.9.0 → v6.10.0 upgrade smoke and Windows `cargo xtask test full`.
