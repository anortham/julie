## Added diff-scoped xtask test workflow

Implemented a new `cargo xtask test changed` path so local iteration no longer defaults to the full `dev` tier after every edit.

- **What changed:** Added `xtask/src/changed.rs` with git-diff collection and path-to-bucket selection, added a `Changed` CLI variant, and exposed `run_named_buckets()` so the runner can execute an arbitrary bucket list.
- **Docs updated:** `AGENTS.md`, `CLAUDE.md`, and `README.md` now teach `cargo xtask test changed` as the local loop and reserve `cargo xtask test dev` for batch-level regression checks.
- **Tests added:** New `xtask/tests/changed_tests.rs` covers CLI parsing, localized bucket selection, docs-only no-op behavior, search-core dogfood promotion, and dev fallback for shared infrastructure. Existing xtask docs/runner tests were updated for the new command.

## Why

Agents were following repo guidance that still said `cargo xtask test dev` was the default after every normal change. That turned the local loop into repeated multi-minute suite runs and killed momentum.

## How

- Mapped changed paths from git diff plus untracked files.
- Selected the smallest matching bucket set when paths are localized.
- Fell back to the `dev` tier when shared or unmapped infrastructure files changed.
- Ignored docs-only and memory files.
- Tightened an early over-selection rule so `src/tests/tools/workspace/**` no longer pulls `workspace-init` by itself.

## Impact

- The repo now has a first-class command that matches the workflow people wanted agents to use.
- The guidance and the tooling now agree, which removes a major source of repetitive broad test runs.
- A live smoke of `cargo xtask test changed` on the working tree selected `cli`, `tools-workspace`, `workspace-init`, and `integration` and finished in 206.3s before the final workspace-test over-selection trim; the trim itself is covered by targeted xtask tests.
