## v7.4.0 Release: Review-Driven Fixes

Cross-model review (Claude Opus + GPT-5.4 via Codex CLI) of the CLI + annotation normalization + early warnings implementation surfaced 5 issues. All fixed in this release.

### Fixes Applied

- **CLI `signals` command** (Medium): Added `julie-server signals` with `--file-pattern`, `--fresh`, `--limit`, `--json`, `--format markdown`. Early warning reports were dashboard-only; now accessible from shell/CI
- **Performance: skip unannotated symbols** (Low): `has_auth_marker_in_owner_chain()` was walking the owner chain for every symbol. Moved inside the entrypoint annotation guard so unannotated symbols skip the walk entirely
- **Performance: memoize annotation sets** (Low): `annotation_sets(config)` rebuilt HashSets per symbol. Now cached by language in a HashMap before the loop. Auth keys folded into the same struct
- **ASCII lowercase consistency** (Low): `annotations.rs:172` used Unicode `to_lowercase()` while indexing used `to_ascii_lowercase()`. Aligned to ASCII
- **File pattern test coverage** (Low): Added test verifying glob-scoped analysis vs unscoped

`Cargo.toml`, `CLAUDE.md`, `docs/site/index.html` all at v7.4.0
