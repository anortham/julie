## Fixed CLI Phantom Workspace Creation

### WHAT
- Added CLI preflight validation so missing or non-directory workspace roots fail before daemon IPC.
- Added daemon `WorkspacePool::get_or_init` validation so daemon sessions cannot create absent workspace directories while initializing a pool entry.
- Updated workspace ID migration to merge stale alias rows into an existing canonical row instead of failing on a primary-key conflict.
- Replaced hardcoded missing-workspace CLI test paths with per-test temp child paths.

### WHY
A CLI test using `/tmp/julie_nonexistent_ws_99998` reached a live daemon. The daemon initialized that missing path, created `.julie`, and wrote rows for both `/tmp/...` and `/private/tmp/...` due macOS path canonicalization timing.

### HOW
- Added red tests for missing pool roots, duplicate migration targets, and deterministic CLI missing-workspace handling.
- Moved missing-path rejection ahead of daemon use in CLI execution.
- Kept failed daemon pool initialization from writing registry rows or source-side `.julie` directories.
- Migrated child tables before either updating the old workspace ID or merging into an existing target row.

### IMPACT
Dashboard phantom workspace entries stop recurring, duplicate `/tmp` alias rows can be reconciled on daemon startup, and CLI missing workspace behavior is deterministic.
