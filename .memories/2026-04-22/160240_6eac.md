## Pre-review checkpoint for search-matrix branch

### WHAT
Prepared the requested Claude pre-merge review for the `codex/search-matrix` worktree after verification passed.

- `cargo xtask test changed` passed after falling back to `dev`
- `cargo xtask test dogfood` passed
- External reviewer choice remains `claude`
- The branch has no local commits beyond base `2f331b518d8468ce622efeb48a78672997d64d2b`, so the review input must be built from the working tree diff, not `main..HEAD` alone

### WHY
A `main..HEAD` review would omit the entire implementation because all branch changes are still uncommitted. The review has to see tracked modifications plus new untracked files or it is fake.

- Use tracked `git diff --no-ext-diff`
- Append synthetic diffs for untracked files with `git diff --no-index -- /dev/null <file>`
- Build matching stat output the same way
- Feed that combined diff plus plan path and test status into the Claude adversarial review prompt

- The review will cover the real branch state instead of an empty diff
- Any verified findings from Claude can be fixed before branch finish-up
- This keeps the requested review intact without forcing an early commit
