## Fast Search File Mode Design

Added a design for first-class indexed file lookup in `fast_search`.

- **Canonical target:** `search_target="files"`
- **Alias:** `search_target="paths"` normalizes to `files`
- **Engine:** Tantivy-led retrieval, not SQLite scans
- **Coverage rule:** file docs must cover all indexed file rows, not only rows with `content IS NOT NULL`

### Why

Live search-matrix seeds show agents using `fast_search` for file and path hunts such as dated plan filenames, fixture names, and dotted path-ish lookups. Those queries are poor fits for current `content` and `definitions` targets.

### How

The design calls for:

- parsing `search_target` into an explicit enum instead of raw string fallthrough
- adding `basename` and `path_text` to the Tantivy file-doc schema
- adding a dedicated `SearchIndex::search_files(...)` lane
- returning token-lean file results with `line=None` and kind `file`
- rejecting `context_lines` for file mode
- normalizing telemetry to canonical `files`

### Impact

This gives Julie a deterministic file lookup mode, removes a class of path-shaped `content` misses, and gives the search-matrix harness a clean new target to measure.
