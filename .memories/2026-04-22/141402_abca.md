## Added telemetry-earned out-of-scope content hint

### WHAT
Implemented Task 4 from the search-quality recovery plan.
- Added `HintKind::OutOfScopeContentHint`.
- Added a pure `build_content_zero_hit_hint(...)` selector in `src/tools/search/hint_formatter.rs`.
- Updated `FastSearchTool::execute_with_trace` to use fixed precedence for content zero-hit hints: syntax hint, then out-of-scope hint, then multi-token hint.
- Added tests for precedence, trace propagation, and telemetry serialization.

### WHY
Fresh daemon telemetry after rebuilding the release showed that `file_pattern_filtered / no_in_scope_candidates` is a real live bucket, not a synthetic lab artifact. Without a dedicated hint, those misses fell through to the generic multi-token advice, which is the wrong nudge.

### HOW
- Queried `~/.julie/daemon.db` and grouped `fast_search` content zero-hits by `zero_hit_reason`, `file_pattern_diagnostic`, and `hint_kind`.
- Confirmed the 24h bucket contained real `no_in_scope_candidates` rows.
- Added a pure hint-selection helper so precedence is testable outside `execute_with_trace`.
- Kept syntax mistakes highest priority and left `CandidateStarvation` on the existing multi-token path.

### IMPACT
- Scoped zero-hits with `NoInScopeCandidates` now tell the agent the search found no candidate files inside the requested scope.
- Telemetry records `out_of_scope_content_hint`, so Task 6 can measure whether this improves without-recourse behavior.
- Existing syntax hint behavior stayed intact.
