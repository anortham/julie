## Fixed quoted-phrase content verification across code punctuation

### WHAT
Added a regression fix so quoted content queries survive Julie's line verifier when code separators sit between the tokens, such as `router.use(...)` for the query `"router use"`.

### WHY
The search side tokenized quoted queries into terms like `router` and `use`, but the line verifier demanded the raw quoted substring on a line. That disagreement drained valid candidates into `line_match_miss`, which is why the `express` matrix case was failing.

### HOW
- Added a narrow unit regression in `line_match_strategy_tests.rs`
- Added a live pipeline regression in `zero_hit_reason_tests.rs`
- Updated `src/tools/search/query.rs` so quoted `Substring` queries strip balanced wrapping quotes and verify as ordered tokenizer terms on the line
- Kept the rest of the strategy surface stable, so hints and existing strategy labels did not need a large rewrite mid-batch
- Re-ran the breadth matrix and the search regression tiers

### IMPACT
The `express` quoted-phrase matrix case now returns hits instead of `line_match_miss`, and the breadth summary no longer reports `line_match_miss_cluster`. Dogfood and dev both passed after the fix.
