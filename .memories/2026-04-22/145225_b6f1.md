## Search matrix investigation design

- **WHAT:** Wrote a new design spec at `docs/plans/2026-04-22-search-matrix-investigation-design.md` for a repo-local search matrix harness.
- **WHY:** Current daemon telemetry is useful for mining `fast_search` failure shapes, but it only records Julie tool calls and cannot tell us whether a zero-hit was later recovered by non-Julie tools.
- **HOW:** The design splits work into a local mining stage that extracts candidate query shapes from `~/.julie/daemon.db` and a committed matrix stage that runs curated `fast_search` cases across a broad repo corpus under `~/source`. It reuses existing telemetry parsing, search-analysis code, xtask entrypoints, and search-quality fixture conventions.
- **IMPACT:** This gives the project a controlled way to investigate search quality across languages and layouts without pretending Julie-only telemetry can answer agent-level recourse questions. It also creates a path to promote stable cases into dogfood and regression coverage.
