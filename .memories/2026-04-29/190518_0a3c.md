## Updated Julie model routing policy

Edited `RAZORBACK.md` in the Julie repo to refine Codex model routing.

### WHY
Recent OpenAI model comparison work showed `gpt-5.4-mini` fits bounded implementation, `gpt-5.3-codex` fits coding review and terminal-heavy debugging, and `gpt-5.5` should remain the planning and highest-risk correctness model.

### HOW
- Normalized Codex model names to lowercase model IDs.
- Kept strategy on `gpt-5.5 medium/high`.
- Kept bounded implementation on `gpt-5.4-mini high` and mechanical work on `gpt-5.4-mini low/medium`.
- Added `gpt-5.3-codex xhigh` for code review, adversarial review, failed-worker diagnosis, terminal-heavy bug fixing, and first escalation.
- Preserved `gpt-5.5 high/xhigh` for top-risk correctness, plan failure, architecture, public API, security, and verification strategy problems.

### IMPACT
Julie now has a sharper cost-speed-quality routing policy for Codex subagents without changing Razorback's generic skills.
