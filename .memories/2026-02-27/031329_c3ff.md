## Wired spawn_reference_embedding into three workspace commands

### WHAT
Added embedding pipeline calls to `handle_add_command`, `handle_index_command`, and `handle_refresh_command` so reference workspaces get embeddings generated after indexing.

### HOW
- **add_remove.rs**: Call `spawn_reference_embedding` after stats update, conditionally append embed count to success message
- **index.rs**: Hoisted `indexed_workspace_id: Option<String>` above the `if let Some(workspace)` block, set it when `is_reference_workspace` is true, then call embedding after message construction
- **refresh_stats.rs**: Added `WorkspaceType` import, check `workspace_entry.workspace_type == WorkspaceType::Reference`, call embedding and append status

### IMPACT
Reference workspaces now automatically get embeddings generated in the background after add, index, or refresh operations. Users see "Embedding N symbols in background..." in the response.
