## Reviewed Task 6 feature wiring and test gating changes

- Focused review on `Cargo.toml` feature wiring and embedding-related test gating updates.
- Validated behavior with targeted test runs under three feature sets:
  - default features (`embeddings-ort`)
  - `--no-default-features`
  - `--no-default-features --features embeddings-candle`
- Identified one quality issue in test gating: a default-feature assertion test is itself behind `#[cfg(feature = "embeddings-ort")]`, so it disappears exactly when the default feature regresses.

### Evidence
- `cargo test --lib --no-default-features test_sqlite_vec_registration_and_version`
- `cargo test --lib test_windows_ort_policy_mentions_directml`
- `cargo test --lib --no-default-features --features embeddings-candle test_candle_feature_can_be_enabled`

### Impact
- Feature gates compile in multiple configurations, but current test gating leaves a regression hole for default-feature coverage.
