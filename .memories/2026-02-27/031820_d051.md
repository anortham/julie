## Enable Embeddings for Reference Workspaces — Complete

Implemented full feature across 7 files to make semantic search work on reference workspaces.

### What was done
- **Task 1:** Added `embedding_count: i64` to `DatabaseStats`, populated via `embedding_count()` in `get_stats()`
- **Task 2:** Created `spawn_reference_embedding()` helper in `src/tools/workspace/indexing/embeddings.rs` — opens fresh DB, spawns pipeline fire-and-forget
- **Tasks 3-5:** Wired helper into `handle_add_command`, `handle_index_command` (for reference workspaces), and `handle_refresh_command` 
- **Tasks 6-7:** Added embedding counts to `handle_stats_command` (Embeddings: X/Y) and `handle_health_command` (Embeddings: X/Y (Z%) or None)

### Key design decisions
- Reuse primary workspace's `EmbeddingProvider` (shared, stateless ONNX model)
- Open fresh `SymbolDatabase` for reference workspace (don't use primary's DB)
- Fire-and-forget `tokio::spawn` + `spawn_blocking` pattern (matches primary workspace approach)
- No file watchers for reference workspaces — embeddings re-run on refresh
