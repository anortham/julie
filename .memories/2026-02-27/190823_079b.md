## Selected default CoreML model source for Apple Silicon

### WHAT
- Set default CoreML model source to `michaeljelly/bge-small-en-coreml-v1.5` for macOS arm64 Candle initialization.
- Set default CoreML artifact path to `coreml/feature-extraction/float16_model.mlpackage`.
- Added default tokenizer/config source for CoreML path: `BAAI/bge-small-en-v1.5`.
- Added helper `default_coreml_model_id_for_platform(os, arch)` and tests for platform-based default selection.

### WHY
- User chose “Download model at first run” and asked for a real default repo selection (option 1).
- CoreML model repos discovered generally do not bundle tokenizer/config; split source resolves this while keeping first-run download behavior.

### HOW
- `src/embeddings/candle_provider.rs`:
  - added constants for default CoreML model + tokenizer repos
  - `try_new()` now auto-resolves default CoreML repo on `macos+aarch64` when env override is absent
  - `try_new_coreml_state()` now downloads model from CoreML repo and tokenizer/config from tokenizer repo
- `src/tests/core/embedding_provider.rs`:
  - added `test_default_coreml_model_id_for_platform_prefers_apple_silicon`
- Existing helper tests and provider tests re-run green.

### IMPACT
- Apple Silicon has a concrete default download-first CoreML source instead of requiring manual env setup.
- CoreML path is more likely to initialize successfully by using a known tokenizer/config source.
