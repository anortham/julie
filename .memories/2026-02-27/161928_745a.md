## Added workspace-level embedding provider regression tests for Task 2 gaps

Added two async tests in `src/tests/core/embedding_provider.rs` that validate behavior through `JulieWorkspace::initialize` (not just pure resolver helpers).

- **What:**
  - `test_workspace_init_explicit_candle_unavailable_sets_unresolved_runtime_status` (`#[cfg(not(feature = "embeddings-candle"))]`)
  - `test_workspace_init_auto_on_apple_silicon_falls_back_to_ort_with_truthful_status` (`#[cfg(all(target_os = "macos", target_arch = "aarch64", feature = "embeddings-ort", not(feature = "embeddings-candle")))]`)
- **Why:** Ensure runtime status accurately reflects resolution/degradation semantics during full workspace initialization, especially for explicit vs auto backend selection.
- **How:**
  - Used env-driven init path (`JULIE_EMBEDDING_PROVIDER`, `JULIE_SKIP_SEARCH_INDEX=1`)
  - Asserted explicit `candle` unavailable path remains unresolved/degraded with no silent fallback
  - Asserted `auto` on Apple Silicon resolves to `ort` and status remains truthful (`resolved_backend=ort`, degraded_reason aligned with provider init success/failure)
- **Impact:** Prevents regressions where workspace init could misreport backend resolution or silently mask unsupported explicit provider requests.
