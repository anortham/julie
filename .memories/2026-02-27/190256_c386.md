## Implemented download-first Candle/CoreML path with safe fallback

### WHAT
- Added `candle-coreml` to `embeddings-candle` feature wiring in `Cargo.toml`.
- Implemented a real Candle provider in `src/embeddings/candle_provider.rs` with:
  - transformer runtime (HF download on first run)
  - optional CoreML runtime (HF download on first run when `JULIE_CANDLE_COREML_MODEL_ID` is set)
  - warmup inference, dimension enforcement (384), and runtime degradation reporting
- Added helper functions for CoreML env/runtime selection and input-name parsing.
- Added tests for helper behavior and CoreML dependency availability.

### WHY
- User requested real Candle integration and explicitly asked for download-on-first-run behavior.
- Existing Metal path had op-coverage failures; needed robust fallback behavior while keeping provider functional.

### HOW
- `src/embeddings/candle_provider.rs`:
  - added `coreml_runtime_requested()` + `parse_coreml_input_names()`
  - added optional CoreML state builder and warmup checks
  - kept transformer path as fallback and merged degraded reasons
  - centralized embedding inference pipeline for rank-2/rank-3 outputs + L2 normalization + 384-dim guard
- `src/tests/core/embedding_provider.rs`:
  - added failing-then-passing tests for CoreML helper parsing and runtime selection
  - updated auto-selection test expectation for current fallback wording
- `src/tests/core/embedding_deps.rs`:
  - added `test_candle_coreml_dependency_is_available`

### IMPACT
- Candle backend is no longer scaffold-only.
- CoreML dependency is present and test-verified.
- First-run model download is supported for both transformer and CoreML runtime paths.
- Runtime remains resilient when acceleration path is unavailable.
