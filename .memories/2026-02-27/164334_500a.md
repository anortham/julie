## Fixed embedding health status truthfulness in workspace health output

### WHAT
- Updated `check_embedding_runtime_health()` to report status based on both `embedding_runtime_status` and `embedding_provider`.
- Added coverage in workspace health tests for degraded provider-backed runtime and provider-missing runtime.

### WHY
- Health output previously reported `INITIALIZED` whenever runtime metadata existed, even when no provider was available, which was misleading.

### HOW
- Implemented explicit branching:
  - `provider + runtime` => `INITIALIZED` (or `DEGRADED` when `degraded_reason` exists)
  - `runtime without provider` => `UNAVAILABLE` with explanatory reason
  - `no runtime` => `NOT INITIALIZED`
- Strengthened tests in `src/tests/tools/workspace/mod_tests.rs`:
  - existing runtime test now asserts `Embedding Status: DEGRADED`
  - new test asserts `Embedding Status: UNAVAILABLE` when provider is missing

### IMPACT
- Embedding health reporting now reflects real runtime/provider availability.
- Prevents false-positive initialized status in health diagnostics.
- Regression coverage now locks in expected behavior for degraded and unavailable cases.
