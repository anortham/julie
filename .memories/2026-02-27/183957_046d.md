## Added real Candle provider path + `candle-coreml` dependency

### WHAT
- Added `candle-coreml` as an optional dependency and wired it into `embeddings-candle` feature.
- Replaced Candle scaffold behavior with a real inference path in `src/embeddings/candle_provider.rs`:
  - loads HF model artifacts (`config.json`, `tokenizer.json`, `model.safetensors`)
  - builds Candle BERT model via `candle-transformers`
  - tokenizes, runs forward pass, mean-pools, L2-normalizes, returns embeddings
  - enforces 384-dim guard from model config
- Added warmup inference at initialization to catch Metal runtime incompatibilities early.
- Implemented automatic Metal->CPU fallback for Candle when warmup fails (records degraded reason, keeps provider functional).
- Added/updated tests:
  - real Candle provider creation + query embedding
  - updated Apple Silicon auto-selection expectation to resolve `candle`
  - `candle-coreml` dependency availability smoke test.

### WHY
- User requested real Candle integration (not scaffold) and explicitly requested adding `candle-coreml`.
- Pure Candle Metal path for this model currently fails at runtime (`no metal implementation for layer-norm`), so warmup fallback preserves functionality while exposing degradation.

### HOW
- `Cargo.toml`: feature wiring for `embeddings-candle` now includes `dep:candle-coreml` + Candle stack deps.
- `src/embeddings/candle_provider.rs`: full provider implementation + warmup + fallback.
- `src/tests/core/embedding_provider.rs`: behavior tests updated for real provider.
- `src/tests/core/embedding_deps.rs`: added `candle-coreml` type availability test.

### IMPACT
- Candle backend is now real and usable for embeddings in this branch.
- `candle-coreml` is present and compile-verified in feature builds.
- On current machine/model, Candle may run degraded on CPU due Metal op coverage limits; runtime status includes degradation reason.
