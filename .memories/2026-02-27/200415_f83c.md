## Made candle a default feature — no build flags needed

User was frustrated that hardware acceleration required `--features embeddings-candle` at build time despite the auto-resolver being designed to handle backend selection.

- **Root cause:** `embeddings-candle` was optional, not in default features. The compile-time `cfg!(feature = "embeddings-candle")` baked `candle_available = false` into the binary.
- **Fix:** Added `embeddings-candle` to default features in `Cargo.toml`. Updated CI release workflow so Linux/Windows use `--no-default-features --features embeddings-ort` to avoid macOS-only candle deps.
- **Verified:** Both `cargo check` (default features with candle) and `cargo check --no-default-features --features embeddings-ort` (CI path) compile clean.
