## Root cause found: Candle wiring exists only as uncommitted worktree changes

Investigated both the main workspace and the feature worktree.

- **Main workspace (`/Users/murphy/source/julie`)** currently runs ORT-only defaults:
  - provider default is `ort` in `src/workspace/mod.rs`
  - factory in `src/embeddings/factory.rs` accepts only `ort`
- **Feature worktree (`/Users/murphy/source/julie/.worktrees/feature-candle-mps-directml`)** contains extensive uncommitted implementation for Candle + auto resolver + DirectML policy.
- Key evidence in feature worktree:
  - Untracked `src/embeddings/candle_provider.rs`
  - Modified `src/embeddings/factory.rs` defaulting provider to `auto` and resolving `auto|ort|candle`
  - Modified `Cargo.toml` feature wiring for `embeddings-candle` and ORT DirectML
- Because these changes were never committed/merged, rebuilding from main produced CPU ORT runtime (`Embedding provider initialized: ... CPU`).

This explains the runtime mismatch despite prior implementation effort.
