## Tasks 1-2 Complete: fast_search hybrid search for NL queries

### What
- **Task 1**: Added characterization test `test_is_nl_like_query_examples` to verify NL detection heuristic
- **Task 2**: Replaced dead `should_use_semantic_fallback` bolt-on with direct `hybrid_search()` routing

### Why
The old semantic fallback in `fast_search` had a threshold (`keyword_result_count < 3`) that never triggered — Tantivy always returns 3+ results for NL queries. Meanwhile, `get_context` already used `hybrid_search()` (keyword + semantic via RRF). This fix makes `fast_search` use the same proven path.

### How
1. Added NL detection gate at top of definitions branch in `text_search.rs`
2. NL queries route through `hybrid_search()` which does keyword + semantic + RRF merge
3. Non-NL queries use the existing keyword-only path (unchanged)
4. Removed dead `should_use_semantic_fallback` from `hybrid.rs`
5. Removed 3 tests for the dead function, added 1 characterization test for `is_nl_like_query`

### Impact
- NL definition queries now get proper hybrid search (keyword + semantic via RRF)
- Net -34 lines of code (removed dead fallback logic)
- 763 fast-tier tests pass, 0 failures
