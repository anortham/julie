## Fixed auto backend resolver fallback semantics for Apple Silicon

Updated resolver/test behavior for Task 2 quality issue:

- **HOW:** Kept explicit provider strictness unchanged; only auto-resolution logic was adjusted. Updated resolver tests to assert ORT fallback and added explicit-Candle-unavailable guard test.
- **IMPACT:** Auto config no longer fails unnecessarily on macOS arm64 builds missing Candle; explicit `candle` still errors clearly when unavailable.
