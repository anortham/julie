## Implemented Task 1 backend preference + runtime status (TDD)

### WHAT
- Added **test-first** coverage in `src/tests/core/embedding_provider.rs` for:
  - `EmbeddingConfig::default()` provider now `"auto"`
  - provider parsing helper accepts `auto|ort|candle`
  - provider parsing rejects unknown values and includes valid set text
  - new embedding runtime status type shape
- Added new embeddings runtime types in `src/embeddings/mod.rs`:
  - `EmbeddingBackend` enum
  - `EmbeddingRuntimeStatus` struct
- Added provider parsing/validation helper in `src/embeddings/factory.rs`:
  - `parse_provider_preference(&str) -> Result<EmbeddingBackend>`
  - unknown provider error includes `auto|ort|candle`
  - `EmbeddingProviderFactory::create` now uses parsed preference
- Updated `JulieWorkspace` in `src/workspace/mod.rs`:
  - new field `embedding_runtime_status: Option<EmbeddingRuntimeStatus>`
  - cloned in `Clone` impl
  - initialized to `None` in constructors
  - `initialize_embedding_provider` now builds config from `EmbeddingConfig::default()`, tracks requested/resolved backend, and records degraded reason on failure
- Updated one required struct literal in tests (`src/tests/tools/search_quality/helpers.rs`) to include the new workspace field.

### WHY
- Task requires backend preference foundation before Candle integration, strict provider validation, and explicit runtime state reporting for initialization/degrade behavior.

### HOW
- Followed red-green cycle:
  1. Added failing tests referencing new helper/types and default behavior.
  2. Implemented minimal types + helper + workspace status plumbing.
  3. Fixed compiler fallout from new workspace field and moved-value issue.
  4. Re-ran targeted tests for new behavior.

### IMPACT
- Backend preference defaults to `auto` and is validated consistently.
- Unknown provider configuration now returns actionable error text listing valid options.
- Workspace now persists runtime embedding init status (requested/resolved backend + acceleration + optional degrade reason), enabling later diagnostics and UI/reporting work.
