## Completed codebase exploration for embedding features plan

### Files explored
- `src/embeddings/pipeline.rs` - `run_embedding_pipeline()` takes `&Arc<Mutex<SymbolDatabase>>` + `&dyn EmbeddingProvider` (workspace-agnostic)
- `src/embeddings/mod.rs` - `EmbeddingProvider` trait, `OrtEmbeddingProvider` re-export
- `src/handler.rs` - `spawn_background_embedding()` is primary-workspace-only, uses `workspace.embedding_provider` and `workspace.db`
- `src/workspace/mod.rs` - `JulieWorkspace` struct has `embedding_provider: Option<Arc<dyn EmbeddingProvider>>`, helper methods like `workspace_db_path()`
- `src/tools/workspace/commands/registry/add_remove.rs` - `handle_add_command()` indexes reference workspace but never triggers embeddings
- `src/tools/workspace/commands/registry/refresh_stats.rs` - `handle_refresh_command()` and `handle_stats_command()` - stats format, no embedding counts
- `src/tools/workspace/commands/registry/health.rs` - `check_database_health()` uses `get_stats()`, no embedding info
- `src/tools/workspace/commands/index.rs` - `handle_index_command()` triggers embeddings only via `run_auto_indexing()` path (primary only)
- `src/tools/workspace/indexing/index.rs` - `index_workspace_files()` has pattern for opening reference workspace DB
- `src/database/types.rs` - `DatabaseStats` struct lacks `embedding_count` field
- `src/database/vectors.rs` - `embedding_count()` method exists, returns `Result<i64>`
- `src/database/helpers.rs` - `get_stats()` method constructs `DatabaseStats`

### Key findings
1. Pipeline is already workspace-agnostic - just needs correct DB + provider
2. Reference workspace DB opening pattern exists in `index_workspace_files()` (line 232-257)
3. `DatabaseStats` is the natural place to add embedding_count
4. Three trigger points needed: add, index (for reference), refresh
