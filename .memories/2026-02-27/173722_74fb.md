## Fixed Task 6 feature-gating gaps for candle-only test builds

### WHAT
- Resolved unresolved `OrtEmbeddingProvider` imports under candle-only builds by adding ORT feature gates to four ORT-dependent test files:
  - `src/tests/tools/search_quality/hybrid_search_dogfood.rs`
  - `src/tests/tools/search_quality/semantic_similarity_dogfood.rs`
  - `src/tests/integration/embedding_incremental.rs`
  - `src/tests/integration/embedding_pipeline.rs`
- Added `#[cfg(feature = "embeddings-ort")]` to `test_default_build_enables_ort_backend_feature` in `src/tests/core/embedding_deps.rs` so no-default candle-only runs do not assert default-feature behavior.

### WHY
- Candle-only matrix run (`--no-default-features --features embeddings-candle`) was failing at compile time due test modules importing ORT-only symbols without gating.
- After import errors were fixed, `embedding_deps` still failed because a default-build assertion ran in candle-only mode.

### HOW
- Used minimal crate-level file gating (`#![cfg(feature = "embeddings-ort")]`) for ORT-specific test modules instead of changing production embedding code.
- Feature-gated only the default-feature assertion test in `embedding_deps`.

### IMPACT
- `cargo test --lib embedding_deps --no-default-features --features embeddings-candle` now passes cleanly.
- Core embedding feature matrix checks now pass for default, ORT-only, candle-only, and both-features configurations.
- Changes are isolated to test cfg behavior and keep runtime implementation untouched.
