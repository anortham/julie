## Phase 3 Design + Implementation Plan Complete

Brainstormed and designed Phase 3: Semantic Similarity in deep_dive.

### Key Design Decisions
- **Tool surface:** `deep_dive` at `full` depth (not `fast_refs` — that's a safety/impact tool)
- **Scope:** Within-workspace only (cross-workspace is future)
- **Vector source:** Stored-vector lookup via `get_embedding()` — no model inference at query time
- **Depth:** `full` only (progressive disclosure: overview → context → full+semantic)
- **No clustering:** KNN point-to-point only, `semantic_group` column stays unused

### Implementation Plan (6 tasks)
1. `get_embedding()` on SymbolDatabase
2. `SimilarEntry` struct + similarity lookup in `build_symbol_context`
3. `format_similar_section` in formatting layer
4. MCP tool description update
5. Dogfood integration test
6. Final verification
