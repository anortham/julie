## Task 6 Complete: scoped_identifier in Rust relationship extraction

### What
Added `scoped_identifier` handling to `extract_call_relationships()` in `relationships.rs`. Qualified calls like `crate::module::func()` now create `Calls` relationships by extracting the bare name from the last path segment.

### Why
The relationship extractor only handled `identifier` (direct calls) and `field_expression` (method calls). Qualified path calls were silently ignored, leaving gaps in the call graph.

### How
- Added a third `else if` branch for `func_node.kind() == "scoped_identifier"`
- Uses `child_by_field_name("name")` to extract the last segment (e.g., `func` from `crate::module::func`)
- Falls back to full text if no `name` field (defensive)
- Delegates to existing `handle_call_target()` for resolution

### Tests
- `test_scoped_call_creates_relationship_with_bare_name` - qualified path call
- `test_simple_call_relationship_still_works` - regression guard
- `test_deeply_nested_scoped_call_creates_relationship` - deeply nested path
