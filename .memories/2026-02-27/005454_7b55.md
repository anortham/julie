## Task 1 Complete: `get_embedding` on SymbolDatabase

Added `get_embedding(symbol_id: &str) -> Result<Option<Vec<f32>>>` to `src/database/vectors.rs`.

- **What**: New method to retrieve a single symbol's stored embedding vector
- **Why**: `deep_dive` needs to look up a symbol's own vector to use as a KNN query for finding semantically similar symbols
- **How**: Uses `rusqlite::OptionalExtension` for graceful `None` on missing rows, `f32::from_le_bytes` on 4-byte chunks for deserialization
- **Tests**: `test_get_embedding_returns_stored_vector` and `test_get_embedding_returns_none_for_missing` in `src/tests/core/vector_storage.rs`
- **Regressions**: 759/759 fast-tier tests pass
