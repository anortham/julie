## Reviewed semantic/embedding implementation for primary + reference workspaces

- Audited plan parity against implementation and searched for architectural gaps in semantic search, workspace routing, and embedding lifecycle.
- Ran parallel deep reviews for:
  - plan-vs-code mismatch
  - primary/reference workspace behavior
  - ORT backend design + future Candle readiness

### Key findings captured
- `hybrid_search` semantic KNN path does not apply `SearchFilter` constraints (`language`/`file_pattern`) before merge.
- Incremental embedding on file modify/create can leave stale vectors because delete-before-reembed is missing for modified files.
- Primary manual `manage_workspace index`/`refresh` paths do not kick off embedding pipeline (only reference and startup auto-index paths do).
- Orphan index detection excludes only references, risking false orphaning of primary index dirs.
- ORT provider is wired directly in workspace initialization (no provider factory/config), which makes adding Candle cross-cutting.

### Why this matters
These issues can reduce semantic result quality, produce stale/stale-index behavior, and slow future multi-backend support.
