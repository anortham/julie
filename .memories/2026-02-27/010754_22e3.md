## Task 2 Complete: SimilarEntry + similarity lookup in build_symbol_context

Added semantic similarity data layer to deep_dive's `SymbolContext`:

### What was added
- **`SimilarEntry` struct** in `data.rs` (line 36-42): holds a `Symbol` + `score: f32` (0.0..1.0)
- **`similar` field** on `SymbolContext` (line 33): `Vec<SimilarEntry>`, populated at "full" depth only
- **`build_similar()` helper** (line 378-424): private function that does get_embedding -> knn_search -> get_symbols_by_ids pipeline
- **Fixed `empty_context()`** in formatting_tests to include `similar: vec![]`

### How build_similar works
1. Get the symbol's own embedding via `db.get_embedding(&symbol.id)`
2. KNN search for `SIMILAR_LIMIT + 1` to account for self-match
3. Filter out self, take up to 5 results
4. Fetch full Symbol objects via `get_symbols_by_ids`
5. Build entries in KNN order with `score = (1.0 - distance) as f32`
6. Returns empty vec if no embedding exists (graceful degradation)
