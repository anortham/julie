## Completed Task 5 (feature-gated Candle provider scaffold + dim guard + fallback policy)

### WHAT
- Added `src/embeddings/candle_provider.rs` under `embeddings-candle` feature.
- Added explicit 384-dimension guard and improved scaffold diagnostics for unknown effective dimensions.
- Wired Candle backend into factory selection path.
- Added runtime fallback behavior: when requested backend is `auto`, resolved backend is `candle`, and Candle init fails, non-strict mode can fall back to ORT when available.
- Added tests for Candle selection path, dimension guard semantics, and fallback/no-fallback behavior.

### WHY
- Needed concrete Candle backend path to support Apple Silicon default policy.
- Needed to avoid disabling embeddings when Candle initialization fails in mixed-feature builds.

### HOW
- `src/embeddings/candle_provider.rs`:
  - added provider scaffold and `validate_output_dimensions(...)`
  - explicit error messaging for scaffold/unavailable vs mismatch
- `src/embeddings/factory.rs`:
  - added fallback helper after init failure for `auto` mode
- `src/workspace/mod.rs`:
  - integrated fallback execution + runtime status updates
- `src/tests/core/embedding_provider.rs`:
  - feature-gated imports/tests for candle-only and ort-only builds
  - tests for dim guard messaging and auto fallback semantics

### IMPACT
- Candle backend is now structurally integrated and selectable.
- Mixed ORT+Candle builds avoid hard failure when Candle init fails under auto mode.
- Test matrix is more robust across feature combinations.
