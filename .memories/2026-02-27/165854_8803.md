## Completed Task 4 (strict acceleration mode)

### WHAT
- Implemented strict acceleration gating controlled by `JULIE_EMBEDDING_STRICT_ACCEL`.
- Added parsing helper for strict env values (`1|true|on`, case-insensitive).
- Enforced provider disablement in strict mode when runtime is unresolved, degraded, or not accelerated.
- Hardened provider preference parsing to trim surrounding whitespace.
- Added tests for strict/non-strict behavior and parsing edge cases.

### WHY
- Strict mode must be useful for CI/local validation to catch silent CPU fallback.
- Without explicit non-accelerated gating, strict mode could still leave CPU embeddings active.

### HOW
- `src/embeddings/factory.rs`:
  - added `strict_acceleration_enabled_from_env_value`
  - added/updated `should_disable_for_strict_acceleration` decision helper
  - trimmed input in `parse_provider_preference`
- `src/workspace/mod.rs`:
  - integrated strict-mode checks during embedding provider init
  - appends clear strict-mode reason when provider is disabled
- `src/tests/core/embedding_provider.rs`:
  - strict helper tests (including non-accelerated case)
  - env parsing truthy/falsy tests
  - strict workspace-init behavior tests

### IMPACT
- Strict acceleration mode now behaves as intended: no accelerated runtime => embeddings disabled with explicit reason.
- Config parsing is less fragile for common env formatting mistakes.
