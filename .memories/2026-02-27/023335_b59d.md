## Dogfood Bug Fixes Complete: Both Bugs Fixed

Two bugs found during semantic embedding dogfood testing, both now fixed with full TDD.

### Bug 1: fast_search Semantic Fallback Was Dead
- **Root cause:** Threshold `keyword_result_count < 3` never met — Tantivy always returns 3+ for NL queries
- **Fix:** NL definition queries now route through `hybrid_search()` (same as get_context) instead of the dead bolt-on fallback
- **Removed:** `should_use_semantic_fallback` function and 3 tests for it
- **Net:** -34 lines of dead code

### Bug 2: fast_refs Missed Qualified Rust Calls
- **Root cause:** `crate::module::func()` stored as full path identifier, not bare `func`
- **Fix:** Added `scoped_identifier` handling in both `extract_identifier_from_node` and `extract_call_relationships` — extracts last segment via `child_by_field_name("name")`
- **Tests:** 6 new tests (3 identifiers, 3 relationships)
