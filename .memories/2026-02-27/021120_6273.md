## Fixed scoped_identifier extraction in Rust call expressions

**Bug:** When Rust code calls a function via a qualified path like `crate::search::hybrid::func()`, the identifier extractor was storing the entire path `"crate::search::hybrid::func"` as the identifier name. This broke `fast_refs` lookups which search by bare name.

**Root cause:** In `extract_identifier_from_node()`, the `call_expression` handler only had special cases for `field_expression` (method calls) and a generic fallback. `scoped_identifier` nodes (tree-sitter's representation of `a::b::c`) fell through to the generic fallback which called `get_node_text()` on the entire node.

**Fix:** Added a `scoped_identifier` branch that uses `child_by_field_name("name")` to extract just the last segment. Applied to both the `name` extraction and the `identifier_node` selection.

**Tests:** 3 new tests in `crates/julie-extractors/src/tests/rust/identifiers.rs`:
- `test_scoped_call_extracts_last_segment` — `crate::search::hybrid::should_use_semantic_fallback()`
- `test_simple_call_still_works` — regression guard for `do_something()`
- `test_nested_scoped_call` — `std::collections::HashMap::new()`

All 1310 extractor tests pass.
