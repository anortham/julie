## Completed Task 7 (runtime status surfaced in health + stats)

### WHAT
- Completed runtime-status surfacing in both workspace commands:
  - `manage_workspace health`
  - `manage_workspace stats`
- Standardized output fields across initialized/degraded/unavailable states:
  - Backend
  - Device
  - Accelerated
  - Degraded
- Added dedicated stats tests and strengthened health tests for provider-missing scenarios.

### WHY
- Runtime fallback/degradation must be visible to operators, not hidden in logs.
- Consistent field shape is required for reliable downstream parsing and debugging.

### HOW
- `src/tools/workspace/commands/registry/refresh_stats.rs`:
  - added runtime section formatting for stats output
- `src/tools/workspace/commands/registry/health.rs`:
  - ensured provider-missing path emits `Degraded:` (not only `Reason:`)
- Tests:
  - `src/tests/tools/workspace/runtime_status_stats.rs` (new)
  - `src/tests/tools/workspace/mod_tests.rs` (health assertions)
  - `src/tests/mod.rs` updated to register new test module

### IMPACT
- Users can now quickly confirm whether embeddings are accelerated, degraded, or unavailable from both health and stats commands.
- Reduces debugging time for DirectML/Candle fallback paths.
