## Implemented semantic/embedding hardening plan in isolated worktree

Completed the planned remediation set for semantic search correctness, workspace parity, stale vector handling, and backend pluggability.

### WHAT changed
- Fixed hybrid semantic filtering so KNN candidates now respect `SearchFilter` constraints before RRF merge.
- Enabled embedding scheduling after **primary** workspace `index` and `refresh` (parity with reference flow).
- Fixed orphan index detection to exclude primary workspace IDs.
- Added incremental re-embed path that removes stale/orphan vectors before storing updated vectors.
- Hardened vector safety:
  - `embed_symbols_for_file` now errors on vector-count mismatch.
  - `get_embedding` now rejects malformed blob lengths.
- Added provider abstraction seam:
  - new `EmbeddingConfig` + `EmbeddingProviderFactory`
  - workspace initialization now uses factory (default `ort`, env-overridable)
- Added Cargo feature gates for backend isolation:
  - `embeddings-ort` (default)
  - `embeddings-candle` placeholder
  - `fastembed` made optional and tied to `embeddings-ort`

### WHY it mattered
- Prevented semantic results from violating caller constraints.
- Removed behavior mismatch between primary and reference workspaces.
- Prevented accidental cleanup of primary index data.
- Eliminated stale embedding accumulation across incremental updates.
- Added explicit failure modes for corrupted/mismatched vector data.
- Reduced coupling to ORT and created a clean seam for future Candle/MPS work.
