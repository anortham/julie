## Fix: Scala/Elixir not indexed + language list sync + zero warnings

**WHAT**: Fixed missing Scala/Elixir support in bulk indexer and watcher, synchronized all 8 language detection lists across the codebase, and resolved all 6 build warnings.

**WHY**: When Scala and Elixir extractors were added, the extractors crate and factory were updated but 3 places in the main crate were missed:
- `src/tools/workspace/language.rs` — bulk indexer mapped .scala/.ex to "text" → no symbol extraction
- `src/watcher/filtering.rs` — watcher ignored Scala/Elixir file changes
- `src/utils/mod.rs` — detect_language() missing elixir

Additionally, `manager.rs::supported_languages()` had a duplicate "elixir" entry and was missing doc languages, `utils/mod.rs::is_supported_file()` was missing 10+ extensions, and `refactoring/utils.rs` was missing 6 languages.

**HOW**:
- Audited all 8 language detection locations via subagent
- Synchronized every list against the canonical `crates/julie-extractors/src/language.rs`
- Removed duplicate "elixir" entries from 3 routing files and manager.rs
- Removed dead code: `ElixirExtractor::{base, current_module_prefix, add_pending_relationship}`, `has_case_keyword`
- Removed unused import `compute_line_changes` from rename.rs
- Added test assertions for scala/elixir in watcher extension test

**IMPACT**: Cats and Phoenix reference workspaces will now properly extract Scala and Elixir symbols on re-index. Build is warning-free.
