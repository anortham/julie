## Fix: Remove `lib/` from vendor detection heuristic

**WHAT**: Removed `"lib"` from the vendor candidate directory names in `analyze_vendor_patterns()`, preventing `lib/` from being auto-excluded in `.julieignore`.

**WHY**: `lib/` is a **primary source directory** in Elixir, Ruby, Dart, and Haskell. The vendor detection was flagging it as third-party code and auto-generating a `.julieignore` that excluded it. This caused Phoenix's entire `lib/phoenix/` source tree (93 files) to be invisible — zero symbols extracted from the core framework code.

**HOW**:
- Removed `"lib"` from both match blocks in `analyze_vendor_patterns()` (direct check + ancestor walk)
- `"libs"` (plural) stays — it's a much stronger vendor signal
- Added regression test `test_analyze_vendor_patterns_does_not_flag_lib_directory` with Elixir project layout
- Verified Phoenix re-indexes correctly: 457 files, 23,967 symbols, 3,860 relationships

**IMPACT**: Any Elixir, Ruby, Dart, or Haskell project will now properly index `lib/` source code. Existing projects with auto-generated `.julieignore` files that exclude `lib/` will need manual cleanup (comment out the `lib/` line).
