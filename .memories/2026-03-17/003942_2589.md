## Scala & Elixir Language Support — Complete

Added Scala (#28) and Elixir (#29) to Julie's language roster, bringing it to **33 entries** (29 languages + tsx/jsx/markdown/json aliases).

### What
- **Scala extractor** (7 files, ~1400 lines): classes, case classes, traits, objects, enums (Scala 3), functions/methods, val/var, imports, packages, type aliases, given instances, extensions, relationships, identifiers, type inference
- **Elixir extractor** (7 files, ~1500 lines): call-dispatch pattern for defmodule/def/defp/defmacro/defprotocol/defimpl/defstruct, module attributes (@type/@callback/@spec/@behaviour), import directives, relationships, identifiers, type inference from @spec
- **Registration**: Both wired into all 10 shared registration points
- **Tests**: 20 Scala + 14 Elixir = 34 new tests, all passing

### Why
GPT's external review identified Scala and Elixir as popular languages missing from Julie's roster. Both have mature tree-sitter grammars.

### How
- **3-phase parallel execution**: Phase 1 (me) registered both languages in shared files to avoid merge conflicts. Phase 2 dispatched two parallel subagents in isolated worktrees — one for Scala, one for Elixir. Phase 3 merged and verified.
- **Scala**: Kotlin-templated (JVM sibling, nearly identical AST structure)
- **Elixir**: Novel call-dispatch pattern (Ruby `calls.rs` inspired) — all definitions are generic `call` nodes requiring semantic inspection of call targets
- **Key Elixir discovery**: `arguments` and `do_block` are child node types, NOT named fields — `child_by_field_name("arguments")` returns None

### Impact
- Julie now supports 33 languages
- All 8 dev tier buckets pass with zero regressions (189.7s)
- CLAUDE.md updated to reflect new language count
