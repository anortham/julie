## Complete Elixir Search Quality Fix — All 4 Bugs Addressed

### Bugs Fixed (4 total)

**Bug 1: Elixir pending relationships silently dropped**
- `relationships.rs`: no-op `AddPendingImmut` trait removed, helpers take `&mut`, push directly to `pending_relationships`
- Import/Export symbols filtered from target matching so `use Phoenix.Router` creates a pending relationship instead of self-referencing

**Bug 2: Definition search promotion too strict for qualified names**
- `scoring.rs`: Added `is_name_match()` that matches last component of dot-qualified names ("Router" matches "Phoenix.Router")
- Expanded `DEFINITION_KINDS` to include module, namespace, type, constant, delegate

**Bug 3: Definition search overfetch window too small**
- `text_search.rs`: Increased from `limit*10 (max 200)` to `limit*20 (max 500)` for non-file-pattern definition search
- Ensures high-centrality modules with qualified names make it into the candidate pool for promotion

**Bug 4: deep_dive resolution assumes parent/child for dot-notation**
- `data.rs`: Added "try full name first" step before qualified parent/child parsing
- Filters for definition kinds (Module, Class, Trait, etc.) to prefer real definitions over JS homonyms
- Extracted `apply_context_file_filter()` and `is_definition_kind()` helpers
