## Fix: Wire up test detection for Scala and Elixir extractors

**WHAT**: Fixed test detection for Scala (added ScalaTest/path-based detection) and Elixir (wired up `is_test_symbol` in `extract_def`).

**WHY**: Both extractors were producing symbols but never flagging any as tests:
- **Scala**: `detect_java_kotlin` only checked for `@Test` annotations (JUnit). Cats uses ScalaTest — trait-based, no annotations.
- **Elixir**: `detect_elixir` logic existed in `test_detection.rs` but the Elixir extractor's `calls.rs` never called `is_test_symbol`.

**HOW**:
- Added `detect_scala()` in `test_detection.rs` — tries JUnit annotations first, then falls back to `is_test_path()` for ScalaTest/MUnit/Specs2, plus `test` name prefix
- Split `"scala"` out of `"java" | "kotlin" | "scala"` match arm to use new detector
- Added `is_test_symbol` call in Elixir `extract_def()` — marks functions in test files
- Verified: 63 test detection unit tests pass, 8/8 dev buckets pass

**IMPACT**: `deep_dive` on Scala/Elixir symbols will now show proper test coverage metrics. Change risk scores will be more accurate since `test_weakness` can drop below 1.0.
