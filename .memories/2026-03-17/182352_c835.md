# QML Verification Complete — 5 Bugs Fixed, All Live Verified

## WHAT
Completed QML language verification against KDE Kirigami (200 .qml files, 6207 symbols). Found and fixed 5 bugs via TDD, including 2 systemic fixes benefiting all 33 languages.

## Fixes

### Fix #1: Import extraction missing
- Added `ui_import` handler to `qml/mod.rs` — extracts `source` field from import nodes
- Result: +790 new Import symbols (5417 → 6207)

### Fix #2: Test detection not working (systemic)
- Added `autotests/` directory and `tst_` file prefix to `is_test_path()` in `test_detection.rs`
- Benefits all 33 languages using Qt-style test conventions

### Fix #3: TypeUsage identifiers for component instantiations
- Added `ui_object_definition` handler to `qml/identifiers.rs` for nested (non-root) objects
- Enables centrality scoring for QML components

### Fix #4: Component name derived from file path
- Root class name now derived from file stem instead of base type
- `ScrollablePage.qml` → class `ScrollablePage` (extends `KC.Page`) instead of class `KC.Page`

### Fix #5: Centrality 0.00 for qualified-name references (systemic)
- Centrality boost SQL used exact name matching (`i.name = symbols.name`)
- QML uses qualified refs (`Kirigami.ScrollablePage`) that didn't match unqualified symbol names
- Fix: added suffix matching (`i.name LIKE '%.' || symbols.name`) in `compute_reference_scores()` Step 1b
- **Benefits all languages with dot-qualified references**: QML, Elixir, Scala, Java, C#, etc.
