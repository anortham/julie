## Zig Verification Complete — 3 More Fixes (6 Total This Session)

### Fix #4: Import identifiers contribute to centrality (systemic)
- **WHAT**: Expanded centrality boost SQL to count `import` identifiers (weight 2.0) alongside `type_usage` (weight 1.0)
- **WHY**: In Zig (and most languages), cross-file references are primarily imports (`@import`, `import { Foo }`, `use Foo`). These were captured as `import` kind identifiers but not counted for centrality.
- **HOW**: Changed `i.kind = 'type_usage'` to `i.kind IN ('type_usage', 'import')` with weighted `CASE` in `compute_reference_scores()` Step 1b
- **IMPACT**: Benefits all 33 languages. DocumentStore (15 imports) went from 0.00 → 1.00 centrality.

### Fix #5: Zig identifier extractor — TypeUsage for type annotations
- **WHAT**: Added type annotation identifier extraction to `zig/identifiers.rs`
- **WHY**: Like GDScript, the Zig extractor only handled `call_expression` and `field_expression`. Type annotations (`field: Type`, `param: *Type`, `!ReturnType`, `var x: Type`) were missed.
- **HOW**: Added `"identifier"` node handler that detects type position by checking parent context (`container_field`, `parameter`, `pointer_type`, `error_union_type`, `optional_type`, `variable_declaration` after `:`). Added `is_after_colon()` helper and `is_zig_builtin_type()` filter.
- **IMPACT**: DocumentStore went from 5 → 133 dependents. Server went from 13 → 83 dependents.
