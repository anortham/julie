# Razor Verification Complete — Orphan Parent Bug Fixed

## WHAT
Verified Razor extractor against dotnet/blazor-samples 9.0 (623 .razor files, 6901 symbols). Found and fixed 1 critical bug.

## The Bug: `get_symbols` invisible `@code` block content

### Root Cause
In `razor/mod.rs` `visit_node()`, the `razor_block` arm:
1. Called `extract_code_block()` to create a parent symbol
2. Passed that parent's ID to `extract_csharp_symbols()` for children
3. Hit `return` — which **skipped** `symbols.push()` at the bottom of the function

Result: child symbols (methods, properties, fields) had `parent_id` pointing to a phantom symbol that was never stored. `get_symbols` depth filtering couldn't resolve the parent chain, so it excluded all `@code` block content.

### Impact
Any agent using `get_symbols` on a Razor file saw **zero C# code** — only directives and template expressions. The actual symbols existed in the DB and were reachable via `fast_search`/`deep_dive`, but the primary file navigation tool was blind.

### Fix
Pass outer `parent_id` instead of phantom code block ID. `@code` block symbols now appear as top-level file members.
