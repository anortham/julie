## Refactor: Consolidate all language detection to single source of truth

**WHAT**: Eliminated 5 duplicate language detection implementations by delegating to the canonical `crates/julie-extractors/src/language.rs`.

**WHY**: Adding Scala and Elixir required modifying 5+ files because extension→language mappings were duplicated across 8 locations. This DRY violation caused the indexing bug where `.scala`/`.ex` files were silently mapped to "text" and produced zero symbols.

**HOW**:
- Added `supported_extensions()` and `supported_languages()` to the canonical `language.rs`
- Expanded `detect_language_from_extension()` with all variant extensions (mts, cts, pyi, phtml, etc.)
- **`src/tools/workspace/language.rs`** → delegates to canonical, keeps filename fallback locally
- **`src/watcher/filtering.rs`** → `build_supported_extensions()` now calls `supported_extensions()`
- **`src/utils/mod.rs`** → both `is_supported_file()` and `detect_language()` delegate
- **`src/tools/refactoring/utils.rs`** → delegates entirely
- **`crates/julie-extractors/src/manager.rs`** → delegates to `supported_languages()`

**IMPACT**: Adding a 34th language now requires editing exactly ONE file: `crates/julie-extractors/src/language.rs`. All downstream consumers automatically pick it up.
