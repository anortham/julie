## TypeScript verification complete — 3 bugs fixed

### WHAT
Completed the full 8-check TypeScript verification sweep against colinhacks/zod. Fixed 3 new bugs discovered during checks 7 and 8.

### WHY
Systematic language verification requires all checks to PASS before moving to the next language. Two checks initially failed (7.3 neighbors=0, 8.2 test exclusion for non-function symbols).

### HOW
**Bug 1 — `expand_graph` missing identifier-based neighbors (Check 7.3):**
- Root cause: `expand_graph()` in `pipeline.rs` queried only the `relationships` table. TypeScript type usages and call refs live in the `identifiers` table. ZodType had 94 identifiers but 0 relationships.
- Fix: After relationship expansion, also query `get_identifiers_by_names()` for `type_usage`, `import`, `call` kinds. Use `containing_symbol_id` as neighbor ID. Relationship entries take priority via `or_insert_with()`.
- Systemic benefit: TypeScript, GDScript, Zig, Scala all gain richer get_context neighbors.

**Bug 2 — `filter_test_symbols` missing path-based fallback (Check 8.2):**
- Root cause: `filter_test_symbols()` in `text_search.rs` only checked `metadata["is_test"]`, which extractors only set on function symbols. Interfaces, types, classes in `.test.ts` files had no `is_test` metadata → bypassed `exclude_tests=true`.
- Fix: Added `|| is_test_path(&s.file_path)` as fallback in `filter_test_symbols`. Systemic benefit: all languages.

### IMPACT
- TypeScript fully verified: all 8 checks now PASS*
- `get_context` now finds identifier-based neighbors for TypeScript (was 0, now shows callers/type users)
- Test exclusion in definition search now correctly filters all symbols from test files
- Dev test tier clean: 8 buckets passed
