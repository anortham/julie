### Bug Fix 1: Watcher-side lockfile filtering
- `filtering.rs::should_index_file` and `events.rs::should_index_file` / `should_process_deletion` didn't check `BLACKLISTED_FILENAMES`
- Watcher could re-index pnpm-lock.yaml and package-lock.json on every save event
- **Fix**: Added `BLACKLISTED_FILENAMES` guard to all 3 watcher filter functions
- **Test**: `test_should_index_file_skips_lockfiles` in `watcher::filtering::tests`

### Bug Fix 2: `libs/` vendor false positive
- `analyze_vendor_patterns` treated `libs/` as a vendor directory (like `vendor/`, `node_modules/`)
- **Impact**: Nx/Angular monorepos use `libs/` as the standard source directory alongside `apps/` — same class of bug as `packages/` that was silently excluding all JS/TS monorepo source code
- **Fix**: Removed `"libs"` from both `matches!` blocks in `discover_indexable_files`
- **Test**: Updated `test_analyze_vendor_patterns_detects_libs_directory` → `test_analyze_vendor_patterns_does_not_flag_libs_directory`, updated 2 other tests to use `vendor/` as example vendor dir
