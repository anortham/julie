# QML Language Verification — 4 Bugs Fixed

## WHAT
Verified QML extractor against KDE Kirigami (200 .qml files, 5417 symbols). Found and fixed 4 bugs via TDD.

## Fixes

### Fix #1: Import extraction missing
- **WHAT**: Added `ui_import` handler to `qml/mod.rs` `traverse_node()`
- **HOW**: Extracts `source` field from `ui_import` nodes (handles both `identifier` and `nested_identifier`)
- **IMPACT**: QML import statements now captured as `Import`-kind symbols

### Fix #2: Test detection not working (systemic)
- **WHAT**: Added `autotests/` directory and `tst_` file prefix to `is_test_path()` in `test_detection.rs`
- **WHY**: Qt/KDE convention not recognized by generic test path detection
- **IMPACT**: Benefits all 33 languages — any project using Qt-style test conventions

### Fix #3: TypeUsage identifiers for component instantiations
- **WHAT**: Added `ui_object_definition` handler to `qml/identifiers.rs`
- **HOW**: Detects nested (non-root) objects and creates `TypeUsage` identifiers for type names
- **IMPACT**: Enables centrality scoring for QML components via type_usage + import centrality boost

### Fix #4: Component name derived from file path
- **WHAT**: Root class name now derives from file stem instead of base type
- **WHY**: In QML, file name IS the component name (`ScrollablePage.qml` = `ScrollablePage`). Base type stored in `extends BaseType` signature.
- **IMPACT**: Definition search now finds QML components by name
