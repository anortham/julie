## TypeScript Verification — 4 Bugs Fixed

### WHAT
Verified TypeScript extractor against colinhacks/zod (498 files, 30,039 symbols). Found and fixed 4 bugs via TDD.

### Bugs Fixed

#### Bug #1 & #2: `packages/` excluded as vendor (CRITICAL)
- `BLACKLISTED_DIRECTORIES` (shared.rs:65) contained `"packages"` — hard-filters the walker
- `analyze_vendor_patterns` (discovery.rs:248,281) also had `"packages"` in vendor candidates
- **Impact**: ALL source files in JS/TS monorepos (npm/pnpm/Lerna/Nx/Turborepo) silently excluded
- zod went from 36 files → 498 files after fix

#### Bug #3: Zero `type_usage` identifiers in TypeScript
- `identifiers.rs` only handled `call_expression` and `member_expression`
- Missing `type_identifier` node handler → zero TypeUsage identifiers
- Centrality Step 1b contributed nothing for TypeScript — all interfaces/types at 0.00
- **Fix**: Added `type_identifier` arm emitting `IdentifierKind::TypeUsage`
- Added `is_ts_noise_type()` filter for common generics (Promise, Array, Record, etc.)
- Verified builtins (string, number) naturally excluded — TS tree-sitter uses `predefined_type` for those

#### Bug #4: Lockfiles indexed despite being generated
- `pnpm-lock.yaml` (ext .yaml) and `package-lock.json` (ext .json) slip through extension blacklist
- pnpm-lock.yaml alone: 9,861 symbols = 43% of zod's total before fix
- **Fix**: Added `BLACKLISTED_FILENAMES` constant in shared.rs, checked in `should_index_file()`
