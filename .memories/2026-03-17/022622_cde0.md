## Fixed Elixir Search Quality: Pending Relationships + Definition Promotion

### Dogfood Testing Results
Ran comprehensive tool quality tests across 3 workspaces:
- **Julie (Rust)** — primary, baseline quality
- **Cats (Scala)** — reference workspace, 22k symbols, excellent quality
- **Phoenix (Elixir)** — reference workspace, 24k symbols, **broken search**

### Bug 1: Elixir Pending Relationships Silently Dropped (FIXED)
- **Root cause (layer 1):** `add_pending_relationship_immut` in `relationships.rs` was a **no-op** — all cross-file relationships were discarded
- **Root cause (layer 2):** Even with mutation fixed, `symbols.iter().find(|s| s.name == target)` matched the local **Import symbol** (created for `use Phoenix.Router`) instead of recognizing this as a cross-file reference
- **Fix:** Changed helper functions to `&mut ElixirExtractor`, push directly to `pending_relationships`, AND filter out Import/Export kinds when matching targets
- **Impact:** Cross-file `use` and function call relationships now produce pending relationships for the resolver, which will fix centrality scores

### Bug 2: Definition Search Promotion Too Strict (FIXED)
- **Root cause:** `promote_exact_name_matches()` did `result.name == query` — `"Phoenix.Router" != "Router"` so Elixir modules never got promoted
- **Secondary issue:** `DEFINITION_KINDS` was missing `"module"`, `"namespace"`, `"type"`, `"constant"`, `"delegate"`
- **Fix:** Added `is_name_match()` helper that matches last component of dot-qualified names, and expanded DEFINITION_KINDS
- **Impact:** Searching "Router" now correctly finds and promotes `defmodule Phoenix.Router`

### Tests Added
- `test_elixir_cross_file_use_produces_pending_relationship` — verifies `use` creates pending relationships
- `test_elixir_cross_file_call_produces_pending_relationship` — verifies unqualified calls create pending relationships  
- `test_qualified_name_last_component_match` — "Router" matches "Phoenix.Router"
- `test_qualified_name_suffix_match` — "Channel.Server" matches "Phoenix.Channel.Server"
- `test_qualified_name_no_partial_component_match` — "Router" does NOT match "RouterHelper"
