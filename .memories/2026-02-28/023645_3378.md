## Embeddings Fallback Logic Fix — Session Complete

Fixed asymmetric backend fallback in `src/embeddings/factory.rs`. When `resolve_backend_preference()` changed to prefer ORT on all platforms, the fallback path wasn't updated to match.

**The Bug:** Only Candle→ORT fallback existed. When ORT init failed on macOS, the function returned `None` instead of trying Candle.

**The Fix:** Added symmetric ORT→Candle fallback when `Auto` resolves to ORT and ORT init fails (lines ~85-92 in factory.rs). Added test `test_auto_fallback_target_is_candle_when_ort_init_fails_and_candle_is_available()` to verify.

**Verification:**
- Rebuilt and restarted Julie — ORT initialized successfully this time
- 19,488 symbols embedded in ~61s via ORT CPU
- Fallback test passes; existing 8 ORT test failures are pre-existing environment issues

**Hybrid Search Status:** Verified working end-to-end. Logs show NL queries correctly triggering hybrid search (keyword + semantic via RRF). Example: "how to find all uses of a symbol" merged 5 keyword + 10 semantic results.
