## Reviewed Task 6 sidecar provider changes (quality gate)

### WHAT
Performed a focused code-quality review of Task 6 files:
- `src/embeddings/sidecar_provider.rs`
- `src/embeddings/factory.rs`
- `src/embeddings/mod.rs`
- `src/tests/core/embedding_sidecar_provider.rs`
- `src/tests/mod.rs`

Also ran targeted tests with sidecar feature enabled:
- `cargo test --lib embedding_sidecar_provider --features embeddings-sidecar`
- `cargo test --lib embedding_sidecar_protocol --features embeddings-sidecar`

### WHY
Task 6 introduces a new subprocess-based embedding backend, which carries lifecycle, protocol, and threading risks that can fail silently under load or in production envs.

### HOW
- Verified provider request/response path and locking model.
- Checked factory resolution/wiring and feature gating.
- Audited env boot assumptions (`python3`, module import path behavior).
- Assessed tests for concurrency/lifecycle/protocol negative coverage.

### IMPACT
- Identified high-impact gaps before merge:
  - No request timeout or startup readiness check can cause indefinite hangs.
  - `PYTHONPATH` handling can break default sidecar import when user already sets `PYTHONPATH`.
  - Test coverage currently misses lifecycle/concurrency/error-envelope robustness.
- Confirmed baseline protocol contract validations exist and pass in targeted tests.
