## Implemented Task 7 sidecar supervision/bootstrap + workspace-init semantics

### WHAT
- Added `src/embeddings/sidecar_supervisor.rs` with managed sidecar bootstrap helpers.
- Integrated supervisor launch config into `SidecarEmbeddingProvider::try_new()` so default startup uses managed venv Python instead of raw `python3`.
- Exported supervisor module/types in `src/embeddings/mod.rs`.
- Added/updated workspace-init tests in `src/tests/core/embedding_provider.rs`:
  - `test_workspace_init_sidecar_bootstrap_failure_falls_back_to_ort`
  - `test_workspace_init_strict_accel_disables_sidecar_when_unaccelerated`
  - adjusted auto-policy expectations to sidecar-first with ORT fallback semantics.

### WHY
Task 7 requires sidecar runtime bootstrap to be supervised and deterministic, while preserving workspace initialization fallback behavior and strict-accel semantics.

### HOW
- Implemented sidecar root resolution:
  - env override: `JULIE_EMBEDDING_SIDECAR_ROOT`
  - default: `<repo>/python/embeddings_sidecar`
- Implemented managed venv resolution:
  - env override: `JULIE_EMBEDDING_SIDECAR_VENV`
  - default cache-based path: `${JULIE_EMBEDDING_CACHE_DIR|$HOME/.cache/julie}/embeddings/sidecar/venv`
- Added venv creation when missing via `python3 -m venv` (overrideable by `JULIE_EMBEDDING_SIDECAR_BOOTSTRAP_PYTHON`).
- Added lightweight package install into managed venv using:
  - `python -m pip install --disable-pip-version-check --no-deps --editable <sidecar_root>`
  - install-marker file to avoid reinstalling every startup.
- Kept explicit program/script overrides functional (`JULIE_EMBEDDING_SIDECAR_PROGRAM` / `..._SCRIPT`) and only defaulted to managed venv when no program override is provided.
- Error messages in bootstrap path now consistently include `sidecar bootstrap` so workspace fallback status captures actionable reason text.

### IMPACT
- Auto mode now attempts managed sidecar bootstrap first and cleanly falls back to ORT when sidecar bootstrap/init fails.
- Strict acceleration correctly disables sidecar embeddings when runtime is CPU/unaccelerated.
- Workspace-init tests now align with sidecar-first policy without assuming sidecar init always succeeds.
