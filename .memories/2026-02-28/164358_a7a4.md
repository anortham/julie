## Addressed Task 5 code-quality blockers for Python sidecar runtime

- **WHAT:** Applied requested quality fixes across Task 5 files (`runtime.py`, `pyproject.toml`, `test_runtime.py`) and expanded runtime test coverage.
- **WHY:** Reviewer feedback identified policy mismatch (lazy imports vs hard deps), edge-case robustness gaps (`batch_size`, MPS probe), and missing behavioral tests.
- **HOW:**
  - Moved heavy runtime packages to optional extra: `[project.optional-dependencies].runtime` while keeping base `dependencies = []`.
  - In `build_runtime()`, added explicit validation that `batch_size` is a positive integer.
  - Hardened `_select_device()` to call availability probes only when callable, preventing crashes for malformed/non-callable `mps.is_available`.
  - Added deterministic tests for:
    - device precedence `cuda -> mps -> cpu`
    - non-callable MPS probe fallback behavior
    - output-count mismatch error path
    - missing dependency error message clarity (monkeypatched import failure)
    - `embed_query()` 384-length vector shape
- **IMPACT:** Runtime contract remains strict 384-d while being safer against edge-case device probes and misconfiguration. Dependency policy now aligns with lazy-import design and keeps tests/tooling lightweight.
