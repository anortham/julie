## Task 3 implemented: sidecar protocol contracts + validation helpers (TDD)

### WHAT
- Added failing tests for sidecar batch response validation:
  - `test_embed_batch_response_rejects_dimension_mismatch`
  - `test_embed_batch_response_rejects_count_mismatch`
- Wired test module in `src/tests/mod.rs`.
- Implemented new protocol module `src/embeddings/sidecar_protocol.rs` with:
  - Protocol constants (`SIDECAR_PROTOCOL_SCHEMA`, `SIDECAR_PROTOCOL_VERSION`, `SIDECAR_EXPECTED_DIMS`)
  - Request/response envelope structs
  - Embed payload/result structs
  - Validation helpers for query and batch response dimensions + counts
- Updated `src/embeddings/mod.rs` to expose `sidecar_protocol` module and re-export protocol types/helpers.

### WHY
Task 3 requires a strict contract boundary before process supervision/client code exists. Dimension/count validation must fail fast so future sidecar provider code cannot silently accept invalid vectors.

  1. Added tests importing `crate::embeddings::sidecar_protocol` before module existed.
  2. Ran targeted test filter and confirmed compile-time failure (`E0432 unresolved import`).
  3. Added minimal protocol types and validation functions.
  4. Re-ran same targeted filter and confirmed both tests pass.

### IMPACT
- Sidecar protocol contract now exists in Rust with explicit validation semantics.
- Future sidecar provider implementation can call `validate_batch_response`/`validate_query_response` directly.
- Keeps scope limited to protocol layer only (no process spawning yet).
