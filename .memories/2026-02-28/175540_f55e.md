## Applied Task 10 documentation quality fixes for sidecar operations guide

### WHAT
- Updated `docs/operations/embedding-sidecar.md` with requested operational caveat and diagnostics clarifications.
- Added explicit note that default sidecar root is derived from build-time `CARGO_MANIFEST_DIR`, which can mismatch on prebuilt binaries.
- Added release-binary override guidance: `JULIE_EMBEDDING_SIDECAR_PROGRAM`, optional `JULIE_EMBEDDING_SIDECAR_SCRIPT`/`JULIE_EMBEDDING_SIDECAR_MODULE`, or `JULIE_EMBEDDING_SIDECAR_ROOT`.
- Clarified diagnostics semantics:
  - `Accelerated` can be `unknown` in stats for non-primary workspaces.
  - `Embedding Status: ...` labels are health-output labels, not stats labels.

### WHY
- User requested doc quality corrections to avoid operational ambiguity for packaged binaries and to align health vs stats wording with actual runtime outputs.

### HOW
- Edited the existing operations doc in place and validated presence of required caveat/override/fallback/diagnostics text with grep checks.

### IMPACT
- Operations guidance is more accurate for deployment scenarios and clearer for interpreting diagnostics across `health` and `stats`.
