## Implemented centrality propagation from interfaces to implementations

In C# DI codebases, all references go through interfaces (`ILuceneIndexService`), leaving concrete implementations (`LuceneIndexService`) with zero centrality. This caused `get_context` to never select important implementation classes as pivots.

### What
- Added Step 2 to `compute_reference_scores()` in `src/database/relationships.rs`
- After computing direct incoming reference scores, propagates 70% of interface/base class centrality to implementations
- SQL: `UPDATE symbols SET reference_score += SUM(target.reference_score * 0.7) WHERE outgoing implements/extends`

### How
- Single SQL UPDATE operating on outgoing `implements`/`extends` relationships
- Propagation factor: 0.7 (70% of interface centrality flows to implementor)
- Self-refs excluded (`from_symbol_id != to_symbol_id`)
- Only fires for symbols that have implements/extends relationships (WHERE EXISTS guard)
