## Fixed three sidecar bugs preventing MPS-accelerated embeddings

### Bug 1: tqdm/safetensors stdout corruption
`sentence_transformers.SentenceTransformer()` loading writes tqdm progress bars to fd 1 (stdout) via native C code, corrupting the JSON-RPC protocol channel.

- **Fix:** `os.dup2(2, 1)` fd-level redirect in `main.py` during model init. Python-level `sys.stdout = sys.stderr` was insufficient since safetensors bypasses Python IO.

### Bug 2: `_as_vectors` crashes on `list[torch.Tensor]`
`sentence-transformers` v5.x with `convert_to_numpy=False` returns `list[torch.Tensor]` instead of a stacked tensor. `_as_vectors` didn't handle tensor elements.

- **Fix:** Added `if hasattr(row, "tolist"): row = row.tolist()` for each element in `runtime.py`.

### Bug 3: Opaque error logging
Pipeline logged `"Embedding batch failed"` with no inner error chain. Used `{e}` instead of `{e:#}` for anyhow errors.

- **Fix:** `with_context(|| format!(...))` in pipeline.rs, `{e:#}` in embeddings.rs.

### Bonus: Python 3.12 preference
Changed `python_interpreter_candidates()` to prefer `python3.12` → `python3.13` → `python3.11` → `python3` for cross-platform PyTorch compat.

- **All tests pass:** 20 Python + 20 Rust sidecar + 6 pipeline = 46 tests green
