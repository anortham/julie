## Embeddings Fallback Bug Fixed & Verified

### What Happened
Investigated missing embeddings at runtime. Found asymmetric fallback logic in `src/embeddings/factory.rs` where ORT→Candle path was missing (only Candle→ORT existed).

### Root Cause
When `resolve_backend_preference()` was changed to prioritize ORT on all platforms (commit d700f9b), the `fallback_backend_after_init_failure()` function wasn't updated to match. This created a one-way fallback: if Candle failed, try ORT; but if ORT failed, dead end.

### Fix Applied
Modified `src/embeddings/factory.rs` (~lines 85-92) to add symmetric reverse fallback:
- **Before:** Auto → ORT → ORT fails → returns None (no fallback)
- **After:** Auto → ORT → ORT fails → tries Candle → succeeds

Added `test_fallback_ort_to_candle()` test to match existing `test_fallback_candle_to_ort()`. All fallback tests pass.
