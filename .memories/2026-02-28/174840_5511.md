## Task 9 quality fixes applied (benchmark correctness, fallback signal assertion, shared sidecar helper)

### WHAT
- Updated benchmark scaffold `scripts/benchmarks/embedding_sidecar_baseline.sh` to:
  - run sidecar path with explicit `--features embeddings-sidecar`
  - fail when zero tests matched (`running 0 tests`)
  - fail when expected sidecar test line is missing (ensures test actually ran)
  - replace `seq` loop with bash arithmetic `while ((...))`
- Added fallback signal verification in `src/tests/tools/hybrid_search_tests.rs`:
  - capture WARN logs with a local `tracing_subscriber` writer
  - assert warning text `Semantic search failed, falling back to keyword-only`
- Removed duplicate fake-sidecar helper logic by introducing shared module:
  - new `src/tests/integration/sidecar_test_helpers.rs`
  - reused by `src/tests/integration/embedding_pipeline.rs`
  - reused by `src/tests/integration/embedding_incremental.rs`
  - wired via `src/tests/mod.rs`

### WHY
- Review-required correctness gaps: benchmark could silently pass without running intended test, and hybrid timeout test only validated behavior but not emitted fallback signal.
- Duplicate sidecar test scaffolding increased maintenance risk and drift.

### HOW
- Used TDD for fallback-signal assertion:
  1. introduced call to nonexistent log-capture helper (RED compile failure),
  2. implemented minimal capture helper (`MakeWriter` + in-memory buffer),
  3. re-ran targeted tests to GREEN.
- Refactored sidecar setup into a shared integration helper module with portable Python interpreter detection and fake JSON-line sidecar process.

### IMPACT
- Benchmark harness is now safer and less likely to produce false confidence.
- Hybrid timeout regression now guards both result degradation and observability signal.
- Integration sidecar tests share a single deterministic helper path, reducing duplication-related regressions.
