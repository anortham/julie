## Embeddings Fallback Fix & Semantic Search Verified

### What Was Done

Fixed asymmetric embedding backend fallback and added observability logging to hybrid search pipeline.

**The Bug:** When `Auto` preference resolved to ORT and ORT init failed, the fallback logic returned `None` instead of trying Candle. Fallback only worked Candle→ORT, never ORT→Candle.

**The Fix:** Added symmetric reverse case in `fallback_backend_after_init_failure()` — when ORT fails after Auto resolution, now tries Candle.

**Observability:** Added info-level logging at hybrid search entry point, merge stats, and upgraded failure cases to warn level.

### Verification

- **Embeddings:** 19,490 symbols embedded via ORT CPU (BGE-small-en-v1.5, 384d)
- **Hybrid Search:** Verified firing in logs with RRF merge of keyword + semantic results
- **Example:** Query "gracefully handle failures when backend is unavailable" correctly ranked `factory.rs:86` (the fallback function we just added) as top semantic match
- **End-to-end:** ORT init → vector storage → sqlite-vec queries → RRF merge working
