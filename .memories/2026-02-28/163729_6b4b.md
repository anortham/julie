## Task 4 quality blockers fixed in Python sidecar protocol scaffold

### WHAT
Addressed protocol contract and robustness issues in `python/embeddings_sidecar` without expanding runtime scope.

- Updated `sidecar/protocol.py` to emit Rust-compatible response envelopes with:
  - `schema = "julie.embedding.sidecar"`
  - `version = 1`
  - `request_id`
  - exactly one of `result` or `error`
- Added structured error payloads: `{"code": ..., "message": ...}`
- Added request-id propagation from either `request_id` or alias `id`
- Added defensive request validation:
  - request must be object
  - `method` must be string
  - `params` must be object when present
  - strict method-specific param typing for `embed_query` and `embed_batch`
- Kept stdio loop resilient for malformed JSON and malformed request payloads
- Expanded `tests/test_protocol.py` for envelope correctness, unknown method errors, malformed params, shutdown loop behavior, and malformed JSON loop handling

### WHY
Rust side validates strict response envelope invariants. Sidecar responses needed to match that contract now to prevent integration breakage in later Rust client/provider tasks.

### HOW
Used TDD cycle:
1. Expanded tests first, including envelope and loop failure cases.
2. Ran focused pytest and confirmed failing import/contract state.
3. Implemented minimal protocol envelope builders + validation logic.
4. Re-ran focused pytest until green.

### IMPACT
- Python scaffold now matches Rust protocol contract expectations at envelope level.
- Error handling is deterministic/structured for client-side handling.
- Sidecar loop no longer depends on permissive coercion and handles bad input robustly.
- Runtime remains lightweight placeholder (no torch/model loading).
