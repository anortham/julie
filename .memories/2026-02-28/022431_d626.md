## Embeddings Fallback Fix + Hybrid Search Observability (Complete)

### What Happened
Investigated missing embeddings in runtime. Found ORT backend init failed with `fallback_used=false`, indicating Candle was never attempted as fallback despite being available.

### Root Cause & Fix
**Asymmetric fallback logic in `src/embeddings/factory.rs`:**
- `fallback_backend_after_init_failure()` only handled Candle→ORT
- `resolve_backend_preference()` prefers ORT on all platforms (post-d700f9b)
- When ORT init failed, function returned `None` instead of trying Candle
- Classic bug: recovery path didn't match new preference order

**Fixed in factory.rs (~lines 85-92):** Added symmetric ORT→Candle fallback when `Auto` resolves to ORT and ORT init fails. Added `test_fallback_ort_to_candle()` test.

### Observability Improvements
Added info-level logging to hybrid search path (permanent improvements):
1. **text_search.rs:286** — `info!` when hybrid mode activates with query
2. **hybrid.rs:merge point** — `info!` showing keyword vs semantic result counts
3. **hybrid.rs:failure path** — upgraded to `warn!` so semantic search failures are visible
