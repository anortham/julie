## Fixed sidecar bootstrap failure — hatchling package discovery

The Python sidecar was silently failing to initialize, causing fallback to ORT.

- **Root cause:** `pyproject.toml` declared `name = "embeddings-sidecar"` but the actual Python package directory is `sidecar/`, not `embeddings_sidecar/`. Hatchling's auto-discovery couldn't find the package, so `pip install --editable ".[runtime]"` failed.
- **Fix:** Added `[tool.hatch.build.targets.wheel] packages = ["sidecar"]` to `pyproject.toml`
- **Verification:** Editable install succeeds, sidecar responds to health check with MPS device, all Python (20) and Rust (20) sidecar tests pass
- **Impact:** Sidecar can now bootstrap and use GPU-accelerated embeddings via PyTorch/MPS
