## Deferred embedding provider initialization — unblocks fast startup

Moved `initialize_embedding_provider()` out of `initialize_all_components()` so that SQLite, Tantivy, and the file watcher start immediately without waiting for the Python sidecar bootstrap (venv creation, pip install, model download, PyTorch import).

- **Before:** Workspace init blocked ~48s on sidecar bootstrap before indexing could begin
- **After:** Indexing starts immediately (~11s); sidecar bootstraps lazily when the embedding pipeline runs after indexing completes
- **Where:** `spawn_workspace_embedding()` now calls `initialize_embedding_provider()` via write lock on the handler's workspace, right before the first embedding run
- **Trade-off:** File watcher's copy of `embedding_provider` is `None` until restart (incremental re-embedding on file changes won't work until next session). Acceptable since the full pipeline runs after indexing anyway.
- **Tests:** Fixed 4 embedding_provider tests + 3 behind cfg guards; 854 Rust + 20 Python all pass
