## Embeddings Fallback & Semantic Search Verification

**What was investigated:**
Fixed asymmetric fallback logic in embedding backend selection. Verified that semantic search pipeline is fully operational.

**The Bug (Fixed):**
- `fallback_backend_after_init_failure()` in `src/embeddings/factory.rs` only handled Candle→ORT, never ORT→Candle
- When `resolve_backend_preference()` returned ORT as primary on all platforms and ORT init failed, no fallback was attempted
- Result: On machines where ORT couldn't init, Candle was never tried despite being available

**The Fix:**
Added symmetric reverse fallback case:
```rust
if requested_backend == EmbeddingBackend::Auto
    && resolved_backend == EmbeddingBackend::Ort
    && capabilities.candle_available
{
    return Some(EmbeddingBackend::Candle);
}
```
Tests added: `test_fallback_ort_to_candle()` to match existing `test_fallback_candle_to_ort()`.

**Verification After Rebuild:**
- ✅ 19,488 symbols embedded via ORT (CPU)
- ✅ ORT initialized successfully this time (transient failure in previous session)
- ✅ Embedding provider active: `has_embeddings=true`, `runtime=ort (ONNX Runtime)`
- ✅ Hybrid search confirmed working: logs show "RRF merge: X keyword + Y semantic results"

**How Semantic Search Works:**
1. Only on `search_target="definitions"`
2. Requires NL-like query (lowercase English phrases, no camelCase/snake_case)
3. When triggered: embeds query → KNN search for semantic neighbors → RRF merge with keyword results

**Test Queries That Work:**
- "how do I gracefully handle when something fails"
- "efficiently find all references without reading files"
- "token budgeting for large outputs"

**Key Insight:** Hybrid search was already active and working in logs before the fix. The fix ensures that if ORT ever fails again on a new machine, Candle will automatically be attempted as fallback.
