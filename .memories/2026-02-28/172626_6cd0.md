## Applied Task 7 quality fixes for sidecar supervision/runtime portability

### WHAT
- Fixed sidecar readiness-failure leak path by terminating child process before returning initialization error.
- Hardened supervisor defaults to be cross-platform for cache directory and bootstrap interpreter discovery.
- Replaced hardcoded `python3` usage in sidecar-related tests with portable interpreter detection helper + env override.

### WHY
- Readiness probe failures could leave a spawned child process running.
- Supervisor path selection should work across Linux/macOS/Windows environments and CI layouts.
- Tests should not assume `python3` exists (common Windows portability issue).

### HOW
- `src/embeddings/sidecar_provider.rs`
  - In `spawn_from_command`, wrapped `probe_readiness()` with explicit error branch that calls `process.terminate()` before returning `Err`.
- `src/embeddings/sidecar_supervisor.rs`
  - `managed_cache_base_dir()` now checks in order:
    1. `JULIE_EMBEDDING_CACHE_DIR`
    2. `XDG_CACHE_HOME`
    3. `LOCALAPPDATA`
    4. `APPDATA`
    5. `HOME/.cache/julie`
    6. `std::env::temp_dir()/julie`
  - Added interpreter detection for venv bootstrap:
    - honors `JULIE_EMBEDDING_SIDECAR_BOOTSTRAP_PYTHON`
    - otherwise probes candidate interpreters (`python3`/`python`, plus `py` on Windows) via `--version`.
- `src/tests/core/embedding_provider.rs` and `src/tests/core/embedding_sidecar_provider.rs`
  - Added `test_python_interpreter()` helper:
    - env override: `JULIE_TEST_PYTHON`
    - portable candidate probing
  - Replaced direct `python3` literals with helper return value.

### IMPACT
- No orphan sidecar child remains after failed readiness probe.
- Managed sidecar bootstrap path is now platform-agnostic for both cache and interpreter assumptions.
- Sidecar tests are portable across environments lacking `python3` alias.
