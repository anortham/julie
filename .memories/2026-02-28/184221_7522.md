## Fixed sidecar init timeout — 5s too short for cold start

The sidecar was bootstrapping correctly (pyproject.toml fix worked) but the health probe timed out at 5 seconds during cold start. PyTorch import + model loading takes 5-15s, and first-time model download can take 30s+.

- **Fix:** Added separate `DEFAULT_SIDECAR_INIT_TIMEOUT_MS = 120_000` (2 minutes) for the initial `probe_readiness()` health check, while keeping the 5s timeout for normal embed requests
- **Configurable via:** `JULIE_EMBEDDING_SIDECAR_INIT_TIMEOUT_MS` env var
- **All tests pass:** 854 passed (fast tier), 20 sidecar-specific tests pass
