## Embedding Fallback Fix + Hybrid Search Observability

Fixed asymmetric backend fallback in `src/embeddings/factory.rs`. ORT→Candle recovery now works when Auto resolves to ORT and ORT init fails.

### What was broken
- `fallback_backend_after_init_failure()` only handled Candle→ORT, never ORT→Candle
- When `resolve_backend_preference()` returned ORT (primary on all platforms), and ORT init failed, the function returned `None` instead of trying Candle
- On macOS this meant embeddings would silently go offline when ORT couldn't initialize

### Root cause
Fallback was designed when Candle was primary (ORT as safety net). When commit d700f9b flipped priorities to make ORT primary everywhere, the fallback path wasn't updated — asymmetry bug where recovery logic didn't match new preference order.

### Fix
Added symmetric reverse fallback case (~lines 85-92): if Auto resolved to ORT and ORT init fails, try Candle. Matches existing Candle→ORT pattern.

### Testing
- Added `test_fallback_ort_to_candle()` test
- Ran embedding suite: all 3 fallback tests pass
- 8 pre-existing ORT test failures (environment issue, not regression)
- Rebuilt and restarted: ORT initialized successfully, 19,488 vectors generated via BGE-small-en-v1.5

### Hybrid search observability
Added info/warn-level logging to hybrid search path for production visibility:
- Entry point logs when hybrid mode activates
- Merge stats log keyword vs semantic result counts
- Failure cases upgraded to warn

Changes written but not yet committed (rebuild used old code without logging).
