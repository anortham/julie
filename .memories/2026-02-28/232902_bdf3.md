## Fixed cross-file inheritance relationship extraction for C#

The centrality propagation from last checkpoint had no effect because the `implements`/`extends` relationships were never being created for cross-file cases — which is nearly ALL C# inheritance since interfaces live in separate files.

### Root Cause
`extract_inheritance_relationships` resolved base types against the current file's `symbols` list only. `class LuceneIndexService : ILuceneIndexService` → lookup `ILuceneIndexService` in local symbols → not found (it's in `ILuceneIndexService.cs`) → silently skipped. No relationship created.

Meanwhile, constructor params and field types correctly used `PendingRelationship` for cross-file resolution.

### Fix
- Added `else` branch to create `PendingRelationship` when base type not found in local symbols
- Uses `is_interface_name()` (C# `IFoo` convention) to infer `Implements` vs `Extends` for pending relationships
- Restructured function with Phase 1 (collect data) / Phase 2 (create relationships) to satisfy borrow checker
