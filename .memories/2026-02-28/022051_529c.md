## Embeddings Fallback + Hybrid Search Verification Complete

**What:** Fixed asymmetric backend fallback in embedding factory + added hybrid search observability logging + verified end-to-end semantic search working.

**Why:** Previous session's ORT init failure left embeddings offline with no recovery. Hybrid search pipeline was complete but opaque in logs, making it impossible to diagnose if semantic search was actually firing.

**How:** 
- Added reverse fallback case: `ORT → Candle` when Auto resolves to ORT and ORT init fails (mirrors existing `Candle → ORT` path)
- Added `test_fallback_ort_to_candle()` test matching existing pattern  
- Added info-level logging at 3 points: hybrid gate (shows query), merge stats (keyword vs semantic counts), failure cases (warn level)
- Staged and committed 4 files: `factory.rs` (fallback), `hybrid.rs` (logging), `text_search.rs` (logging), `embedding_provider.rs` (test)

**Impact:**
- **Fallback fixed:** ORT failures now gracefully recover to Candle instead of leaving embeddings offline
- **Observability added:** Production logs now show when hybrid search fires and result composition (keyword:semantic ratio)
- **Semantic search verified:** Live testing confirms end-to-end flow: ORT embeddings → sqlite-vec storage → hybrid retrieval → RRF merge
- **Test coverage:** New test case for reverse fallback prevents regression

**Evidence:**
- Logs show: `Hybrid search (keyword + semantic) for NL query: '...' Hybrid merge: 10 keyword + 10 semantic results → RRF merge`
- Query "gracefully handle failures when backend is unavailable" returned `factory.rs:86` (our new fallback function) as top result — semantic understanding verified
- All fallback tests pass; 8 pre-existing ORT test failures unchanged (environment-specific)
- Embeddings fully generated: 19,490 vectors via ORT CPU in ~61s

**Next:**

- Hybrid search observability is permanent improvement for future debugging
