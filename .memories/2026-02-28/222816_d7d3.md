## Completed deep-dive analysis for 3 dogfood search/scoring fixes

### Issue 1: Definition Search Over-fetch
- **Bug location**: `src/tools/search/text_search.rs` lines 139-146 (reference workspace) and 360-366 (primary workspace)
- `.take(limit_usize)` happens BEFORE centrality boost and promotion when `file_pattern` is set
- Without `file_pattern`, the no-filter path correctly does: centrality → promote → truncate
- The `kind` field on `SymbolSearchResult` is a string ("class", "import", etc.) via `SymbolKind::Display`
- Enhancement: `promote_exact_name_matches` should also prefer definition kinds (class, struct, interface, trait, enum) over imports

### Issue 2: `is_nl_like_query` Duplication
- `scoring.rs:240` — the authoritative version, uses `looks_like_identifier_token()` to reject identifiers
- `expansion.rs:68` — simpler duplicate, only checks `words.len() > 1 && all words have alphabetic chars`
- The expansion version DOES NOT reject identifier-like queries (e.g. "SearchIndex" would return true)
- Used by: `expansion.rs:27` (expand_query_terms), `scoring.rs:131` (apply_nl_path_prior), `index.rs:655` (rerank_candidate_limit), `text_search.rs:90,288` (hybrid decision)
- Fix: Delete the expansion.rs version, make scoring.rs version pub(crate), import it in expansion.rs

### Issue 3: C# Field/Property Type Relationships
- `csharp/relationships.rs` visit_relationships handles: class/interface/struct → inheritance, constructor → DI params, invocation/member_access → calls
- **Missing**: `field_declaration` and `property_declaration` node types
- `helpers.rs` already has `extract_field_type()` (line 174) and `extract_property_type()` (line 130) that extract type strings
- Can reuse `extract_parameter_type_name()` pattern (lines 248-341) which handles identifier, generic_name, nullable_type, qualified_name, array_type
- Test file pattern: `crates/julie-extractors/src/tests/csharp/constructor_relationships.rs` uses `extract_full()` helper
- C# test mod.rs at `crates/julie-extractors/src/tests/csharp/mod.rs` needs new module registered
