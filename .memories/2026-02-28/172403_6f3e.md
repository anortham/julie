## Task 7 code-quality review findings (sidecar bootstrap/supervision)

Completed a focused review of Task 7 touched areas:
- `src/embeddings/sidecar_supervisor.rs`
- `src/embeddings/sidecar_provider.rs`
- `src/tests/core/embedding_provider.rs`
- module wiring in `src/embeddings/mod.rs`, `src/embeddings/factory.rs`, `src/tests/mod.rs`

### What looked good
- Auto policy + fallback semantics are wired coherently (`Auto -> Sidecar -> ORT/Candle` when init fails).
- Strict acceleration disable path is covered by workspace-init tests.
- Sidecar protocol/provider tests cover envelope validation, request-id mismatch, concurrency, and drop-triggered shutdown.

### Required blockers identified
- **Bootstrap safety leak:** sidecar child process is not terminated when readiness probe fails during construction (`SidecarEmbeddingProvider::spawn_from_command`), risking orphan processes.
- **Cross-platform bootstrap defaults:** managed venv bootstrap defaults to `python3` and cache base uses `HOME` directly, which is brittle on Windows and non-standard environments.

### Additional quality gaps
- Test harnesses hardcode `python3` in sidecar tests, making cross-platform behavior under-tested.
- No direct tests for `sidecar_supervisor` path/bootstrap behavior (env overrides, path derivation, interpreter resolution).
