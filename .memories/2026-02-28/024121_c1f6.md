## Fixed Embedding Backend Fallback — ORT → Candle Path

**Problem:** Embeddings failed to initialize with "Failed to initialize fastembed ONNX model". Logs showed `fallback_used=false` — ORT was tried, failed, but Candle was never attempted as a fallback.

**Root Cause:** The fallback logic was asymmetric:
- `resolve_backend_preference` prefers ORT on all platforms
- `fallback_backend_after_init_failure` only handled Candle→ORT, never ORT→Candle
- When `Auto` resolved to ORT and ORT init failed, no fallback path existed

**Solution:** Added the reverse fallback case in `src/embedding/runtime.rs` (line ~104):
```rust
// New: ORT failed, Auto was requested, Candle is available → try Candle
if requested_backend == EmbeddingBackend::Auto
    && resolved_backend == EmbeddingBackend::Ort
    && capabilities.candle_available
{
    return Some(EmbeddingBackend::Candle);
}
```

**Testing:**
