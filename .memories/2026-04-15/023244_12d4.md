## Tightened semantic unresolved-edge policy

Completed the next tree-sitter hardening slice in the `treesitter-world-class-hardening` worktree.

- **What changed:**
  - Go now drops stdlib package call noise for `fmt.Println` while preserving project package unresolved calls like `utils.HelperFunction`.
  - PowerShell now drops built-in cmdlet noise for `Write-Output` and `Get-ChildItem` while preserving unresolved project functions, including user-defined Verb-Noun names.
  - Bash and R now have explicit regression tests that lock builtin/runtime filtering as contract behavior.
- **Why:** Structured unresolved data was already preserved across the branch, but some extractors still emitted obvious runtime noise as pending graph edges. That polluted relationship quality and created confidently wrong candidates.
- **How:**
  - Executed the implementation plan in `docs/plans/2026-04-14-semantic-unresolved-edge-policy.md`.
  - Used subagent-driven development for the Go, PowerShell, Bash, and R slices with inline review.
  - Tightened extractor policy only where the extractor had strong evidence: import-backed stdlib filtering in Go, exact-name builtin cmdlet filtering in PowerShell, and explicit contract tests for Bash and R.
  - Verified targeted regressions and then ran `cargo xtask test dev`.
- **Impact:** The branch now emits fewer bogus unresolved edges without sacrificing plausible cross-file project calls. This raises graph quality and leaves the next hardening slice in better shape.
