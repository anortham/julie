## Fixed parallel integration flake in stale-index verification

Resolved the remaining branch-level blocker that was preventing `cargo xtask test system` and `cargo xtask test full` from passing.

- **What:** Serialized the env-mutating integration tests onto the shared `embedding_env` lock.
- **Why:** The stale-index tests were green in isolation and under serialized integration runs, but failed under the parallel `integration` bucket inside xtask tiers. The common factor was unsafe process-environment mutation during parallel test execution.
- **How:** Added `#[serial_test::serial(embedding_env)]` to the env-mutating tests in `src/tests/integration/documentation_indexing.rs`, the two stale-index freshness tests in `src/tests/integration/stale_index_detection.rs`, and the daemon lifecycle slow-init test in `src/tests/integration/daemon_lifecycle.rs`. Also corrected the stale comment that claimed per-test thread isolation was sufficient for env safety.
- **Impact:** `cargo xtask test system` is green again, and the full branch-level verification now passes. This removes the last known verification blocker for the tree-sitter hardening branch and confirms the issue was test interference, not extractor logic.
