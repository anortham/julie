## Completed structured unresolved relationship hardening wave

### WHAT
Finished the remaining structured unresolved relationship migration in the `treesitter-world-class-hardening` worktree.

- Migrated the remaining legacy extractor wave to emit `StructuredPendingRelationship` alongside degraded legacy `PendingRelationship` compatibility payloads.
- Added shared invariant coverage for structured pending extend/offset/rekey behavior.
- Updated registry canonical extraction so the migrated wave now preserves `structured_pending_relationships` instead of dropping them during registry assembly.
- Removed fresh warning noise introduced by the migration batch.

### WHY
The extractor stack was stuck in a half-upgraded state: some languages preserved unresolved target structure, while others still collapsed pending calls to flat `callee_name` strings. That made semantic policy work premature and risky because the input data was inconsistent.

### HOW
- Expanded `relationship_precision.rs`, `path_identity.rs`, and `api_surface.rs` to lock the shared contract down.
- Migrated `c`, `cpp`, `rust`, `zig`, `go`, `python`, `ruby`, `gdscript`, and `dart` extractors to keep structured pending state and emit unresolved targets through `BaseExtractor::create_pending_relationship(...)`.
- Added or tightened cross-file regression tests for each migrated language to assert structured target shape only where the parser exposes real context.
- Reworked `registry.rs` macros so the remaining migrated wave reaches canonical extraction through structured-aware entrypoints.
- Removed the unused `warn` import, the unused legacy file-extractor macro, and dead `add_pending_relationship` methods left behind in `python`, `ruby`, and `gdscript`.

### IMPACT
- Canonical extraction now preserves structured unresolved targets across the remaining legacy wave.
- The branch is past the producer-migration phase and is ready for the next slice: semantic unresolved-edge policy tightening.
- Fresh dev-tier verification is green after the cleanup pass.
