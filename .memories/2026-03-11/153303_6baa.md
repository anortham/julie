## Phase 7: CI/CD & Distribution for Tray App

### WHAT
- Added `build-tray` job to `.github/workflows/release.yml` that builds julie-tray on all 3 platforms
- Created `tauri-app/package.json` with `@tauri-apps/cli` dev dependency
- Updated `tauri.conf.json` bundle icons to include `.ico` and `.icns`
- Updated `.gitignore` for generated icon files and node_modules

### WHY
The tray app needs to ship alongside julie-server in each release. Without CI/CD, users would have no way to get the tray app binary.

### HOW
- **Parallel jobs**: `build` (server) and `build-tray` (tray) run concurrently, `release` waits for both
- **Platform bundles**: macOS → `.dmg`, Windows → NSIS `.exe` installer, Linux → `.AppImage` + `.deb`
- **Icon generation**: `npx tauri icon` runs in CI to generate `.ico`/`.icns` from committed `icon.png`
- **Linux deps**: Installs `libwebkit2gtk-4.1-dev`, `libappindicator3-dev`, `librsvg2-dev`, `patchelf`
- **Release notes**: Updated to document both server and tray app downloads with platform table
- **Consistent naming**: `julie-tray-v{VERSION}-{target}.{ext}` pattern
