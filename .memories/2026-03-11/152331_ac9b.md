## Phase 6: Dashboard Diagnostics Endpoint + Export Button

### WHAT
- Created `src/api/diagnostics.rs` — new `GET /api/diagnostics/report` endpoint
- Registered route, OpenAPI schemas, and tag in `src/api/mod.rs`
- Added "Export Diagnostic Bundle" card + button to `ui/src/views/Dashboard.vue`
- Rebuilt the UI (`npm run build`)

### WHY
Users need an easy way to export logs and system state for debugging. The tray app has a zip-based bundle export, but the web dashboard needed its own — JSON-based — so users can click one button from the browser.

### HOW
- Server endpoint collects: system info (version/OS/arch/uptime), daemon health (PID file, watcher count), per-project status (symbol/file counts), embedding state (backend/acceleration/degraded reason), recent log lines (last 200 from today's log + daemon stdout/stderr)
- All paths sanitized (home dir → `~`)
- Dashboard button fetches JSON, creates a `Blob`, triggers browser download as `julie-diagnostic-{timestamp}.json`
- Follows existing dashboard patterns: plain `fetch()` + Vue `ref()`, `.check-btn` CSS class

### IMPACT
- Completes Phase 6 of the Tauri tray app plan (dashboard enhancements)
- Users can now export diagnostics from the web dashboard with one click
- All 1298 tests pass, UI builds clean
