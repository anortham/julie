## Added graceful daemon shutdown on Windows via named events

The daemon was dying silently on Windows with no shutdown log. Root cause: the daemon is spawned with `CREATE_NO_WINDOW`, so `tokio::signal::ctrl_c()` (which wraps `SetConsoleCtrlHandler`) never fires. When `julie stop` used `taskkill /F` (equivalent to `TerminateProcess`/`SIGKILL`), the process was killed instantly, skipping all cleanup and log flushing.

### Changes

- **`src/daemon/shutdown_event.rs`** (new): Win32 named event wrapper using `CreateEventW`/`SetEvent`/`WaitForSingleObject`. `ShutdownEvent::create()` + `wait()` for the daemon side, `signal_shutdown()` for the stop side.
- **`src/paths.rs`**: Added `daemon_shutdown_event()` returning `Local\julie-daemon-shutdown-{hash}`. Extracted `julie_home_hash()` (FNV-1a) to share between pipe name and event name.
- **`src/daemon/mod.rs`**: Creates event at startup, `spawn_blocking` waiter bridges Win32 blocking wait into `tokio::select!` via a `stop_notify: Arc<Notify>`.
- **`src/daemon/lifecycle.rs`**: `stop_daemon()` signals the named event first, falls back to `taskkill /F` for old daemons without event support.
- **Tests**: 4 new tests in `src/tests/daemon/shutdown_event.rs` (naming, isolation, create+signal roundtrip, nonexistent event) + 1 hash consistency test in paths.

### E2E verified

```
Shutdown event received from `julie stop`, stopping daemon
Daemon shutting down active_sessions=0
Embedding service shut down
Daemon stopped
```

Previously: no log output at all on shutdown.
