## v6.1.4 Dashboard overhaul + daemon reliability fixes

Massive session covering dashboard, daemon, and incremental indexing.

### Dashboard: Observatory Theme + Wiring
- **New theme**: Warm amber-on-charcoal replacing low-contrast purple/blue. Outfit font for display, JetBrains Mono for code. Animated badges, gradient volume bars, glow-on-hover cards.
- **Search playground**: Was a stub returning empty results since creation. Wired to actual Tantivy indexes via WorkspacePool. Supports definition and content search across all loaded workspaces.
- **Live Activity SSE**: Was broken (event type mismatch "message" vs "activity"). Fixed, now sends pre-rendered HTML fragments.
- **Metrics table**: Was static. Added 5s polling via htmx.
- **Status page**: Was static. Added /status/live JSON endpoint + 5s JS polling for uptime, sessions, workspaces.
- **Context Saved card**: New metric showing source bytes examined vs output returned. Found source_bytes was never written to daemon.db (passed as None in the daemon.db write path, while correctly written to per-workspace symbols.db).
- **SessionChange events**: Were defined but never emitted. Now sent on connect/disconnect.
- **Audit cleanup**: Removed dead DashboardEvent::LogEntry variant, orphan activity_event.html, unused formatUptime(), dead SSE endpoints (/events/status, /events/metrics), dead CSS rules, dead kind badge branches.

### Daemon Reliability
- **Workspace status "pending"**: `normalize_workspace_paths()` had `if !cfg!(windows) { return Ok(0); }` gating both Windows path fixes AND cross-platform status restoration. Removed the gate; status restoration now runs on all platforms.
- **Stale binary auto-restart**: Three-layer fix. (1) Added mtime check at disconnect time. (2) When idle (0 sessions) and stale binary detected on new connection, shut down immediately before accepting (was accepting then stuck for the session duration). (3) Both checks are now in the running binary.
- **Vector count/embedding model**: Never tracked in daemon.db. Added startup bulk backfill (scans all workspace symbols.db files), per-session backfill, and embedding pipeline completion callback. COALESCE in update_workspace_stats prevents indexing from clobbering these values.

### Incremental Indexing on Session Connect
- **Problem**: File watcher only catches live events. Files changed while daemon was down were invisible until force re-index.
- **Root cause**: `check_if_indexing_needed` (mtime comparison) was gated behind `is_indexed` flag. `handle_index_command` had its own `is_indexed` guard that short-circuited. `get_database_mtime` used wrong path in daemon mode (stdio path instead of ~/.julie/).
- **Fix**: (1) `on_initialized` now spawns staleness check even when already indexed. (2) `get_database_mtime` checks both daemon and stdio paths. (3) `handle_index_command` falls through to `index_workspace_files` instead of returning "already indexed". The blake3 hash comparison in `filter_changed_files` was already well-built; it just couldn't be reached.
- **Verified**: 100 changed files detected and incrementally re-indexed in under 5 seconds on restart.
