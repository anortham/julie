## Shelved codehealth risk/coverage metrics

Removed security risk, change risk, and test coverage metrics from all user-facing and agent-facing surfaces across 10 commits (-1,691 lines, 23 files).

- **What:** Stripped unreliable codehealth metrics from indexing pipeline, deep_dive output, get_context output, query_metrics tool, dashboard, marketing site, and 2 skills
- **Why:** Test coverage showed 3% on a well-tested project (direct references only, no transitive closure). Security risk labeled most symbols HIGH due to heuristic miscalibration in database-heavy codebases. Users seeing bad data lose trust in the entire system.
- **How:** Subagent-driven development with Sonnet implementers, 9 sequential tasks. Brainstormed the decision first, then wrote a formal plan.
- **Impact:** Agents no longer see misleading "Security Risk: HIGH" on harmless functions. Indexing is ~2-5s faster per pass. query_metrics now only serves session/history (real operational data). Test quality analysis (thorough/adequate/thin/stub) is preserved since it's independent and reliable.

### Key decisions
- Keep analysis source files on disk but don't compile them (dormant, not deleted)
- Keep `compute_test_quality_metrics` (rates test functions, independent)
- Keep snapshot infrastructure but simplified to symbol/file counts only
- Keep centrality computation (core search ranking, just removed from dashboard)
- Fundamental blockers to revisiting: incomplete call graph, no type information
