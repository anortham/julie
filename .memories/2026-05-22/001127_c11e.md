## FTS Phase 1: T1–T4 implementation complete, ready for T5 bakeoff

Closing the FTS ranking gap vs Eros's `lancedb-fts` from the May-21 multi-lang bakeoff (267/406 vs 374/406 top1). Phase 1 plan at `docs/plans/2026-05-21-fts-ranking-fixes-phase1.md` is doing cheap wins + measurement infrastructure before Phase 2's unified-target refactor.

### Completed tasks (on branch `fts-ranking-fixes-phase1`)

- **T1** (`906b6965`): standalone CLI was probing the Python embedding sidecar on every NL definitions query, adding ~8.6s. Root cause: `bootstrap_standalone_handler` claimed to "skip embeddings" but the NL search path bypassed that intent. Fix: `JulieServerHandler::mark_standalone_embedding_skipped()` sets `embedding_runtime_status` to a sentinel so the probe guard fires immediately. Daemon mode unchanged. Standalone now ~130ms warm.
- **T3** (`d3f0edd1`): `JULIE_ABLATE_STEMMING` and `JULIE_ABLATE_CAMEL_EMIT` env-var gates on `CodeTokenizer`. `TokenizerCompatibilitySignature` includes both flags so toggling forces index rebuild. Default-off byte-identical to prior behavior.
- **T2** (`04c42753`): extended `EXACT_TITLE_BOOST` (+100) across all three search targets. New `SymbolDatabase::titles_for_files(paths)` batched lookup (chunked at 500). Content path: per-file symbol-title exact-match boost. Files path: `apply_symbol_title_boost_to_file_results` wired through `execute_file_search` with best-effort DB access. Definitions path already correct via `promote_exact_name_matches`. **No scoring constants changed.**
- **T4** (`aec993d3`): `Ablation::{None,NoStemming,NoCamel,Both}` enum + `--ablation` CLI flag on `cargo xtask search-matrix run`. `EnvGuard` RAII restores env on `Drop`. Force reindex per non-baseline ablation. `ablation_label` field added to `SearchMatrixBaselineExecution` with serde default. `diff_baseline_reports()` markdown-table helper.
