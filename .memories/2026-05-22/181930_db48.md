## Phase 2 unified-schema cutover complete + merged to main

FTS Phase 2 (T1..T12) replaced the per-target search dispatch (definitions/files/content) with a single unified Tantivy schema + query path + reranker. 7 codex-adversarial-review findings applied. Merged to local main, ready for release build + dogfood.

### T12 acceptance gate

- **Pre-fix baseline** (f46cee0b): **352/406 top1 (86.7%)**, Top5 371/406. Margin **+2**.
- **Post-fix re-eval** (current HEAD): **355/406 top1 (87.4%)**, Top5 376/406. Margin **+5**.
- The codex-fix batch lifted top1 by +3 hits (overfetch fix recovered hits dropped by the truncation-before-filter bug; role-based test exclusion removed inline `#[cfg(test)]` symbols that were outranking real production hits).

Eval methodology: julie-only Python script that mirrors `_first_matching_rank` + `_path_matches` from eros's `compare.py`, without the slow lancedb candidates (which made the previous bakeoff infeasible — ~4-9 hr vs ~45 s for julie-only). Script + artifacts committed under `docs/plans/evidence/`.

### Codex review findings (all fixed)

- **#1 HIGH** (07a9e718): unified pass overfetch — was requesting only `limit` raw hits then filtering, now requests `max(limit*4, 50)` then filters.
- **#2 HIGH** (07a9e718): exclude_tests now checks `role=="test"` not just `is_test_path()` — catches inline `#[cfg(test)]` symbols.
- **#3 HIGH** (8368d5d7): atomic compat rebuild — lock moved from inside-the-deleted-dir to a sibling lockfile in parent dir, `fs2::lock_exclusive` blocks losers, atomic rebuild via tmpdir + rename.
- **#4 HIGH** (756aa9c7): watcher reprojects relationship partners on incremental update — captures old/new partner symbol IDs, unions, reprojects (capped at 100).
- **#5 MEDIUM** (97699ed4): projection now propagates `relationship_text` SQL errors (was `unwrap_or_default`).
- **#6 MEDIUM** (8368d5d7): reranker score writeback uses `Vec<Option<f32>>` indexed by ordinal — eliminates `(file_path, name)` key collision between file rows and same-named symbols.
- **#7 MEDIUM** (4de9706e): walker inherits git-root `.gitignore` in nested workspaces — separates ancestor `.gitignore` (allowed up to git root) from ancestor `.julieignore` (still blocked).
