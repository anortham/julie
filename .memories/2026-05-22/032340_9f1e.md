## T6: rerank_unified added with Eros-recipe field-score boosts

### WHAT
- Added `pub fn rerank_unified(query: &ParsedQuery, candidates: &[Candidate]) -> Vec<Ranked>` to `src/search/reranker.rs`
- Swapped T5 placeholder `rerank_symbol_score` loop in `search_unified` (index.rs) to batch `rerank_unified` call
- Fixed `is_file_doc` bug: was `role == "docs"`, now correctly `hit.kind == "file"`
- Added 4 new constants: `FILE_BASENAME_EXACT_BOOST=120`, `FILE_BASENAME_TERM_BOOST=30`, `PATH_FRAGMENT_BOOST=25`, `BASENAME_EXACT_TERM_BOOST=40`, `FILE_KIND_EXACT_BONUS=30`
- Created `src/tests/tools/search/unified_reranker_test.rs` with 3 acceptance tests
- Registered test module in `mod.rs`

### WHY
Phase 2 unified-schema rollout. `search_unified` needed a reranker that correctly handles mixed-kind candidates (symbols + file rows) in one pass.

### HOW
- `rerank_unified`: per-candidate score = `tantivy_score + full_string_compact_boost + per_term_boosts - role_demotion`
- File rows (`is_file_doc=true`): compact-form basename/stem match → `+FILE_BASENAME_EXACT_BOOST(120) + FILE_KIND_EXACT_BONUS(30)`; symbol rows: compact-name match → `+EXACT_TITLE_BOOST(100) + kind_boost`
- Per-term: title exact +100, title partial +50, path fragment +25, basename-stem exact +30, basename exact +40
- Score writeback uses `HashMap<(path,title) -> score>` to map ranked output back to original hit order before re-sort
- `compact_alnum_lc` from `crate::search::index` reused via `use crate::search::index::compact_alnum_lc`

### IMPACT
- All 3 acceptance tests PASS (GREEN): `exact_name_beats_partial`, `file_basename_exact_beats_other_file_path_fragment`, `vendor_demoted`
- `cargo check --lib` clean (2 warnings only, no errors)
- `rerank_symbol_score`, `rerank_content_score`, `apply_symbol_title_boost_to_file_results` untouched (survive to T9)
