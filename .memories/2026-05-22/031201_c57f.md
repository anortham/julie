## T5 complete: unified query path (build_unified_query + search_unified + execute_search_unified)

### WHAT
Added four new public functions across four files implementing the Phase 2 unified BM25 search path:
- `build_unified_query` in `src/search/query.rs`
- `UnifiedHit` struct + `SearchIndex::search_unified` in `src/search/index.rs`
- `unified_search_impl` in `src/tools/search/text_search.rs`
- `execute_search_unified` in `src/tools/search/execution.rs`
- Tests in `src/tests/tools/search/unified_query_path_test.rs`

### WHY
Phase 2 T5: read side of the unified schema. One BM25 sweep across all 7 FTS fields, no `doc_type` filter, returning mixed-kind hits with `kind` carried per hit.

### HOW
- `build_unified_query`: 7 fields, per-field weights from Eros defaults (name=4.0, sig=2.0, path_text/doc_comment/rel_text/pretok=1.5, code_body=1.0), term groups with ORIGINAL/ALIAS/NORMALIZED group weights, AND mode with OR fallback
- `search_unified`: over-fetches by NL_RERANK_OVERFETCH_FACTOR=4, materializes all UnifiedHit fields from stored docs, applies post-fetch filters (language/kind/file_pattern/exclude_tests), uses rerank_symbol_score as T5 placeholder
- `unified_search_impl`: mirrors text_search_impl workspace-routing pattern, returns (Vec<Symbol>, usize)
- `execute_search_unified`: mirrors execute_definition_search, iterates workspaces, merges, sorts by score desc
- No existing functions modified; all additive

### IMPACT
- `cargo check` clean (one dead_code warning for execute_search_unified — expected, T8 wires it)
- Both acceptance criteria tests GREEN: `returns_mixed_kinds` + `file_exact_beats_symbol_partial`
- T8 can now wire `execute_search_unified` without touching any existing function
