## T7: Populate relationship_text from relationships table

### WHAT
Implemented T7 of the FTS Phase 2 unified-schema rollout. Symbol rows in the Tantivy index now have their `relationship_text` field populated from the SQLite `relationships` table.

### HOW
- **`src/search/projection/apply.rs`**: Added `collect_relationship_names_bounded()` — one batch call each to `get_outgoing_relationships_for_symbols` and `get_relationships_to_symbols`, resolves partner symbol names via `get_symbols_by_ids`, deduplicates, joins with spaces, truncates at 512 bytes on whitespace boundary. Added `truncate_to_whitespace_boundary()` helper. Added `apply_documents_with_db()` for DB-backed projection (used in tests). Added `RELATIONSHIP_TEXT_MAX_BYTES = 512` constant. Modified `apply_documents_with_context` to accept `relationship_map: &HashMap<String, String>`. Modified `symbol_doc_to_search_document` to accept `relationship_text: String` parameter. File rows unchanged (always empty).
- **`src/search/projection.rs`**: Re-exported `collect_relationship_names_bounded` and `apply_documents_with_db` as `pub(crate)`. Updated `project_documents`, `project_documents_with_locks`, `rebuild`, and `ensure_current_inner` to precompute relationship maps before calling `apply_documents_with_context`.
- **`src/watcher/handlers.rs`**: Threaded `Arc::clone(db)` into the `spawn_blocking` closure, acquires lock inside to pass `&db_guard` to `apply_uncommitted_documents_from_symbols`.
- **`src/watcher/runtime.rs`**: Same pattern for `retry_dirty_tantivy`.
- **`src/tests/tools/search/relationship_text_test.rs`**: New test file with 3 tests. Registered in `mod.rs`.

### Key discoveries
- `relationship_text` is indexed but NOT stored in Tantivy schema (`code_text_not_stored`). So `UnifiedHit.relationship_text` always reads back as `""`. Test verifies functional effect (caller_function appears in unified search results for callee_target) rather than the stored value.
- `apply_documents` (public, test-only path) passes empty maps — no DB needed since tests that call it don't need relationship enrichment.
- `apply_uncommitted_documents_from_symbols` gained a required `db: &SymbolDatabase` parameter — Rust forced all callers to update (both watcher paths caught at compile time).

### IMPACT
All callers of `apply_documents_with_context` now thread relationship maps through. Production watcher and catch-up indexer paths all enriched.
