## T8 follow-up #2: adapt/delete remaining test failures after unified cutover

### WHAT
Resolved the remaining 27 (now 14 in this batch) test failures from T8's atomic cutover by adapting tests to the new unified-path contract and fixing real propagation gaps in `execute_search_unified`.

### WHY
T8 (commit `55b733f2`) routed all queries through `execute_search_unified`, removing `search_target`. Follow-up #1 (commit `8fc1d61e`) fixed the 5 named regressions but left 27 line_mode-era tests asserting behaviour the unified path no longer provides. Per the user's standing rule "no lazy verification dismissals — fix ALL issues in first pass", all of them had to be resolved before T8 ships, not deferred to T9.

### HOW
**Code changes (`src/tools/search/execution.rs`, `src/tools/search/mod.rs`):**
- Refactored `execute_search_unified` to track filter attribution (`pre_file_pattern_filter_total` and `pre_test_filter_total`) and implement scope rescue.
- When `NoInScopeCandidates` triggers and the pattern is not whitespace-separated multi-glob, re-run the search without `file_pattern`. If hits surface, mark `scope_relaxed=true`, persist `original_file_pattern` and `original_zero_hit_reason`.
- Stamped `or_disjunction_detected` on every run via `query::clean_or_disjunction_terms`.
- Wired `build_content_zero_hit_hint` so `hint_kind` (e.g. `OutOfScopeContentHint`) lands on the trace.
- `mod.rs:execute_with_trace` now prepends `build_scope_rescue_header` when the trace is scope-relaxed, and renders the targeted content zero-hit hint text instead of the generic "No results found" message.

**Test triage:**
- **Category A** (3 tests deleted/rewritten): `line_match_strategy_tests` stemming-dependent assertions that T3 removed; deleted `live_tokenized_file_level_match_avoids_line_match_miss` in `zero_hit_reason_tests.rs` for the same reason.
- **Category B** (9 line_mode tests): adapted index-requirement message assertions to the neutral "Search requires a Tantivy index..." wording; rewrote 5 content-search tests to use symbol-bearing fixtures (the unified path doesn't index comment-only files).
- **Category C** (7 propagation tests): rewrote tests 1/3/4 to verify that pre-rescue `FilePatternFiltered` survives on `trace.original_zero_hit_reason`; updated tests 5/6 to expect unified mixed-kind hit counts (symbol+file rows); fixed test 7's fixture to wrap content in a `def` so the Python extractor produces a symbol.
