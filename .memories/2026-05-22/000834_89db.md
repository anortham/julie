## T4: search_matrix ablation bakeoff extension

### WHAT
Extended the `search_matrix` xtask runner to A/B-test the tokenizer ablation gates added in T3. Full implementation of Task 4 from `docs/plans/2026-05-21-fts-ranking-fixes-phase1.md`.

### HOW

**New types:**
- `Ablation` enum in `xtask/src/cli.rs` — four variants: `None`, `NoStemming`, `NoCamel`, `Both`
- `EnvGuard` RAII struct in `xtask/src/cli.rs` — saves/restores `JULIE_ABLATE_STEMMING` and `JULIE_ABLATE_CAMEL_EMIT` on drop
- `Ablation::apply_env()` — sets env vars BEFORE workspace pool construction (critical: CodeTokenizer reads them at `new()`)

**Modified structs:**
- `SearchMatrixBaselineExecution` — added `ablation_label: String` with `#[serde(default)]` (additive, existing reports still parse)
- `SearchMatrixCommand::Baseline` — added `ablation: Ablation` field (default = `Ablation::None`)

**Modified functions:**
- `run_search_matrix_command` — destructures new `ablation` field, generates labeled filename (`{profile}-baseline-{label}.json`)
- `run_search_matrix_baseline_with_home` — added `ablation: &Ablation` param, passes to async runner
- `run_baseline_async` — calls `ablation.apply_env()` before pool creation; force-reindexes each workspace when ablation is non-None; stamps `ablation_label` on every execution record

**CLI parsing:**
- `ParsedSearchMatrixOptions` — added `ablation: Option<Ablation>`
- `parse_search_matrix_options` — handles `--ablation <variant>`
- `parse_search_matrix_command` — rejects `--ablation` for `mine`, defaults to `Ablation::None` for `baseline`

**Report helpers:**
- `search_matrix_report.rs` — `render_baseline_report_markdown` now shows ablation label in header and per-row
- New `diff_baseline_reports()` pub function — side-by-side per-case top-rank diff, prints markdown table to a `Write`

**Tests:**
- `xtask/tests/search_matrix_ablation_tests.rs` — 10 tests: 6 CLI parsing (including rejection of invalid variant and `--ablation` on `mine`), 2 label round-trip through JSON, 1 env-var hygiene (vars unset after run), 1 invariant test (different ablation labels produce distinct stamps in output)
- Updated `xtask/tests/search_matrix_contract_tests.rs` — added `Ablation` import, fixed `SearchMatrixCommand::Baseline` pattern match, added `&Ablation::None` to all 4 `run_search_matrix_baseline_with_home` call sites

### IMPACT
- `cargo xtask search-matrix run --profile <name> --ablation <none|no-stemming|no-camel|both>` works
- Each ablation variant writes a distinctly-named JSON+MD report
- `ablation_label` present in every execution record; existing reports (missing field) still parse via serde default
- T5 (bakeoff run) can now produce 4 labeled reports per profile and diff them
