## Workspace warnings cleanup + fixed exposed test failures

Cleared ~111 build warnings across `julie` (lib + lib tests) and `julie-extractors`. Most warnings were dead code surfaced by Phase 2's unified-search cutover (T8 deleted the search_target dispatch but left orphan formatters, rerankers, and tests behind).

### What landed

- Deleted dead helpers: legacy `format_lean_*` / `format_definition_*` family, unused glob compilation + file-rank helpers, `tantivy_symbol_to_symbol`
- Deleted dead test files: `definition_overfetch_tests.rs`, `definition_promotion_tests.rs`, `tantivy_kind_conversion_tests.rs` (all targeted deleted symbols)
- Migrated `clear_cache_for_test() + acquire_gate()` callers to per-test `Registry::new()` (`watcher_mutation_gate.rs`, `workspace/commands/index.rs`)
- Stripped dead fields from `LineModeSearchResult` / `LineModeFetchOutcome` / `LineModeScopedOutcome`: `relaxed`, `zero_hit_reason`, `file_pattern_diagnostic`, `original_zero_hit_reason`
- Renamed `test_tryCatch_error_handling` ‚Üí `test_try_catch_error_handling` (snake_case)
- Updated `xtask/test_tiers.toml` + contract goldens for removed `tools-search-promotion` command
- Regenerated `LANGUAGE_CERTIFICATION_REPORT.md`

### Test-failure rabbit hole (the part that took the time)

User pushed back: *"test failures are not optional"*. The warnings cleanup exposed 5 pre-existing failures in target_workspace + workspace_isolation_smoke + target_workspace contract test:

- **Contract conflict between two tests for unknown workspace_id:**
  - `test_invalid_target_workspace_id_error` (older, 2026-04-18) expected `Err`
  - `test_invalid_workspace_id_returns_error` (newer, 2026-05-10) expected `Ok` with `"Workspace not indexed yet"`
  - The newer smoke test is the canonical contract (explicitly documented). Fixed by:
    - Adding `unknown_target_workspace_message()` helper distinct from `missing_index_message()` (semantics: "needs index" vs "needs refresh")
    - Routing DB-missing target to the new message in stdio mode
    - Updating the older Err-expecting test to match the smoke contract
- **Isolation tests checking line-body markers** (`PRIMARY_WORKSPACE_MARKER` text strings in `println!()` bodies) didn't work with unified search's symbol-based output. Rewrote to assert against unique fixture symbol names (`primary_marker_function` / `reference_marker_function`) ‚Äî survives output format changes
- **`test_live_workspace_surface_has_no_legacy_workspace_language`** caught my own regression: my new comments in `mod.rs` used the banned phrase "reference workspace" ‚Üí renamed to "target workspace"
