## T4: Switch projection to emit SearchDocument with union write

### WHAT
Switched `apply_documents_with_context` (and transitively `apply_documents` and `apply_uncommitted_documents_from_symbols`) from calling `add_symbol_with_context` + `add_file_content` to building `SearchDocument`s and calling `add_search_doc`.

### WHY
Phase 2 unified-schema rollout. T4 routes all projection writes through the single `SearchDocument` union shape, which writes both the old-path fields AND the new Phase 2 fields (`pretokenized_code`, `relationship_text`). This enables `pretokenized_code` to be populated at index time for all projection paths (catch-up indexer, watcher, rebuild).

### HOW
- Added two private helpers in `apply.rs`:
  - `symbol_doc_to_search_document(doc: &SymbolDocument, context: &SymbolIndexContext) -> SearchDocument` — builds union shape, populates `pretokenized_code` from `name + signature + code_body`, leaves `relationship_text` empty (T7 populates it)
  - `file_doc_to_search_document(doc: &FileDocument) -> SearchDocument` — builds file row, populates `pretokenized_code` from first ≤2000 bytes of content
- `normalize_file_path` / `basename_for_path` inlined (trivial one-liners, private in `index.rs`)
- `classify_role` / `test_subrole` imported from `crate::search::scoring` (pub(crate))
- `truncate_utf8_bytes` imported from `crate::search::index` (pub)
- Old methods (`add_symbol_with_context`, `add_file_content`) kept alive — T9 deletes them

### Key decision: pretokenize input for file rows
Used `truncate_utf8_bytes(&doc.content, 2000)` as the input to `pretokenize_code` for file rows. Judgment: avoids index-time bloat for large files; consistent with the 2000-byte cap on `code_body` for symbol rows.

### IMPACT
- All three `apply_*` helpers now write `SearchDocument` union shape
- `pretokenized_code` populated for symbol and file rows
- Old read paths (`search_symbols`, `search_content`) still work — union coverage preserved
- Watcher (`apply_uncommitted_documents_from_symbols`) and catch-up indexer both go through new path
