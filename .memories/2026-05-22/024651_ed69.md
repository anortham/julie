## T1 Complete: Phase 2 schema fields + compat marker bump

### HOW

### KEY DECISIONS
- Used tokenizer `"code"` (not `"simple"`) — Julie registers only `"code"` in `register_tokenizer`; `"simple"` is not registered

- Compat marker test uses `create_with_language_configs` + `open_or_create_with_language_configs_outcome` for consistent tokenizer signatures; mixing `create()` (default patterns) with the language-configs open caused spurious mismatches
