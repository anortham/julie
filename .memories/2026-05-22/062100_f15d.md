## T8 Follow-Up: Restore Semantic Behaviors in Unified Path

### WHAT
Fixed 5 failing `fast_search_regression_tests` plus 3 related test failures (8 total → 0) introduced by the T8 atomic cutover. All acceptance criteria from the review feedback met.

### WHY
T8 dropped semantic behaviors that the per-target paths used to apply: NL-default-exclude-tests, file_pattern override for test exclusion, line-mode line numbers in locations format, and path-segment ranking. The review required these be restored in a single follow-up commit.

### HOW

**Failure 1 + 2** (`content_nl_default_excludes_tests_*`, `content_auto_exclude_tests_respects_*`):
- Made `effective_content_exclude_tests` `pub(crate)` in `line_mode.rs`
- `execute_search_unified` now applies it: NL queries default to excluding tests unless caller scopes to a test file_pattern
- Track pre-filter total separately; if filtering tests dropped all candidates, set `ZeroHitReason::TestFiltered`

**Failure 3** (`content_locations_format_omits_matching_line_text`):
- When `return_format == "locations"` and no exact-name symbol matches, fall back to `line_mode_matches` and format via `format_content_locations_only`
- New helper: `FastSearchTool::try_line_mode_locations()`
- Restores line-accurate file:line output that the old `execute_content_search` provided

**Failure 4** (`file_search_missing_index_names_file_mode`):
- Updated the test to accept the neutral "Search requires a Tantivy index" message that the unified surface now emits (per review's preferred approach)

**Failure 5** (`file_search_preserves_hidden_directory_ranking_in_tool_output`):
- Added `FILE_PATH_SEGMENT_EXACT_BOOST = 160.0` to `rerank_unified`
- Boosts file rows when the raw query exactly matches a `/`-separated path component, handling dot-prefixed directories like `.cargo/config.toml` for query `.cargo` (compact-normalisation would otherwise strip the dot)

**Test updates for unified semantics:**
- `test_fast_search_deserializes_without_search_target`: context_lines default is `Some(1)` post-T8 (no more per-target defaults)
- `fast_search_unified_returns_file_hits_for_filename_query`: switched query from `mod.rs` to `browser_client` (more discriminating)
- `content_hits_have_neutral_zero_score`: re-targeted to verify unified path returns hits for every seeded file (the 0.0-score invariant was tied to the deleted line-mode-only content path)
