### Phase 2 design — locked, awaiting user spec review

Investigation doc (`docs/investigation/2026-05-21-fts-ranking-gap-vs-lancedb.md`) is the architectural source of truth. Design doc resolves three implementation-level questions:

- **Tokenizer:** drop stemming and aggressive emission; simple tokenizer at query time; CamelCase/snake_case split moved to index-time-only `pretokenized_code` field. Ablation infra survives as measurement vehicle.
- **`relationship_text`:** populated for v1 from Julie's `relationships` table (symbol-rows); file-rows aggregate from contained symbols with drop-if-cost-spikes fallback.
- **`body_excerpt`:** truncated at 2000 bytes for both row types, matching Eros.
- **Dispatch:** `search_target` deleted from `fast_search` (no API-compat concern — agents read tool descriptions fresh each session). One `execute_search`, one reranker pass over mixed-kind candidates. `SymbolDocument` + `FileDocument` replaced by single `SearchDocument`.
- **Migration:** `SEARCH_COMPAT_MARKER_VERSION` bump 3 → 4. Existing workspaces auto-rebuild on next open via the proven compat-marker mechanism.
