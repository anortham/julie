## T2 — SearchDocument struct + add_search_doc method (union shape)

### WHAT
Added `SearchDocument` struct and `add_search_doc` method to `src/search/index.rs` as part of Phase 2 unified-schema rollout.

### WHY
Phase 2 requires a single unified Tantivy document type that covers both symbol and file rows, so old search paths continue to work after T4 cuts projection over.

### HOW
- `SearchDocument` is a union of all fields from `SymbolDocument` + `FileDocument`, plus Phase 2 additions (`pretokenized_code`, `relationship_text` — empty in T2).
- `add_search_doc` writes ALL fields: doc_type discriminator, shared fields (id, name, language, file_path, basename, kind, role, test_role), symbol fields (signature, doc_comment, code_body, annotations, start_line), file fields (content, path_text), and Phase 2 fields.
- Two constructors: `SearchDocument::for_symbol(symbol, annotation_keys, annotations_text, owner_names_text)` and `SearchDocument::for_file(file_info)`.
- `truncate_utf8_bytes(s, max_bytes)` helper truncates on UTF-8 char boundaries — pub so tests can call it directly.
- `stem_of_basename(basename)` helper strips extension from basename for the file-row `name` field.
- Old `SymbolDocument`/`FileDocument`/`add_symbol_with_context`/`add_file_content` untouched.

### IMPACT
- T2 acceptance criteria met: cargo check clean, 3/3 tests pass (GREEN).
- Unblocks T4 (projection cutover) — `add_search_doc` is the write path T4 will use.
