## T12 acceptance gate: discovered prior bakeoffs measured STALE binaries

### WHAT
Every bakeoff in this session (#1 212/406 → #5 333/406) was measuring a julie binary built from a pre-T8 commit that still accepted `--target`. Current binary (post-T8, no `--target`) returns exit 2 against eros's harness, so a true current-julie bakeoff has never been recorded.

### WHY
- T8 cutover (commit `55b733f2`) deleted `--target` from `julie-server search`
- `python/eros/eval/compare.py:_run_auto_julie_query` still constructs `["julie-server","--workspace",...,"search","--target",<category>,...]`
- The on-disk `~/source/julie/target/release/julie-server` was a stale pre-T8 build. Subsequent `cargo build --release` (10:04 CDT today) replaced it, but the rebuild happened MID-BAKEOFF #5, so #5 itself is a mixed measurement.
- Direct repro: `subprocess.run([binary, ..., 'search','--target','definitions','-n','5','q'])` returns exit 2 + clap UnknownArgument on the current binary.

### HOW (fixes shipped)
1. **`fix(cli): restore --target as deprecated no-op for backward compat`** (`029bd4a7`) — added hidden `target: Option<String>` field on `SearchArgs`, kept T8 routing semantics. Updated `cli_search_no_target_test::target_flag_accepted_as_no_op`.
2. **Discovered second pollution layer**: `~/.julieignore` contained the anchored pattern `source/codex-plugin-cc/plugins/` (auto-generated 2026-05-07 when julie was apparently run from `~/`). `ignore` crate inherits custom-ignore files from parent dirs via `parents(true)` (the default). Codex-plugin-cc indexed 19 files vs the 60 it should — silently dropping 41 plugins/ files including `app-server-protocol.d.ts` which 3+ corpus queries target.
3. **Moved `~/.julieignore` + `~/source/.julieignore` to `~/.julie-stale-ignores-backup-2026-05-22/`** before relaunching.
4. **`fix(walker): stop inheriting ignore files from PARENT directories`** (`f46cee0b`) — set `parents(false)` on `WalkBuilder`. `git_global(true)` retained for XDG. Added two regression tests covering ancestor `.julieignore` and `.gitignore`.

### IMPACT
- Alamofire indexed 22 → 478 files (paths the corpus expected — `docs/docsets/.../js/jazzy.js` etc — now actually indexed)
- Codex-plugin-cc indexed 19 → 60 files (plugins/codex tree restored)
- Direct julie probe on 27 sampled queries (5 per category): 22/27 = 81%
- Bakeoff #6 in progress with restored `--target` + clean ancestor state

### UNKNOWN
- Whether the broader index (478 vs 22 for Alamofire) helps or hurts top-1 ranking. The OLD bakeoffs' 333/406 was an artifact of indexing FEWER files, which may have made ranking easier accidentally.
