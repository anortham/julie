## Fixed two Codex adversarial-review findings on fts-phase2-unified-schema

### Finding #3 (HIGH): Concurrent compat rebuild race

**WHAT:** Rewrote `recreate_index_with_lock` in `src/search/index.rs` with a stable cross-process lock.

**WHY:** The old sentinel `.recreating` lived INSIDE the index directory. `recreate_index` calls `remove_dir_all(path)` which deletes the sentinel — so a concurrent opener that already checked the sentinel was now racing against a directory teardown, causing open failures or corrupt state. Also `open_with_tokenizer` was calling bare `recreate_index` (bypassing the lock entirely).

**HOW:**
- Lock file moved to PARENT directory: `<dirname>.julie-rebuild.lock` — survives `remove_dir_all`
- Uses `fs2::FileExt::lock_exclusive()` (already a crate dep) — blocks second caller until first finishes
- After acquiring lock, re-checks compatibility (loser returns early with already-rebuilt index)
- Atomic rebuild via `<dirname>.tmp-rebuild` + `std::fs::rename` — no torn state visible to other openers
- Fixed `open_with_tokenizer` to call `recreate_index_with_lock` instead of bare `recreate_index`

### Finding #6 (MEDIUM): Reranker score writeback key collision

**WHAT:** Replaced `HashMap<(file_path, name), f32>` writeback in `search_unified_full` with ordinal-index writeback.

**WHY:** A file row has `name = basename stem` (e.g. "foo" for `src/foo.rs`). A function named `foo` in that same file produces the same `(path, title)` key. The old writeback silently overwrote one candidate's score with the other's, flipping ranks.

**HOW:**
- Added `original_index: usize` field to `Ranked` struct
- `rerank_unified` uses `.iter().enumerate()` to set `original_index` per candidate
- `search_unified_full` writes back via `reranked_scores[r.original_index] = Some(r.final_score)` then zips with hits

### Tests
- `concurrent_rebuild_both_succeed` — two threads race on stale index, both succeed
- `reranker_writeback_disambiguates_same_name_same_path` — file row + function with same (path, title) get distinct scores

Both tests pass.
