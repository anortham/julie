## Phase 2 (type_arguments) breadth complete + golden/cert side-effects root-caused

Miller-bridge extraction enrichments, branch `feat/miller-bridge-extraction-enrichments`. Phase 2 (ordered/nested generic type-arguments → `type_arguments` table) is now implemented across **all 18 generic-bearing languages** and the generated gates are reconciled.

### Verification (all green)
- **81 reader tests** across 18 langs (`cargo nextest run -p julie-extractors type_arguments`) + **5 main-crate** DB/migration/persistence tests (`--lib type_arguments`). Note: `--lib` only hits the main `julie` binary; reader tests live in the `julie-extractors` binary (use `-p julie-extractors`).
- `cargo check --all-targets`: **0 warnings**.
- #16 (review-finding fixes) confirmed: `scala::...::function_type_arg_preserves_ordinal` + `gdscript::...::nested_generic_preserves_order_and_nesting` PASS.

### Golden + cert root-cause (this session)
Two generated gates broke because two readers (QML, Razor) *newly added* a `type_identifier`/`identifier` dispatch arm for type-arg anchoring — a side effect that also emits type-usage identifiers:
- **Razor** (`razor/cross_file` golden +3): `ItemFromOther`, `OtherProject`, `Models` — all user types/namespaces, builtin-filtered via `is_csharp_builtin_type`, C#-consistent. **Legitimate → blessed.** Cert report total identifiers 212→215, razor row 4→7 match exactly.
- **QML** (`qml/basic` golden +2): both `string` (a builtin primitive). QML's new arm had **no builtin filter** (unlike C#/Python/Razor) → noise. **Fixed the reader** (added `is_qml_builtin_type`, TDD RED→GREEN: `qml::identifiers::tests::test_builtin_property_types_are_not_recorded_as_type_usage`), reverted the qml golden. Builtins never carry type args, so type-arg capture unaffected.

### Pre-existing bug FOUND (off-path, flag to user)
`qml::modern::tests::test_extract_property_value_sources` fails **4 vs 1** Property symbols: `PropertyAnimation on value { from; to; duration }` bindings miscounted as Property symbols. Proven pre-existing — QML *symbol* extraction (`qml/mod.rs`) is byte-identical to `main` (branch only committed a 4-line `get_type_argument_usages()` getter). Not in dev/changed/nano buckets, so it silently rotted. Different subsystem from type-arg/identifier work.
