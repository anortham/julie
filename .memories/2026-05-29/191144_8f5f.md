## VB.NET Generic Type Arguments Investigation

Investigated VB.NET's support for capturing use-site generic type arguments in Julie's type_arguments table (Miller bridge Phase 2).

### WHAT
VB.NET fully supports generic types with syntax `Type(Of Arg1, Arg2)` (e.g., `List(Of String)`, `Dictionary(Of String, Integer)`). Currently the extractor **does NOT capture use-site generic type arguments** and actively strips them during type normalization.

### WHY
Julie is expanding use-site generic capture from C#/TypeScript to other languages. VB.NET is a candidate because it has real generics and tree-sitter support, but current state was unknown.

### HOW - Verification Done
1. Examined VB.NET extractor at `/Users/murphy/source/julie/crates/julie-extractors/src/vbnet/` — identifiers.rs, helpers.rs, types.rs, relationships.rs, members.rs
2. Found generic argument stripping in `normalize_type_name()` (helpers.rs:195-203) that truncates at `(Of` marker
3. Located tree-sitter grammar at https://github.com/LuisCarreras/tree-sitter-vb-dotnet
4. Verified tree-sitter node kinds:
   - `generic_type`: node wrapping a type with `(Of ...)` arguments
   - `type_argument_list`: the argument container (children can be generic_type, namespace_name, array_type, nullable_type, etc.)
   - `new_expression`: has `type` field that can be generic_type
   - `invocation`: method calls that might have generic targets

### CURRENT STATE: 'none'
- identifiers.rs: no record_type_arguments calls anywhere
- Has invocation_expression handling but does not check for/record generics
- No type_usage identifier tracking (unlike C#/TS)
- helpers.rs:83-88 extracts type_parameters for DECLARATIONS but use-site args are stripped

### EFFORT: Medium
- Pattern fully proven in C# (record_outermost_generic_type_arguments at line 107)
- TypeScript provides alternative pattern for annotation context (record_outermost_generic_type_arguments_ts)
- Work required:
  1. Add identifier arm to detect type_usage identifiers within generic_type nodes
  2. Create decompose_vb_type_arg function (3-5 line stubs needed)
  3. Call record_type_arguments at generic_type + new_expression hook points
  4. Possibly handle invocation expressions with generic targets
- NOT 'high' because C# pattern is directly reusable; NOT 'low' because VB.NET lacks any type_usage tracking currently

### TREE-SITTER PROOF
Verified via jq on `/tmp/tree-sitter-vb-dotnet/src/node-types.json`:
- generic_type has children: namespace_name, type_argument_list
- type_argument_list accepts: array_type, generic_type, namespace_name, nullable_type, primitive_type, tuple_type
- new_expression.type field can be generic_type
- This mirrors C# structure closely
