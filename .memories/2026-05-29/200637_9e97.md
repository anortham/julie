**Key gotcha — shared `extract_type_arguments` (base/type_arguments.rs):** assigns `ordinal` ONLY when `decompose` returns `Some` (ordinal++ inside the Some branch). So a decomposer returning `None` for a middle arg SHIFTS subsequent ordinals down (not a gap — a shift). Therefore every decomposer's `_ =>` fallback MUST return `Some(leaf text)` for unknown NAMED nodes, never None. Rust/Go/GDScript/Java/Swift do this correctly (Rust's `lifetime => None` is correct by design — lifetimes aren't type args).
