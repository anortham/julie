## Blocked Sensitive Workspace Roots

- **WHAT**: Added shared root-safety guard for Julie workspace roots and wired it before daemon pool initialization, request-root primary binding, and `manage_workspace` `index`/`open`/`register` path persistence. Added regression coverage proving `WorkspacePool::get_or_init` rejects `/` without leaving a daemon registry row. Fixed a stale docs-site Codex panel contract while running the dev-selected changed gate.
- **WHY**: `/Users/murphy` appeared as a pending registered workspace because some paths only guarded CWD fallback; `WorkspacePool::get_or_init` and workspace commands could still upsert sensitive roots.
- **HOW**: Moved sensitive root detection from `handler.rs` into `src/workspace/root_safety.rs`, kept the CWD-specific error text, added generic rejection for all workspace roots, and inserted checks before any upsert/bind/index path.
- **IMPACT**: Sensitive roots like `/`, home directories, `/Users`, `/home`, `/root`, and Windows system/user container paths are now refused before daemon registry rows are created. `manage_workspace list` no longer shows the stale `/Users/murphy` row in this session.
