## Phase 1 — Capability Schema + Validators (DONE)

Phase 1 of the best-in-class tree-sitter run is complete. Three commits land schema + validators on the worktree HEAD before any per-language work runs.

### What changed

- **Task 1.1** (commit `6279a6fd`): Migrated `capabilities.json.evidence` from bare-string `"docs/findings/COMPILED-FINDINGS.md"` to typed `{kind, value, command}` objects across all 42 gap rows. Added typed `EvidenceRef` enum (Test/Fixture/Commit + DeadString legacy fallback) in `crates/julie-extractors/src/tests/capability_matrix.rs`. New test `capability_matrix_evidence_is_typed_object` rejects bare strings and validates per-variant invariants. Added `planned_closure_task: Option<String>` field for Task 1.2.
- **Task 1.2** (commit `c729e863`): Added three new tests:
  - `capability_matrix_evidence_resolves` — checks fixture paths exist on disk, commit SHAs resolve via `git cat-file -e`, test names exist in nextest inventory (lazy-loaded only when needed).
  - `capability_matrix_no_not_implemented_exceptions` — bans "not implemented", "not yet supported", "todo", "coming soon" in exception reasons.
  - `capability_matrix_open_rows_have_planned_closure_task` — every open row carries a `planned_closure_task` whose value appears in the plan body; closed/exception rows must NOT carry the field.
  - Populated `planned_closure_task` on all 42 open rows: Phase 4a (25 general-prog), Phase 4b/.html/.razor (component/template), Phase 4c (css/regex), Phase 4d (markdown/yaml), Phase 3.{1,2,3} (sql/json/toml).
- **Task 1.3** (commit `a0325092`): Resolved html/razor/sql contradictions:
  - `html` had a "N/A — domain limitation" `required_closure` that contradicted Phase 4b's plan to emit pending for `<script src=...>` cross-file refs. Rewrote reason+required_closure to match the Phase 4b.html closure path.
  - `razor` got tightened `required_closure` naming the `razor_pending_relationships_handled_by_csharp_embed` locking test and the deferred status=exception flip path.
  - `sql` got plan-prescribed required_closure ("Move SQL out of NO_PENDING_CAPABILITIES…").
- **Bonus** (commit `c2eb72b6`): Found-on-the-path xtask breakage: `xtask/src/tree_sitter_certification_data.rs` had a duplicated `CapabilityGap.evidence: String` deserializer that started failing once capabilities.json shipped typed objects. Mirrored the same `EvidenceRef` enum into the xtask, dispatched `validate_gap` per variant (preserving the "gap evidence path does not exist" wording the existing tree_sitter_certification test asserts), and updated the synthetic test fixture in `write_minimal_repo` to typed evidence. Regenerated `docs/LANGUAGE_CERTIFICATION_REPORT.md` so the markdown evidence column now renders typed `kind:value` strings.

### Why this matters

Phase 1 is the doc-rot prevention layer. Without typed evidence + planned_closure_task + exception-reason validation, later phases can close the same gap by mutually exclusive paths or hide deferred work behind "not implemented" exception strings. The four new validators make that impossible — every gap row now declares "what closes me" (`planned_closure_task` for open rows) or "what locks my exception" (typed test/commit evidence for exception rows) in a machine-checked field.
