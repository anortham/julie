## Fixed: stale-binary daemon restart no longer force-aborts live sessions

Completed the **defer + reap** fix (user-chosen, complete option) for the recovery-marker storm where stale-binary auto-restart force-aborted live MCP sessions.

### Changes (9 files, all on branch main, uncommitted)
1. **lifecycle.rs** `mark_restart_pending`: signal restart channel iff `active_sessions == 0` (was: on `first_request`). Decouples arm-the-latch from fire-the-restart. Arming with live sessions only sets the flag.
2. **session.rs**: `SessionRecord.last_activity: Instant`; `touch_session`/`touch_session_at`/`evict_idle(now, threshold)`; `session_idle_timeout()` (env `JULIE_DAEMON_SESSION_IDLE_TIMEOUT_SECS`, default 300s).
3. **mcp_session.rs**: per-request `touch_session` in `handler_for_request` + `handle_notification` so active sessions are never reaped.
4. **app/helpers.rs**: `spawn_session_idle_reaper` (restart-scoped: only reaps while `restart_pending`; fires `mark_restart_pending(0)` when count hits 0) + `session_reaper_interval` (threshold/5 clamped [200ms,30s]).
5. **app.rs / app/handle.rs**: spawn reaper in `serve`, stash `session_reaper_handle`, abort in both LIFO teardown lists.
6. **tests**: new `test_restart_channel_fires_only_after_last_session_disconnects`; rewrote 2 lifecycle tests that encoded the old contract; 3 new session tests; revised `restart_listener` end-to-end test to assert CLEAN reaping exit + NO recovery marker (renamed `daemon_reaps_idle_session_to_restart_cleanly_when_it_never_disconnects`).

### Behavior now
Stale binary + live sessions → arm only, no force-abort. Sessions disconnect naturally OR (if idle and a restart is pending) get reaped → count→0 → restart fires → drain Clean. Honors documented "exit after last session disconnects" AND the prior fix's hang-forever guard (via reaper).
