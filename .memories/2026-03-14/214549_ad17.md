## Fixed two watcher bugs found during dogfooding

### What
1. **Bug 2:** Added `**/.worktrees/**` to `build_ignore_patterns()` and `WorkspaceConfig::default()` — prevents filewatcher from indexing git worktree files
2. **Bug 3:** Added Tantivy search index cleanup to `handle_file_deleted_static` — file deletions now clean up both SQLite AND Tantivy, preventing orphaned search results

### Why
Discovered during dogfooding the `exclude_tests` feature: search results contained duplicate entries from `.worktrees/smart-test-filtering/` paths. Investigation revealed two root causes:
- The filewatcher uses hardcoded `glob::Pattern` ignore patterns that didn't include `.worktrees/` (unlike the full indexer which respects `.gitignore`)
- File deletions only cleaned SQLite, leaving ghost entries in Tantivy forever

### How
- **Bug 2:** One-line additions to `filtering.rs` and `workspace/mod.rs`
- **Bug 3:** Added `search_index` parameter to `handle_file_deleted_static`, scoped `db_lock` to drop before `spawn_blocking`, added `remove_by_file_path` + `commit` following the same pattern as `handle_file_created_or_modified_static`. Updated all callers (mod.rs × 2, handlers.rs rename, test files × 2).

### Impact
- No more duplicate search results from worktrees
- File deletions now properly clean up Tantivy (fixes a general bug, not just worktree-specific)
- Design debt noted: watcher still doesn't respect `.gitignore` (tracked as future work)
