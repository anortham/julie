## Test Intelligence (Phase 2) — Complete

### What
Implemented Layer C (test-to-code linkage) and Layer D (change risk scoring) for Code Health Intelligence:
- **`compute_test_coverage()`** — Links test symbols to production symbols via relationships + identifiers, aggregates quality tiers, stores in `metadata["test_coverage"]`
- **`compute_change_risk_scores()`** — Computes 0.0–1.0 risk score from centrality (P95-normalized), visibility, test weakness, symbol kind. Stores in `metadata["change_risk"]` with HIGH/MEDIUM/LOW labels
- **`deep_dive` integration** — Shows full risk breakdown: "Change Risk: HIGH (0.82) — 14 callers, public, thin tests" with factor details
- **`get_context` integration** — Appends `[HIGH risk]` / `[MEDIUM risk]` / `[LOW risk]` labels on pivot lines

### How
- Two new files: `src/analysis/test_coverage.rs` (~270 lines) and `src/analysis/change_risk.rs` (~240 lines)
- Both hooked into indexing pipeline after existing `compute_test_quality_metrics()`
- 16 new tests (8 coverage + 8 risk) — all pass
- No schema migrations — all data in existing `metadata` JSON column
- Identifier name-match fallback uses directory proximity disambiguation with `(test_id, ident_name)` grouping key

### Impact
AI agents now see per-symbol change risk in `deep_dive` and `get_context` output, answering "how dangerous is it to touch this?" without any extra tool calls. Combined with Phase 1 (test detection + quality tiers), Julie now provides complete test intelligence across 31 languages.
