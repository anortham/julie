## Fixed path relevance scoring inversion + TODO.md cleanup

### Bug fix: `PathRelevanceScorer` test penalty inversion
- **Root cause:** `calculate_score()` had an `in_test_directory` exemption that prevented the test penalty from applying to files in dedicated test directories (`tests/`, `spec/`, etc.). Combined with the 0.15 penalty on collocated test files (in `src/`), this created an inversion: `tests/user_test.rs` scored 0.4 while `src/user.test.ts` scored 0.15.
- **Fix:** Removed the `in_test_directory` exemption and changed penalty from 0.15 to 0.5. The directory score (1.0 for `src/`, 0.4 for `tests/`) already handles graduated ranking; the penalty just needs to uniformly de-prioritize test files vs production code.
- **Result:** Clean hierarchy: production (1.2) > collocated test (0.5) > dedicated test dir (0.2) > docs (0.2) > deps (0.1)
