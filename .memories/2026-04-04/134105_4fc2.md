### BUG FOUND: Primary workspace has no filewatcher in daemon mode

**Evidence:** `grep "watcher_pool.*created and started" daemon.log` shows watchers for flask, jq, sinatra, cobra, serena (all REFERENCE workspaces). Zero watchers for the primary julie workspace.

**Impact:** Primary workspace index goes stale the moment any file is edited during a session. `deep_dive` on newly created symbols fails with "No symbol found". Critical for edit tools: agent uses edit_file to modify code, then can't find the new symbols.

**Root cause location:** `src/daemon/watcher_pool.rs` likely only creates watchers when reference workspaces are attached, not for the primary workspace. The handler's `initialize` path runs catch-up indexing (mtime/blake3 comparison) on session connect but never starts a watcher for the primary workspace.

**How to reproduce:**
1. Start a session in daemon mode
2. Edit any .rs file with the built-in Edit tool
3. `touch` the file
4. Check daemon.log: zero file event entries
5. `deep_dive` on a newly added symbol: "No symbol found"

**Files to investigate:**
- `src/daemon/watcher_pool.rs` (watcher creation logic)
- `src/handler.rs:1235` (workspace initialization, staleness check)
- `src/daemon/mod.rs` (session connection handling)
