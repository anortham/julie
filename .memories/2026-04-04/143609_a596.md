## Fixed critical filewatcher bug: gitignore pattern leak from cache directories

### Root Cause (3-model consensus: Claude + Codex + Gemini)

The file watcher's `build_gitignore_matcher()` in `src/watcher/filtering.rs` calls `collect_gitignore_files()` which walks subdirectories up to 8 levels deep. It discovered `.gitignore` files inside hidden cache directories (`.ruff_cache`, `.pytest_cache`, `.venv`) that contain a bare `*` pattern (standard "ignore everything in this cache dir" behavior).

The `ignore` crate 0.4.25 converts bare `*` to `**/*` at parse time when added via `GitignoreBuilder::add()`. Because Julie flattens all nested `.gitignore` files into a single root-scoped builder (`GitignoreBuilder::new(workspace_root)`), the `**/*` pattern matches EVERY file in the workspace. This caused `should_index_file()` to reject ALL source files as "gitignored", silently breaking the file watcher's event processing pipeline.

### Diagnostic Evidence Chain

1. **Daemon logs**: Watcher created and started for all workspaces, but zero "Background task processing" events ever logged after daemon restarts
2. **notify callback**: Added diagnostic logging, confirmed FSEvents fires and `tx.send()` succeeds
3. **Event detector**: Confirmed `rx.recv()` receives events from the channel
4. **should_index_file**: Added per-check logging, found REJECTED (gitignored) for `src/watcher/filtering.rs`
5. **is_gitignored**: Added pattern diagnostic, identified `*` from `.ruff_cache/.gitignore` as the matching pattern
6. **Codex verification**: Confirmed the `ignore` crate's `GitignoreBuilder` API is designed for same-directory patterns; Julie's cross-directory flattening is the root misuse

### Fix

In `collect_gitignore_files()`, skip hidden directories (names starting with `.`) in addition to blacklisted directories and `.git`. These are universally tool caches whose `.gitignore` files are internal housekeeping and should not influence file watching.
