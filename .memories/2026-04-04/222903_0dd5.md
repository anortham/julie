## Added web-research skill (v6.6.2)

New skill that teaches agents to use browser39 + Julie's existing tools for token-efficient web research.

### What
- Created `.claude/skills/web-research/SKILL.md` - a skill (not MCP tool) that orchestrates browser39 fetching + filewatcher indexing + Julie selective reading
- Updated both READMEs (julie + julie-plugin) with browser39 as optional prereq
- Updated plugin CI workflow for 11 skills
- Copied skill to plugin repo

### Why
Agents waste thousands of tokens fetching full web pages via WebFetch. Julie already has markdown extraction, full-text search, and embeddings. The missing piece was getting web content onto disk as markdown.

### Key Decision
**Skill-based approach, not a native MCP tool.** Zero Rust code. browser39 fetches and converts to markdown, filewatcher auto-indexes, existing Julie tools provide selective reading. Decoupled evolution, clean dependency story.

### How It Works
1. browser39 fetches URL -> markdown
2. Agent saves to `docs/web/{domain}/{path}.md`
3. Filewatcher auto-indexes (symbols, search, embeddings)
4. Agent uses `get_symbols`/`fast_search`/`get_context` with `file_pattern="docs/web/**"`
5. Delete files when done, filewatcher deindexes

### Validated
- browser39 GitHub repo: 8 symbols extracted, selective reading works
- Rust Book ownership chapter: 9 sections at max_depth=2, full content extraction
- Filewatcher latency: ~2 seconds to index new files
