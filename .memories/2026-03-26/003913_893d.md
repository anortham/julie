## v6.0.6 Released: Three Bug Fixes + Serena Comparison

### Fixes
- **RRF merge metadata bias** (`hybrid.rs`): Semantic loop now uses `insert` instead of `or_insert`, so SQLite-sourced metadata overwrites stale Tantivy stored fields for duplicate symbols.
- **Embedding backfill on daemon restart** (`index.rs`): "Already indexed" early-return path now spawns `spawn_workspace_embedding`. Pipeline's incremental filter handles resume naturally.
- **C# property name extraction** (`members.rs`): `extract_property` used `.find(|c| c.kind() == "identifier")` which grabbed the type identifier for custom types. Fixed with `child_by_field_name("name")` + skip-past-type fallback.

### Serena Comparison (Practical)
Used Serena's tools throughout the session for navigation. Key observations:
- Only 3 core tools used: `find_symbol` (18x), `search_for_pattern` (7x), `get_symbols_overview` (4x)
- Editing tools (`replace_symbol_body`, `insert_after_symbol`) went unused in favor of built-in Edit
- `find_referencing_symbols` returned empty results when rust-analyzer was still indexing
- JSON response format adds ~28% bloat from escaping vs Julie's plain text
- `find_symbol(include_body=true)` is convenient but blows up for ambiguous names (99,899 char response for `extract_property`)
- Evaluated adding `include_body` to `fast_search`, concluded current two-call pattern (fast_search -> get_symbols) is more token-efficient due to natural funneling
- `deep_dive` already fills the one-call "show me this symbol" niche
