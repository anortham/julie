## Fixed dashboard workspace status bugs + added live polling

### Bug: Forward-slash paths stuck at "pending"

Three-part root cause:
1. **Adapter normalized slashes** (`adapter/mod.rs:71`): `.replace('\\', "/")` sent forward-slash paths to the daemon, stored as-is in daemon.db
2. **Index handler early-return didn't update status**: After daemon restart, `workspace_pool.get_or_init` reset status to "pending" via `upsert_workspace`. The handler detected workspace was already indexed (`is_indexed = true` from local DB symbols), returned "already indexed" early, and never set daemon.db status back to "ready"
3. **Upsert unconditionally downgraded "ready" to "pending"**: Even for workspaces that were already indexed, reconnecting reset their status

### Fixes applied

- **`src/tools/workspace/commands/index.rs`**: Early-return "already indexed" path now calls `update_workspace_status(ws_id, "ready")` before returning
- **`src/adapter/mod.rs`**: Removed redundant `.replace('\\', "/")` (generate_workspace_id normalizes internally)
- **`src/daemon/database.rs`**: 
  - `upsert_workspace` SQL uses CASE to prevent downgrading "ready" to "pending"
  - New `normalize_workspace_paths()` method: on startup, converts forward-slash paths to native backslashes and restores stuck "ready" status
- **`src/daemon/mod.rs`**: Wired normalization into daemon startup after existing ID migration

### Dashboard live polling

Added 5-second JS polling to the Projects page so status badges, symbol/file counts, and summary bar update without page refresh. Used targeted `getElementById` updates (not htmx table replacement) to preserve Alpine.js expand/collapse state.

- New route: `GET /projects/statuses` returns JSON with badge HTML + stats per workspace
- New partials: `project_summary.html`, `project_table.html` (extracted from projects.html)
- `project_row.html`: Added unique IDs to status/stats cells for targeted updates

### Also fixed

- **Dashboard label**: Changed "Total symbols" to "Code symbols" in project detail view (the count excludes imports/exports/docs, so the old label was misleading)
