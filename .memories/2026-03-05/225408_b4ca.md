## Fixed 4 findings from GPT code review

### P1 + P3: Workspace root threading
- **What:** Added `workspace_root: PathBuf` field to `JulieServerHandler`, threaded from `main.rs`'s `get_workspace_root()` resolution
- **Why:** CLI `--workspace` flag was parsed in `main.rs` but handler independently used `current_dir()`, causing workspace/logging mismatch. Agent instructions (`JULIE_AGENT_INSTRUCTIONS.md`) were loaded from cwd instead of workspace root.
- **How:** `new(workspace_root: PathBuf)` constructor, `load_agent_instructions(&self)` reads from `self.workspace_root`. Added `new_for_test()` convenience constructor for existing test sites.

### P2a: Removed redundant tool_execution_lock
- **What:** Removed `tool_execution_lock: Arc<tokio::sync::Mutex<()>>` from handler and all 7 tool methods
- **Why:** rmcp's `SinkStreamTransport::send()` already serializes writes via `Arc<Mutex<Si>>`. Our lock serialized *all computation*, not just I/O — blocking concurrent read-only tool calls.
- **How:** Pure removal — field, constructor init, `tool_lock_is_free()`, and all `_guard` acquisitions deleted.

### P2b: Skip embeddings during auto-indexing
- **What:** Added `skip_embeddings: bool` parameter to `handle_index_command`, controlled via `call_tool_with_options()`
- **Why:** Auto-indexing triggered sidecar Python runtime startup (huggingface.co downloads), causing 80+ second delays
- **How:** `run_auto_indexing()` calls `call_tool_with_options(self, true)`. Explicit `manage_workspace index` still triggers embeddings.
