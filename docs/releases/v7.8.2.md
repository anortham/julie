# v7.8.2 — Tool-output compaction, line-match correctness, plugin sync

**Theme:** Quality-of-life patch release. Tool output formatters compact same-file runs (token savings for agents), `line_matches` gets a correctness fix for compound identifiers, `edit_file` learns cross-workspace targeting, and a new `cargo xtask sync-plugin` command replaces the manual skill-mirror dance for plugin distribution.

No API changes that require user action. No fixture format changes. No re-indexing required.

## Headline changes

### Tool-output compaction (token savings for agents)

`deep_dive`, `get_context` (readable + compact), `blast_radius`/impact, and `fast_search` (locations and full line mode) now group consecutive same-file hits under a single file header instead of repeating the path on every row. Pure formatting change — no semantic difference, just fewer tokens per result. Test coverage added across all five formatters.

### `line_matches` compound-identifier fix

A search for `workspace_is_primary` previously matched lines containing only an unrelated sub-token like `workspace_label:` because the tokenizer split the compound on `_` and any sub-token was treated as sufficient.

Fixed: compound terms (containing punctuation or camelCase boundaries with multiple sub-tokens) now require a contiguous case-insensitive substring match. Simple terms keep the existing tokenized matching. New `line_match_strategy_tests.rs` covers positive matches, negative matches against decomposed sub-tokens, and excluded-term semantics.

If you previously relied on the loose matching for short identifier searches, the fix tightens precision without affecting simple-token lookups.

### `edit_file` cross-workspace targeting

`edit_file` now accepts a `workspace` parameter and routes via `resolve_workspace_filter` like `rewrite_symbol`. Previously the schema rejected `workspace` due to `deny_unknown_fields`. New `test_edit_file_routes_to_target_workspace` verifies the target-workspace edit lands and the primary workspace is left untouched.

### `call_path` returns compact text instead of pretty JSON

`call_path` output now uses a compact text format (`found=… hops=… 1. A --call--> B at file`). Easier for agents to read, fewer tokens, and the structural information that matters is preserved. Internal consumers (call_path's own tests) were updated; no external Rust callers, no `.cjs` plugin scripts, and the agent-facing instructions describe the tool semantically rather than by parse format. If you were parsing `call_path` JSON in your own tooling, switch to text.

### Developer ergonomics

- **`cargo xtask sync-plugin`** — new subcommand that mirrors `.claude/skills/` from the source repo into `~/source/julie-plugin/skills/` (full mirror with deletes), and reports hook divergence without writing. Replaces the manual `cp` step previously documented for plugin-skill updates. `--dry-run` supported.
- **`cargo xtask test nano`** — new tier running `core-database` + `core-fast` only (~25s). Fastest sanity tier for the edit-test loop, sits below `smoke` and `dev`.
- **README, CLAUDE.md, AGENTS.md** — Codex CLI and OpenCode plugin install paths documented.

### Cleanups

- Stranded debug artifact (`probe.cjs` in `.claude/hooks/`) removed.
- Minor clippy nit (`field_reassign_with_default`) in `xtask/src/sync_plugin.rs`.
- ~80% of the line delta in this release is `cargo fmt` reformatting in daemon, watcher, and PID files. No semantic change in those files.

## Known issues / caveats

- **Daemon force-restart can drop active MCP sessions.** When the `julie-server` release binary is rebuilt while a Claude Code session has an active MCP connection, the daemon's stale-binary auto-restart drains active sessions for 10 seconds, then force-shuts-down with `Session drain timeout exceeded, forcing shutdown — in-flight writes may be lost`. Sessions running heavy work (embeddings, indexing, large search) often can't drain in 10s. The stdio adapter does not always reconnect cleanly, and `mcp__julie__*` tools become unavailable in the affected session until restart. Tracked in `TODO.md` under "Daemon Reliability"; not addressed in this release. Workaround: avoid `cargo build --release` while a Claude Code session is open against the local `julie-server` binary, or restart the session after rebuilding.
