# Julie v7.16.0

This release upgrades Julie to `julie-extractors` v2.16.0 and turns the
upstream enrichment data into first-class code-intelligence surfaces.

## Structural Pattern Intelligence

- Added the `patterns` MCP and standalone CLI tool for listing, searching, and
  summarizing stored structural facts.
- Supports pattern ID, free-text, path, language, metadata, facet, and workspace
  filters.
- Pattern queries use the target workspace's persisted facts rather than
  reparsing files or accepting grammar-specific tree-sitter queries.

## Source-Region Search

- Added `regions` filtering to lexical `fast_search` and the standalone
  `search` command.
- Searches can be restricted to upstream-classified source regions such as
  comments, documentation, string literals, and embedded content.
- Region-filtered zero hits now report a distinct reason through search
  diagnostics and telemetry.
- Semantic and hybrid searches reject region filters explicitly because those
  backends operate on symbols rather than source lines.

## Complexity And Enrichment Persistence

- Persists upstream source regions, structural facts, and complexity metrics in
  typed SQLite tables.
- Full indexing, watcher updates, and the external extraction path now replace
  all enrichment domains atomically with symbols and relationships.
- `deep_dive` now reports compact per-symbol complexity data, including
  decisions, loops, nesting, parameters, and line counts.
- Workspace cleanup and file replacement remove stale enrichment rows.

## Runtime And Compatibility

- Updated every direct extractor dependency to `julie-extractors` v2.16.0.
- Advanced the Julie database schema to version 29 for the three enrichment
  domains.
- Existing databases migrate automatically; semantic indexes are invalidated
  by the updated extraction contract and rebuilt through the normal workspace
  repair flow.
- No MCP configuration change.
