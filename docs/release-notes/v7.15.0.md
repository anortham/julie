# Julie v7.15.0

Minor release that re-pins the `julie-extractors` dependency from v2.1.0 to
v2.4.0, picking up three upstream releases of extraction quality and file
selection fixes across all 34 languages.

## Extractor Upgrade (v2.1.0 → v2.4.0)

- **Broad language data-quality release (v2.3.0):** upstream moved from coarse
  "supported" language claims to fixture-backed, per-domain extraction
  evidence across the full language matrix. Includes stronger Ruby
  relationships, QML type evidence, Vue single-file component span fixes, and
  expanded annotation, doc-comment, literal, and source-region coverage.
- **Git-exact ignore-rule precedence (v2.4.0):** nested `.julieignore` files
  are now honored at any depth (previously capped) with the same semantics as
  nested `.gitignore`; each ignore file is anchored at its own directory
  exactly as git does, fixing root-relative pattern leaks; whitelists can no
  longer re-include files under excluded parent directories.
- **v3 extraction contract (v2.2.0/v2.2.1):** the extractor library now
  carries `structural_facts` and `complexity_metrics` domains (including
  framework-aware facts for ASP.NET minimal APIs, htmx, and Alpine). Julie
  does not yet consume these domains; they flow through the extraction
  pipeline as inert data.

## Automatic Reindex On Upgrade

`SEMANTIC_INDEX_ENGINE_VERSION` now embeds the new extraction contract
(`…source-regions-v1.structural-facts-v1.complexity-metrics-v1`), so existing
workspaces detect the stored-index mismatch and reindex automatically on first
contact with the new binary. Expect a one-time background reindex per
workspace after upgrading; search stays available during the catch-up.

## Runtime And Compatibility

- No Julie SQLite schema change and no external extract-contract change.
- Manual MCP configs should still point at `julie-server`.
- Plugin users get the paired `julie-server` and `julie-embedding-host`
  binaries as usual.

| Signal | Value |
|--------|-------|
| `LATEST_SCHEMA_VERSION` | **28** |
| `EXTRACT_CONTRACT_VERSION` | **3** |
| `hash_algorithm` | **blake3** |

Consumers already compatible with v7.14.x's `(28, 3, blake3)` external extract
surface need no changes.

## Honest Caveats

- The one API break in the upgrade (`ExtractionResults` gaining
  `structural_facts`/`complexity_metrics`) only touched the watcher's
  text-only construction site; extraction behavior for selected files is
  otherwise upstream-controlled.
- The new structural-facts and complexity-metrics data is not yet surfaced by
  any Julie tool — this release is about tracking the current extractor
  contract and its quality fixes, not new features.
