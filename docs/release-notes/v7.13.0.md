# Julie v7.13.0

Cross-language extraction enrichments for the Miller bridge. This minor
release lands three new extraction capabilities — C# attribute
annotations, ordered generic type-argument capture, and carrier-gated
string-literal capture — plus test-role classification on the extract
and watcher paths, a batch of per-language extractor fixes, and a new CI
gate over the full per-language extractor suite.

**Contract move:** schema `26 → 28`, extract contract `1 → 2`. This is the
release Miller re-pins to as `(28, 2)` at its M4 resolver milestone. It is
the only release whose extract contract is `2` and schema is `28`; the
batch is shipped as a single coordinated step so no intermediate
`(27, 1)` / `(26, 2)` pair ever escapes.

## Highlights

- **Generic type-argument capture (schema 27, `type_arguments` table).**
  Use-site generic type arguments are now captured with order and
  nesting preserved (`IRepository<User>` no longer collapses to
  `IRepository`). Rolled out across **all generic-bearing languages**, not
  just the C#/TS/Python that seeded it — the cross-language parity
  mandate is enforced in the plan ledger.
- **String-literal call-arg capture (schema 28, `literals` table,
  contract → 2).** Carrier-gated string literals passed to call sites are
  now persisted end-to-end. Coverage spans **26 applicable languages**
  (8 verified-not-applicable, recorded in the breadth ledger): Python,
  C#, TS, JS, Vue, Go, Ruby, Java, PHP, Rust (call + `sqlx` macro), Dart,
  Kotlin, Scala, VB.NET, Razor, GDScript, Lua, Elixir, Swift, R, C, C++,
  Zig, QML, Bash, PowerShell. Includes TS↔JS HTTP-client carrier parity
  (`ky`/`got`/`ofetch`) and Bash/PowerShell interpolation normalization.
- **C# attribute annotations on every symbol kind (Phase 1).** Type and
  member attributes (`[Table]`, `[Route]`, `[Column]`, `[Key]`,
  `[JsonProperty]`, …) are now emitted as `symbol_annotations` rows on
  fields, events, constants, properties, delegates, destructors, and the
  rest — previously only methods and constructors carried them, and every
  other kind silently dropped its attributes.
- **Test-role classification on extract + watcher paths.** Symbols are
  classified by test role at extraction time, with base-type
  test-container detection, a canonical `base_types` signal, framework
  arms, and a shared call-style core. Cross-language call-style
  materialization covers C/C++/Lua/R/Elixir; a centralized
  `classify_call_exact` matcher backs the Wave-3 adapters and closes the
  qualified-callee false-positive class.
- **New `extractor-units` CI gate.** The 2291-test per-language extractor
  suite now runs under a dedicated bucket in the **dev** and **full**
  tiers. Previously this suite ran in no bucket, so 24 pre-existing
  per-language failures were invisible to the tiers — all 24 are fixed
  and the suite is green.

## Fixes

- **24 pre-existing per-language extractor failures** repaired (surfaced
  once the suite was gated), plus a further 11 per-language extraction
  bugs fixed in the test-role pass.
- **C++ pointer/reference-return free functions** are now extracted as
  symbols instead of being skipped.
- **QML** `PropertyAnimation` binding keys are no longer miscounted as
  `Property` symbols.
- **VB.NET literal carriers** now actually load — `vbnet.toml` is embedded
  so the `[literal_carriers]` config ships with the binary.
- **Workspace root discovery** now stops the upward walk at **all** VCS
  roots (`.git`, `.hg`, `.svn`, …), not just `.git`. This fixed the
  non-hermetic `tools-workspace-targeting` rebind tests (#33) and
  hardened root resolution for non-git checkouts.
- Addressed extraction-enrichment review findings.

## Miller / consumer compatibility

Miller gates its read layer on `(schema_version, extract_contract_version)`
with exact equality. The pairing for this release is:

| Signal | Value |
|--------|-------|
| `LATEST_SCHEMA_VERSION` | **28** (027 = `type_arguments`, 028 = `literals`) |
| `EXTRACT_CONTRACT_VERSION` | **2** |

Consumers pinned to `(26, 1)` (v7.12.2) will correctly reject this release
until they re-pin to `(28, 2)`. Per Miller's plan that is a one-line
change at M4; these enrichments do not block earlier Miller milestones.
