# Julie v7.13.2

External extract contract release for Miller.

## Why This Release Exists

Miller now consumes Julie through `julie-server extract` as a pinned,
process-owned extractor. This release makes the extract output self-describing
enough for Miller to reject incompatible indexes before it serves stale or
misread data.

## External Extract Contract

`julie-server extract info --json` and scan/update/delete reports now include:

- `extract_contract_version`: `3`
- `schema_version`: `28`
- `hash_algorithm`: `blake3`

Miller pins this exact pairing:

```text
julie-server v7.13.2
schema_version = 28
extract_contract_version = 3
hash_algorithm = blake3
```

Consumers pinned to contract `2` should reject this release until they are
updated to understand `hash_algorithm`.

## Notes

This is a contract metadata release. It does not change Julie's source parsing
or schema shape from the v7.13.x line.
