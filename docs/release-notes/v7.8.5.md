# Julie v7.8.5

Plugin hook cleanup release. The julie-server binary is functionally
identical to v7.8.4; this version exists so users on the v7.8.4 plugin
get an update notification that delivers the hook fix.

## Why this release exists

The v7.8.4 julie-plugin shipped two PreToolUse hooks that fired globally
instead of only inside indexed Julie workspaces:

- `pretool-broad-tests.cjs` (Claude Code) — blocked unfiltered
  `cargo test --lib` / `cargo nextest run --lib` in any project the
  plugin was installed under, then suggested `cargo xtask test changed`
  as the alternative. That command only exists in the Julie repo, so
  the discipline was being projected onto unrelated cargo projects with
  no working substitute.
- `codex-pretooluse.cjs` (Codex CLI) — same blocking logic on the Codex
  side.

`pretool-agent.cjs` had the same kind of leak — emitting Julie-specific
tool guidance on every Agent call regardless of workspace — though it
only added context rather than blocking.

## Plugin changes (julie-plugin v7.8.5)

1. `pretool-broad-tests.cjs` and `codex-pretooluse.cjs` are removed.
   The narrowest-test-first discipline they enforced is documented in
   the upstream `CLAUDE.md` / `AGENTS.md` (TDD methodology + subagent
   test rules), which is the right layer for project-wide conventions.
   Hooks should be reserved for blast-radius safety, not coding
   discipline.
2. `pretool-agent.cjs` now gates on `findIndexedWorkspace(cwd)` and
   silently exits when run outside a registered Julie workspace.
3. `bin/install-codex.cjs` no longer installs a `PreToolUse` entry into
   `~/.codex/hooks.json`; `codex-sessionstart` and `codex-posttooluse`
   still install as before.

## Julie source changes

- Removed the dev-only `.claude/hooks/pretool-broad-tests.cjs` from this
  repo for symmetry with the plugin cleanup.
- Plan/design/rubric documents for the in-flight best-in-class
  tree-sitter capability program were committed earlier in the release
  cycle. These are internal planning artifacts, not user-facing API
  changes.

There are no behavioral or API changes in the julie-server binary
between v7.8.4 and v7.8.5.

## Upgrade notes

- Plugin users on v7.8.4: the Claude Code plugin updater should offer
  v7.8.5 next time you open a session that loads the plugin. Accept the
  update to get the workspace-scoped hooks.
- Codex users on v7.8.4: re-run `node bin/install-codex.cjs` in the
  refreshed plugin install. It removes the stale `PreToolUse` entry
  from `~/.codex/hooks.json` automatically (sentinel-based merge).
- Julie source consumers: no migration. The binary behavior is
  unchanged.

## Verification

- Plugin hook smoke checks (Claude Code):
  - `pretool-agent.cjs` outside a registered workspace: exit 0, no
    output.
  - `pretool-grep-redirect.cjs` outside a registered workspace: exit 0,
    no output.
- Plugin hooks.json and Codex hooks.json each parsed as valid JSON
  after the cleanup edits.
- No julie source code paths changed, so the v7.8.4 verification
  evidence remains valid for v7.8.5.
