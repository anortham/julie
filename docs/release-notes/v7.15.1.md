# Julie v7.15.1

Patch release that re-pins `julie-extractors` from v2.4.0 to v2.5.1.

## Extractor Upgrade (v2.4.0 -> v2.5.1)

- Picks up the upstream UTF-16 source-decoding fix for BOM-marked SQL files.
- UTF-16LE and UTF-16BE SQL sources with a byte order mark are decoded before
  extractor parsing.
- Incremental indexing diagnostics still use the original source byte hash and
  byte count.
- Malformed UTF-16 input still reports a read failure with the original content
  hash and byte count.

## Runtime And Compatibility

- No Julie SQLite schema change.
- No external extract-contract change.
- No parser dependency change.
- `SEMANTIC_INDEX_ENGINE_VERSION` is unchanged because
  `julie_extractors::EXTRACTION_CONTRACT_VERSION` is unchanged.
- Manual MCP configs should still point at `julie-server`.
