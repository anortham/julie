# Julie v7.8.3

Focused reliability and onboarding release for MCP clients that do not send
workspace roots, plus restored Intel Mac binary distribution.

## Highlights

- Fixed first-call workspace indexing for Codex Desktop and other no-roots MCP
  clients. When Julie has to defer cwd-based auto-indexing, the first primary
  code-intelligence tool call now waits for the deferred repair/index pass
  instead of racing ahead and reporting `Workspace not indexed yet`.
- Restored macOS Intel release artifacts. Releases now ship
  `x86_64-apple-darwin` alongside Apple Silicon macOS, Linux x86_64, and Windows
  x86_64.
- Improved non-Claude setup docs and plugin launcher guidance. Codex and
  OpenCode users now get launcher-based MCP examples instead of placeholder
  `/path/to/julie-server` instructions.

## MCP Client Startup Fix

This release fixes the deferred cwd indexing race introduced by clients that do
not provide MCP roots. The affected path was most visible in desktop-style
clients where Julie starts without a reliable workspace root and binds the
workspace on the first primary tool request.

Expected behavior now:

- `fast_search`, `deep_dive`, `get_symbols`, editing tools, and other
  primary-workspace tools can trigger and await deferred indexing on first use.
- Explicit `manage_workspace(operation="index")` still owns its own indexing
  path and avoids double-running the repair.
- Filesystem root (`/`) remains rejected as a startup workspace hint.

## Plugin and Harness UX

The `julie-plugin` update for this release includes:

- Codex session-start guidance that reports whether the current cwd already
  appears to be inside an indexed Julie workspace.
- Codex installer output with an exact user-level MCP command:
  `codex mcp add julie -- node /path/to/julie-plugin/hooks/run.cjs`.
- OpenCode installer output with a ready-to-copy `opencode.json` MCP block.
- Updated README guidance: `JULIE_WORKSPACE` is no longer presented as required
  for Codex Desktop. It is now documented as an override for no-roots clients
  that launch Julie from the wrong directory.
- Corrected plugin docs for the current four-skill bundle.

## Maintainer Workflow

Added maintainer-only `xtask` helpers:

- `cargo xtask dev-link` links installed plugin binaries back to the local
  release build for faster dogfooding.
- `cargo xtask dev-restart` gracefully stops the daemon so the next MCP request
  picks up the rebuilt binary.
