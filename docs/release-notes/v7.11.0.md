# Julie v7.11.0

Search release: unified lexical search, optional semantic/hybrid search, a
smaller default `fast_search` payload, richer search telemetry, and daemon
file-descriptor headroom for larger dogfood workspaces.

## Highlights

- **Unified lexical search path.** `fast_search` now routes through one unified
  Tantivy path instead of maintaining separate definition/content/file flows.
  The index stores pre-tokenized code, relationship text, title fields, role
  metadata, and unified document rows so exact symbols, related definitions,
  content matches, and file rows are ranked together consistently.
- **Better lexical ranking.** Title-exact boosts, relationship-text matching,
  basename/file-row scoring, test-intent handling, and Eros-derived field-score
  boosts improve the top results without needing a larger result set.
- **Optional `backend` parameter on `fast_search`.** Omit `backend` for normal
  lexical search. Use `backend="lexical"` for pure lexical bakeoffs,
  `backend="semantic"` for concept-to-symbol lookup, or `backend="hybrid"` for
  symbol semantic search with lexical scoring mixed in. Semantic/hybrid results
  are symbol-backed only.
- **Bounded semantic fallback.** Default `fast_search` stays lexical, but when
  an identifier-like, unscoped query returns zero lexical hits and embeddings are
  ready, Julie may show labeled semantic fallback candidates. Plain natural
  language zero-hit noise does not trigger fallback.
- **Smaller default result payload.** MCP `fast_search` now defaults to 6
  results instead of 10. Agents can still request any explicit `limit` from 1 to
  500. The CLI default remains 10 because terminal output is not automatically
  injected into model context.
- **Search debugging and dashboard updates.** Search telemetry now records
  backend fallback state. The dashboard search compare path uses the shared
  execution params, and Julie skills/agent instructions document lexical,
  semantic, and hybrid search behavior.
- **Daemon file descriptor headroom.** On Unix, daemon startup attempts to raise
  the soft `RLIMIT_NOFILE` to at least 4096 when the hard limit allows it. This
  avoids low default descriptor ceilings during multi-workspace dogfood runs.

## Search Behavior Changes

### Unified Search

The public `search_target` split is gone from the main `fast_search` path.
Search now decides how to rank mixed results from the query, file pattern, role,
test role, and indexed document fields. The CLI keeps `--target` as a
deprecated no-op so older scripts do not fail, but new calls should omit it.

The unified schema adds indexed fields for:

- `pretokenized_code`
- `relationship_text`
- title/name/path signals
- role and test-role metadata
- unified document projection rows

That lets one search surface definitions, content, files, and relationship
matches without running separate target-specific searches and merging them late.

### Semantic and Hybrid Backends

`fast_search` now accepts:

```json
{
  "query": "retry policy builder",
  "backend": "semantic"
}
```

Backend rules:

- omitted: normal lexical search with bounded semantic fallback on
  identifier-like zero-hit queries
- `lexical`: pure Tantivy/SQLite lexical behavior, no semantic fallback
- `semantic`: embedding KNN symbol search only
- `hybrid`: semantic symbol retrieval with lexical scoring blended in

If explicit semantic/hybrid search is requested before embeddings are available,
Julie falls back to lexical and labels that fallback in the result.

### Context Control

The new default `limit=6` is deliberate. Dogfood checks across Julie, Eros,
Express, Flask, and Newtonsoft.Json showed that the upgraded ranking kept the
useful candidates in the top six while results seven through ten were commonly
duplicates, docs, or lower-confidence context. Broad exploration remains
available by passing a larger `limit` or using `get_context`.

## Tooling and Evaluation

- Added a semantic-value scorecard under `docs/eval/semantic-value/` for
  comparing lexical, semantic, and hybrid behavior across larger repositories.
- Extended `xtask search_matrix` and changed-test bucket mapping so search
  experiments and regressions are easier to run without broad ad hoc commands.
- Updated Julie skills for `explore-area`, `impact-analysis`, and
  `search-debug` so agents know when to choose lexical vs. semantic vs. hybrid.
- Added search telemetry fields for strategy/backend behavior and fallback
  attribution.

## Fixes and Hardening

- Reproject relationship partners on incremental watcher updates.
- Propagate relationship-text SQL errors instead of silently swallowing them.
- Make search projection rebuilds atomic around compatibility marker changes.
- Use unique reranker keys to avoid accidental dedupe collisions.
- Restore exact definition enrichment and locations/full ordering parity after
  unified-path cutover.
- Include test artifacts when the query itself names a test file or test
  concept.
- Drop over-broad `bin/`, `plugin`, and `plugins` auto-vendor exclusions that
  hid legitimate code in some repositories.
- Restore daemon indexes after search ablation runs.
- Restore deprecated CLI `--target` as a no-op for compatibility.

## Upgrade Notes

- Existing indexes remain usable. New unified fields and relationship text are
  populated through projection/index refresh and normal watcher activity.
- `fast_search(search_target=...)` callers should remove the parameter. Unknown
  MCP params are ignored by Julie, but the parameter no longer controls routing.
- `fast_search` default output is shorter because the default `limit` is now 6.
  Pass `limit=10` or larger when you intentionally want more breadth.
- Semantic/hybrid search depends on embeddings. First index of a large workspace
  may need time for embedding generation; steady-state filewatcher updates keep
  embeddings current.
- The release archive still ships `julie-server`, `julie-adapter`, and
  `julie-daemon`.
