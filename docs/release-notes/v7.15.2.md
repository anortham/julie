# Julie v7.15.2

Patch release that re-pins `julie-extractors` from v2.5.1 to v2.5.3.

## Extractor Upgrade (v2.5.1 -> v2.5.3)

- Picks up upstream Razor component identity fixes from v2.5.2. `.razor`
  files now emit a filename-derived public top-level component symbol, with
  `@namespace` reflected in the component signature when present.
- Picks up upstream GDScript implicit script-class fixes from v2.5.2 for
  top-level resource-path extends statements such as
  `extends "res://base_controller.gd"`.
- Picks up upstream SQLite cleanup indexes from v2.5.3 for large incremental
  artifact updates and delete-heavy writer paths.

## Runtime And Compatibility

- No Julie SQLite schema change.
- No upstream durable SQLite, JSONL, JSON report, or artifact integer schema
  change.
- Parser dependency versions are unchanged.
- `SEMANTIC_INDEX_ENGINE_VERSION` now embeds the upstream
  `file-derived-component-symbols-v1` extraction-contract marker, so existing
  workspaces can detect the extractor output-shape change and reindex.
- Manual MCP configs should still point at `julie-server`.
