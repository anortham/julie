# Julie v7.14.1

Patch maintenance release for the in-process server cleanup that followed the
HTTP daemon removal.

## Runtime Cleanup

- Renamed the remaining internal `daemon` module and tests to `registry` so the
  source tree matches the current architecture: `julie-server` serves MCP
  in-process over stdio, with no forked daemon runtime.
- Renamed `DaemonPaths` to `RegistryPaths` and removed obsolete daemon path
  helpers for pid, lock, port, transport, token, discovery, migration, and
  shutdown-event files.
- Removed stale `restart_pending` plumbing from handler construction, health
  checks, dashboard status data, and related tests.
- Kept live registry DB and embedding-host path behavior intact, including the
  Windows pipe-name hash helper used by the embedding host.

## Test And Xtask Cleanup

- Renamed the calibrated xtask bucket from `daemon` to `registry`.
- Updated changed-test routing, search-matrix contract fixtures, and no-upward
  dependency guards to use registry terminology.
- Added/updated integration tripwires so deleted daemon runtime files and the
  new registry module surface stay enforced together.

## Runtime And Compatibility

- No schema or external extract-contract change.
- Manual MCP configs should still point at `julie-server`.
- Plugin users should continue to use the plugin launcher; this release is still
  distributed as the paired `julie-server` and `julie-embedding-host` binaries.

| Signal | Value |
|--------|-------|
| `LATEST_SCHEMA_VERSION` | **28** |
| `EXTRACT_CONTRACT_VERSION` | **3** |

Consumers already compatible with v7.14.0's `(28, 3, blake3)` external extract
metadata do not need a schema re-pin for this release.
