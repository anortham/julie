# Julie v7.10.0

External SQLite extraction CLI for non-MCP hosts, daemon split into
`julie-adapter` + `julie-daemon` with shared `julie-server` shim, default-on
search reranker, workspace-scoped connection pool, decoupled embedding pipeline,
and a wave of adapter/daemon reliability fixes.

## Highlights

- **`julie-server extract` CLI.** New process-facing extractor for hosts in
  Go/C#/other runtimes that want Julie's parser data in a caller-owned SQLite
  database without running the MCP server. Five subcommands (`scan`, `update`,
  `delete`, `analyze`, `info`) write the canonical schema, are fully idempotent,
  and use a per-DB exclusive write lock. Does not touch the daemon, MCP
  transport, Tantivy, or embeddings. See `docs/EXTERNAL_EXTRACT.md`.
- **Daemon split.** The single `julie-server` binary is now three: `julie-adapter`
  (the stdio MCP entry point), `julie-daemon` (the long-lived background
  process), and `julie-server` (a compatibility shim that auto-detects and
  dispatches). Existing MCP configs continue to work via the shim; the adapter
  publishes/discovers daemons via a token-protected discovery file and an
  advisory lock prevents legacy/new daemon coexistence.
- **Search reranker default-on.** A new reranker stage scores results with
  intent-classified weights for definition vs. content searches, role/test_role
  awareness, and capability flags. The Tantivy schema gained explicit role and
  test_role fields. NL path priors are applied exactly once.
- **Workspace connection pool.** Daemon read paths migrate off the legacy
  `Arc<Mutex<SymbolDatabase>>` onto a per-workspace pooled connection model.
  Cross-await struct sites that previously held the mutex across `.await`
  boundaries are now pool-borrowed, removing a class of single-writer
  contention. A concurrent-MCP regression test guards against future deadlock
  reintroduction.
- **Embedding pipeline decoupled.** Indexing no longer blocks on embedding
  generation. Vector counts are reconciled on deferred terminal paths so an
  embedding sidecar bootstrap that finishes after an index call doesn't leave
  the workspace stuck at zero vectors.

## What's New

### External Extract CLI

`julie-server extract` is a fully separate use case from the MCP server.
Commands:

- `extract scan --root <dir> --db <file>` — incremental scan; full rebuild with
  `--force`.
- `extract update --root <dir> --db <file> --file <path>` — single-file update,
  designed for host watchers.
- `extract delete --root <dir> --db <file> --file <path>` — remove rows for one
  path; missing rows return `not_found` with exit 0.
- `extract analyze --db <file>` — DB-derived reference scores and test linkage.
- `extract info --db <file>` — read-only metadata: version, schema state,
  totals, analysis state. Safe to call concurrently with mutations.

Guarantees:

- Idempotent. Unchanged inputs return `unchanged` and commit nothing.
- Per-DB exclusive write lock at `<db>.julie-extract.lock` (30 s default).
  `extract info` is read-only and skips the lock.
- Single canonical root per DB. Pointing the same DB at a different root fails
  unless you use `scan --force`.
- No silent data loss. A parser failure that would erase known-good rows exits
  non-zero and preserves existing data.
- Respects `.gitignore`, `.julieignore`, hard blacklist, and a 1 MiB per-file
  cap.

### Daemon Architecture

- `julie-adapter`: stdio JSON-RPC adapter. Discovers or spawns a daemon,
  bridges traffic to its localhost Streamable HTTP endpoint.
- `julie-daemon`: the long-lived background process. Owns shared workspace
  indexes, the embedding provider, and file watchers.
- `julie-server`: compatibility shim. Detects whether it was invoked as an MCP
  client (no args → adapter mode) or with a subcommand, and dispatches.
- Discovery file with atomic IO, token gating, and a kernel advisory lock
  (`DaemonLockGuard`) prevents two daemons from binding the same JULIE_HOME.
- A legacy-migration gate refuses coexistence with pre-split daemons.
- Bounded drain on shutdown with recovery markers and proper 503/502 wiring
  during restart.
- Idle workspaces evict after 60 s; the daemon drain timeout is now bounded.

### Search Reranker

- Intent classifier (`parse_query`) routes queries to definition or content
  scoring.
- Rerank scoring function combines BM25, role/test_role match, centrality,
  and capability flags.
- Tantivy schema enriched with role + test_role fields. Capability flags field
  was dropped after the reranker proved sufficient.
- NL path priors now apply exactly once (previously double-applied on some
  query shapes).
- Test-intent override + path prior tuning for queries like "test for X".
- Ablation harness and corpus (`docs/eval/`) lets us measure scorer changes
  before flipping defaults.

### Adapter Reliability

- Retry on pre-output transport errors with exponential backoff.
- Preserve lost request line on transport send failure (don't drop the
  in-flight request).
- Skip malformed client JSON instead of retrying it.
- Recover the daemon after a post-output transport loss (the calling MCP
  session no longer dies if the daemon hiccups mid-response).
- Adapter's startup lock is held until the spawned daemon publishes liveness,
  closing a race where a second adapter could spawn a duplicate daemon.

### Daemon Reliability

- `mark_restart_pending` always notifies the restart channel — the daemon no
  longer silently misses a restart signal during certain shutdown orderings.
- `restart_notify` listener is wired to `stop_notify`, so a restart-pending
  daemon shuts down promptly when its last session disconnects.
- `request_db` no longer locks the legacy `workspace.db` for target-workspace
  reads. Target-workspace reads run through the connection pool.

### Extractor + Performance

- Cache extractor hot paths (`perf: cache extractor hot paths`).
- Gate name-only `test_` detection on test paths to avoid mis-classifying
  production code as tests.
- Close extractor type fixture gaps caught by capability-bar tests.

## Upgrade Notes

- **No MCP API changes.** All existing tools, parameters, and responses are
  unchanged.
- **New binaries in the archive.** Hosts that hardcoded `julie-server` keep
  working. Hosts that want to skip the shim can launch `julie-adapter`
  directly.
- **Reindex not required.** Existing `.julie/indexes/` databases are
  forward-compatible; the reranker reads existing schema fields and the new
  role/test_role columns are populated incrementally on next watcher events
  or `manage_workspace(operation="refresh")`.
- **External extract DBs are caller-owned.** If you adopt `julie-server extract`,
  Julie owns the schema inside that file but the host owns the file lifecycle.
  Hosts should check `schema_version` and `extract_contract_version` from
  `extract info` before relying on a DB.
- **First daemon launch after upgrade** will rotate the discovery file and
  token file under `~/.julie/`. The legacy-migration gate refuses to start if a
  pre-split daemon is still running; stop it first.
