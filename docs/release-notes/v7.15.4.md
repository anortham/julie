# Julie v7.15.4

This release updates Julie to `julie-extractors` v2.14.0 and adopts the
release's parser, grammar, extraction, and relationship-span improvements.

## Parser And Extraction Updates

- Updated the extractor dependency from v2.5.3 to v2.14.0 and Tree-sitter from
  0.26.8 to 0.26.11.
- Adopted the certified T-SQL grammar fork, including bracketed identifiers,
  batches, procedures, triggers, and `MERGE`, while retaining general SQL
  extraction coverage.
- Adopted the certified Razor grammar fork and public expression-node support.
- Added consumer fixtures for C# 14 file-based applications, Swift 0.7.3,
  R 1.3.0, current Razor syntax, and current T-SQL syntax.
- Synchronized Julie's semantic index engine version with extractor contract 3,
  including the newer route, HTTP-boundary, containing-symbol, backend-HTTP,
  and T-SQL structural facts.

## Relationship Precision

- Structured pending relationships now preserve their upstream source span in
  relationship metadata.
- Same-line calls from one caller to one target use the byte span in their
  relationship identity, so distinct call sites no longer replace each other.
- Legacy spanless relationships retain their existing identity format.

## Runtime And Compatibility

- No Julie SQLite schema change.
- Existing workspaces require a reindex because the semantic index engine
  version changed.
- No MCP configuration change.
