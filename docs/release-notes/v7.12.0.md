# Julie v7.12.0

Cleanup release: typed boundaries across the MCP handler, hardened workspace
path resolution, source-aware language detection unified across extractors,
five new ADRs that pin down load-bearing invariants, and a large test
reorganization that cuts noise without changing behavior.

No user-facing tool semantics changed. Existing MCP clients and CLI invocations
keep working unchanged.

## Highlights

- **Typed error boundary at the MCP handler.** A typed error mapper now sits
  between tool implementations and MCP wire output. `classify_tool_failure`
  finished rollout on the remaining tool wrappers, so terminal vs. recoverable
  vs. validation failures are classified once at the boundary instead of being
  re-encoded ad hoc per wrapper. (`ADR-0002`)
- **Workspace path resolver is strict and typed.** Paths that resolve outside
  the active workspace root are now rejected with a typed error instead of
  silently scoped or partially handled. Resolver failures across the handler
  are typed end-to-end. (`ADR-0001`, `feat(workspace): type resolver failures`)
- **Per-path edit lock invariant.** Multi-file edits now serialize per path so
  concurrent rewrites of the same file cannot interleave. (`ADR-0004`,
  `fix(editing): serialize multi-file commits`)
- **Prepared-once edit/rewrite invariant.** The shared edit/rewrite path caches
  prepared work once per call instead of re-preparing per candidate.
  (`ADR-0003`)
- **Source-aware language detection is shared across extractors.** Pipeline
  language detection moved out of `pipeline.rs` into a single
  `language_spec` module used by all extractor entry points. (`ADR-0005`,
  `feat(extractors): share source-aware language detection`)
- **Search line-mode integrity.** Content-location line tracing was restored,
  semantic test filtering was unified, and line enrichment degradation is now
  surfaced through search telemetry instead of silently swallowed.
- **Resolver respects stored caller language.** The cross-language resolver now
  uses the caller's stored language metadata rather than re-inferring from
  context, fixing a class of cross-language jump misses.

## Architecture Decision Records

Five ADRs were added under `docs/adr/`, recording invariants the cleanup
branch enforced in code so the next refactor cannot quietly drop them:

- `ADR-0001-strict-workspace-path-resolver-contract.md`
- `ADR-0002-typed-mcp-boundary-error-classifier.md`
- `ADR-0003-prepared-once-edit-rewrite-invariant.md`
- `ADR-0004-per-path-edit-lock-invariant.md`
- `ADR-0005-source-aware-language-detection-precedence.md`

## Test Suite Reorganization

A large fraction of the diff is test reorganization with no behavior change:

- Oversized test files were split into focused modules so each file fits the
  500/1000-line house limit.
- Inline test database setup was migrated to the shared db builders. Constructor,
  propagation, reference, relationship, outgoing, path, type-usage, metrics,
  vector-storage, embedding, hybrid, analysis, identifier-query, deep-dive,
  get-context-graph, and change-risk fixtures now go through one path.
- Dashboard intelligence, watcher event queue, dashboard integration, daemon
  database, real-world validation, target-workspace fast-refs, global
  targeting, and workspace discovery test groups were split into smaller
  files.

This is groundwork: the next time these areas change, the failing surface is
smaller and the fixtures are consistent. Production behavior is unchanged.

## Bug Fixes

- `fix(resolver): use stored caller language metadata` — cross-language
  navigation honors the caller's recorded language instead of re-inferring.
- `fix(search): surface line enrichment degradation` — degraded line
  enrichment is reported instead of silently dropping line context.
- `fix(editing): serialize multi-file commits` — multi-file edits no longer
  race on the same path.
- `fix(search): trace line hits for content locations` — content-location
  results now carry line tracing.
- `fix(search): unify semantic test filtering` — semantic search applies the
  same test-intent filter as lexical, eliminating divergent zero-hit cases.

## Internal Cleanups

- Removed the legacy primary-database accessor and the stale handler workspace
  root field; both were dead after the typed-boundary work.
- Removed verified dead code uncovered during the audit.
- Daemon storage was split into focused `workspace` and `codehealth` modules.
- The search and editing modules share a single test predicate and cache
  prepared work once per call instead of per candidate.

## Verification

`cargo xtask test dev` on the v7.12.0 version-bump tree: 34/34 buckets passed
in 533.7s.

## Upgrade Notes

- No configuration changes required. No CLI flags removed or renamed.
- No MCP tool signatures changed.
- Workspace indexes from v7.11.0 do not require rebuild.
- Strict path resolution may reject inputs that resolved to a no-op in older
  versions if the path was outside the active workspace root. The new error
  is a typed validation failure with the offending path; correct the input
  rather than retry.
