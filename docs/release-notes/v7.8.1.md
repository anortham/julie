# v7.8.1 — Tree-Sitter Review-Finding Closure + Index Health Hardening

**Theme:** Patch release closing 12 review findings raised against the v7.8.0 tree-sitter remediation. Eight of them come from the formal review matrix (TS-RF-001..008); four more are follow-on issues caught by an adversarial Codex review on the remediation diff itself.

No API changes. No fixture format changes. No re-indexing required.

## Headline fixes

### Tree-sitter review findings (TS-RF-001..008)

Closes the eight findings tracked in `docs/TREE_SITTER_REVIEW_FINDINGS_STATUS.md` with executable evidence rather than documented gaps.

- **TS-RF-001 — relationship ID collisions.** Relationship IDs now include callsite identity (`file_path`, `line_number`) so a caller invoking the same callee on multiple lines no longer collapses to one edge.
- **TS-RF-002 — embedded child rekey.** HTML `<script>`/`<style>` and Vue style remaps refresh span-derived IDs and rekey embedded child `parent_id` pointers after host offsets are applied.
- **TS-RF-003 — Vue parses-once invariant.** Vue identifier extraction parses the SFC exactly once per pass, carries section offsets through byte and line spans, and computes containing symbols from adjusted host coordinates.
- **TS-RF-004 — `<script setup>` spans.** Variables, imports, function declarations, and standalone macro symbols inside `<script setup>` now carry exact file-relative line/byte spans with column normalization.
- **TS-RF-005 — HTML embedded body offsets.** Offsets come from the embedded text node's tree-sitter bounds instead of substring matching. Duplicate attribute text inside the host can no longer steal the span.
- **TS-RF-006 — SQL view_source / trigger_target.** SQL emits these relationships from the AST when the parser is happy and falls back to a regex matcher only on `ERROR` nodes.
- **TS-RF-007 — Regex fallback span fidelity.** Regex fallback symbols build their `NormalizedSpan` from the actual text-match range via `from_content_range` + `create_symbol_from_span`, not from the root parser node.
- **TS-RF-008 — Capability matrix requires evidence.** A previously-open SQL relationships gap can no longer disappear silently; the capability matrix now requires golden fixture evidence before allowing the entry to close.

### Index health hardening (post-review fixes)

An adversarial Codex review on the TS-RF remediation diff surfaced four follow-on issues. Closed in the same release to keep the audit trail clean.

- **B1 (HIGH) — `ProjectionFailure` invariant silently cleared.** `set_dirty_projection_count(0)` removed both `TantivyDirty` and `ProjectionFailure` together, so an abandoned file's failure indication vanished the moment any other dirty file succeeded. Introduced `abandoned_projection_files: BTreeSet<String>` in `IndexingRuntimeState`. `ProjectionFailure` now tracks that set's emptiness, not the dirty counter. The watcher records a path on abandonment and clears it on per-file recovery; `begin_operation` resets the set. Three inline regression tests cover the invariant.
- **B2 (MEDIUM) — phantom-fd detail suppressed.** `render_report` printed `"Database not initialized"` whenever `canonical_store` was `Unavailable`, swallowing the rich `"ON-DISK STATE MISSING: ...Restart daemon and force-reindex to recover."` detail string. The report now prints `canonical_store.detail` when present and falls back to the generic line only when detail is empty.
- **B3 (LOW-MED) — Vue containing_symbol_id non-determinism.** When two symbols tied on priority and size, `find_containing_symbol_id_for_span` produced different results across runs because the stable sort preserved `HashMap` iteration order. Added an `a.id.cmp(&b.id)` tiebreaker.
- **L1 (DOC) — SQL CREATE_VIEW regex CTE edge case.** On `ERROR`-node fallback, the non-greedy first-`FROM` capture in `CREATE_VIEW_SOURCE_RE` grabs CTE/subquery sources instead of the view's actual base table. Documented up-front so the trade-off is visible to readers.
