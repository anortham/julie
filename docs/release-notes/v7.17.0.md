# Julie v7.17.0

This release turns the persisted web structural facts into navigable code
intelligence, hardens indexing/projection consistency, splits several large
modules for maintainability, and overhauls the test control plane for faster,
more honest local gates. It also fixes a silent search-staleness regression
found in pre-release review.

## Web-Mode Navigation (New)

- `blast_radius` gains a `web` mode that traverses the mixed code + web-edge
  graph, surfacing cross-file HTTP client-call → route-handler callers alongside
  ordinary impacts.
- Web structural facts (HTTP client calls, route handlers, SQL query/table
  facts) are wired into `trace`, `impact`, and `search`.
- Web callers are rendered as a bounded, paginated section with spillover, so a
  high-fanout endpoint no longer dumps an unbounded list into one response.

## Indexing & Projection Consistency

- Dropped the outer `Mutex` around `SearchIndex` so concurrent readers no longer
  serialize on a single lock; the rebuild clear→apply→commit window is covered
  by a purpose-built test proving readers always observe the last committed
  generation.
- Hardened projection consistency and added startup projection reconciliation
  after a successful workspace repair.
- Health now reports generic (projection-type, language/layout-agnostic)
  projection freshness.

## Reliability Fix (found in pre-release review)

- Fixed a silent search-staleness regression: a watcher file handler that errored
  *after* the SQLite commit but before the Tantivy apply left the projection
  falsely `Ready` while search silently missed the file, unrecoverable by the
  hash-based staleness scan. The handler error path now queues the file for
  Tantivy retry so the batch stamps `Stale` and self-heals. Covered by a new
  regression test.

## Maintainability

- Split `SearchIndex` and the watcher runtime internals into focused modules.
- Removed dead code and a misleading docstring in the web-edge pipeline.

## Test Control Plane

- New fast tier and `changed` selection with an explicit **OverBudget** gate
  (fails closed instead of silently falling back to `dev`), optional `--scale`
  union, and separate warm / prebuild / cold-wall accounting.
- Added the `xtask-eval` harness split for product-linked search-matrix/eval
  runs.
- Pinned a reproducible Rust toolchain release.
- Fixed a stale `changed` mapping that routed `src/startup.rs` to a bucket that
  no longer exists (silently dropping that selection); the guarding test now
  runs against the real manifest so the mapping can no longer reference a
  non-existent bucket.
- Closed a coverage gap: the `external_extract` contract/schema tests (including
  the schema-version lockstep tripwire) now run in the dev-tier `registry`
  bucket, so schema drift can no longer rot unnoticed.

## Upgrade Notes

- Database schema advances **29 → 30** (new `web_edges` table). Existing
  databases migrate automatically on open.
- No MCP configuration change.
- The reader-facing `extract` contract version is unchanged (v3).
