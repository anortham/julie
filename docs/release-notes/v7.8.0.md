# v7.8.0 — Tree-Sitter Extractor Audit Remediation

**Theme:** Cross-cutting correctness wave across all 34 language extractors, capability-matrix honesty work, and rigor tests for previously unguarded contracts.

This release closes the verified findings from the tree-sitter extractor audit (`docs/findings/COMPILED-FINDINGS.md`) and locks the corrected behavior into exact semantic tests.

## Headline changes

### Extractor correctness across 34 languages
- **SQL** — JOIN relationships now emit edges from the enclosing FROM table to each joined table (no more self-edges). View-source and trigger-target relationships extracted from CREATE VIEW and CREATE TRIGGER. Dead synthetic external IDs (e.g. `external_users`) are no longer emitted; cross-file FK references are suppressed at the source until structured pending promotion lands. SQL relationship line numbers now follow the project's 1-based convention.
- **Data formats** — JSON doc-comment truncation no longer panics on multi-byte UTF-8 sequences crossing the truncation boundary. YAML alias resolution requires exact anchor name match (no more `*foo` matching `&foobar`). Markdown heading levels preserved in metadata; first heading wins for duplicate slugs; self-references filtered.
- **Multi-binding extraction** — Java, C#, VB.NET, Go, Bash, and Python now emit one Symbol per declarator/binding instead of collapsing to the first.
- **Identity & TypeInfo** — Symbol IDs are stable across re-extraction; TypeInfo merge no longer loses fields on union; visibility fallback uses language-specific defaults.
- **Ruby** — Property accessors and pending edges preserved; rspec test detection.
- **C / C++** — Method-call identifier normalization; relationship resolution disambiguates duplicate names; visibility tightening.
- **Visibility & doc comments** — Inner-doc support for Rust; doc-comment style detection per language.
- **Dart, R, Elixir** — Documentation harvesting and language-idiom corrections.
- **TypeScript / JavaScript** — Class/function/interface/import/identifier coverage.
- **Identifier & call coverage** — Generic args, decorators, parameters, assignments threaded across languages.
- **Vue / HTML / CSS / Razor embedded extraction** — Embedded JS/CSS now delegates to language-native extractors with full-file offsets preserved. Vue Options API methods/computed/data/props emit real symbols. Vue template extracts ref/slot/v-model symbols. CSS modern at-rules and pseudo selectors. Razor `@using` targets namespace-specific IDs.
- **Scripting** — PowerShell, Bash, Lua, GDScript modeling improvements.
- **JVM, Swift, Scala, PHP** — Idiom-level corrections.
- **Degraded parser failure** — Total tree-sitter parse failure now returns a degraded `ExtractionResults` with a parse diagnostic instead of losing the file.

### Capability-matrix honesty
The capability matrix at `fixtures/extraction/capabilities.json` now records honest documentation for every false `target_capabilities` claim. Previously, 12 languages silently lowered the support bar without explaining why; now every gap entry distinguishes:
- **Domain limitations** (e.g. "JSON has no static type system; types capability is intentionally unsupported") — permanent design choices.
- **Real TODOs** (e.g. "JSON Schema $ref relationship extraction is not yet implemented" or "SQL FK pending-promotion deferred — see sql/relationships.rs around line 115") — actionable closure language.

19 new gap entries across html, css, lua, qml, r, razor, sql, regex, markdown, json, toml, yaml.

### Resolver scoping
Rust namespace scoring for `std`, `core`, `alloc` now gates to Rust pending files and Rust candidate symbols only — TypeScript code receiving a path with `std/` no longer gets a Rust-flavored namespace boost.

Rust re-export crate-root detection anchored to workspace crate layouts instead of any path containing `<crate>/src`, preventing vendor-path false-positives.

### New regression tests
- `test_vue_identifier_extraction_parses_script_section_once` — locks down the Vue parses-once invariant (script section walked exactly once per extraction call).
- `test_data_format_relationships_have_exact_targets_or_no_edge` — JSON/TOML/YAML/Markdown relationships must have real targets or emit no edge.
- `test_capability_matrix_records_known_gaps_for_languages_with_unfixed_findings` — every false target_capability must have a matching documented gap entry.
- `test_capability_matrix_pending_claim_requires_pending_output_in_fixtures` — pending_relationships=true must have either fixture evidence or a documented gap.

## Important fixes from pre-merge review

The pre-merge review pass and an adversarial Codex review caught two material issues that landed in the final fix layer:

- **SymbolKind panic at the Tantivy boundary** — `SymbolKind::from_string` panicked on schema-evolved or corrupt rows. Replaced with `try_from_string` plus a `tracing::warn!` and `SymbolKind::Variable` fallback. A single bad row can no longer crash a search request.
- **EnumMember silent downgrade** (Codex finding) — the DB-rescue path in `content_search_with_index` was using `format!("{:?}", s.kind).to_lowercase()`, which produced `"enummember"` while `try_from_string` expects `"enum_member"`. After the panic fix, this would have silently degraded rescued EnumMember symbols to Variable. Switched to `s.kind.to_string()` (Display impl, snake_case).

## Honest caveats

- **Razor fragile fallback** — flagged by the multi-lane review but not addressed: the verbatim finding was lost to context compaction during review consolidation, and dispatching a fix without ground truth would have been guesswork. If the fragility surfaces in dogfood, it can be tracked separately.
- **2 of 4 Task 16 meta-tests not implemented** — `test_relationship_tests_assert_exact_source_target_kind_and_count` and `test_symbol_tests_assert_exact_name_kind_parent_and_range` would require auditing 30+ per-language test modules and are properly multi-day work; deliberately out of scope for this release.
- **9 Minor items** from the multi-lane review (style/cosmetic per language) — out of scope for this release; can be follow-ups against `main`.
- **SQL cross-file FK pending promotion is deferred** — current behavior suppresses unresolved FK targets at extraction. Capability gap entry on SQL records this with concrete required-closure language.

## Upgrade notes

No API changes. No fixture format changes that would invalidate stored indexes. Re-indexing is not required, but downstream consumers who depended on the old (broken) extractor output for any of the languages listed above will see corrected results.
